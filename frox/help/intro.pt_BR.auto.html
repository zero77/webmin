Proxy FTP da Frox
O Frox é um servidor que pode interceptar e encaminhar conexões FTP feitas por clientes que são roteados pelo seu servidor. Isso permite que você armazene em cache o conteúdo do FTP baixado com frequência, imponha o controle de acesso sobre quais clientes têm permissão para usar o FTP e limite a quantidade de largura de banda que pode ser consumida. Isso como o equivalente ao Squid, mas para o protocolo FTP em vez de HTTP.

Não é particularmente útil, a menos que seu sistema seja o gateway entre uma rede interna e a Internet - se você está apenas procurando uma maneira de armazenar em cache o conteúdo FTP em um único sistema, é melhor usar o Squid.

Para que o Frox funcione corretamente, uma regra de firewall deve ser criada para redirecionar todo o tráfego roteado na porta 21 para a porta em que o Frox atende. Se você estiver executando o Linux e usando o módulo Linux Firewall para gerenciar seu firewall, este módulo poderá configurar esta regra para você facilmente.
