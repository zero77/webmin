Frox FTP-Proxy
Frox ist ein Server, der FTP-Verbindungen von Clients abfangen und weiterleiten kann, die über Ihren Server geleitet werden. Auf diese Weise können Sie häufig heruntergeladene FTP-Inhalte zwischenspeichern, die Zugriffskontrolle darüber erzwingen, welche Clients FTP verwenden dürfen, und die Bandbreite begrenzen, die verbraucht werden kann. Dies entspricht dem Squid, jedoch für das FTP-Protokoll anstelle von HTTP.

Dies ist nur dann besonders nützlich, wenn Ihr System ein Gateway zwischen einem internen Netzwerk und dem Internet ist. Wenn Sie nur nach einer Möglichkeit suchen, FTP-Inhalte auf einem einzelnen System zwischenzuspeichern, ist es besser, stattdessen Squid zu verwenden.

Damit Frox ordnungsgemäß funktioniert, muss eine Firewall-Regel erstellt werden, um den gesamten gerouteten Datenverkehr auf Port 21 auf den Port umzuleiten, den Frox abhört. Wenn Sie Linux ausführen und das Linux-Firewall-Modul zum Verwalten Ihrer Firewall verwenden, kann dieses Modul diese Regel problemlos für Sie einrichten.
