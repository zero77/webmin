FTP 프록시
Frox는 서버를 통해 라우팅되는 클라이언트의 FTP 연결을 가로 채서 전달할 수있는 서버입니다. 이를 통해 일반적으로 다운로드되는 FTP 컨텐츠를 캐시하고 FTP 사용이 허용되는 클라이언트에 대한 액세스 제어를 시행하며 소비 할 수있는 대역폭의 양을 제한 할 수 있습니다. 이것은 Squid와 동일하지만 HTTP 대신 FTP 프로토콜에 해당합니다.

시스템이 내부 네트워크와 인터넷 사이의 게이트웨이가 아닌 경우 특히 유용하지 않습니다. 단일 시스템에서 FTP 컨텐츠를 캐시하는 방법을 찾고 있다면 Squid를 대신 사용하는 것이 좋습니다.

Frox가 제대로 작동하려면 포트 21의 모든 라우팅 된 트래픽을 Frox가 수신하는 포트로 경로 재지 정하는 방화벽 규칙을 작성해야합니다. Linux를 실행 중이고 Linux 방화벽 모듈을 사용하여 방화벽을 관리하는 경우이 모듈에서이 규칙을 쉽게 설정할 수 있습니다.
