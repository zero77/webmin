Proxy FTP de Frox
Frox es un servidor que puede interceptar y reenviar conexiones FTP realizadas por clientes que se enrutan a través de su servidor. Esto le permite almacenar en caché el contenido FTP comúnmente descargado, imponer el control de acceso sobre qué clientes pueden usar FTP y limitar la cantidad de ancho de banda que se puede consumir. Esto como el equivalente de Squid, pero para el protocolo FTP en lugar de HTTP.

No es particularmente útil a menos que su sistema sea una puerta de enlace entre una red interna e Internet; si solo está buscando una forma de almacenar en caché el contenido FTP en un solo sistema, es mejor usar Squid.

Para que Frox funcione correctamente, se debe crear una regla de firewall para redirigir todo el tráfico enrutado en el puerto 21 al puerto que Frox escucha. Si está ejecutando Linux y está utilizando el módulo Linux Firewall para administrar su firewall, este módulo puede configurar esta regla fácilmente.
