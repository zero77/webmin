ereduak

Sarbide-kontrol hizkuntzak patroi hauek ezartzen ditu:
  • '' 'Batekin hasten den katea. izaera. Ostalari izena datorren izenaren azken osagaiak zehaztutako patroiarekin bat datorrenean. Adibidez, ".tue.nl" eredua "wzv.win.tue.nl" ostalariaren izenarekin bat dator.
  • '' 'Batekin amaitzen den katea. izaera. Ostalari helbide bat datorren bere zenbakizko eremuak emandako katearekin bat datoz. Adibidez, '131.155 eredua'. Eindhoven Unibertsitateko sareko (ia 131.155.xx) helbide bakoitzarekin bat egiten du.
  • '@' Karaktere batekin hasten den katea NIS (lehen YP) sareko talde izenarekin tratatzen da. Ostalariaren izena datorren zehaztutako netgroup ostalaria bada. Netgroup partiduak ez dira onartzen demonio prozesuaren izenetarako edo bezeroen erabiltzaile izenetarako.
  • 'Nnnn / mmmm' formaren adierazpena 'net/maskara' bikote gisa interpretatzen da. IPv4 ostalariaren helbidea bat dator "net" bit eta AND helbidearen eta "maskararen" berdina bada. Adibidez, '131.155.72.0/255.255.254.0' sare / maskara eredua '131.155.72.0' barrutiko helbide guztiekin bat dator '131.155.73.255'.
  • '[N: n: n: n: n: n: n: n] / m' formaren adierazpena '[net]/prefixlen' bikote gisa interpretatzen da. IPv6 ostalariaren helbidea 'aurrizki' 'net' zatiak helbidearen 'aurrizki' zatien parekoa bada. Adibidez, [net]/prefixlen eredua '[3ffe: 505: 2: 1 ::]/64' helbide guztiekin bat dator '3ffe: 505: 2: 1:' barrutian. bidez '3ffe: 505: 2: 1: ffff: ffff: ffff: ffff'.
  • '/' Karaktere batekin hasten den katea fitxategi izen gisa tratatzen da. Ostalariaren izena edo helbidea datorren fitxategian zerrendatutako ostalari-izen edo helbide-eredu batekin bat datorrenarekin bat dator. Fitxategiaren formatua zeroen lerro bat edo gehiago da, zuri-espazioak bereizten dituzten ostalari izen edo helbide patroiak dituzten zero edo gehiago. Fitxategi izenaren patroia ostalari izen bat edo helbide eredua erabil daiteke.
  • Komodinak '*' eta '?' ostalari edo IP helbideekin bat etor daiteke. Bat egiteko metodo hau ezin da 'net/maskara' bat datorrenarekin bat egin, ostalariaren izenarekin bat datorrenarekin '.' edo IP helbidearekin bat dator '.' rekin amaituz.

komodinak

Sarbide kontrol hizkuntzak komodina esplizituak onartzen ditu:
ALL Wildcard unibertsala, beti dator.
LOCAL Puntu baten karaktereik ez duen ostalari bat dator.
EZEZAGUNA Bere izena ezezaguna den edozein erabiltzailek bat egiten du eta izen edo helbide ezezaguna duen edozein ostalari bat dator. Eredu hau kontu handiz erabili behar da: baliteke ostalariaren izenak ez izatea erabilgarri aldi baterako izenen zerbitzariaren arazoengatik. Sareko helbide bat ez da erabilgarri egongo softwarea zertaz hitz egin nahi duen.
ezagutzen Izena ezagutzen duen edozein erabiltzailek bat egiten du eta izen eta helbide ezagunak dituen edozein ostalari bat dator. Eredu hau kontu handiz erabili behar da: baliteke ostalariaren izenak ez izatea erabilgarri aldi baterako izenen zerbitzariaren arazoengatik. Sareko helbide bat ez da erabilgarri egongo softwarea zertaz hitz egin nahi duen.
Paranoid Izenak ez badu bere helbidearekin bat egiten duen edozein ostalari bat dator. -DPARANOID (modu lehenetsia) tcpd eraikitakoan, horrelako bezeroen eskaerak baztertzen ditu sarbide kontrol taulak begiratu aurretik. Eraiki -DPARANOID gabe horrelako eskaeren gaineko kontrol handiagoa nahi duzunean.

operadoreak

EXCEPT Erabileraren asmoa forma hau da: 'zerrenda_1 EXCEPT zerrenda_2'; eraikuntza honek zerrenda_1arekin bat datorrenarekin bat egiten du, zerrenda_2rekin bat egin ezean. EXCEPT operadorea daemon_lists eta bezero_listeetan erabil daiteke. Salbuespen operadorea habia egin daiteke: kontrol hizkuntzak parentesi artean erabiltzea ahalbidetuko balu, "A SALBATU B salbuespen c" bezala analizatuko litzateke (salbuespen bat (B salbu k). "

Maskorraren komandoak

Lehendabizikoarekin bat datorren sarbide kontrolaren arauak shell komandoa badu, komando hau% ordezkapenak (ikus hurrengo atala). Emaitza /bin/sh haurren prozesu baten bidez lortzen da sarrera, irteera eta /dev/null konektatutako errore estandarrak. Zehaztu '&' komandoaren amaieran itxaron nahi ez baduzu.

Shell komandoek ez dute inetd-aren PATH ezarpenean oinarritzen. Horren ordez, bide izen absolutuak erabili beharko lituzkete, edo PATH = what- ever adierazpenarekin hasi beharko lukete.

% Hedapenak

Honako hedapen hauek shell komandoen barruan daude erabilgarri:
%a (%A) Bezeroa (zerbitzaria) ostalariaren helbidea.
%c Bezeroari buruzko informazioa: erabiltzailea @ ostalaria, erabiltzailea @ helbidea, ostalariaren izena edo helbide soila, eskuragarri dagoen informazioaren arabera.
%d Daemon prozesuaren izena (argv[0] balioa).
%h (%H) Bezeroa (zerbitzaria) ostalariaren izena edo helbidea, ostalariaren izena ez badago.
%n (%N) Bezeroa (zerbitzaria) ostalariaren izena (edo "ezezaguna" edo "paranoia").
%p Daemon prozesuaren id.
%s Zerbitzariaren informazioa: daemon @ ostalaria, daemon @ helbidea edo daemon izena besterik ez dago, informazioa eskuragarri dagoen arabera.
%u Bezeroaren erabiltzaile izena (edo "ezezaguna")
%% % 'Bakar batera zabaltzen da? izaera.
