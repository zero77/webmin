Wrapper TCP

introduzione

Questo modulo utilizza un linguaggio di controllo di accesso semplice basato su modelli client (nome / indirizzo host, nome utente) e server (nome processo, nome / indirizzo host).

Una versione estesa della lingua di controllo dell'accesso è descritta nel documento hosts_options (5). Le estensioni vengono attivate al momento della creazione del programma creando con -DPROCESS_OPTIONS.

File di controllo di accesso

Il software di controllo accessi consulta due file. La ricerca si interrompe alla prima corrispondenza:
  • L'accesso sarà concesso quando una coppia (demone, client) corrisponde a una voce nel file /etc/hosts.allow.
  • Altrimenti, l'accesso verrà negato quando una coppia (demone, client) corrisponde a una voce nel file /etc/hosts.deny.
  • In caso contrario, l'accesso sarà concesso.
Un file di controllo di accesso inesistente viene trattato come se fosse un file vuoto. Pertanto, il controllo dell'accesso può essere disattivato fornendo nessun file di controllo dell'accesso.
