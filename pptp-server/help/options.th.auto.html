ตัวเลือก PPP
เนื่องจากโปรโตคอล PPTP ใช้ PPP ภายในเพื่อส่งผ่านข้อมูลจริงการกำหนดค่าของ PPP daemon จึงส่งผลต่อวิธีการทำงานของมัน ตัวเลือกในหน้านี้พิจารณาว่าจำเป็นต้องมีการตรวจสอบความถูกต้องหรือไม่และหากเป็นเช่นนั้นรูปแบบของการรับรองความถูกต้องจะได้รับการยอมรับ แนะนำให้ใช้ CHAP เนื่องจากเป็นมาตรฐาน defacto สำหรับ PPTP อย่างไรก็ตามเมื่อเปิดใช้งานการรับรองความถูกต้องคุณจะต้องตั้งค่าการเข้าสู่ระบบสำหรับลูกค้าในหน้าบัญชี PPP

ตัวเลือกที่ด้านล่างของหน้าสามารถใช้เพื่อเปิดใช้งานการเข้ารหัส MPPE และโปรโตคอลการตรวจสอบความถูกต้อง MS-CHAP ที่ลูกค้า Windows ใช้เป็นค่าเริ่มต้น MPPE เพิ่มความปลอดภัยให้กับการเชื่อมต่อ PPTP แต่ต้องการให้ทั้ง PPP daemon และเคอร์เนลระบบปฏิบัติการรองรับ PPPd เวอร์ชั่น 2.4.2 ขึ้นไปมีการรองรับ MPPE ดั้งเดิมในขณะที่มีการแก้ไขสำหรับเวอร์ชั่นเก่า อย่างไรก็ตามตัวเลือกไฟล์การกำหนดค่าจริงจะแตกต่างกันระหว่างเวอร์ชั่น PPPd ทั้งสองนี้ โมดูลนี้จะใช้ตัวเลือกที่ถูกต้องเสมอ
