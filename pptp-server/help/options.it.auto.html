Opzioni PPP
Poiché il protocollo PPTP utilizza PPP internamente per trasportare effettivamente i dati, la configurazione del demone PPP influenza il modo in cui si comporta. Le opzioni in questa pagina determinano se è richiesta o meno l'autenticazione e, in tal caso, quali forme di autenticazione sono accettate. Si consiglia CHAP, poiché è lo standard defacto per PPTP. Tuttavia, una volta attivata l'autenticazione è necessario impostare gli accessi per i client nella pagina Account PPP.

Le opzioni nella parte inferiore della pagina possono essere utilizzate per abilitare la crittografia MPPE e i protocolli di autenticazione MS-CHAP utilizzati per impostazione predefinita dai client Windows. MPPE aggiunge sicurezza alle connessioni PPTP, ma richiede che sia il demone PPP che il kernel del sistema operativo lo supportino. Le versioni PPPd 2.4.2 e successive hanno il supporto MPPE nativo, mentre esiste una patch per le versioni precedenti. Tuttavia, le opzioni del file di configurazione effettive differiscono tra queste due versioni di PPPd. Questo modulo utilizzerà comunque sempre le opzioni corrette.
