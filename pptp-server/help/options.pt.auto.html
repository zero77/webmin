Opções de PPP
Como o protocolo PPTP usa o PPP internamente para realmente transportar dados, a configuração do daemon PPP afeta como ele se comporta. As opções nesta página determinam se a autenticação é necessária ou não e, em caso afirmativo, quais formas de autenticação são aceitas. O CHAP é recomendado, pois é o padrão defacto para PPTP. No entanto, depois que a autenticação é ativada, você deve configurar logins para clientes na página Contas PPP.

As opções na parte inferior da página podem ser usadas para habilitar os protocolos de criptografia MPPE e autenticação MS-CHAP usados por padrão pelos clientes Windows. O MPPE adiciona segurança às conexões PPTP, mas requer que o daemon PPP e o kernel do sistema operacional o suportem. As versões 2.4.2 e superior do PPPd têm suporte nativo ao MPPE, enquanto existe um patch para versões mais antigas. No entanto, as opções reais do arquivo de configuração diferem entre essas duas versões do PPPd. Este módulo sempre usará as opções corretas.
