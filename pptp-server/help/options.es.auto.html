Opciones de PPP
Debido a que el protocolo PPTP usa PPP internamente para transportar datos, la configuración del demonio PPP afecta su comportamiento. Las opciones en esta página determinan si se requiere autenticación o no, y si es así, qué formas de autenticación se aceptan. Se recomienda CHAP, ya que es el estándar de facto para PPTP. Sin embargo, una vez que se activa la autenticación, debe configurar los inicios de sesión para los clientes en la página Cuentas PPP.

Las opciones en la parte inferior de la página se pueden usar para habilitar el cifrado MPPE y los protocolos de autenticación MS-CHAP utilizados por defecto por los clientes de Windows. MPPE agrega seguridad a las conexiones PPTP, pero requiere que tanto el demonio PPP como el núcleo del sistema operativo lo admitan. Las versiones PPPd 2.4.2 y superiores tienen soporte nativo de MPPE, mientras que existe un parche para versiones anteriores. Sin embargo, las opciones reales del archivo de configuración difieren entre estas dos versiones PPPd. Sin embargo, este módulo siempre usará las opciones correctas.
