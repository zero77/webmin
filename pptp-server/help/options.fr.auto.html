Options PPP
Étant donné que le protocole PPTP utilise PPP en interne pour transporter réellement les données, la configuration du démon PPP affecte son comportement. Les options de cette page déterminent si l'authentification est requise ou non, et si oui, quelles formes d'authentification sont acceptées. CHAP est recommandé, car il s'agit de la norme de facto pour PPTP. Cependant, une fois l'authentification activée, vous devez configurer les connexions pour les clients sur la page Comptes PPP.

Les options en bas de la page peuvent être utilisées pour activer le cryptage MPPE et les protocoles d'authentification MS-CHAP utilisés par défaut par les clients Windows. MPPE ajoute de la sécurité aux connexions PPTP, mais nécessite que le démon PPP et le noyau du système d'exploitation le prennent en charge. Les versions PPPd 2.4.2 et supérieures ont un support MPPE natif, tandis qu'un correctif existe pour les anciennes versions. Cependant, les options réelles du fichier de configuration diffèrent entre ces deux versions PPPd. Ce module utilisera toujours les bonnes options.
