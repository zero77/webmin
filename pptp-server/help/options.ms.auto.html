Pilihan PPP
Kerana protokol PPTP menggunakan PPP secara dalaman untuk benar-benar mengangkut data, konfigurasi daemon PPP mempengaruhi bagaimana ia bertindak. Pilihan di halaman ini menentukan sama ada pengesahan diperlukan atau tidak, dan jika demikian apa bentuk pengesahan yang diterima. CHAP disyorkan, kerana ini adalah standard defacto untuk PPTP. Namun, setelah pengesahan diaktifkan, anda mesti menyediakan log masuk untuk pelanggan di halaman Akaun PPP.

Pilihan di bahagian bawah halaman dapat digunakan untuk mengaktifkan enkripsi MPPE dan protokol pengesahan MS-CHAP yang digunakan secara lalai oleh klien Windows. MPPE menambahkan keselamatan pada sambungan PPTP, tetapi memerlukan daemon PPP dan kernel sistem operasi menyokongnya. PPPd versi 2.4.2 ke atas mempunyai sokongan MPPE asli, sementara patch ada untuk versi lama. Walau bagaimanapun, pilihan fail konfigurasi sebenarnya berbeza antara kedua versi PPPd ini. Modul ini akan selalu menggunakan pilihan yang betul.
