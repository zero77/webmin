Contrôle d'accès
Selon la version de SSH que vous utilisez, un ou deux types de contrôle d'accès peuvent être configurés sur cette page.

Pour le contrôle d'accès basé sur l'hôte, vous pouvez autoriser ou refuser les hôtes en entrant une liste de noms d'hôtes (comme www.foo.com ), d'adresses IP ou de modèles de noms d'hôtes (comme *.foo.com ). Ce type de contrôle d'accès n'est cependant pas pris en charge par toutes les versions du serveur SSH.

Pour le contrôle d'accès basé sur un utilisateur ou un groupe, les noms d'utilisateur peuvent être des comptes de connexion locaux (comme jcameron ), des modèles de compte (comme jcam* ) ou une combinaison de connexion et de nom d'hôte client (comme jcameron@www.foo.com ). Les groupes autorisés ou refusés peuvent également être des noms de groupe simples ou des modèles de nom de groupe, mais ne peuvent pas inclure de noms d'hôte.
