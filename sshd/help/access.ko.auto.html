액세스 제어
사용중인 SSH 버전에 따라이 페이지에서 하나 또는 두 종류의 액세스 제어를 설정할 수 있습니다.

호스트 기반 액세스 제어의 경우 호스트 이름 (예 : www.foo.com ), IP 주소 또는 호스트 이름 패턴 ( *.foo.com )을 입력하여 호스트를 허용하거나 거부 할 수 있습니다. 이러한 종류의 액세스 제어는 모든 SSH 서버 버전에서 지원되지는 않습니다.

사용자 또는 그룹 기반 액세스 제어의 경우 사용자 이름은 로컬 로그인 계정 (예 : jcameron ), 계정 패턴 (예 : jcam * ) 또는 로그인 및 클라이언트 호스트 이름의 조합 (예 : jcameron@www.foo.com ) 일 수 있습니다. 허용되거나 거부 된 그룹은 일반 그룹 이름 또는 그룹 이름 패턴 일 수 있지만 호스트 이름은 포함 할 수 없습니다.
