Authentification
Cette page sert à configurer les options liées à l'authentification pour votre serveur SSH. Certaines des options les moins évidentes sont:
Autoriser l'authentification par mot de passe
Si ce paramètre est défini sur Non, les utilisateurs devront configurer leur répertoire .ssh avec une clé publique avant de pouvoir se connecter. S'il est défini sur Oui, l'utilisateur sera autorisé à entrer un mot de passe pour l'authentification si la validation ne peut pas être effectuée à l'aide d'une clé publique.

Autoriser l'authentification RSA?
Si ce paramètre est défini sur Non, les utilisateurs devront toujours saisir leur mot de passe même si leur clé publique a été configurée.

Vérifiez les autorisations sur les fichiers clés?
S'il est défini sur Oui, le serveur SSH vérifie le répertoire .ssh chaque utilisateur et tous les répertoires parents pour s'assurer qu'ils ne sont pas accessibles en écriture de groupe, et le fichier .ssh/identity pour s'assurer qu'il n'est pas lisible par quelqu'un d'autre. S'il est défini sur Non, aucune vérification des autorisations de fichier ne sera effectuée.
