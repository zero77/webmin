루트 액세스
서버가 루트 사용자를 존중할 호스트를 제어합니다. 클라이언트가 목록에 없으면 공유를 읽기 전용 또는 읽기 / 쓰기로 마운트 할 수있는 경우에도 NFS 서버는 클라이언트의 루트 사용자를 익명 사용자 로 취급합니다.
호스트는 다음과 같은 방법으로 지정할 수 있습니다.

호스트 이름 또는 IP 주소 (예 : ftp.foo.com 또는 1.2.3.4 )
주어진 이름이나 주소로 호스트를 찾습니다
넷 그룹 (예 : 엔지니어링 )
netgroup의 구성원 인 모든 호스트와 일치
DNS 도메인 (예 : .foo.com )
도메인의 모든 호스트와 일치
네트워크 (예 : @ 10.254.1 )
네트워크의 모든 호스트와 일치
네트워크 / 네트 마스크 (예 : @ 10.254.1 / 24 )
네트워크의 모든 호스트와 일치

또한 위의 표기법에 접두사 -를 붙일 수 있습니다. 즉, 호스트, 넷 그룹, 도메인 또는 네트워크가이 공유를 마운트 할 수 없습니다 . 이것은 호스트 집합에 의한 액세스는 허용하지만 한 멤버는 거부하는 데 유용 할 수 있습니다.
