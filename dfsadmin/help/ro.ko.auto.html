읽기 전용 액세스
이 공유에 대한 읽기 전용 액세스 권한이있는 호스트를 결정합니다. 이 옵션에는 세 가지 선택이 있습니다.
  • 없음
    읽기 전용 액세스 권한이있는 호스트가 없습니다.
  • 모든 호스트
    모든 호스트는이 공유를 읽기 전용으로 마운트 할 수 있습니다. 인터넷에 연결되어 있으면 전 세계 누구나 내 보낸 디렉토리에서 파일을 읽을 수 있습니다.
  • 나열된 호스트
    아래에 나열된 호스트 또는 IP 주소는이 공유를 읽기 전용으로 마운트 할 수 있습니다. 아래 나열된 일치하는 호스트는이 공유를 읽기 전용으로 마운트 할 수 있습니다. 호스트는 다음과 같은 방법으로 지정할 수 있습니다.

    호스트 이름 또는 IP 주소 (예 : ftp.foo.com 또는 1.2.3.4 )
    주어진 이름이나 주소로 호스트를 찾습니다
    넷 그룹 (예 : 엔지니어링 )
    netgroup의 구성원 인 모든 호스트와 일치
    DNS 도메인 (예 : .foo.com )
    도메인의 모든 호스트와 일치
    네트워크 (예 : @ 10.254.1 )
    네트워크의 모든 호스트와 일치
    네트워크 / 네트 마스크 (예 : @ 10.254.1 / 24 )
    네트워크의 모든 호스트와 일치

    또한 위의 표기법에 접두사 -를 붙일 수 있습니다. 즉, 호스트, 넷 그룹, 도메인 또는 네트워크가이 공유를 마운트 할 수 없습니다 . 이것은 호스트 집합에 의한 액세스는 허용하지만 한 멤버는 거부하는 데 유용 할 수 있습니다.
