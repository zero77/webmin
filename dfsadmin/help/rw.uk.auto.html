Доступ для читання та запису
Це визначає, які хости мають доступ до цієї спільної доступу. Для цього варіанту є три варіанти:
  • Жоден
    Жоден хост не має доступу для читання і запису.
  • Усі господарі
    Будь-який хост може змонтувати цю частку для читання-запису Якщо ви підключені до Інтернету, це означає, що будь-хто у світі може читати, записувати та видаляти файли в експортованому каталозі. Використовуйте цю опцію обережно.
  • Перераховані господарі
    Хости або IP-адреси, перелічені нижче, можуть змонтувати цю частку для читання-запису. Відповідні хости, перелічені нижче, можуть змонтувати цю частку для читання-запису. Хости можна вказати наступними способами:

    Ім'я хоста або IP-адреса (наприклад, ftp.foo.com або 1.2.3.4 )
    Збігає хост із заданим іменем або адресою
    Netgroup (як інженерія )
    Відповідає будь-якому хосту, який є членом netgroup
    Домен DNS (наприклад, .foo.com )
    Відповідає будь-якому хосту в домені
    Мережа (наприклад, @ 10.254.1 )
    Відповідає будь-якому хосту в мережі
    Мережа / мережева маска (наприклад, @ 10.254.1 / 24 )
    Відповідає будь-якому хосту в мережі

    Крім того, будь-який із зазначених вище позначень може мати префікс - це означає , що господар, сетевая_группа, домен або мережі не дозволяється встановлювати цю частку. Це може бути корисно для дозволу набору хостів, але заборонити одному учаснику.
