Systemprotokolle NG
Mit diesem Modul können Sie das Syslog-NG-Programm konfigurieren, das den klassischen Unix Syslog-Daemon ersetzt. Es ist dafür verantwortlich, Protokollnachrichten aus verschiedenen Quellen (z. B. Programmen oder anderen Systemen) zu übernehmen und diese Nachrichten dann in die richtigen Protokolldateien zu schreiben.

Die Syslog-NG-Konfiguration besteht aus vier verschiedenen Objekttypen, die zusammen bestimmen, welche Protokolle wo geschrieben werden. Sie sind :

Protokollquellen
Jede Quelle gibt eine oder mehrere Möglichkeiten an, wie Protokollnachrichten vom Server empfangen werden können, z. B. vom Netzwerk, vom Linux-Kernel oder von anderen Programmen auf dem System.
Ziele protokollieren
Ein Ziel ist eine Datei, ein Remote-Server, ein angemeldeter Benutzer oder ein Programm, das Protokollnachrichten empfangen kann. Wenn Sie nur ein Ziel definieren, wird nichts darauf geschrieben. Sie müssen es zuerst mit einem Protokollziel verknüpfen.
Protokollfilter
Ein Filter vergleicht Nachrichten basierend auf einer Vielzahl von Bedingungen, z. B. ihrer Einrichtung (Quellprogrammtyp), Priorität (Schweregrad), Programm, Host und Netzwerkadresse des Absenders. Filter werden in Protokollzielen verwendet, um zu steuern, welche Nachrichten an welche Ziele gesendet werden.
Ziele protokollieren
Ziele sind, wo alles zusammenkommt. Jedes Ziel definiert null oder mehr Quellen, Filter und Ziele, die angeben, dass Nachrichten von den angegebenen Quellen, die mit den angegebenen Filtern übereinstimmen, an die angegebenen Ziele geschrieben werden sollen.
