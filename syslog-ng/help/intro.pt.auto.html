Logs do sistema NG
Este módulo permite configurar o programa Syslog-NG, que é um substituto para o daemon clássico do Unix Syslog. É responsável por receber as mensagens de log de várias fontes (como programas ou outros sistemas) e depois gravar essas mensagens nos arquivos de log corretos.

A configuração do Syslog-NG é composta por quatro tipos diferentes de objetos, que quando combinados determinam quais logs são gravados onde. Eles são :

Origens de log
Cada fonte especifica uma ou mais maneiras pelas quais as mensagens de log podem ser recebidas pelo servidor, como da rede, kernel do Linux ou outros programas no sistema.
Destinos de registro
Um destino é um arquivo, servidor remoto, usuário conectado ou programa que pode receber mensagens de log. Apenas definir um destino não faz com que nada seja gravado nele - você deve vinculá-lo a um Destino de Log primeiro.
Filtros de registro
Um filtro corresponde às mensagens com base em uma variedade de condições, como suas instalações (tipo de programa de origem), prioridade (nível de gravidade), programa, host e endereço de rede do remetente. Os filtros são usados nos destinos de log para controlar quais mensagens vão para quais destinos.
Destinos de log
Os alvos são onde tudo acontece. Cada destino define zero ou mais fontes, filtros e destinos, que especificam que as mensagens das fontes fornecidas que correspondem aos filtros fornecidos devem ser gravadas nos destinos fornecidos.
