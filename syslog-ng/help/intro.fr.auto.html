Journaux système NG
Ce module vous permet de configurer le programme Syslog-NG, qui remplace le démon Unix Syslog classique. Il est chargé de prendre les messages de journal de diverses sources (tels que des programmes ou d'autres systèmes), puis d'écrire ces messages dans les fichiers journaux corrects.

La configuration Syslog-NG est composée de quatre types d'objets différents qui, lorsqu'ils sont combinés ensemble, déterminent quels journaux sont écrits où. Elles sont :

Sources de journal
Chaque source spécifie une ou plusieurs façons dont les messages de journal peuvent être reçus par le serveur, par exemple depuis le réseau, le noyau Linux ou d'autres programmes sur le système.
Destinations de journal
Une destination est un fichier, un serveur distant, un utilisateur connecté ou un programme pouvant recevoir des messages de journal. La simple définition d'une destination ne fait rien y écrire - vous devez d'abord la lier à une cible de journal.
Filtres de journaux
Un filtre fait correspondre les messages en fonction de diverses conditions, telles que leur installation (type de programme source), la priorité (niveau de gravité), le programme, l'adresse réseau de l'hôte et de l'expéditeur. Les filtres sont utilisés dans les cibles de journal pour contrôler quels messages vont vers quelles destinations.
Cibles de journal
Les cibles sont l'endroit où tout se réunit. Chaque cible définit zéro ou plusieurs sources, filtres et destinations, qui spécifient que les messages des sources données correspondant aux filtres donnés doivent être écrits vers les destinations données.
