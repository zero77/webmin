Hosts permitidos
Nesta página, você pode controlar quais hosts têm permissão para acessar seu servidor de banco de dados e que tipo de autenticação eles precisam fornecer. Cada regra de controle de acesso pode ser aplicada a um único host, a uma rede inteira, a todos os hosts de rede ou apenas aos usuários que se conectam a partir desta máquina. Além disso, cada regra pode ser aplicada a um único banco de dados selecionado ou a todos os bancos de dados no servidor.

Ao escolher o modo de autenticação para hosts permitidos, as seguintes opções estão disponíveis:

Senha em texto sem formatação
O usuário deve fornecer uma senha que corresponda à definida na página Usuários do PostgreSQL .

Senha criptografada
O usuário deve fornecer uma senha que, quando criptografada, corresponda à definida nos Usuários do PostgreSQL .

Nenhuma autenticação necessária
Nenhuma senha precisa ser fornecida.

Rejeitar conexão
Não permita conexões a partir deste host. Isso é útil para negar o acesso de hosts específicos se você tiver concedido acesso a uma rede inteira.
