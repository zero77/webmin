Επιτρεπόμενοι οικοδεσπότες
Σε αυτήν τη σελίδα μπορείτε να ελέγξετε ποιοι κεντρικοί υπολογιστές επιτρέπεται να έχουν πρόσβαση στο διακομιστή βάσης δεδομένων σας και τι είδους έλεγχο ταυτότητας θα πρέπει να παρέχουν. Κάθε κανόνας ελέγχου πρόσβασης μπορεί να εφαρμοστεί σε έναν μόνο κεντρικό υπολογιστή, σε ένα ολόκληρο δίκτυο, σε όλους τους κεντρικούς υπολογιστές δικτύου ή μόνο σε χρήστες που συνδέονται από αυτό το μηχάνημα. Επιπλέον, κάθε κανόνας μπορεί να εφαρμοστεί σε μία μόνο επιλεγμένη βάση δεδομένων ή σε όλες τις βάσεις δεδομένων του διακομιστή.

Κατά την επιλογή της λειτουργίας ελέγχου ταυτότητας για επιτρεπόμενους κεντρικούς υπολογιστές, είναι διαθέσιμες οι ακόλουθες επιλογές:

Plaintext κωδικός πρόσβασης
Ο χρήστης πρέπει να παρέχει έναν κωδικό πρόσβασης που να ταιριάζει με αυτόν που έχει οριστεί στη σελίδα χρηστών PostgreSQL .

Κρυπτογραφημένος κωδικός πρόσβασης
Ο χρήστης πρέπει να παρέχει έναν κωδικό πρόσβασης που όταν κρυπτογραφείται ταιριάζει με αυτό που έχει οριστεί στους χρήστες PostgreSQL .

Δεν απαιτείται έλεγχος ταυτότητας
Δεν χρειάζεται να παρέχεται κωδικός πρόσβασης.

Απόρριψη σύνδεσης
Μην επιτρέπετε καθόλου συνδέσεις από αυτόν τον κεντρικό υπολογιστή. Αυτό είναι χρήσιμο για την άρνηση πρόσβασης από συγκεκριμένους κεντρικούς υπολογιστές εάν έχετε παραχωρήσει πρόσβαση σε ένα ολόκληρο δίκτυο.
