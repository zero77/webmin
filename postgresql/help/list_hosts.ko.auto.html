허용 된 호스트
이 페이지에서 데이터베이스 서버에 액세스 할 수있는 호스트와 제공해야 할 인증 종류를 제어 할 수 있습니다. 각 액세스 제어 규칙은 단일 호스트, 전체 네트워크, 모든 네트워크 호스트 또는이 컴퓨터에서 연결하는 사용자에게만 적용될 수 있습니다. 또한 각 규칙은 선택된 단일 데이터베이스 또는 서버의 모든 데이터베이스에 적용될 수 있습니다.

허용 된 호스트에 대한 인증 모드를 선택할 때 다음 옵션을 사용할 수 있습니다.

일반 텍스트 비밀번호
사용자는 PostgreSQL 사용자 페이지에서 설정 한 것과 일치하는 암호를 제공해야합니다.

암호화 된 비밀번호
사용자는 암호화 할 때 PostgreSQL 사용자에 설정된 암호와 일치하는 암호를 제공해야합니다.

인증이 필요하지 않습니다
비밀번호를 전혀 제공하지 않아도됩니다.

연결 거부
이 호스트로부터의 연결을 전혀 허용하지 마십시오. 전체 네트워크에 대한 액세스 권한을 부여한 경우 특정 호스트의 액세스를 거부하는 데 유용합니다.
