Zulässige Hosts
Auf dieser Seite können Sie steuern, welche Hosts auf Ihren Datenbankserver zugreifen dürfen und welche Art von Authentifizierung sie bereitstellen müssen. Jede Zugriffssteuerungsregel kann für einen einzelnen Host, für ein gesamtes Netzwerk, für alle Netzwerkhosts oder nur für Benutzer gelten, die von diesem Computer aus eine Verbindung herstellen. Darüber hinaus kann jede Regel entweder für eine einzelne ausgewählte Datenbank oder für alle Datenbanken auf dem Server gelten.

Bei Auswahl des Authentifizierungsmodus für zulässige Hosts stehen folgende Optionen zur Verfügung:

Nur-Text-Passwort
Der Benutzer muss ein Kennwort angeben, das mit dem auf der Seite PostgreSQL-Benutzer festgelegten Kennwort übereinstimmt.

Verschlüsseltes Passwort
Der Benutzer muss ein Kennwort angeben, das bei der Verschlüsselung mit dem in PostgreSQL-Benutzern festgelegten Kennwort übereinstimmt.

Keine Authentifizierung erforderlich
Es muss überhaupt kein Passwort angegeben werden.

Verbindung ablehnen
Erlaube überhaupt keine Verbindungen von diesem Host. Dies ist nützlich, um den Zugriff von bestimmten Hosts zu verweigern, wenn Sie Zugriff auf ein gesamtes Netzwerk gewährt haben.
