Hôtes autorisés
Sur cette page, vous pouvez contrôler quels hôtes sont autorisés à accéder à votre serveur de base de données et quel type d'authentification ils devront fournir. Chaque règle de contrôle d'accès peut s'appliquer à un seul hôte, à un réseau entier, à tous les hôtes du réseau ou uniquement aux utilisateurs se connectant à partir de cette machine. En outre, chaque règle peut s'appliquer à une seule base de données sélectionnée ou à toutes les bases de données sur le serveur.

Lorsque vous choisissez le mode d'authentification pour les hôtes autorisés, les options suivantes sont disponibles:

Mot de passe en clair
L'utilisateur doit fournir un mot de passe qui correspond à celui défini dans la page Utilisateurs de PostgreSQL .

Mot de passe crypté
L'utilisateur doit fournir un mot de passe qui, une fois chiffré, correspond à celui défini dans PostgreSQL Users .

Aucune authentification requise
Aucun mot de passe ne doit être fourni du tout.

Refuser la connexion
N'autorisez pas les connexions de cet hôte. Ceci est utile pour refuser l'accès à partir d'hôtes spécifiques si vous avez accordé l'accès à un réseau entier.
