LDAP erabiltzaileak eta taldeak
Zure sistema Unix erabiltzaileak eta taldeak gordetzeko LDAP zerbitzaria erabiltzeko konfiguratuta badago, modulu hau LDAP datu-base bateko erabiltzaileak kudeatzeko erabil daiteke. LDAP sarritan NISen alternatiba gisa erabiltzen da sareko sistema ugaritan erabiltzailearen informazioa sinkronizatzeko, datu-base batek erabiltzaile bakoitzari atributu osagarriak gorde ditzake Unix erabiltzaile-izen estandarra, shell eta abar.

Jakina, LDAP zerbitzari bat zure sisteman edo sareko ostalariren batean egon behar da modulu hau erabiltzeko. Gainera, ostalari bat edo gehienak konfiguratu behar dira erabiltzailearentzako eta / edo taldeko autentifikazioetarako LDAP erabiltzeko. Zeregin hauetako bat ere ezin da moduluaren barruan egin - eskuz egin behar dira, edo Webmin beste modulu batzuekin batera.

Samba ere LDAP zerbitzaria erabiltzeko konfiguratuta baduzu, modulu honek LDAP erabiltzaile bakoitzarentzako beharrezko pasahitz atributuak sortu eta kudeatu ditzake Samba zerbitzarian ere saioa egin ahal izateko. Funtzio hau funtziona dezan, zure LDAP zerbitzaria konfiguratu behar da bere eskemako erabiltzaile bakoitzeko Samba atributu eta objektu klase gehigarriak onartzen ditu. Samba saioa hastean bakarrik ? Aukera Samba atributuak sortuko da Bai ezarrita.

Moduluak erabiltzaileak ere sor ditzake Cyrus IMAP zerbitzarian, baldin eta LDAP datu base beraren aurka autentikatzeko konfiguratuta badago. Erabiltzaile berri bakoitzerako postontziak sortuko ditu, erabiltzailea bere postontzietara harpidetu eta administratzaileentzako sarbidea emango die. LDAP erabiltzailea ezabatzen bada, bere IMAP postontzia ere egongo da. Ezaugarri honek Cyrus IMAP zerbitzariarekin bakarrik funtzionatzen du. Ez da probatu eta, seguruenik, ez da beharrezkoa beste zerbitzari batzuetarako, erabiltzaileen etxeko direktorioak posta gordetzeko.
