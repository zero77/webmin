Usuários e grupos LDAP
Se o seu sistema estiver configurado para usar um servidor LDAP para armazenar usuários e grupos Unix, este módulo poderá ser usado para gerenciar usuários em um banco de dados LDAP. O LDAP é frequentemente usado como uma alternativa ao NIS para sincronizar as informações do usuário em vários sistemas em uma rede, pois um banco de dados pode armazenar atributos adicionais para cada usuário, além do nome de usuário padrão do Unix, shell e assim por diante.

Naturalmente, um servidor LDAP deve estar em execução no seu sistema ou em algum host da sua rede para que este módulo seja usado. Além disso, um ou mais hosts devem ser configurados para usar LDAP para autenticação de usuário e / ou grupo. Nenhuma dessas tarefas pode ser realizada no módulo - elas devem ser feitas manualmente ou com outros módulos Webmin para esse fim.

Se você tiver o Samba configurado para usar também um servidor LDAP, este módulo também poderá criar e gerenciar os atributos de senha necessários para cada usuário LDAP para poder efetuar login no servidor Samba também. Para que esse recurso funcione, seu servidor LDAP deve ser configurado para suportar os atributos adicionais do Samba e a classe de objeto para cada usuário em seu esquema. Somente quando o Samba entra? opção for definida como Sim , os atributos do Samba serão criados.

O módulo também pode criar usuários em um servidor Cyrus IMAP, se ele estiver configurado para autenticação no mesmo banco de dados LDAP. Ele criará caixas de correio para cada novo usuário, assinará o usuário nas caixas de correio e concederá acesso de administrador a elas. Se um usuário LDAP for excluído, sua caixa de correio IMAP também será. Atualmente, esse recurso funciona apenas com o servidor Cyrus IMAP - ele não foi testado e provavelmente nem é necessário para outros servidores, que usam os diretórios pessoais dos usuários para armazenamento de mensagens.
