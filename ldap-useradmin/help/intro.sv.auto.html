LDAP-användare och grupper
Om ditt system är konfigurerat för att använda en LDAP-server för att lagra Unix-användare och grupper, kan den här modulen användas för att hantera användare i en LDAP-databas. LDAP används ofta som ett alternativ till NIS för att synkronisera användarinformation mellan flera system i ett nätverk, eftersom en databas kan lagra ytterligare attribut för varje användare utöver standard Unix-användarnamn, skal osv.

Naturligtvis måste en LDAP-server köras på ditt system eller på någon värd i ditt nätverk för att den här modulen ska kunna användas. Dessutom måste en eller de flesta värdar konfigureras för att använda LDAP för användar- och / eller gruppverifiering. Ingen av dessa uppgifter kan utföras inom modulen - de måste göras manuellt eller med andra Webmin-moduler för ändamålet.

Om du också har Samba konfigurerat att använda en LDAP-server kan den här modulen också skapa och hantera nödvändiga lösenordattribut för varje LDAP-användare för att kunna logga in på Samba-servern också. För att den här funktionen ska fungera måste din LDAP-server konfigureras för att stödja ytterligare Samba-attribut och objektklass för varje användare i dess schema. Bara när Samba loggar in? alternativet är inställt på Ja kommer Samba attribut att skapas.

Modulen kan också skapa användare på en Cyrus IMAP-server om den är konfigurerad för att verifiera mot samma LDAP-databas. Det kommer att skapa brevlådor för varje ny användare, prenumerera användaren på sina brevlådor och ge admin åtkomst till dem. Om en LDAP-användare raderas kommer hans IMAP-brevlåda också att vara. För närvarande fungerar den här funktionen bara med Cyrus IMAP-servern - den har inte testats och är förmodligen inte ens nödvändig för andra servrar, som använder användarnas hemkataloger för postlagring.
