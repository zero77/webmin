Utenti e gruppi LDAP
Se il sistema è configurato per utilizzare un server LDAP per la memorizzazione di utenti e gruppi Unix, questo modulo può essere utilizzato per gestire gli utenti in un database LDAP. LDAP viene spesso utilizzato come alternativa a NIS per sincronizzare le informazioni utente su più sistemi in una rete, in quanto un database può memorizzare ulteriori attributi per ciascun utente oltre al nome utente Unix standard, alla shell e così via.

Naturalmente, un server LDAP deve essere in esecuzione sul sistema o su un host sulla rete per utilizzare questo modulo. Inoltre, uno o più host devono essere configurati per utilizzare LDAP per l'autenticazione dell'utente e / o del gruppo. Nessuna di queste attività può essere eseguita all'interno del modulo: devono essere eseguite manualmente o con altri moduli Webmin allo scopo.

Se Samba è configurato per utilizzare anche un server LDAP, questo modulo può anche creare e gestire gli attributi di password necessari per ogni utente LDAP per poter accedere anche al server Samba. Affinché questa funzione funzioni, il server LDAP deve essere configurato per supportare gli attributi Samba e la classe di oggetti aggiuntivi per ciascun utente nel suo schema. Solo quando si accede a Samba? l'opzione è impostata su Sì verranno creati gli attributi Samba.

Il modulo può anche creare utenti su un server IMAP Cyrus, se è impostato per l'autenticazione con lo stesso database LDAP. Creerà caselle di posta per ogni nuovo utente, abbonerà l'utente alle sue caselle e concede loro l'accesso di amministratore. Se un utente LDAP viene eliminato, lo sarà anche la sua casella di posta IMAP. Attualmente questa funzione funziona solo con il server IMAP Cyrus: non è stata testata e probabilmente non è nemmeno necessaria per altri server che utilizzano le directory home degli utenti per l'archiviazione della posta.
