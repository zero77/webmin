Usuarios y grupos de LDAP
Si su sistema está configurado para usar un servidor LDAP para almacenar usuarios y grupos de Unix, este módulo se puede usar para administrar usuarios en una base de datos LDAP. LDAP a menudo se usa como una alternativa a NIS para sincronizar la información del usuario en múltiples sistemas en una red, ya que una base de datos puede almacenar atributos adicionales para cada usuario además del nombre de usuario, shell, etc. estándar de Unix.

Naturalmente, un servidor LDAP debe estar ejecutándose en su sistema o en algún host de su red para que este módulo sea utilizado. Además, uno o la mayoría de los hosts deben estar configurados para usar LDAP para la autenticación de usuarios y / o grupos. Ninguna de estas tareas puede realizarse desde el módulo: deben realizarse manualmente o con otros módulos de Webmin para tal fin.

Si también ha configurado Samba para usar un servidor LDAP, este módulo también puede crear y administrar los atributos de contraseña necesarios para que cada usuario LDAP también pueda iniciar sesión en el servidor Samba. Para que esta característica funcione, su servidor LDAP debe estar configurado para admitir los atributos Samba adicionales y la clase de objeto para cada usuario en su esquema. ¿Solo cuando el inicio de sesión de Samba? la opción se establece en Sí, se crearán los atributos de Samba.

El módulo también puede crear usuarios en un servidor Cyrus IMAP, si está configurado para autenticarse en la misma base de datos LDAP. Creará buzones para cada nuevo usuario, suscribirá al usuario a sus buzones y le otorgará acceso de administrador. Si se elimina un usuario LDAP, su buzón IMAP también lo estará. Actualmente, esta función solo funciona con el servidor Cyrus IMAP: no se ha probado y probablemente ni siquiera sea necesaria para otros servidores, que utilizan los directorios principales de los usuarios para el almacenamiento de correo.
