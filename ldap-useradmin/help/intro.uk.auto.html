Користувачі та групи LDAP
Якщо ваша система налаштована на використання сервера LDAP для зберігання користувачів та груп Unix, цей модуль може використовуватися для управління користувачами в базі даних LDAP. LDAP часто використовується як альтернатива NIS для синхронізації інформації користувачів у кількох системах мережі, оскільки база даних може зберігати додаткові атрибути для кожного користувача на додаток до стандартного імені користувача Unix, оболонки тощо.

Звичайно, для використання цього модуля повинен працювати сервер LDAP у вашій системі або на якомусь хості у вашій мережі. Крім того, один або більшість хостів повинні бути налаштовані для використання LDAP для аутентифікації користувачів та / або групи. Жодне з цих завдань неможливо виконати з модуля - їх потрібно виконувати вручну або з іншими модулями Webmin для цього.

Якщо у вас Samba налаштований і для використання сервера LDAP, цей модуль також може створювати та керувати необхідними атрибутами пароля для кожного користувача LDAP, щоб мати можливість увійти на сервер Samba. Щоб ця функція працювала, ваш LDAP-сервер повинен бути налаштований для підтримки додаткових атрибутів Samba та класу об'єктів для кожного користувача в його схемі. Тільки коли вхід у Samba? для параметра встановлено значення Так, чи будуть створені атрибути Samba.

Модуль також може створювати користувачів на сервері Cyrus IMAP, якщо він налаштований для автентифікації на тій самій базі даних LDAP. Він створить поштові скриньки для кожного нового користувача, підпише користувача на його поштові скриньки та надасть адміністратору доступ до них. Якщо користувач LDAP буде видалений, його також буде поштова скринька IMAP. В даний час ця функція працює лише з сервером Cyrus IMAP - вона не перевірена і, мабуть, навіть не потрібна іншим серверам, які використовують домашні каталоги користувачів для зберігання пошти.
