Карыстальнікі і групы LDAP
Калі ваша сістэма настроена на выкарыстанне сервера LDAP для захоўвання карыстальнікаў і груп Unix, гэты модуль можа быць выкарыстаны для кіравання карыстальнікамі ў базе дадзеных LDAP. LDAP часта выкарыстоўваецца ў якасці альтэрнатывы NIS для сінхранізацыі інфармацыі карыстальніка па некалькіх сістэмах у сетцы, бо база дадзеных можа захоўваць дадатковыя атрыбуты для кожнага карыстальніка ў дадатак да стандартнага карыстальніка Unix, абалонкі і гэтак далей.

Натуральна, што сервер LDAP павінен працаваць у вашай сістэме або на нейкім хосце ў вашай сеткі, каб гэты модуль быў выкарыстаны. Акрамя таго, адзін або большасць хастоў павінны быць настроены для выкарыстання LDAP для аўтэнтыфікацыі карыстальнікаў і / або групы. Ні адно з гэтых задач не можа быць выканана з модуля - яны павінны быць выкананы ўручную альбо з іншымі модулямі Webmin для гэтага.

Калі ў вас ёсць Samba, наладжаны таксама на выкарыстанне сервера LDAP, гэты модуль можа таксама ствараць і кіраваць неабходнымі атрыбутамі пароля для кожнага карыстальніка LDAP, каб мець магчымасць увайсці на сервер Samba. Каб гэтая функцыя працавала, ваш сервер LDAP павінен быць наладжаны для падтрымкі дадатковых атрыбутаў Samba і класа аб'ектаў для кожнага карыстальніка ў яго схеме. Толькі калі ўваходзіць Samba? параметр усталяваны ў Так, ці будуць створаны атрыбуты Samba.

Модуль таксама можа ствараць карыстальнікаў на серверы Cyrus IMAP, калі ён створаны для аўтэнтыфікацыі на той жа базе дадзеных LDAP. Ён будзе ствараць паштовыя скрыні для кожнага новага карыстальніка, падпісваць яго на паштовыя скрыні і прадастаўляць адміністратарам доступ да іх. Калі карыстальнік LDAP будзе выдалены, яго паштовая скрыня IMAP таксама будзе. У цяперашні час гэтая функцыя працуе толькі з серверам Cyrus IMAP - яна не прайшла праверку і, верагодна, нават не патрэбна для іншых сервераў, якія выкарыстоўваюць хатнія каталогі карыстальнікаў для захоўвання пошты.
