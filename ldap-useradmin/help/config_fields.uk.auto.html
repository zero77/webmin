Додаткові властивості користувача LDAP, щоб дозволити редагування
Якщо у вашій базі даних LDAP використовуються атрибути, які не підтримуються безпосередньо Webmin, цей параметр конфігурації може використовуватися для додавання додаткових полів на сторінку редагування користувача для управління ними. Це може бути корисно для редагування властивостей Qmail + LDAP, таких як mailForwardingAddress та mailMessageStore.

Атрибути для редагування повинні бути введені у форматі:

name1 опис1
name2 опис2

Ім'я - це ім'я атрибуту LDAP, а опис - текст, який відображатиметься в Webmin. Ім'я може необов'язково супроводжуватися знаком +, що вказує на те, що цей атрибут може допускати кілька значень.
