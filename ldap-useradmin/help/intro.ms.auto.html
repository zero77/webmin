Pengguna dan Kumpulan LDAP
Sekiranya sistem anda dikonfigurasi untuk menggunakan pelayan LDAP untuk menyimpan pengguna dan kumpulan Unix, modul ini dapat digunakan untuk mengurus pengguna dalam pangkalan data LDAP. LDAP sering digunakan sebagai alternatif kepada NIS untuk menyegerakkan maklumat pengguna di pelbagai sistem dalam rangkaian, kerana pangkalan data dapat menyimpan atribut tambahan untuk setiap pengguna sebagai tambahan kepada nama pengguna Unix, shell dan sebagainya.

Secara semula jadi, pelayan LDAP mesti dijalankan pada sistem anda atau pada beberapa hos di rangkaian anda agar modul ini dapat digunakan. Sebagai tambahan, satu atau kebanyakan host mesti dikonfigurasikan untuk menggunakan LDAP untuk pengesahan pengguna dan / atau kumpulan. Kedua-dua tugas ini tidak dapat dilakukan dari dalam modul - ia mesti dilakukan secara manual, atau dengan modul Webmin lain untuk tujuan tersebut.

Sekiranya anda telah mengkonfigurasi Samba untuk menggunakan pelayan LDAP juga, modul ini juga dapat membuat dan mengurus atribut kata laluan yang diperlukan untuk setiap pengguna LDAP agar dapat masuk ke pelayan Samba juga. Agar ciri ini berfungsi, pelayan LDAP anda mesti dikonfigurasikan untuk menyokong atribut Samba tambahan dan kelas objek untuk setiap pengguna dalam skema. Hanya apabila log masuk Samba? pilihan ditetapkan ke Ya akankah atribut Samba dibuat.

Modul ini juga dapat membuat pengguna di pelayan Cyrus IMAP, jika disiapkan untuk mengesahkan terhadap pangkalan data LDAP yang sama. Ini akan membuat peti mel untuk setiap pengguna baru, melanggan pengguna ke peti melnya dan memberikan akses kepada mereka kepada pentadbir. Sekiranya pengguna LDAP dihapus, peti mel IMAP juga akan ada. Pada masa ini, ciri ini hanya berfungsi dengan pelayan Cyrus IMAP - belum diuji dan mungkin tidak diperlukan untuk pelayan lain, yang menggunakan direktori rumah pengguna untuk penyimpanan surat.
