Utilisateurs et groupes LDAP
Si votre système est configuré pour utiliser un serveur LDAP pour stocker des utilisateurs et des groupes Unix, ce module peut être utilisé pour gérer les utilisateurs dans une base de données LDAP. LDAP est souvent utilisé comme alternative à NIS pour synchroniser les informations des utilisateurs sur plusieurs systèmes d'un réseau, car une base de données peut stocker des attributs supplémentaires pour chaque utilisateur en plus du nom d'utilisateur Unix standard, du shell, etc.

Naturellement, un serveur LDAP doit être exécuté sur votre système ou sur un hôte de votre réseau pour que ce module soit utilisé. De plus, un ou la plupart des hôtes doivent être configurés pour utiliser LDAP pour l'authentification des utilisateurs et / ou des groupes. Aucune de ces tâches ne peut être effectuée à partir du module - elles doivent être effectuées manuellement ou avec d'autres modules Webmin à cet effet.

Si vous avez également configuré Samba pour utiliser un serveur LDAP, ce module peut également créer et gérer les attributs de mot de passe nécessaires pour que chaque utilisateur LDAP puisse également se connecter au serveur Samba. Pour que cette fonctionnalité fonctionne, votre serveur LDAP doit être configuré pour prendre en charge les attributs Samba supplémentaires et la classe d'objets pour chaque utilisateur de son schéma. Uniquement lorsque la connexion Samba? est définie sur Oui , les attributs Samba seront-ils créés.

Le module peut également créer des utilisateurs sur un serveur Cyrus IMAP, s'il est configuré pour s'authentifier auprès de la même base de données LDAP. Il créera des boîtes aux lettres pour chaque nouvel utilisateur, abonnera l'utilisateur à ses boîtes aux lettres et leur accordera un accès administrateur. Si un utilisateur LDAP est supprimé, sa boîte aux lettres IMAP le sera également. Actuellement, cette fonctionnalité ne fonctionne qu'avec le serveur Cyrus IMAP - elle n'a pas été testée et n'est probablement même pas nécessaire pour les autres serveurs, qui utilisent les répertoires personnels des utilisateurs pour le stockage du courrier.
