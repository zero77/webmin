ผู้ใช้และกลุ่ม LDAP
หากระบบของคุณถูกกำหนดค่าให้ใช้เซิร์ฟเวอร์ LDAP สำหรับการจัดเก็บผู้ใช้และกลุ่ม Unix โมดูลนี้สามารถใช้เพื่อจัดการผู้ใช้ในฐานข้อมูล LDAP LDAP มักใช้เป็นทางเลือกสำหรับ NIS สำหรับการซิงโครไนซ์ข้อมูลผู้ใช้ข้ามหลายระบบในเครือข่ายเนื่องจากฐานข้อมูลสามารถจัดเก็บแอตทริบิวต์เพิ่มเติมสำหรับผู้ใช้แต่ละคนนอกเหนือจากชื่อผู้ใช้ Unix มาตรฐานเชลล์และอื่น ๆ

โดยปกติแล้วเซิร์ฟเวอร์ LDAP จะต้องทำงานในระบบของคุณหรือในบางโฮสต์ในเครือข่ายของคุณเพื่อให้โมดูลนี้ใช้ นอกจากนี้หนึ่งหรือส่วนใหญ่โฮสต์จะต้องกำหนดค่าให้ใช้ LDAP สำหรับการตรวจสอบผู้ใช้และ / หรือกลุ่ม ไม่สามารถทำภารกิจเหล่านี้ได้จากภายในโมดูล - ต้องทำด้วยตนเองหรือกับโมดูล Webmin อื่น ๆ เพื่อจุดประสงค์

หากคุณได้กำหนดค่า Samba ให้ใช้เซิร์ฟเวอร์ LDAP ด้วยโมดูลนี้ยังสามารถสร้างและจัดการแอตทริบิวต์รหัสผ่านที่จำเป็นสำหรับผู้ใช้ LDAP แต่ละคนเพื่อให้สามารถเข้าสู่เซิร์ฟเวอร์ Samba ได้เช่นกัน เพื่อให้ฟีเจอร์นี้ใช้งานได้เซิร์ฟเวอร์ LDAP ของคุณต้องได้รับการกำหนดค่าเพื่อรองรับแอตทริบิวต์ Samba เพิ่มเติมและคลาสอ็อบเจ็กต์สำหรับผู้ใช้แต่ละรายในสคีมา เมื่อ เข้าสู่ระบบ Samba เท่านั้น ? ตัวเลือกถูกตั้งค่าเป็น ใช่ คุณลักษณะของ Samba จะถูกสร้างขึ้น

โมดูลยังสามารถสร้างผู้ใช้บนเซิร์ฟเวอร์ Cyrus IMAP หากมีการตั้งค่าให้ตรวจสอบสิทธิ์กับฐานข้อมูล LDAP เดียวกัน มันจะสร้างกล่องจดหมายสำหรับผู้ใช้ใหม่แต่ละคนสมัครสมาชิกผู้ใช้ไปยังกล่องจดหมายของเขาและให้สิทธิ์การเข้าถึงแก่ผู้ดูแลระบบ หากผู้ใช้ LDAP ถูกลบกล่องจดหมาย IMAP ของเขาก็จะเป็นเช่นกัน ปัจจุบันคุณลักษณะนี้ใช้ได้กับเซิร์ฟเวอร์ Cyrus IMAP เท่านั้น - ยังไม่ได้ทดสอบและอาจไม่จำเป็นสำหรับเซิร์ฟเวอร์อื่น ๆ ซึ่งใช้โฮมไดเรกทอรีของผู้ใช้สำหรับการจัดเก็บอีเมล
