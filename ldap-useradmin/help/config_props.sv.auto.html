LDAP-egenskaper för alla nya användare
Det här alternativet kan användas för att specificera egenskaper som kommer att tilldelas LDAP-användare när de skapas, utöver de som skapas av Webmin. De måste anges i följande format:

namn1: värde1
namn2: värde2

Följande ersättningar kan användas som ersätts när användaren skapas:

$ {USER} Användarens inloggningsnamn
$ {PLAINPASS} Användarens okrypterade lösenord, om ett sådant gavs
$ {REAL} Användarens verkliga namn
$ {UID} Användarens Unix UID-nummer
$ {GID} Användarens Unix-grupp-ID-nummer
$ {HOME} Användarens hemkatalog
$ {SHELL} Användarens inloggningsskal
