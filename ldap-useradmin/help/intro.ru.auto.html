Пользователи и группы LDAP
Если ваша система настроена на использование сервера LDAP для хранения пользователей и групп Unix, этот модуль можно использовать для управления пользователями в базе данных LDAP. LDAP часто используется в качестве альтернативы NIS для синхронизации информации о пользователях между несколькими системами в сети, поскольку база данных может хранить дополнительные атрибуты для каждого пользователя в дополнение к стандартному имени пользователя Unix, оболочке и так далее.

Естественно, сервер LDAP должен быть запущен в вашей системе или на каком-либо хосте в вашей сети, чтобы использовать этот модуль. Кроме того, один или большинство хостов должны быть настроены на использование LDAP для аутентификации пользователя и / или группы. Ни одна из этих задач не может быть выполнена из модуля - они должны быть выполнены вручную или с другими модулями Webmin для этой цели.

Если вы настроили Samba на использование сервера LDAP, этот модуль также может создавать необходимые атрибуты пароля и управлять ими для каждого пользователя LDAP, чтобы иметь возможность также входить на сервер Samba. Чтобы эта функция работала, ваш сервер LDAP должен быть настроен на поддержку дополнительных атрибутов Samba и класса объектов для каждого пользователя в его схеме. Только когда самба логинится? для параметра установлено значение Да, будут ли созданы атрибуты Samba.

Модуль также может создавать пользователей на сервере Cyrus IMAP, если он настроен для аутентификации в той же базе данных LDAP. Он создаст почтовые ящики для каждого нового пользователя, подпишет пользователя на его почтовые ящики и предоставит им доступ администратора. Если пользователь LDAP удален, его почтовый ящик IMAP также будет. В настоящее время эта функция работает только с сервером Cyrus IMAP - она не была протестирована и, вероятно, даже не нужна для других серверов, которые используют домашние каталоги пользователей для хранения почты.
