Utilizatori și grupuri LDAP
Dacă sistemul dvs. este configurat pentru a utiliza un server LDAP pentru stocarea utilizatorilor și grupurilor Unix, acest modul poate fi utilizat pentru a gestiona utilizatorii într-o bază de date LDAP. LDAP este adesea utilizat ca o alternativă la NIS pentru sincronizarea informațiilor utilizatorilor pe mai multe sisteme dintr-o rețea, deoarece o bază de date poate stoca atribute suplimentare pentru fiecare utilizator în plus față de numele de utilizator standard, shell și așa mai departe.

Desigur, un server LDAP trebuie să fie executat pe sistemul dvs. sau pe o anumită gazdă din rețeaua dvs. pentru ca acest modul să fie utilizat. În plus, una sau majoritatea gazdelor trebuie configurate pentru a utiliza LDAP pentru autentificarea utilizatorului și / sau a grupului. Niciuna dintre aceste sarcini nu se poate face din interiorul modulului - trebuie să fie efectuate manual sau cu alte module Webmin în acest scop.

Dacă aveți Samba configurat să utilizeze și un server LDAP, acest modul poate crea și gestiona atributele de parolă necesare pentru fiecare utilizator LDAP pentru a se putea conecta la serverul Samba. Pentru ca această funcție să funcționeze, serverul dvs. LDAP trebuie să fie configurat pentru a accepta atributele Samba suplimentare și clasa de obiecte pentru fiecare utilizator din schema sa. Doar atunci când se conectează Samba? opțiunea este setată pe Da, vor fi create atributele Samba.

Modulul poate crea de asemenea utilizatori pe un server Cyrus IMAP, dacă este configurat să se autentifice cu aceeași bază de date LDAP. Va crea căsuțe poștale pentru fiecare utilizator nou, va abona utilizatorul la căsuțele sale poștale și va acorda acces la administrator. Dacă un utilizator LDAP este șters, cutia poștală IMAP va fi la fel de bine. În prezent, această caracteristică funcționează numai cu serverul Cyrus IMAP - nu a fost testată și probabil nu este chiar necesară pentru alte servere, care folosesc directoarele de acasă ale utilizatorilor pentru stocarea poștelor.
