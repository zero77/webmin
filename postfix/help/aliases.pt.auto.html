Aliases de Correio
Os aliases fornecem um mecanismo em todo o sistema para redirecionar emails para destinatários locais.

O campo Alias to... contém um ou mais dos seguintes, separados por vírgula:

  • address : o correio é encaminhado para o address , compatível com o padrão RFC 822.
  • /file/name : o email é anexado a /file/name . Consulte local (8) para obter detalhes da entrega em arquivo. A entrega não se limita a arquivos regulares. Por exemplo, para descartar mensagens indesejadas, desvia-as para /dev/null .
  • |command : o correio é canalizado para o command . Os comandos que contêm caracteres especiais, como espaço em branco, devem ser colocados entre aspas duplas. Consulte local (8) para obter detalhes da entrega ao comando. Quando o comando falha, uma quantidade limitada de saída do comando é enviada de volta ao remetente.
  • :include:/file/name : o correio é enviado para os destinos listados no arquivo nomeado. Linhas em :include: arquivos têm a mesma sintaxe que o lado direito das entradas de alias.
