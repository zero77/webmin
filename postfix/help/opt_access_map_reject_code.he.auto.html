תגובת שרת SMTP בגין הפרת מפות גישה
access_map_reject_code
פרמטר זה מציין את קוד התגובה של שרת SMTP כאשר דואר מפר את המגבלה של access map שהיא:
הגישה נדחתה: הפרה של Access_map (בלקוח SMTP או בשם המארח HELO, השולח או כתובת הדוא"ל של הנמען)

שים לב שערך ברירת המחדל לאחר access_map_reject_code המשמש ל access_map_reject_code הוא קוד 554, וקוד זה ב- 'RFC 822 סטנדרטי' אמור להיות 'פעולה מבוקשת נדחתה: גישה נדחתה'. אז עם Postfix, הפרת כללי access map לא משקפת בדיוק את ההסבר RFC822 של הקוד המדווח.

כדי למנוע בלבול, באפשרותך לשנות זאת ל -570 לדוגמא (סיבה נוספת היא שקוד 554 הוא גם ערך ברירת המחדל של התיקון לאחר תקלות מסוג אחר), אחרת אל תשנה זאת אלא אם כן יש לך הבנה מלאה של RFC 822.
