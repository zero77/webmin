정식 매핑
선택적인 표준 파일은 로컬 및 로컬이 아닌 주소에 대한 주소 매핑을 지정합니다. cleanup (8) 데몬이 매핑을 사용합니다. 주소 매핑은 재귀 적입니다.

파일은 postmap (1) 명령에 대한 입력으로 사용됩니다. dbm 또는 db 형식의 색인 파일 인 결과는 메일 시스템에서 빠른 검색에 사용됩니다.

정식 매핑은 메시지 헤더 주소 (예 : 메시지 안에 나타나는 주소)와 메시지 봉투 주소 (예 : SMTP 프로토콜 명령에 사용되는 주소) 모두에 영향을줍니다. 원하는 경우 Sendmail 규칙 세트 S3 생각하십시오.

일반적으로 표준 테이블을 사용하여 로그인 이름을 Firstname.Lastname 으로 바꾸거나 레거시 메일 시스템에서 생성 한 주소를 정리합니다.

표준 매핑은 가상 도메인 지원과 혼동되어서는 안됩니다. 이를 위해 가상 도메인 맵을 사용하십시오.

표준 매핑은 로컬 앨리어싱과 혼동해서는 안됩니다. 이를 위해 메일 별명 맵을 사용하십시오.

정식 매핑의 형식은 다음과 같으며 매핑은 나열된 순서대로 시도됩니다.

  • user@domain 매핑 된 address :
    user@domain 이 address 로 바뀝니다.이 양식이 가장 우선합니다. 이 양식은 레거시 메일 시스템에서 생성 된 주소를 정리하는 데 유용합니다. Firstname.Lastname 스타일 주소를 생성하는 데에도 사용할 수 있지만 더 간단한 솔루션은 아래를 참조하십시오.
  • user address 매핑 됨 :
    user@site 는 site가 $myorigin 과 같 $myorigin site가 $mydestination 나열되어 있거나 $inet_interfaces 나열되어 있으면 주소로 대체됩니다. 이 양식은 로그인 이름을 Firstname.Lastname 바꾸는 데 유용합니다.
  • @domain address 매핑 됨 :
    도메인의 모든 주소는 주소로 바뀝니다. 이 양식은 우선 순위가 가장 낮습니다.
위의 모든 양식에서 address가 @otherdomain 양식 인 경우 결과는 otherdomain의 동일한 사용자입니다.

주소 확장 : 테이블 조회가 실패하고 localpart 주소에 선택적 수신자 구분 기호 (예 : user+foo@domain )가 포함 된 경우 확장되지 않은 주소 (예 : user@domain )에 대한 검색이 반복되고 일치하지 않는 확장이 테이블 조회 결과. 일치하는 순서는 user+foo@domain , user@domain , user+foo , user 및 @domain 입니다.
