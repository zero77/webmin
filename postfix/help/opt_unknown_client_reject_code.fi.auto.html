SMTP-palvelimen vastaus tuntemattoman asiakkaan hylkäämiseen
unknown_client_reject_code
Tämä parametri määrittelee SMTP-palvelimen vastauskoodin, kun asiakas, jolla ei ole osoitetunnistetta osoitetta, rikkoo reject_unknown_clients rajoitussääntöä, joka on:
Käyttö evätty: Tuntematon SMTP-asiakaskoneen isäntänimi (ilman DNS A- tai MX-tietuetta)

Huomaa, että postfix oletusarvo käytetty unknown_client_reject_code on koodin 450 käytetään myös muita virheitä. Tämä koodi 'standardissa RFC 822' tarkoittaa 'pyydettyä postitoimintaa ei suoritettu: postilaatikko on varattu tai käyttö evätty' ja pyydä lähettäjää yrittämään myöhemmin uudelleen (toivoen, että DNS vastaa myöhemmin oikein).

Älä muuta tätä, ellei sinulla ole täysin ymmärrystä RFC 822: sta.
