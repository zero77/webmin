Antwort des SMTP-Servers auf Ablehnung des Hostnamens
invalid_hostname_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn ein Client gegen die Anti-UCE-Einschränkungsregel " reject_invalid_hostname verstößt.
Schlechte Syntax für Client-HELO-Hostnamen (nicht RFC-kompatibel)

Beachten Sie, dass der für invalid_hostname_reject_code verwendete Postfix-Standardwert Code 501 ist und dieser Code in 'Standard-RFC 822' 'Syntaxfehler in Parametern oder Argumenten' bedeuten sollte. Bei Postfix spiegelt ein 501-Statuscode nicht genau die RFC822-Erklärung wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 574 ändern (ein weiterer Grund ist, dass Code 501 auch der Postfix-Standardwert für andere Arten von Fehlern ist). Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
