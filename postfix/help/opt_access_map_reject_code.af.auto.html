SMTP-bediener se reaksie op oortreding van toegangskaart
access_map_reject_code
Hierdie parameter spesifiseer die SMTP-bediener se antwoordkode wanneer 'n e-pos die access map oortree, naamlik:
Toegang geweier: toegang_kaartoortreding (op SMTP-kliënt of HELO-gasheernaam, sender of e-posadres van die ontvanger)

Let daarop dat die standaardwaarde van die postfix wat gebruik word vir access_map_reject_code kode 554 is, en hierdie kode in 'standaard RFC 822' moet beteken 'Gevraagde access_map_reject_code verwerp: toegang geweier'. Dus met Postfix weerspieël 'n oortreding van die access map nie presies die RFC822-verduideliking van gerapporteerde kode nie.

Om verwarring te voorkom, kan u dit byvoorbeeld na 570 verander ('n ander rede is dat kode 554 ook die standaardwaarde van die postfix is vir ander soort foute), anders moet u dit nie verander nie, tensy u 'n volledige begrip van RFC 822 het.
