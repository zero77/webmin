Адказ SMTP-сервера пры адхіленні кліента
reject_code
Гэты параметр указвае код адказу SMTP-сервера, калі кліент парушае правіла smtpd_client_restrictions :
Доступ адхілены: імя хаста SMTP кліента адхілена

Звярніце ўвагу, што значэнне па змаўчанні Postfix, якое выкарыстоўваецца для reject_code з'яўляецца кодам 554, і гэты код у "стандартным RFC 822" павінен азначаць "Запытаная дзеянне пошты адхілена: доступ забаронены". Такім чынам, у Postfix, парушэнне правілаў smtpd_client_restrictions не адлюстроўвае дакладна тлумачэнне smtpd_client_restrictions паведамляемага кода.

Каб пазбегнуць блытаніны, вы можаце змяніць гэта на 575, напрыклад (іншая прычына заключаецца ў тым, што код 554 таксама з'яўляецца значэннем па змаўчанні постфікса для іншых відаў памылак), інакш не мяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
