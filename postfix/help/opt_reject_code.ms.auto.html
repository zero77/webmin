Respons pelayan SMTP pada penolakan pelanggan
reject_code
Parameter ini menentukan kod respons pelayan SMTP ketika klien melanggar peraturan smtpd_client_restrictions yang:
Akses ditolak: nama hos pelanggan SMTP ditolak

Perhatikan bahawa nilai lalai postfix yang digunakan untuk reject_code adalah kod 554, dan kod ini dalam 'standard RFC 822' harus bermaksud 'Tindakan mel diminta ditolak: akses ditolak'. Oleh itu, dengan Postfix, pelanggaran peraturan smtpd_client_restrictions tidak menggambarkan dengan tepat penjelasan RFC822 mengenai kod yang dilaporkan.

Untuk mengelakkan kekeliruan, anda boleh mengubahnya menjadi 575 misalnya (sebab lain adalah bahawa kod 554 juga merupakan nilai lalai postfix untuk jenis kesalahan lain), jika tidak, jangan ubah ini melainkan anda mempunyai pemahaman lengkap tentang RFC 822.
