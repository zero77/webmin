Réponse du serveur SMTP en cas de violation du fqdn
non_fqdn_reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un courrier viole la reject_non_fqdn_hostname ou reject_non_fqdn_sender ou reject_non_fqdn_recipient qui sont:
Accès refusé: domaine non valide pour le nom d'hôte HELO du client, l'adresse e-mail de l'expéditeur ou du destinataire (pas le FQDN)

Notez que la valeur par défaut de postfix utilisée pour non_fqdn_reject_code est le code 504, et ce code dans la «norme RFC 822» devrait signifier «paramètre de commande non implémenté». Ainsi, avec Postfix, une violation de la règle non_fqdn_reject_code ne reflète pas exactement l'explication RFC822 du code signalé.

Pour éviter toute confusion, vous pouvez changer cela en 578 par exemple, sinon, ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
