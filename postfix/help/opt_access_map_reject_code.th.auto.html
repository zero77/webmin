การตอบสนองของเซิร์ฟเวอร์ SMTP เกี่ยวกับการละเมิดแผนที่การเข้าถึง
access_map_reject_code
พารามิเตอร์นี้ระบุรหัสการตอบกลับของเซิร์ฟเวอร์ SMTP เมื่อเมลละเมิดข้อ จำกัด การ access map ที่:
ปฏิเสธการเข้าถึง: การละเมิด access_map (บนไคลเอ็นต์ SMTP หรือชื่อโฮสต์ HELO, ผู้ส่งหรือที่อยู่อีเมลของผู้รับ)

โปรดทราบว่าค่าเริ่มต้น postfix ที่ใช้สำหรับ access_map_reject_code คือรหัส 554 และรหัสนี้ใน 'มาตรฐาน RFC 822' ควรหมายถึง 'การกระทำเมลที่ร้องขอถูกปฏิเสธ: การเข้าถึงถูกปฏิเสธ' ดังนั้นด้วย Postfix access map ละเมิดกฎการ access map ไม่ได้สะท้อนถึงคำอธิบาย RFC822 ของรหัสที่รายงาน

เพื่อหลีกเลี่ยงความสับสนคุณสามารถเปลี่ยนเป็น 570 ได้เช่นกัน (อีกเหตุผลหนึ่งคือรหัส 554 เป็นค่าเริ่มต้น postfix สำหรับข้อผิดพลาดประเภทอื่น ๆ ) มิฉะนั้นอย่าเปลี่ยนสิ่งนี้เว้นแต่คุณจะเข้าใจ RFC 822 เป็นอย่างดี
