fqdn 위반에 대한 SMTP 서버 응답
non_fqdn_reject_code
이 매개 변수는 메일이 reject_non_fqdn_hostname 과 같은 reject_non_fqdn_sender reject_non_fqdn_hostname 또는 reject_non_fqdn_sender 또는 reject_non_fqdn_recipient 규칙을 위반할 때 SMTP 서버 응답 코드를 지정합니다.
액세스 거부 : 클라이언트 HELO 호스트 이름, 발신자 또는 수신자 전자 메일 주소에 대한 잘못된 도메인 (FQDN 아님)

non_fqdn_reject_code 사용되는 접미사 기본값은 코드 504이며 'standard RFC 822'의이 코드는 'Command parameter not implementation'을 의미해야합니다. 따라서 Postfix에서 non_fqdn_reject_code 규칙 위반은보고 된 코드의 RFC822 설명을 정확하게 반영하지 않습니다.

혼동을 피하기 위해 예를 들어 578로 변경할 수 있습니다. 그렇지 않으면 RFC 822를 완전히 이해하지 않는 한이를 변경하지 마십시오.
