Ответ SMTP-сервера на запрещенную ретрансляцию
relay_domains_reject_code
Этот параметр указывает код ответа сервера SMTP, когда письмо нарушает правило relay_domains_reject , а именно:
Доступ запрещен: ретрансляция не авторизована или локальный хост не является шлюзом

Обратите внимание, что значением по умолчанию для постфикса, используемым для relay_domains_reject_code является код 554, и этот код в «стандарте RFC 822» должен означать «Запрошенное действие почты отклонено: доступ запрещен». Таким образом, с Postfix нарушение правила relay_domains_reject не отражает в точности объяснение RFC822 сообщаемого кода.

Чтобы избежать путаницы, вы можете изменить это значение, например, на 572 (другая причина в том, что код 554 также является значением по умолчанию для постфикса для других типов ошибок), в противном случае не изменяйте это, если у вас нет полного понимания RFC 822.
