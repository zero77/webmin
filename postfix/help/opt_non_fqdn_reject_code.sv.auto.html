SMTP-server svar på fqdn överträdelse
non_fqdn_reject_code
Den här parametern anger SMTP-serverns svarskod när ett e-postmeddelande bryter mot reject_non_fqdn_hostname eller reject_non_fqdn_sender eller reject_non_fqdn_recipient som är:
Åtkomst nekad: Ogiltig domän för HELO-värdnamn, avsändare eller mottagarens e-postadress (inte FQDN)

Observera att standardvärdet för postfix som används för non_fqdn_reject_code är kod 504, och denna kod i 'standard RFC 822' bör betyda 'Kommandoparameter inte implementerad'. Så med Postfix non_fqdn_reject_code en non_fqdn_reject_code regeln som inte är förknippad med en_fqdn_reject_code inte exakt RFC822-förklaringen om rapporterad kod.

För att undvika förvirring kan du ändra detta till 578 till exempel, annars ändra inte detta om du inte har en fullständig förståelse för RFC 822.
