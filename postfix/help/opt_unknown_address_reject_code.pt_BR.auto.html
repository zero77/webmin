Resposta do servidor SMTP na rejeição de domínio
unknown_address_reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um cliente viola os reject_unknown_sender_domain ou reject_unknown_recipient_domain regras de restrição que são:
Acesso negado: domínio desconhecido para o endereço de email do remetente ou do destinatário (sem registro A ou MX do DNS)

Observe que o valor padrão do postfix usado para unknown_address_reject_code é o código 450 também usado para outros erros. Esse código em 'RFC 822 padrão' significa 'Ação de email solicitada não executada: caixa de correio ocupada ou acesso negado' e solicita ao remetente que tente novamente mais tarde (esperando que o DNS responda corretamente mais tarde).

Não altere isso, a menos que você tenha um entendimento completo da RFC 822.
