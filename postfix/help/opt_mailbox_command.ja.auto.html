メールボックス配信の代わりに使用する外部コマンド
mailbox_command
このパラメーターは、メールボックス配信の代わりに使用するオプションの外部コマンドを指定します。コマンドは、適切なHOME 、 SHELL 、およびLOGNAME環境設定で受信者として実行されます。例外： root配信は$default_userとして行われます。

関心のある他の環境変数： USER （受信者のユーザー名）、 EXTENSION （アドレス拡張子）、 DOMAIN （アドレスのドメイン部分）、およびLOCAL （アドレスlocalpart）。

他のPostfix構成パラメーターとは異なり、このパラメーターはパラメーター置換の対象ではありません。これは、シェル構文を簡単に指定できるようにするためです（下の例を参照）。

Postfixに高価なシェルプロセスを実行させるため、シェルメタ文字は避けてください。 Procmailだけで十分です。

システム全体でメールを配信するためにこれを使用する場合、実際のユーザーにルートのメールを転送するエイリアスを設定する必要があります。

例：

  • /some/where/procmail
  • /some/where/procmail -a "$EXTENSION"
