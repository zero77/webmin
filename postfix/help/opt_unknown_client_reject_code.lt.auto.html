SMTP serverio atsakymas dėl nežinomo kliento atmetimo
unknown_client_reject_code
Šis parametras nurodo SMTP serverio atsakymo kodą, kai klientas, neturintis adreso reject_unknown_clients žemėlapio, pažeidžia apribojimo taisyklę „ reject_unknown_clients kuri yra:
Prieiga atmesta: Nežinomas SMTP kliento pagrindinis vardas (be DNS A arba MX įrašo)

Atminkite, kad „postfix“ numatytoji reikšmė, naudojama unknown_client_reject_code yra kodas 450, taip pat naudojama kitoms klaidoms. Šis kodas „standartiniame RFC 822“ reiškia „neatliktas prašomas pašto veiksmas: pašto dėžutė užimta arba prieiga atimta“ ir paprašykite siuntėjo vėliau bandyti dar kartą (tikėdamasis, kad DNS vėliau teisingai atsakys).

Nekeiskite to, nebent jūs visiškai suprantate apie RFC 822.
