การตอบสนองของเซิร์ฟเวอร์ SMTP เกี่ยวกับการปฏิเสธลูกค้าที่ไม่รู้จัก
unknown_client_reject_code
พารามิเตอร์นี้ระบุรหัสการตอบสนองของเซิร์ฟเวอร์ SMTP เมื่อไคลเอ็นต์ที่ไม่มีการแมปแมปชื่อละเมิดกฎข้อ จำกัด reject_unknown_clients ที่:
ปฏิเสธการเข้าถึง: ชื่อโฮสต์ไคลเอ็นต์ SMTP ที่ไม่รู้จัก (ไม่มีระเบียน DNS A หรือ MX)

โปรดทราบว่าค่าเริ่มต้น postfix ที่ใช้สำหรับ unknown_client_reject_code คือรหัส 450 ที่ใช้สำหรับข้อผิดพลาดอื่น ๆ รหัสนี้ใน 'มาตรฐาน RFC 822' หมายถึง 'ไม่ได้ทำการร้องขอจดหมาย: กล่องจดหมายไม่ว่างหรือปฏิเสธการเข้าถึง' และขอให้ผู้ส่งลองอีกครั้งในภายหลัง (หวังว่า DNS จะตอบได้อย่างถูกต้องในภายหลัง)

อย่าเปลี่ยนแปลงสิ่งนี้หากคุณไม่เข้าใจ RFC 822 โดยสมบูรณ์
