Réponse du serveur SMTP au rejet du nom d'hôte
invalid_hostname_reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un client enfreint la règle de restriction anti-UCE reject_invalid_hostname qui est:
Syntaxe incorrecte pour le nom d'hôte HELO du client (non compatible RFC)

Notez que la valeur par défaut de suffixe utilisée pour invalid_hostname_reject_code est le code 501, et ce code dans la «norme RFC 822» devrait signifier «erreur de syntaxe dans les paramètres ou les arguments». Ainsi, avec Postfix, un code d'état 501 ne reflète pas exactement l'explication RFC822.

Pour éviter toute confusion, vous pouvez changer cela en 574 par exemple (une autre raison est que le code 501 est également la valeur par défaut de postfix pour d'autres types d'erreurs), sinon, ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
