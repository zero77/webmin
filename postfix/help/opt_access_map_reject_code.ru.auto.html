Ответ SMTP-сервера на нарушение карты доступа
access_map_reject_code
Этот параметр указывает код ответа сервера SMTP, когда письмо нарушает ограничение access map , а именно:
Доступ запрещен: нарушение access_map (для клиента SMTP или имени хоста HELO, адреса электронной почты отправителя или получателя)

Обратите внимание, что значением постфикса по умолчанию, используемым для access_map_reject_code является код 554, и этот код в «стандарте RFC 822» должен означать «Запрошенное действие почты отклонено: доступ запрещен». Таким образом, с Postfix нарушение правила access map не отражает в точности объяснение RFC822 сообщаемого кода.

Чтобы избежать путаницы, вы можете изменить это значение, например, на 570 (другая причина в том, что код 554 также является значением по умолчанию для постфикса для других типов ошибок), в противном случае не изменяйте это, если у вас нет полного понимания RFC 822.
