Внешняя команда для использования вместо доставки в почтовый ящик
mailbox_command
Этот параметр указывает необязательную внешнюю команду для использования вместо доставки в почтовый ящик. Команда запускается как получатель с правильными настройками среды HOME , SHELL и LOGNAME . Исключение: доставка для root выполняется как $default_user .

Другие переменные среды, представляющие интерес: USER (имя пользователя получателя), EXTENSION (расширение адреса), DOMAIN (доменная часть адреса) и LOCAL (адрес localpart).

В отличие от других параметров конфигурации Postfix, этот параметр не подлежит замене параметров. Это облегчает указание синтаксиса оболочки (см. Пример ниже).

Избегайте метасимволов оболочки, потому что они заставят Postfix запускать дорогостоящий процесс оболочки. Procmail только Procmail достаточно дорога.

ЕСЛИ ВЫ ИСПОЛЬЗУЕТЕ ЭТО, ЧТОБЫ ПОСТАВИТЬ ПОЧТУ В СИСТЕМУ, ВЫ ДОЛЖНЫ УСТАНОВИТЬ АЛИАС, КОТОРЫЙ ПЕРЕДАВАЕТ ПОЧТУ ДЛЯ КОРНЯ НАСТОЯЩЕГО ПОЛЬЗОВАТЕЛЯ.

Примеры:

  • /some/where/procmail
  • /some/where/procmail -a "$EXTENSION"
