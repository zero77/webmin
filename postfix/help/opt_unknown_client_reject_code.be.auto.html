Адказ SMTP-сервера на невядомым кліенце адхіляе
unknown_client_reject_code
Гэты параметр указвае код адказу SMTP-сервера, калі кліент без адлюстравання reject_unknown_clients парушае reject_unknown_clients абмежавання reject_unknown_clients :
Доступ забаронены: Невядомы SMTP кліент кліента (без запісу DNS A або MX)

Звярніце ўвагу, што значэнне па змаўчанні Postfix, якое выкарыстоўваецца для unknown_client_reject_code з'яўляецца кодам 450, які выкарыстоўваецца і для іншых памылак. Гэты код у "стандартным RFC 822" азначае "Запытаныя дзеянні пошты не прадпрымаюцца: паштовая скрыня занятая альбо доступ забаронены" і папрасіць адпраўніка паспрабаваць яшчэ раз пазней (спадзеючыся, што DNS адкажа правільна пазней).

Не мяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
