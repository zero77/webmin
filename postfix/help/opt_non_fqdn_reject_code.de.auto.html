SMTP-Serverantwort bei fqdn-Verletzung
non_fqdn_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn eine E-Mail gegen die reject_non_fqdn_hostname oder " reject_non_fqdn_sender oder " reject_non_fqdn_recipient verstößt.
Zugriff verweigert: Ungültige Domain für Client-HELO-Hostname, Absender- oder Empfänger-E-Mail-Adresse (nicht FQDN)

Beachten Sie, dass der für non_fqdn_reject_code verwendete Postfix-Standardwert Code 504 ist und dieser Code in 'Standard-RFC 822' 'Befehlsparameter nicht implementiert' bedeuten sollte. Bei Postfix non_fqdn_reject_code eine non_fqdn_reject_code nicht genau die RFC822-Erklärung des gemeldeten Codes wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 578 ändern. Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
