Транспортне картографування
Необов'язковий транспортний файл визначає відображення від ієрархій домену до транспорту доставки повідомлень та / або ретрансляторів хостів. Для відображення використовується тривіально переписаний (8) демон.

Формат транспортної таблиці такий:

  • domain відображений на .. transport:nexthop : Пошта для домену доставляється через транспорт до nexthop.
  • .domain відображений на .. transport:nexthop : Пошта для будь-якого піддомену домену доставляється через транспорт до nexthop.
Примітка: записи транспортної карти мають перевагу над доменами, визначеними параметром mydestination . Якщо ви використовуєте додаткову транспортну карту, можливо, безпечніше буде вказати явні записи для всіх доменів, визначених у мідестинації, наприклад:
  • hostname.my.domain ... local:
  • localhost.my.domain ... local:
Інтерпретація поля nexthop залежить від транспорту. У випадку SMTP вкажіть host:service для порту сервера, що не використовується за замовчуванням, і використовуйте [host] або [host:port] , щоб відключити пошук DX MX (обмінник пошти). Форма [] також може використовуватися з IP-адресами замість імен хостів.
