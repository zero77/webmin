SMTP服務器對訪問映射衝突的響應
access_map_reject_code
當郵件違反access map限制時，此參數指定SMTP服務器響應代碼：
拒絕訪問：違反access_map（在SMTP客戶端或HELO主機名，發件人或收件人電子郵件地址上）

請注意，用於access_map_reject_code的後綴默認值為代碼554，“標準RFC 822”中的此代碼應表示“請求的郵件操作被拒絕：訪問被拒絕”。因此，對於Postfix，違反access map規則的行為無法完全反映所報告代碼的RFC822說明。

為避免混淆，您可以將其更改為例如570（另一個原因是代碼554也是其他類型錯誤的後綴默認值），否則，除非您完全了解RFC 822，否則請勿更改此值。
