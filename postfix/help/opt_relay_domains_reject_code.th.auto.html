การตอบสนองของเซิร์ฟเวอร์ SMTP เกี่ยวกับการส่งต่อสิ่งต้องห้าม
relay_domains_reject_code
พารามิเตอร์นี้ระบุรหัสการตอบกลับของเซิร์ฟเวอร์ SMTP เมื่อเมลละเมิดกฎ relay_domains_reject นั่นคือ:
ปฏิเสธการเข้าถึง: รีเลย์ไม่ได้รับอนุญาตหรือไม่ใช่โฮสต์ท้องถิ่นไม่ใช่เกตเวย์

โปรดทราบว่าค่าเริ่มต้นของ postfix ที่ใช้สำหรับ relay_domains_reject_code คือรหัส 554 และรหัสนี้ใน 'มาตรฐาน RFC 822' ควรหมายถึง 'ปฏิเสธการกระทำจดหมายที่ร้องขอ: ปฏิเสธการเข้าถึง' ดังนั้นด้วย Postfix การละเมิดกฎ relay_domains_reject ไม่ได้สะท้อนถึงคำอธิบาย RFC822 ของรหัสที่รายงาน

เพื่อหลีกเลี่ยงความสับสนคุณสามารถเปลี่ยนเป็น 572 เช่นกัน (อีกเหตุผลหนึ่งคือรหัส 554 เป็นค่าเริ่มต้น postfix สำหรับข้อผิดพลาดประเภทอื่น ๆ ) มิฉะนั้นอย่าเปลี่ยนสิ่งนี้เว้นแต่คุณจะเข้าใจ RFC 822 เป็นอย่างดี
