การตอบสนองของเซิร์ฟเวอร์ SMTP เกี่ยวกับการปฏิเสธชื่อโฮสต์ที่ไม่รู้จัก
unknown_hostname_reject_code
พารามิเตอร์นี้ระบุรหัสการตอบกลับของเซิร์ฟเวอร์ SMTP เมื่อไคลเอนต์ละเมิดกฎข้อ จำกัด reject_unknown_hostname นั่นคือ:
ปฏิเสธการเข้าถึง: ชื่อโฮสต์ของไคลเอ็นต์ HELO ที่ไม่รู้จัก (ไม่มีระเบียน DNS A หรือ MX)

โปรดทราบว่าค่าเริ่มต้น postfix ที่ใช้สำหรับ unknown_hostname_reject_code คือรหัส 501 และรหัสนี้ใน 'มาตรฐาน RFC 822' ควรหมายถึง 'ข้อผิดพลาดทางไวยากรณ์ในพารามิเตอร์หรืออาร์กิวเมนต์' ดังนั้นด้วย Postfix รหัสสถานะ 501 จึงไม่สะท้อนคำอธิบายที่ชัดเจนของ RFC822

เพื่อหลีกเลี่ยงความสับสนคุณสามารถเปลี่ยนเป็น 577 ได้เช่นกัน (อีกเหตุผลหนึ่งคือรหัส 501 เป็นค่าเริ่มต้นของ postfix สำหรับข้อผิดพลาดประเภทอื่น ๆ ) มิฉะนั้นอย่าเปลี่ยนสิ่งนี้เว้นแต่คุณจะเข้าใจ RFC 822 เป็นอย่างดี
