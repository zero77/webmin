Respons pelayan SMTP pada penolakan pelanggan yang tidak diketahui
unknown_client_reject_code
Parameter ini menentukan kod respons pelayan SMTP ketika klien tanpa pemetaan alamat untuk nama melanggar peraturan pembatasan reject_unknown_clients yang:
Akses ditolak: Nama hos pelanggan SMTP tidak diketahui (tanpa rekod DNS A atau MX)

Perhatikan bahawa nilai lalai postfix yang digunakan untuk unknown_client_reject_code adalah code 450 juga digunakan untuk kesalahan lain. Kod ini dalam 'standard RFC 822' bermaksud 'Tindakan mel yang diminta tidak diambil: peti mel sibuk atau akses ditolak' dan minta pengirim untuk mencuba lagi nanti (berharap DNS akan menjawab dengan betul kemudian).

Jangan ubah ini melainkan anda mempunyai pemahaman lengkap mengenai RFC 822.
