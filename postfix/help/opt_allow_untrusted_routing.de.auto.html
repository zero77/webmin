Nicht vertrauenswürdiges Routing zulassen
allow_untrusted_routing
Dieser Parameter steuert, ob Postfix E-Mails mit vom Absender angegebenem Routing (Benutzer [@%!] Remote-Site [@%!]) Von nicht vertrauenswürdigen Clients an Ziele relay_domains , die durch den Parameter relay_domains gesegnet sind.

Standardmäßig dürfen nicht vertrauenswürdige Clients kein Routing angeben. Dies schließt eine unangenehme offene Relay-Lücke, in der ein Backup-MX-Host dazu verleitet werden kann, Junk-Mails an einen primären MX-Host weiterzuleiten, der sie dann an die Welt spammt.
