Shell zur Übermittlung an einen externen Befehl
local_command_shell
Dieser Parameter steuert, welche Shell für die Übermittlung an einen externen Befehl verwendet wird. Standardmäßig werden externe Befehle direkt ausgeführt. Befehle werden an /bin/sh nur vergeben, wenn sie Shell-Metazeichen oder integrierte Shell-Befehle enthalten.

Geben Sie eine andere Shell an, wenn Sie mehr Kontrolle darüber benötigen, welche Programme beispielsweise aus .forward Dateien .forward können.

"Sendmail's Restricted Shell" smrsh wird von den meisten smrsh verwendet ( smrsh ist Teil der jüngsten Sendmail-Distributionen).

Hinweis: Wenn eine Shell angegeben wird, wird sie auch dann aufgerufen, wenn der Befehl keine in die Shell integrierten Befehle oder Metazeichen enthält.

Beispiel: /some/where/smrsh -c
