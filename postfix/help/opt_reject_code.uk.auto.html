Відповідь SMTP-сервера на відхилення клієнта
reject_code
Цей параметр визначає код відповіді сервера SMTP, коли клієнт порушує правило smtpd_client_restrictions яке:
Доступ відхилений: ім'я хоста SMTP клієнта відхилено

Зауважте, що значенням за замовчуванням постфікса, що використовується для коду reject_code є код 554, і цей код у "стандартному RFC 822" повинен означати "Запитувана поштова дія відхилена: доступ заборонено". Отже, з Postfix, порушення правила smtpd_client_restrictions не відображає саме пояснення RFC822 звітного коду.

Щоб уникнути плутанини, ви можете змінити це на 575, наприклад (інша причина полягає в тому, що код 554 є також значенням за замовчуванням постфікса для інших видів помилок), інакше не змінюйте це, якщо ви не маєте повного розуміння RFC 822.
