Адказ SMTP-сервера на невядомае імя хаста адхіліць
unknown_hostname_reject_code
Гэты параметр паказвае код адказу SMTP-сервера, калі кліент парушае reject_unknown_hostname абмежавання reject_unknown_hostname :
Доступ адхілены: Імя хаста невядомага кліента (без запісу DNS A або MX)

Звярніце ўвагу, што значэнне па змаўчанні постфікса, якое выкарыстоўваецца для unknown_hostname_reject_code - гэта код 501, і гэты код у "стандартным RFC 822" павінен азначаць "Памылка сінтаксісу ў параметрах або аргументах". Такім чынам, у Postfix код 501 не дакладна адлюстроўвае тлумачэнне RFC822.

Каб пазбегнуць блытаніны, вы можаце змяніць гэта на 577, напрыклад (іншая прычына заключаецца ў тым, што код 501 таксама з'яўляецца значэннем па змаўчанні постфікса для іншых відаў памылак), у адваротным выпадку не мяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
