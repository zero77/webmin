การตอบสนองของเซิร์ฟเวอร์ SMTP ต่อการปฏิเสธของลูกค้า
reject_code
พารามิเตอร์นี้ระบุรหัสการตอบกลับของเซิร์ฟเวอร์ SMTP เมื่อไคลเอนต์ละเมิดกฎ smtpd_client_restrictions นั่นคือ:
ปฏิเสธการเข้าถึง: ปฏิเสธชื่อโฮสต์ไคลเอ็นต์ SMTP

โปรดทราบว่าค่าเริ่มต้นของ postfix ที่ใช้สำหรับ reject_code คือรหัส 554 และรหัสนี้ใน 'มาตรฐาน RFC 822' ควรหมายถึง 'การกระทำเมลที่ร้องขอถูกปฏิเสธ: การเข้าถึงถูกปฏิเสธ' ดังนั้นด้วย Postfix การละเมิดกฎ smtpd_client_restrictions ไม่ได้สะท้อนถึงคำอธิบาย RFC822 ของรหัสที่รายงาน

เพื่อหลีกเลี่ยงความสับสนคุณสามารถเปลี่ยนเป็น 575 (อีกเหตุผลหนึ่งคือรหัส 554 เป็นค่าเริ่มต้น postfix สำหรับข้อผิดพลาดชนิดอื่น) มิฉะนั้นอย่าเปลี่ยนสิ่งนี้เว้นแต่คุณจะเข้าใจ RFC 822 เป็นอย่างดี
