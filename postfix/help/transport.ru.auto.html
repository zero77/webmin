Транспортное картографирование
Необязательный транспортный файл задает сопоставление иерархий доменов с транспортными системами доставки сообщений и / или узлами ретрансляции. Отображение используется демоном trivial-rewrite (8).

Формат транспортной таблицы следующий:

  • domain сопоставлен с .. transport:nexthop : почта для домена доставляется через транспорт к nexthop.
  • .domain сопоставлен с .. transport:nexthop : Почта для любого субдомена домена доставляется через транспорт к nexthop.
Примечание. Записи транспортной карты имеют приоритет над доменами, указанными в параметре mydestination . Если вы используете дополнительную транспортную карту, может быть безопаснее указать явные записи для всех доменов, указанных в mydestination, например:
  • hostname.my.domain ... local:
  • localhost.my.domain ... local:
Интерпретация поля nexthop зависит от транспорта. В случае SMTP укажите host:service для порта сервера не по умолчанию и используйте [host] или [host:port] для отключения поиска DNS MX (mail exchanger). Форма [] также может использоваться с IP-адресами вместо имен хостов.
