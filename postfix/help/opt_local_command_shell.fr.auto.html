Shell à utiliser pour la livraison à une commande externe
local_command_shell
Ce paramètre contrôle le shell qui sera utilisé pour la livraison à la commande externe. Par défaut, les commandes externes sont exécutées directement; les commandes sont données à /bin/sh uniquement lorsqu'elles contiennent des méta-caractères shell ou des commandes intégrées shell.

Spécifiez un autre shell lorsque vous avez besoin de plus de contrôle sur les programmes qui peuvent être exécutés à partir de fichiers .forward par exemple.

"shell restreint de sendmail" smrsh est ce que la plupart des gens utiliseront ( smrsh fait partie des distributions sendmail récentes)

Remarque: lorsqu'un shell est spécifié, il est invoqué même lorsque la commande ne contient aucune commande intégrée au shell ni méta-caractère.

Exemple: /some/where/smrsh -c
