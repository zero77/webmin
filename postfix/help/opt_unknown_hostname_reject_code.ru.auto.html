Ответ SMTP-сервера на неизвестное имя хоста отклонено
unknown_hostname_reject_code
Этот параметр указывает код ответа сервера SMTP, когда клиент нарушает правило ограничения reject_unknown_hostname , а именно:
Доступ запрещен: неизвестное имя хоста HELO клиента (без записи DNS A или MX)

Обратите внимание, что значением по умолчанию для постфикса, используемым для unknown_hostname_reject_code является код 501, и этот код в «стандарте RFC 822» должен означать «Синтаксическая ошибка в параметрах или аргументах». Таким образом, с Postfix код состояния 501 не отражает в точности объяснение RFC822.

Чтобы избежать путаницы, вы можете изменить это значение, например, на 577 (другая причина в том, что код 501 также является значением по умолчанию для постфикса для других типов ошибок), в противном случае не изменяйте это, если у вас нет полного понимания RFC 822.
