Domenii virtuale
Tabelul virtual opțional specifică redirecționări pentru destinatari sau domenii locale și non-locale. Redirecțiile sunt utilizate de demonul de curățare (8). Redirecțiile sunt recursive.

Redirecționarea virtuală se aplică numai adresei plicului destinatarului și nu afectează anteturile mesajelor. Dacă doriți, setul de reguli Sendmail S0 . Utilizați maparea canonică pentru a rescrie adresele de antet și plic în general.

Asistența tipică pentru un domeniu virtual arată astfel:

  • virtual.domain mapat cu ... anything (conținutul din dreapta nu contează)
  • user1@virtual.domain mapat la ... address1
  • user2@virtual.domain mapat la ... address2, address3
Prin aceasta, serverul SMTP acceptă e-mailul pentru virtual.domain și respinge e-mail-ul pentru unknown@virtual.domain ca fiind de unknown@virtual.domain .

Formatul tabelului virtual este următorul, mapările fiind încercate în ordinea listată aici:

  • user@domain mapat la ... address, address, ... : Poșta pentru user@domain este redirecționată către address . Această formă are cea mai mare prioritate.
  • user mapat la ... address, address, ... : e-mail pentru user@site este redirecționat către address atunci când site-ul este egal cu $myorigin când site-ul este listat în $mydestination sau când este listat în $inet_interfaces . Această funcționalitate se suprapune funcționalității bazei de date locale de alias . Diferența este că maparea virtuală poate fi aplicată adreselor non-locale.
  • @domain mapat la ... address, address, ... : Poșta pentru orice utilizator din domain este redirecționată către address . Această formă are precedența cea mai mică.
În toate formularele de mai sus, atunci când adresa are formularul @otherdomain , rezultatul este același utilizator în alte domenii. Aceasta funcționează doar pentru prima adresă din expansiune.

EXTENSIUNE ADRESA: Când căutarea eșuează, iar adresa locală conține delimitatorul de destinatar opțional (de exemplu, user+foo@domain ), căutarea este repetată pentru adresa neexecutată (de exemplu, user@domain ), iar extensia de adresă neegalată este propagată la rezultatul expansiunii. Ordinea potrivită este: domeniu user+foo@domain , user+foo@domain user@domain , user+foo , user și @domain .
