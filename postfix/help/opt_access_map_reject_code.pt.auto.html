Resposta do servidor SMTP em violação do mapa de acesso
access_map_reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um email viola a restrição do access map que é:
Acesso negado: violação do access_map (no cliente SMTP ou nome do host HELO, remetente ou endereço de email do destinatário)

Observe que o valor padrão do postfix usado para access_map_reject_code é o código 554, e esse código no 'padrão RFC 822' deve significar 'Ação de email solicitada rejeitada: acesso negado'. Portanto, com o Postfix, uma violação da regra do access map não reflete exatamente a explicação RFC822 do código relatado.

Para evitar confusão, você pode alterar isso para 570, por exemplo (outro motivo é que o código 554 também é o valor padrão do postfix para outros tipos de erros); caso contrário, não altere isso, a menos que você tenha um entendimento completo da RFC 822.
