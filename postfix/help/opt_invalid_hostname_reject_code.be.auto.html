Адказ SMTP-сервера на імя хаста адхіляецца
invalid_hostname_reject_code
Гэты параметр указвае код адказу SMTP-сервера, калі кліент парушае правіла абмежавання UCE reject_invalid_hostname :
Няправільны сінтаксіс для імя кліента HELO (не сумяшчальны з RFC)

Звярніце ўвагу, што значэнне па змаўчанні Postfix, якое выкарыстоўваецца для invalid_hostname_reject_code з'яўляецца кодам 501, і гэты код у "стандартным RFC 822" павінен азначаць "Памылка сінтаксісу ў параметрах або аргументах". Такім чынам, у Postfix код 501 не дакладна адлюстроўвае тлумачэнне RFC822.

Каб пазбегнуць блытаніны, вы можаце змяніць гэта на 574, напрыклад (іншая прычына заключаецца ў тым, што код 501 таксама з'яўляецца значэннем па змаўчанні постфікса для іншых відаў памылак), у адваротным выпадку не змяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
