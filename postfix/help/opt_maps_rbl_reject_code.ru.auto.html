Ответ SMTP-сервера на нарушение доменов RBL
maps_rbl_reject_code
Этот параметр указывает код ответа SMTP-сервера, когда письмо нарушает правило maps_rbl_domains , а именно:
Доступ запрещен: SMTP-клиент указан в списке RBL

Обратите внимание, что значение постфикса по умолчанию, используемое для maps_rbl_reject_code является кодом 554, и этот код в «стандарте RFC 822» должен означать «Запрошенное действие почты отклонено: доступ запрещен». Таким образом, с Postfix нарушение правила maps_rbl_domains не отражает в точности объяснение RFC822 сообщаемого кода.

Чтобы избежать путаницы, вы можете изменить это значение, например, на 571 (другая причина в том, что код 554 также является значением по умолчанию для постфикса для других видов ошибок), в противном случае не изменяйте его, если у вас нет полного понимания RFC 822.
