Risposta del server SMTP sulla violazione della mappa di accesso
access_map_reject_code
Questo parametro consente di specificare il codice di risposta del server SMTP quando una posta viola la restrizione della access map che è:
Accesso negato: violazione access_map (su client SMTP o nome host HELO, indirizzo e-mail mittente o destinatario)

Si noti che il valore predefinito postfix utilizzato per access_map_reject_code è il codice 554 e questo codice in "RFC 822 standard" dovrebbe significare "Azione posta richiesta rifiutata: accesso negato". Pertanto, con Postfix, una violazione delle regole della access map non riflette esattamente la spiegazione RFC822 del codice segnalato.

Per evitare confusione, è possibile cambiarlo in 570 ad esempio (un altro motivo è che il codice 554 è anche il valore predefinito postfix per altri tipi di errori), altrimenti non modificarlo a meno che non si abbia una completa comprensione di RFC 822.
