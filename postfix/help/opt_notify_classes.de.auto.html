Was für ein Problem, dem Postmeister zu melden
notify_classes
Sie sollten einen Postmaster- Alias einrichten, der auf eine menschliche Person verweist. Dieser Alias muss vorhanden sein, damit Benutzer Probleme bei der E-Mail-Zustellung melden können.

Das Postfix-System selbst meldet Probleme auch an den Postmaster-Alias. Möglicherweise interessieren Sie sich nicht für alle Arten von Fehlerberichten, daher ist dieser Berichtsmechanismus konfigurierbar. Standardmäßig werden dem Postmaster nur schwerwiegende Probleme (Ressource, Software) gemeldet:

Standard: resource, software

Die Bedeutung der Klassen ist wie folgt:

  • bounce: Senden Sie Postmaster-Kopien nicht zustellbarer E-Mails.
  • 2bounce: Senden Sie Double Bounces an den Postmaster.
  • delay: Informieren Sie den Postmeister über verspätete Post. In diesem Fall empfängt der Postmaster nur Nachrichtenkopfzeilen.
  • policy: Informieren Sie den Postmaster über Clientanforderungen, die aufgrund von (UCE-) Richtlinienbeschränkungen abgelehnt wurden. Der Postmaster erhält eine Abschrift der gesamten SMTP-Sitzung.
  • protocol: Informieren Sie den Postmaster über Protokollfehler (Client- oder Serverseite) oder Versuche eines Clients, nicht implementierte Befehle auszuführen. Der Postmaster erhält eine Abschrift der gesamten SMTP-Sitzung.
  • resource: Informieren Sie den Postmaster über E-Mails, die aufgrund von Ressourcenproblemen nicht zugestellt wurden (z. B. Fehler beim Schreiben von Warteschlangendateien).
  • software: Informieren Sie den Postmaster über E-Mails, die aufgrund von Softwareproblemen nicht zugestellt wurden.
