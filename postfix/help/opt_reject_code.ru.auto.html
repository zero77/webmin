Ответ SMTP-сервера на отклонение клиента
reject_code
Этот параметр указывает код ответа сервера SMTP, когда клиент нарушает правило smtpd_client_restrictions , а именно:
Доступ запрещен: имя хоста клиента SMTP отклонено

Обратите внимание, что значением по умолчанию для постфикса, используемым для reject_code является код 554, и этот код в «стандарте RFC 822» должен означать «Запрошенное действие почты отклонено: доступ запрещен». Таким образом, в smtpd_client_restrictions нарушение правила smtpd_client_restrictions не отражает в точности объяснение RFC822 сообщаемого кода.

Чтобы избежать путаницы, вы можете изменить это значение, например, на 575 (другая причина в том, что код 554 также является значением по умолчанию для постфикса для других типов ошибок), в противном случае не изменяйте это, если у вас нет полного понимания RFC 822.
