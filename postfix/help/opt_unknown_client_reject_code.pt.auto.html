Resposta do servidor SMTP em rejeição de cliente desconhecida
unknown_client_reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um cliente sem mapeamento de endereço para nome viola a regra de restrição reject_unknown_clients que é:
Acesso negado: nome do host do cliente SMTP desconhecido (sem registro DNS A ou MX)

Observe que o valor padrão do postfix usado para unknown_client_reject_code é o código 450 também usado para outros erros. Esse código em 'RFC 822 padrão' significa 'Ação de email solicitada não executada: caixa de correio ocupada ou acesso negado' e solicita ao remetente que tente novamente mais tarde (esperando que o DNS responda corretamente mais tarde).

Não altere isso, a menos que você tenha um entendimento completo da RFC 822.
