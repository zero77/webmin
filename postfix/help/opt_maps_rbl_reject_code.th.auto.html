การตอบสนองของเซิร์ฟเวอร์ SMTP เกี่ยวกับการละเมิดโดเมน RBL
maps_rbl_reject_code
พารามิเตอร์นี้ระบุรหัสการตอบกลับของเซิร์ฟเวอร์ SMTP เมื่อจดหมายละเมิดกฎ maps_rbl_domains นั่นคือ:
ปฏิเสธการเข้าถึง: ไคลเอนต์ SMTP ที่ระบุไว้ใน RBL

โปรดทราบว่าค่าเริ่มต้น postfix ที่ใช้สำหรับ maps_rbl_reject_code คือรหัส 554 และรหัสนี้ใน 'มาตรฐาน RFC 822' ควรหมายถึง 'ปฏิเสธการกระทำจดหมายที่ร้องขอ: ปฏิเสธการเข้าถึง' ดังนั้นด้วย Postfix การละเมิดกฎ maps_rbl_domains ไม่ได้สะท้อนถึงคำอธิบาย RFC822 ของรหัสที่รายงาน

เพื่อหลีกเลี่ยงความสับสนคุณสามารถเปลี่ยนเป็น 571 ได้เช่นกัน (อีกเหตุผลหนึ่งคือรหัส 554 เป็นค่าเริ่มต้น postfix สำหรับข้อผิดพลาดประเภทอื่น ๆ ) มิฉะนั้นอย่าเปลี่ยนสิ่งนี้เว้นแต่คุณจะเข้าใจ RFC 822 เป็นอย่างดี
