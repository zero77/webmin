Risposta del server SMTP su inoltro proibito
relay_domains_reject_code
Questo parametro consente di specificare il codice di risposta del server SMTP quando una posta viola la regola relay_domains_reject che è:
Accesso negato: il relè non è autorizzato o l'host locale non è un gateway

Si noti che il valore predefinito postfix utilizzato per relay_domains_reject_code è il codice 554 e questo codice in "RFC 822 standard" dovrebbe significare "Azione posta richiesta rifiutata: accesso negato". Quindi con Postfix, una violazione della regola relay_domains_reject non riflette esattamente la spiegazione RFC822 del codice riportato.

Per evitare confusione, è possibile cambiarlo in 572 per esempio (un altro motivo è che il codice 554 è anche il valore predefinito postfix per altri tipi di errori), altrimenti non modificarlo a meno che non si abbia una comprensione completa di RFC 822.
