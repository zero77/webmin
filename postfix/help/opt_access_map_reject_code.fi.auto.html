SMTP-palvelimen vastaus pääsykartan rikkomukseen
access_map_reject_code
Tämä parametri määrittää SMTP-palvelimen vastauskoodin, kun posti rikkoo access map joka on:
Käyttö evätty: access_map-rikkomus (SMTP-asiakkaan tai HELO-isäntänimessä, lähettäjän tai vastaanottajan sähköpostiosoitteessa)

Huomaa, että access_map_reject_code käytetty postfix-oletusarvo on koodi 554, ja tämän koodin 'standardi RFC 822: ssä' pitäisi tarkoittaa 'Pyydetty postitoiminto hylätty: käyttö evätty'. Joten Postfixin kanssa access map rikkomus ei heijasta tarkasti ilmoitetun koodin RFC822-selitystä.

Sekaannusten välttämiseksi voit muuttaa tämän arvoon esimerkiksi 570 (toinen syy on, että koodi 554 on myös jälkiliitteen oletusarvo muun tyyppisille virheille). Muussa tapauksessa älä muuta tätä, ellet ole ymmärtänyt täysin RFC 822: ta.
