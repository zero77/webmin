Respuesta del servidor SMTP en violación de fqdn
non_fqdn_reject_code
Este parámetro especifica el código de respuesta del servidor SMTP cuando un correo viola la reject_non_fqdn_hostname o reject_non_fqdn_sender o reject_non_fqdn_recipient que son:
Acceso denegado: dominio no válido para la dirección de correo electrónico del remitente o del destinatario HELO del cliente (no FQDN)

Tenga en cuenta que el valor predeterminado de postfix utilizado para non_fqdn_reject_code es el código 504, y este código en 'RFC 822 estándar' debería significar 'Parámetro de comando no implementado'. Entonces, con Postfix, una violación de la regla non_fqdn_reject_code no refleja exactamente la explicación RFC822 del código informado.

Para evitar confusiones, puede cambiar esto a 578, por ejemplo, de lo contrario, no cambie esto a menos que tenga un conocimiento completo de RFC 822.
