SMTP serverio atsakymas dėl prieigos žemėlapio pažeidimo
access_map_reject_code
Šis parametras nurodo SMTP serverio atsakymo kodą, kai paštas pažeidžia access map apribojimą, kuris yra:
Prieiga atmesta: „access_map“ pažeidimas (SMTP kliento arba HELO pagrindinio kompiuterio, siuntėjo ar gavėjo el. Pašto adresuose)

Atminkite, kad „ access_map_reject_code numatytoji reikšmė, naudojama „ access_map_reject_code yra 554 kodas, o šis kodas „standartiniame RFC 822“ turėtų reikšti „Prašomas pašto veiksmas atmestas: prieiga atmesta“. Taigi naudojant „Postfix“ access map taisyklės pažeidimas tiksliai neatspindi pranešto kodo RFC822 paaiškinimo.

Norėdami išvengti painiavos, galite tai pakeisti, pavyzdžiui, į 570 (kita priežastis, kad 554 kodas taip pat yra numatytoji „postfix“ vertė kitokio pobūdžio klaidoms), priešingu atveju nekeiskite to, nebent jūs visiškai suprantate RFC 822.
