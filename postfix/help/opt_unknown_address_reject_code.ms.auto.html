Respons pelayan SMTP pada penolakan domain
unknown_address_reject_code
Parameter ini menentukan kod tindak balas pelayan SMTP apabila pelanggan melanggar reject_unknown_sender_domain atau reject_unknown_recipient_domain peraturan sekatan yang:
Akses ditolak: Domain tidak diketahui untuk alamat e-mel pengirim atau penerima (tanpa rekod DNS A atau MX)

Perhatikan bahawa nilai lalai postfix yang digunakan untuk unknown_address_reject_code tidak unknown_address_reject_code adalah kod 450 juga digunakan untuk kesalahan lain. Kod ini dalam 'standard RFC 822' bermaksud 'Tindakan mel yang diminta tidak diambil: peti mel sibuk atau akses ditolak' dan minta pengirim untuk mencuba lagi nanti (berharap DNS akan menjawab dengan betul kemudian).

Jangan ubah ini melainkan anda mempunyai pemahaman lengkap mengenai RFC 822.
