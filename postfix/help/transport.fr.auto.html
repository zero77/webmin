Cartographie des transports
Le fichier de transport facultatif spécifie un mappage des hiérarchies de domaine vers les transports de remise des messages et / ou les hôtes relais. Le mappage est utilisé par le démon trivial-rewrite (8).

Le format de la table de transport est le suivant:

  • domain mappé à .. transport:nexthop : le courrier pour le domaine est livré via le transport à nexthop.
  • .domain mappé à .. transport:nexthop : le courrier de tout sous-domaine de domaine est remis via le transport à nexthop.
Remarque: les entrées de mappage de transport ont priorité sur les domaines spécifiés dans le paramètre mydestination . Si vous utilisez la carte de transport facultative, il peut être plus sûr de spécifier des entrées explicites pour tous les domaines spécifiés dans mydestination, par exemple:
  • hostname.my.domain ... local:
  • localhost.my.domain ... local:
L'interprétation du champ nexthop dépend du transport. Dans le cas de SMTP, spécifiez host:service pour un port de serveur non par défaut, et utilisez [host] ou [host:port] afin de désactiver les recherches DNS MX (échangeur de messagerie). Le formulaire [] peut également être utilisé avec des adresses IP au lieu de noms d'hôtes.
