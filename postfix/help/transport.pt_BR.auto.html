Mapeamento de transporte
O arquivo de transporte opcional especifica um mapeamento de hierarquias de domínio para transportes de entrega de mensagens e / ou hosts de retransmissão. O mapeamento é usado pelo daemon trivial-rewrite (8).

O formato da tabela de transporte é o seguinte:

  • domain mapeado para .. transport:nexthop : o email do domínio é entregue por meio de transporte para o nexthop.
  • .domain mapeado para .. transport:nexthop : o correio para qualquer subdomínio do domínio é entregue por meio de transporte para o nexthop.
Nota: as entradas do mapa de transporte têm precedência sobre os domínios especificados no parâmetro mydestination . Se você usar o mapa de transporte opcional, pode ser mais seguro especificar entradas explícitas para todos os domínios especificados no mydestination, por exemplo:
  • hostname.my.domain ... local:
  • localhost.my.domain ... local:
A interpretação do campo nexthop depende do transporte. No caso do SMTP, especifique host:service para uma porta de servidor não padrão e use [host] ou [host:port] para desativar as pesquisas de DNS do MX (servidor de mensagens). O formulário [] também pode ser usado com endereços IP em vez de nomes de host.
