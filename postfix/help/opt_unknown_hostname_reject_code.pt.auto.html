Resposta do servidor SMTP na rejeição de nome de host desconhecido
unknown_hostname_reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um cliente viola a regra de restrição reject_unknown_hostname que é:
Acesso negado: nome de host HELO do cliente desconhecido (sem registro DNS A ou MX)

Observe que o valor padrão do postfix usado para unknown_hostname_reject_code é o código 501, e esse código no 'padrão RFC 822' deve significar 'erro de sintaxe nos parâmetros ou argumentos'. Portanto, com o Postfix, um código de status 501 não reflete exatamente a explicação da RFC822.

Para evitar confusão, você pode alterar para 577, por exemplo (outro motivo é que o código 501 também é o valor padrão do postfix para outros tipos de erros); caso contrário, não altere isso, a menos que você tenha um entendimento completo da RFC 822.
