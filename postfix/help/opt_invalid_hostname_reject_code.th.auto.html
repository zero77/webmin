การตอบสนองของเซิร์ฟเวอร์ SMTP เกี่ยวกับการปฏิเสธชื่อโฮสต์
invalid_hostname_reject_code
พารามิเตอร์นี้ระบุรหัสการตอบกลับของเซิร์ฟเวอร์ SMTP เมื่อไคลเอนต์ละเมิดกฎข้อ จำกัด anti-UCE ที่ reject_invalid_hostname นั่นคือ:
ไวยากรณ์ไม่ถูกต้องสำหรับชื่อโฮสต์ของไคลเอ็นต์ HELO (ไม่สอดคล้องกับ RFC)

โปรดทราบว่าค่าเริ่มต้น postfix ที่ใช้สำหรับ invalid_hostname_reject_code คือรหัส 501 และรหัสนี้ใน 'มาตรฐาน RFC 822' ควรหมายถึง 'ข้อผิดพลาดทางไวยากรณ์ในพารามิเตอร์หรืออาร์กิวเมนต์' ดังนั้นด้วย Postfix รหัสสถานะ 501 จึงไม่สะท้อนคำอธิบายที่ชัดเจนของ RFC822

เพื่อหลีกเลี่ยงความสับสนคุณสามารถเปลี่ยนเป็น 574 เช่น (อีกเหตุผลหนึ่งคือรหัส 501 เป็นค่าเริ่มต้น postfix สำหรับข้อผิดพลาดประเภทอื่น ๆ ) มิฉะนั้นอย่าเปลี่ยนสิ่งนี้เว้นแต่คุณจะเข้าใจ RFC 822 เป็นอย่างดี
