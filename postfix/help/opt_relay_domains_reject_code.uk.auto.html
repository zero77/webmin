Відповідь SMTP-сервера щодо забороненої ретрансляції
relay_domains_reject_code
Цей параметр вказує код відповіді сервера SMTP, коли пошта порушує правило relay_domains_reject яке:
Доступ заборонено: Реле не дозволено або не локальний хост, а не шлюз

Зауважте, що за замовчуванням значення postfix, яке використовується для relay_domains_reject_code є кодом 554, і цей код у "стандартному RFC 822" повинен означати "Запрошена поштова дія відхилена: доступ заборонено". Таким чином, з Postfix, порушення правила relay_domains_reject не відображає саме пояснення RFC822 звітного коду.

Щоб уникнути плутанини, ви можете змінити це на 572, наприклад (інша причина полягає в тому, що код 554 - це також значення за замовчуванням постфікса для інших видів помилок), інакше не змінюйте це, якщо ви не маєте повного розуміння RFC 822.
