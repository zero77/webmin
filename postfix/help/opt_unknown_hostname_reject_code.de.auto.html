Antwort des SMTP-Servers auf unbekannten Hostnamen ablehnen
unknown_hostname_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn ein Client gegen die Einschränkungsregel " reject_unknown_hostname verstößt.
Zugriff verweigert: Unbekannter Client-HELO-Hostname (ohne DNS A- oder MX-Eintrag)

Beachten Sie, dass der Postfix-Standardwert für unknown_hostname_reject_code Code 501 ist und dieser Code in 'Standard-RFC 822' 'Syntaxfehler in Parametern oder Argumenten' bedeuten sollte. Bei Postfix spiegelt ein 501-Statuscode nicht genau die RFC822-Erklärung wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 577 ändern (ein weiterer Grund ist, dass Code 501 auch der Postfix-Standardwert für andere Arten von Fehlern ist). Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
