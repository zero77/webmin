Відповідь SMTP-сервера на порушення fqdn
non_fqdn_reject_code
Цей параметр визначає код відповіді сервера SMTP, коли пошта порушує reject_non_fqdn_hostname або reject_non_fqdn_sender або reject_non_fqdn_recipient які є:
Заборонено доступ: Недійсний домен для імені хоста HELO клієнта, відправника чи електронної адреси одержувача (не FQDN)

Зауважте, що значення за замовчуванням Postfix, яке використовується для non_fqdn_reject_code - це код 504, і цей код у "стандартному RFC 822" повинен означати "Параметр команди не реалізований". Отже, з Postfix, порушення правила non_fqdn_reject_code не відображає саме пояснення RFC822 звітного коду.

Щоб уникнути плутанини, ви можете змінити це на 578, наприклад, в іншому випадку не змінюйте це, якщо ви не маєте повного розуміння RFC 822.
