Antwort des SMTP-Servers auf Domain ablehnen
unknown_address_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn ein Client gegen die Einschränkungsregeln " reject_unknown_sender_domain oder " reject_unknown_recipient_domain verstößt.
Zugriff verweigert: Unbekannte Domain für Absender- oder Empfänger-E-Mail-Adresse (ohne DNS A- oder MX-Eintrag)

Beachten Sie, dass der Postfix-Standardwert für unknown_address_reject_code Code 450 ist, der auch für andere Fehler verwendet wird. Dieser Code in "Standard-RFC 822" bedeutet "Angeforderte E-Mail-Aktion nicht ausgeführt: Postfach belegt oder Zugriff verweigert" und fordert den Absender auf, es später erneut zu versuchen (in der Hoffnung, dass der DNS später korrekt antwortet).

Ändern Sie dies nur, wenn Sie mit RFC 822 vollständig vertraut sind.
