Reakcia servera SMTP na porušenie prístupovej mapy
access_map_reject_code
Tento parameter určuje kód odozvy servera SMTP, keď e-mail porušuje obmedzenie access map ktoré je:
Prístup bol odmietnutý: porušenie prístupovej mapy (na klientovi SMTP alebo HELO názov hostiteľa, e-mailovú adresu odosielateľa alebo príjemcu)

Všimnite si, že predvolená hodnota access_map_reject_code používaná pre access_map_reject_code je kód 554 a tento kód v 'štandardnom RFC 822' by mal znamenať 'Požadovaná akcia pošty bola odmietnutá: prístup bol odmietnutý'. Takže s Postfixom porušenie pravidiel access map neodráža presne RFC822 vysvetlenie hláseného kódu.

Aby ste sa vyhli nejasnostiam, môžete to zmeniť napríklad na 570 (ďalším dôvodom je to, že kód 554 je tiež predvolená hodnota postfixu pre iné druhy chýb), inak to nemeňte, pokiaľ nemáte úplné znalosti o RFC 822.
