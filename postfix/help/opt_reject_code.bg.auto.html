Отговор на SMTP сървър при отхвърляне на клиента
reject_code
Този параметър указва кода за отговор на SMTP сървъра, когато клиент наруши правилото smtpd_client_restrictions което е:
Достъпът е отказан: името на хоста на SMTP клиента е отхвърлено

Обърнете внимание, че стойността по подразбиране postfix, използвана за код reject_code е код 554, а този код в „стандартен RFC 822“ трябва да означава „Искано действие на пощата е отхвърлено: достъпът е отказан“. Така че при Postfix, нарушението на правилото smtpd_client_restrictions не отразява точно обяснението RFC822 на отчетения код.

За да избегнете объркване, можете да промените това например на 575 (друга причина е, че код 554 е също стойността по подразбиране на postfix за други видове грешки), в противен случай не променяйте това, освен ако не имате пълно разбиране за RFC 822.
