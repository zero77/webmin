Antwort des SMTP-Servers bei Verletzung der Zugriffskarte
access_map_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn eine E-Mail gegen die access map verstößt:
Zugriff verweigert: Verletzung der Zugriffskarte (auf SMTP-Client oder HELO-Hostname, Absender- oder Empfänger-E-Mail-Adresse)

Beachten Sie, dass der für access_map_reject_code verwendete Postfix-Standardwert Code 554 ist und dieser Code in 'Standard-RFC 822' 'Angeforderte Mail-Aktion abgelehnt: Zugriff verweigert' bedeuten sollte. Bei Postfix spiegelt eine Verletzung der access map nicht genau die RFC822-Erklärung des gemeldeten Codes wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 570 ändern (ein weiterer Grund ist, dass Code 554 auch der Postfix-Standardwert für andere Arten von Fehlern ist). Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
