Shell a ser usado para entrega ao comando externo
local_command_shell
Este parâmetro controla qual shell será usado para entrega ao comando externo. Por padrão, comandos externos são executados diretamente; Os comandos são fornecidos para /bin/sh somente quando eles contêm metacaracteres do shell ou comandos internos do shell.

Especifique um shell diferente quando precisar de mais controle sobre quais programas podem ser executados, por exemplo, arquivos .forward .

O smrsh "shell restrito do sendmail" é o que a maioria das pessoas usará (o smrsh faz parte das distribuições recentes do sendmail)

Nota: quando um shell é especificado, ele é chamado mesmo quando o comando não contém comandos internos do shell ou metacaracteres.

Exemplo: /some/where/smrsh -c
