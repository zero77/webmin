О какой беде сообщать почтмейстеру
notify_classes
Вы должны создать псевдоним администратора почты, который указывает на человека. Этот псевдоним должен существовать, чтобы люди могли сообщать о проблемах доставки почты.

Сама система Postfix также сообщает о проблемах псевдониму postmaster. Возможно, вас не заинтересуют все типы отчетов об ошибках, поэтому этот механизм отчетов можно настраивать. По умолчанию сообщается только о серьезных проблемах (ресурс, программное обеспечение) для администратора почты:

По умолчанию: resource, software

Смысл классов заключается в следующем:

  • bounce: отправить почтмейстеру копии недоставленной почты.
  • 2bounce: отправьте двойной 2bounce: почты.
  • delay: сообщите почтмейстеру о задержанной почте. В этом случае почтмейстер получает только заголовки сообщений.
  • policy: информировать администратора почты о клиентских запросах, которые были отклонены из-за ограничений политики (UCE). Почтмейстер получает стенограмму всего сеанса SMTP.
  • protocol: информировать администратора почты об ошибках протокола (на стороне клиента или сервера) или попытках клиента выполнить невыполненные команды. Почтмейстер получает стенограмму всего сеанса SMTP.
  • resource: Сообщите почтмейстеру о доставке почты из-за проблем с ресурсами (например, ошибки записи файла очереди).
  • software: Сообщите почтовому мастеру о доставке почты из-за проблем с программным обеспечением.
