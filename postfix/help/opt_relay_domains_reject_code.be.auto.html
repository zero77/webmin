Адказ SMTP-сервера на забароненую рэтрансляцыю
relay_domains_reject_code
Гэты параметр паказвае код адказу SMTP-сервера, калі пошта парушае правіла relay_domains_reject , якое:
Доступ забаронены: рэле не дазволена, альбо мясцовы хост не шлюз

Звярніце ўвагу, што значэнне па змаўчанні Postfix, якое выкарыстоўваецца для relay_domains_reject_code з'яўляецца кодам 554, і гэты код у "стандартным RFC 822" павінен азначаць "Запытаная дзеянне пошты адхілена: доступ забаронены". Такім чынам, у Postfix, парушэнне правіла relay_domains_reject не адлюстроўвае дакладна тлумачэнне relay_domains_reject паведамляемага кода.

Каб пазбегнуць блытаніны, вы можаце змяніць гэта на 572, напрыклад (іншая прычына заключаецца ў тым, што код 554 таксама з'яўляецца значэннем па змаўчанні постфікса для іншых відаў памылак), інакш не мяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
