Відповідь SMTP-сервера на невідоме ім'я хоста відхиляється
unknown_hostname_reject_code
Цей параметр вказує код відповіді сервера SMTP, коли клієнт порушує правило обмеження reject_unknown_hostname яке:
Заборонено доступ: ім'я хоста невідомого клієнта (без запису DNS A або MX)

Зауважте, що значення за замовчуванням постфіксу, яке використовується для unknown_hostname_reject_code є кодом 501, і цей код у "стандартному RFC 822" повинен означати "Помилка синтаксису в параметрах або аргументах". Отже, з Postfix, код статусу 501 не відображає саме пояснення RFC822.

Щоб уникнути плутанини, ви можете змінити це на 577, наприклад (інша причина полягає в тому, що код 501 є також значенням за замовчуванням постфікса для інших видів помилок), інакше не змінюйте це, якщо ви не маєте повного розуміння RFC 822.
