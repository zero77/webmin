Risposta del server SMTP al rifiuto del nome host
invalid_hostname_reject_code
Questo parametro consente di specificare il codice di risposta del server SMTP quando un client viola la regola di restrizione anti-UCE reject_invalid_hostname che è:
Sintassi errata per il nome host HELO del client (non conforme a RFC)

Si noti che il valore predefinito postfix utilizzato per invalid_hostname_reject_code è il codice 501 e questo codice in "RFC 822 standard" dovrebbe significare "Errore di sintassi in parametri o argomenti". Quindi con Postfix, un codice di stato 501 non riflette esattamente la spiegazione di RFC822.

Per evitare confusione, è possibile cambiarlo in 574 per esempio (un altro motivo è che il codice 501 è anche il valore predefinito postfix per altri tipi di errori), altrimenti non modificarlo a meno che non si abbia una completa comprensione di RFC 822.
