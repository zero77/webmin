Resposta do servidor SMTP na rejeição do cliente
reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um cliente viola a regra smtpd_client_restrictions que é:
Acesso negado: nome do host do cliente SMTP rejeitado

Observe que o valor padrão do postfix usado para reject_code é o código 554, e esse código no 'padrão RFC 822' deve significar 'Ação de email solicitada rejeitada: acesso negado'. Portanto, com o Postfix, uma violação da regra smtpd_client_restrictions não reflete exatamente a explicação RFC822 do código relatado.

Para evitar confusão, você pode alterar para 575, por exemplo (outro motivo é que o código 554 também é o valor padrão do postfix para outros tipos de erros); caso contrário, não altere isso, a menos que você tenha um entendimento completo da RFC 822.
