Respons pelayan SMTP pada pelanggaran peta akses
access_map_reject_code
Parameter ini menentukan kod respons pelayan SMTP apabila surat melanggar batasan access map iaitu:
Akses ditolak: pelanggaran access_map (pada klien SMTP atau nama host HELO, alamat e-mel pengirim atau penerima)

Perhatikan bahawa nilai lalai postfix yang digunakan untuk access_map_reject_code adalah kod 554, dan kod ini dalam 'standard RFC 822' bermaksud 'Tindakan mel diminta ditolak: akses ditolak'. Jadi dengan Postfix, pelanggaran peraturan access map tidak mencerminkan penjelasan RFC822 dengan tepat mengenai kod yang dilaporkan.

Untuk mengelakkan kekeliruan, anda boleh mengubahnya menjadi 570 misalnya (sebab lain adalah bahawa kod 554 juga merupakan nilai lalai postfix untuk jenis kesalahan lain), jika tidak, jangan ubah ini melainkan anda mempunyai pemahaman lengkap tentang RFC 822.
