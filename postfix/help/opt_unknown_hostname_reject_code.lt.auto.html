SMTP serverio atsakymas nežinomu pagrindinio kompiuterio vardu atmetamas
unknown_hostname_reject_code
Šis parametras nurodo SMTP serverio atsakymo kodą, kai klientas pažeidžia reject_unknown_hostname apribojimo taisyklę, kuri yra:
Prieiga atmesta: nežinomas kliento HELO pagrindinio kompiuterio vardas (be DNS A arba MX įrašo)

Atminkite, kad pašto dėžutės numatytoji reikšmė, naudojama unknown_hostname_reject_code kodui, yra kodas 501, o šis kodas „standartiniame RFC 822“ turėtų reikšti „parametrų ar argumentų sintaksės klaida“. Taigi naudojant „Postfix“ 501 būsenos kodas tiksliai neatspindi RFC822 paaiškinimo.

Norėdami išvengti painiavos, galite tai pakeisti, pavyzdžiui, į 577 (kita priežastis yra ta, kad kodas 501 taip pat yra numatytoji „postfix“ reikšmė kitokio pobūdžio klaidoms), priešingu atveju nekeiskite to, nebent jūs visiškai suprantate RFC 822.
