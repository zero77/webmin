Mappatura dei trasporti
Il file di trasporto opzionale specifica una mappatura dalle gerarchie di dominio ai trasporti di consegna dei messaggi e / o agli host di inoltro. La mappatura viene utilizzata dal demone trivial-rewrite (8).

Il formato della tabella di trasporto è il seguente:

  • domain mappato su .. transport:nexthop : la posta per il dominio viene recapitata tramite il trasporto a nexthop.
  • .domain mappato su .. transport:nexthop : la posta per qualsiasi sottodominio del dominio viene recapitata tramite il trasporto a nexthop.
Nota: le voci della mappa di trasporto hanno la precedenza sui domini specificati nel parametro mydestination . Se si utilizza la mappa di trasporto opzionale, potrebbe essere più sicuro specificare voci esplicite per tutti i domini specificati in mydestination, ad esempio:
  • hostname.my.domain ... local:
  • localhost.my.domain ... local:
L'interpretazione del campo nexthop dipende dal trasporto. Nel caso di SMTP, specificare host:service per una porta del server non predefinita e utilizzare [host] o [host:port] per disabilitare le ricerche DNS MX (scambiatore di posta). Il modulo [] può essere utilizzato anche con indirizzi IP anziché con nomi host.
