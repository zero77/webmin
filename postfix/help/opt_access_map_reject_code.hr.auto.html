Odgovor SMTP poslužitelja na kršenje karte pristupa
access_map_reject_code
Ovaj parametar određuje kôd odgovora SMTP poslužitelja kada pošta krši ograničenje access map koja je:
Pristup odbijen: kršenje access_map (na SMTP klijentu ili imenu glavnog računala HELO, pošiljatelju ili primatelju e-adrese)

Imajte na umu da je zadana vrijednost postfiksa koja se koristi za access_map_reject_code kod 554, a ovaj kôd u 'standardnom RFC 822' trebao bi značiti 'Zahtijevana radnja pošte odbijena: pristup odbijen'. Dakle, uz Postfix, kršenje pravila access map ne odražava točno RFC822 objašnjenje prijavljenog koda.

Kako biste izbjegli zabunu, ovo možete promijeniti na primjer 570 (drugi razlog je taj što je kod 554 također zadana vrijednost postfiksa za ostale vrste pogrešaka), u suprotnom, nemojte to mijenjati ako nemate potpuno razumijevanje RFC 822.
