Comando externo a ser usado em vez da entrega da caixa de correio
mailbox_command
Este parâmetro especifica o comando externo opcional a ser usado em vez da entrega da caixa de correio. O comando é executado como o destinatário com as configurações de ambiente HOME , SHELL e LOGNAME . Exceção: a entrega para root é feita como $default_user .

Outras variáveis de ambiente de interesse: USER (nome de usuário do destinatário), EXTENSION (extensão de endereço), DOMAIN (parte do endereço do domínio) e LOCAL (a parte local do endereço).

Ao contrário de outros parâmetros de configuração do Postfix, esse parâmetro não está sujeito a substituições de parâmetros. Isso facilita a especificação da sintaxe do shell (veja o exemplo abaixo).

Evite os meta-caracteres do shell, porque forçarão o Postfix a executar um processo caro do shell. Somente o Procmail é caro o suficiente.

SE UTILIZAR ISTO PARA ENTREGAR O CORREIO EM TODO O SISTEMA, DEVE INSTALAR UM ALIAS QUE ENVIE O CORREIO PARA A RAIZ A UM USUÁRIO REAL.

Exemplos:

  • /some/where/procmail
  • /some/where/procmail -a "$EXTENSION"
