Відповідь SMTP-сервера на порушення карти доступу
access_map_reject_code
Цей параметр вказує код відповіді сервера SMTP, коли пошта порушує обмеження access map доступу, яке:
Заборонено доступ: порушення access_map (на SMTP-клієнт або ім'я хоста HELO, електронну адресу відправника чи одержувача)

Зауважте, що значенням за замовчуванням Postfix, яке використовується для коду access_map_reject_code є код 554, і цей код у "стандартному RFC 822" повинен означати "Запрошена поштова дія відхилена: доступ заборонено". Отже, з Postfix, порушення правила access map не відображає саме пояснення RFC822 звітного коду.

Щоб уникнути плутанини, ви можете змінити це на 570, наприклад (інша причина полягає в тому, що код 554 - це також значення за замовчуванням постфікса для інших видів помилок), інакше не змінюйте це, якщо ви не маєте повного розуміння RFC 822.
