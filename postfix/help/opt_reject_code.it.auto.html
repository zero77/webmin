Risposta del server SMTP sul rifiuto del client
reject_code
Questo parametro specifica il codice di risposta del server SMTP quando un client viola la regola smtpd_client_restrictions che è:
Accesso negato: nome host client SMTP rifiutato

Si noti che il valore predefinito postfix utilizzato per reject_code è il codice 554 e questo codice in "RFC standard 822" dovrebbe significare "Azione di posta richiesta rifiutata: accesso negato". Quindi, con Postfix, una violazione della regola smtpd_client_restrictions non riflette esattamente la spiegazione RFC822 del codice segnalato.

Per evitare confusione, è possibile cambiarlo in 575 per esempio (un altro motivo è che il codice 554 è anche il valore predefinito postfix per altri tipi di errori), altrimenti non modificarlo a meno che non si abbia una completa comprensione di RFC 822.
