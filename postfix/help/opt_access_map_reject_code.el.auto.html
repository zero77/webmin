Απόκριση διακομιστή SMTP σε παραβίαση χάρτη πρόσβασης
access_map_reject_code
Αυτή η παράμετρος καθορίζει τον κωδικό απόκρισης διακομιστή SMTP όταν ένα μήνυμα παραβιάζει τον περιορισμό του access map που είναι:
Δεν επιτρέπεται η πρόσβαση: παράβαση access_map (σε πελάτη SMTP ή όνομα κεντρικού υπολογιστή HELO, διεύθυνση email αποστολέα ή παραλήπτη)

Σημειώστε ότι η προεπιλεγμένη τιμή μετά την access_map_reject_code χρησιμοποιείται για το access_map_reject_code είναι ο κωδικός 554 και αυτός ο κωδικός στο «τυπικό RFC 822» πρέπει να σημαίνει «Απορρίφθηκε η ενέργεια αλληλογραφίας που ζητήθηκε: η πρόσβαση δεν επιτρέπεται». Έτσι, με το Postfix, μια παραβίαση κανόνα access map δεν αντικατοπτρίζει ακριβώς την εξήγηση RFC822 του αναφερόμενου κώδικα.

Για να αποφύγετε τη σύγχυση, μπορείτε να το αλλάξετε σε 570 για παράδειγμα (ένας άλλος λόγος είναι ότι ο κωδικός 554 είναι επίσης η προεπιλεγμένη τιμή μετά την επιδιόρθωση για άλλα είδη σφαλμάτων), διαφορετικά, μην το αλλάξετε εκτός εάν έχετε πλήρη κατανόηση του RFC 822.
