Réponse du serveur SMTP en cas de violation du plan d'accès
access_map_reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un courrier viole la restriction de access map qui est:
Accès refusé: violation access_map (sur le client SMTP ou le nom d'hôte HELO, l'adresse e-mail de l'expéditeur ou du destinataire)

Notez que la valeur par défaut de postfix utilisée pour access_map_reject_code est le code 554, et ce code dans la «norme RFC 822» devrait signifier «Action de messagerie demandée rejetée: accès refusé». Donc, avec Postfix, une violation de règle de access map ne reflète pas exactement l'explication RFC822 du code signalé.

Pour éviter toute confusion, vous pouvez changer cela en 570 par exemple (une autre raison est que le code 554 est également la valeur par défaut de postfix pour d'autres types d'erreurs), sinon, ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
