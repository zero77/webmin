Risposta del server SMTP sul rifiuto del dominio
unknown_address_reject_code
Questo parametro consente di specificare il codice di risposta del server SMTP quando un client viola le regole di restrizione reject_unknown_sender_domain o reject_unknown_recipient_domain che sono:
Accesso negato: dominio sconosciuto per l'indirizzo e-mail del mittente o del destinatario (senza record DNS A o MX)

Si noti che il valore predefinito postfix utilizzato per unknown_address_reject_code è il codice 450 utilizzato anche per altri errori. Questo codice in "RFC 822 standard" significa "Azione di posta richiesta non eseguita: casella di posta occupata o accesso negato" e chiedere al mittente di riprovare più tardi (sperando che il DNS risponda correttamente in seguito).

Non modificarlo a meno che tu non abbia una conoscenza completa di RFC 822.
