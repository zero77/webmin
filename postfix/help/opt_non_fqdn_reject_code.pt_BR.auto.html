Resposta do servidor SMTP na violação do fqdn
non_fqdn_reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um email viola a reject_non_fqdn_hostname ou reject_non_fqdn_sender ou reject_non_fqdn_recipient que são:
Acesso negado: domínio inválido para o nome de host do cliente HELO, remetente ou endereço de email do destinatário (não o FQDN)

Observe que o valor padrão do postfix usado para non_fqdn_reject_code é o código 504, e esse código no 'padrão RFC 822' deve significar 'Parâmetro de comando não implementado'. Portanto, com o Postfix, uma violação da regra non_fqdn_reject_code não reflete exatamente a explicação RFC822 do código relatado.

Para evitar confusão, você pode alterar para 578, por exemplo, caso contrário, não altere, a menos que tenha um entendimento completo da RFC 822.
