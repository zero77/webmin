Ответ SMTP-сервера на отклонение имени хоста
invalid_hostname_reject_code
Этот параметр указывает код ответа сервера SMTP, когда клиент нарушает правило ограничения reject_invalid_hostname anti-UCE, а именно:
Неверный синтаксис для имени хоста клиента HELO (не соответствует RFC)

Обратите внимание, что значением по умолчанию для постфикса, используемым для invalid_hostname_reject_code является код 501, и этот код в «стандарте RFC 822» должен означать «Синтаксическая ошибка в параметрах или аргументах». Таким образом, с Postfix код состояния 501 не отражает в точности объяснение RFC822.

Чтобы избежать путаницы, вы можете изменить это значение, например, на 574 (другая причина в том, что код 501 также является значением по умолчанию для постфикса для других видов ошибок), в противном случае не изменяйте это, если у вас нет полного понимания RFC 822.
