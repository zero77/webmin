Réponse du serveur SMTP en cas de violation des domaines RBL
maps_rbl_reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un courrier viole la règle maps_rbl_domains qui est:
Accès refusé: client SMTP répertorié dans RBL

Notez que la valeur par défaut de postfix utilisée pour maps_rbl_reject_code est le code 554, et ce code dans la «norme RFC 822» devrait signifier «Action de messagerie demandée rejetée: accès refusé». Ainsi, avec Postfix, une violation de la règle maps_rbl_domains ne reflète pas exactement l'explication RFC822 du code signalé.

Pour éviter toute confusion, vous pouvez changer cela en 571 par exemple (une autre raison est que le code 554 est également la valeur par défaut de postfix pour d'autres types d'erreurs), sinon, ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
