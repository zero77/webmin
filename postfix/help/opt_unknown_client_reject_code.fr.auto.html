Réponse du serveur SMTP au rejet d'un client inconnu
unknown_client_reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un client sans mappage d'adresse à nom viole la règle de restriction reject_unknown_clients qui est:
Accès refusé: nom d'hôte du client SMTP inconnu (sans enregistrement DNS A ou MX)

Notez que la valeur par défaut de postfix utilisée pour unknown_client_reject_code est le code 450 également utilisé pour d'autres erreurs. Ce code dans «RFC 822 standard» signifie «Action de messagerie demandée non effectuée: boîte aux lettres occupée ou accès refusé» et demandez à l'expéditeur de réessayer plus tard (en espérant que le DNS répondra correctement plus tard).

Ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
