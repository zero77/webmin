Ответ SMTP-сервера на отклонение неизвестного клиента
unknown_client_reject_code
Этот параметр указывает код ответа сервера SMTP, когда клиент без сопоставления адреса с именем нарушает правило ограничения reject_unknown_clients , а именно:
Доступ запрещен: неизвестное имя хоста SMTP-клиента (без записи DNS A или MX)

Обратите внимание, что значением по умолчанию для постфикса, используемым для unknown_client_reject_code является код 450, также используемый для других ошибок. Этот код в «стандартном RFC 822» означает «Запрошенное почтовое действие не выполнено: почтовый ящик занят или доступ запрещен» и попросите отправителя повторить попытку позже (надеясь, что DNS ответит правильно позже).

Не изменяйте это, если у вас нет полного понимания RFC 822.
