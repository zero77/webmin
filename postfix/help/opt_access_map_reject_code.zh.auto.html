SMTP服务器对访问映射冲突的响应
access_map_reject_code
当邮件违反access map限制时，此参数指定SMTP服务器响应代码：
拒绝访问：违反access_map（在SMTP客户端或HELO主机名，发件人或收件人电子邮件地址上）

请注意，用于access_map_reject_code的后缀默认值为代码554，“标准RFC 822”中的此代码应表示“请求的邮件操作被拒绝：访问被拒绝”。因此，对于Postfix，违反access map规则的行为无法完全反映所报告代码的RFC822说明。

为避免混淆，您可以将其更改为例如570（另一个原因是代码554也是其他类型错误的后缀默认值），否则，除非您完全了解RFC 822，否则请勿更改此值。
