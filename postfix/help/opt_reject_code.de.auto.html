Antwort des SMTP-Servers auf Client-Ablehnung
reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn ein Client gegen die Regel smtpd_client_restrictions verstößt:
Zugriff verweigert: SMTP-Client-Hostname abgelehnt

Beachten Sie, dass der für reject_code verwendete Postfix-Standardwert Code 554 ist und dieser Code in 'Standard-RFC 822' 'Angeforderte Mail-Aktion abgelehnt: Zugriff verweigert' bedeuten sollte. Bei Postfix smtpd_client_restrictions eine smtpd_client_restrictions nicht genau die RFC822-Erklärung des gemeldeten Codes wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 575 ändern (ein weiterer Grund ist, dass Code 554 auch der Postfix-Standardwert für andere Arten von Fehlern ist). Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
