Risposta del server SMTP sulla violazione di fqdn
non_fqdn_reject_code
Questo parametro consente di specificare il codice di risposta del server SMTP quando un messaggio di posta elettronica viola il reject_non_fqdn_hostname o reject_non_fqdn_sender o reject_non_fqdn_recipient che sono:
Accesso negato: dominio non valido per nome host HELO client, indirizzo e-mail mittente o destinatario (non FQDN)

Si noti che il valore predefinito postfix utilizzato per non_fqdn_reject_code è il codice 504 e questo codice in "RFC 822 standard" dovrebbe significare "Parametro di comando non implementato". Pertanto, con Postfix, una violazione della regola non_fqdn_reject_code non riflette esattamente la spiegazione RFC822 del codice segnalato.

Per evitare confusione, è possibile modificarlo su 578, ad esempio, altrimenti non modificarlo a meno che non si abbia una comprensione completa di RFC 822.
