SMTP-Serverantwort bei Verletzung der RBL-Domänen
maps_rbl_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn eine E-Mail gegen die Regel " maps_rbl_domains verstößt.
Zugriff verweigert: SMTP-Client in RBL aufgeführt

Beachten Sie, dass der für maps_rbl_reject_code verwendete Postfix-Standardwert Code 554 ist und dieser Code in 'Standard-RFC 822' 'Angeforderte Mail-Aktion abgelehnt: Zugriff verweigert' bedeuten sollte. Bei Postfix maps_rbl_domains eine maps_rbl_domains nicht genau die RFC822-Erklärung des gemeldeten Codes wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 571 ändern (ein weiterer Grund ist, dass Code 554 auch der Postfix-Standardwert für andere Arten von Fehlern ist). Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
