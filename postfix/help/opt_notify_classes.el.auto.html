Τι πρόβλημα να αναφέρετε στον διευθυντή
notify_classes
Θα πρέπει να δημιουργήσετε ένα ψευδώνυμο postmaster που να δείχνει ένα ανθρώπινο άτομο. Αυτό το ψευδώνυμο πρέπει να υπάρχει, έτσι ώστε οι χρήστες να μπορούν να αναφέρουν προβλήματα παράδοσης αλληλογραφίας.

Το ίδιο το σύστημα Postfix αναφέρει επίσης προβλήματα στο ψευδώνυμο του postmaster. Μπορεί να μην ενδιαφέρεστε για όλους τους τύπους αναφορών προβλημάτων, επομένως αυτός ο μηχανισμός αναφοράς είναι διαμορφώσιμος. Η προεπιλογή είναι να αναφέρετε μόνο σοβαρά προβλήματα (πόρος, λογισμικό) στον ταχυδρομικό διευθυντή:

Προεπιλογή: resource, software

Η έννοια των τάξεων έχει ως εξής:

  • bounce: Στείλτε αντίγραφα αλληλογραφίας postmaster.
  • 2bounce: Στείλτε διπλά bounces στον postmaster.
  • delay: Ενημερώστε τον ταχυδρομείο για καθυστερημένη αλληλογραφία. Σε αυτήν την περίπτωση, ο διευθυντής λαμβάνει μόνο κεφαλίδες μηνυμάτων.
  • policy: Ενημερώστε τον διευθυντή για αιτήματα πελατών που απορρίφθηκαν λόγω περιορισμών πολιτικής (UCE). Ο διευθυντής λαμβάνει μια μεταγραφή ολόκληρης της περιόδου λειτουργίας SMTP.
  • protocol: Ενημερώστε τον postmaster για σφάλματα πρωτοκόλλου (πλευρά πελάτη ή διακομιστή) ή απόπειρες ενός πελάτη να εκτελέσει μη εκτελεσμένες εντολές. Ο διευθυντής λαμβάνει μια μεταγραφή ολόκληρης της περιόδου λειτουργίας SMTP.
  • resource: Ενημερώστε τον υπεύθυνο ταχυδρομείου για την αλληλογραφία που δεν παραδόθηκε λόγω προβλημάτων πόρων (για παράδειγμα, σφάλματα εγγραφής ουράς).
  • software: Ενημερώστε τον ταχυδρομείο για την αλληλογραφία που δεν παραδόθηκε λόγω προβλημάτων λογισμικού.
