Resposta do servidor SMTP na violação de domínios RBL
maps_rbl_reject_code
Este parâmetro especifica o código de resposta do servidor SMTP quando um email viola a regra maps_rbl_domains que é:
Acesso negado: cliente SMTP listado na RBL

Observe que o valor padrão do postfix usado para maps_rbl_reject_code é o código 554, e esse código no 'padrão RFC 822' deve significar 'Ação de email solicitada rejeitada: acesso negado'. Portanto, com o Postfix, uma violação da regra maps_rbl_domains não reflete exatamente a explicação RFC822 do código relatado.

Para evitar confusão, você pode alterar para 571, por exemplo (outro motivo é que o código 554 também é o valor padrão do postfix para outros tipos de erros); caso contrário, não altere isso, a menos que você tenha um entendimento completo da RFC 822.
