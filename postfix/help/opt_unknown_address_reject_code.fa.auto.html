پاسخ سرور SMTP در رد دامنه
unknown_address_reject_code
این پارامتر هنگامی که مشتری با استفاده از قوانین محدودیت reject_unknown_recipient_domain یا نقض قوانین reject_unknown_sender_domain را نقض می کند ، کد پاسخ سرور SMTP را مشخص می کند:
دسترسی رد شد: دامنه ناشناخته برای آدرس ایمیل فرستنده یا گیرنده (بدون ثبت DNS A یا MX)

توجه داشته باشید که مقدار پیش فرض postfix که برای unknown_address_reject_code است کد 450 نیز برای سایر خطاها استفاده می شود. این کد در "استاندارد RFC 822" به معنای "درخواست نامه درخواست شده نیست. صندوق پستی مشغول است یا دسترسی به آن ممنوع است" و از فرستنده بخواهید که بعداً دوباره امتحان کند (امیدوار است DNS بعداً به درستی جواب دهد).

این را تغییر ندهید مگر اینکه درک کاملی از RFC 822 داشته باشید.
