Risposta del server SMTP sulla violazione dei domini RBL
maps_rbl_reject_code
Questo parametro consente di specificare il codice di risposta del server SMTP quando una posta viola la regola maps_rbl_domains che è:
Accesso negato: client SMTP elencato in RBL

Si noti che il valore predefinito postfix utilizzato per maps_rbl_reject_code è il codice 554 e questo codice in "RFC 822 standard" dovrebbe significare "Azione posta richiesta rifiutata: accesso negato". Quindi con Postfix, una violazione della regola maps_rbl_domains non riflette esattamente la spiegazione RFC822 del codice riportato.

Per evitare confusione, è possibile cambiarlo in 571 ad esempio (un altro motivo è che il codice 554 è anche il valore predefinito postfix per altri tipi di errori), altrimenti non modificarlo a meno che non si abbia una comprensione completa di RFC 822.
