SMTP-bediener se reaksie op kliënt verwerp
reject_code
Hierdie parameter spesifiseer die SMTP-bediener se antwoordkode wanneer 'n kliënt die reël smtpd_client_restrictions oortree wat is:
Toegang geweier: SMTP-kliënt gasheernaam verwerp

Let daarop dat die standaardwaarde van die postfix wat vir reject_code word, kode 554 is, en hierdie kode in 'standaard RFC 822' moet beteken 'Gevraagde reject_code verwerp: toegang geweier'. Met Postfix smtpd_client_restrictions reël-oortreding van smtpd_client_restrictions nie presies die RFC822-verduideliking van gerapporteerde kode nie.

Om verwarring te voorkom, kan u dit byvoorbeeld na 575 verander ('n ander rede is dat kode 554 ook die standaardwaarde van die postfix is vir ander soort foute), anders moet u dit nie verander nie, tensy u 'n volledige begrip van RFC 822 het.
