Shell da utilizzare per la consegna a comandi esterni
local_command_shell
Questo parametro controlla quale shell verrà utilizzata per il recapito al comando esterno. Per impostazione predefinita, i comandi esterni vengono eseguiti direttamente; i comandi vengono dati a /bin/sh solo quando contengono meta caratteri della shell o comandi incorporati della shell.

Specificare una shell diversa quando è necessario un maggiore controllo su quali programmi possono essere eseguiti, ad esempio da file .forward .

smrsh "shell ristretta di sendmail" è ciò che la maggior parte delle persone utilizzerà ( smrsh fa parte delle recenti distribuzioni di sendmail)

Nota: quando viene specificata una shell, viene invocata anche quando il comando non contiene comandi o meta caratteri incorporati nella shell.

Esempio: /some/where/smrsh -c
