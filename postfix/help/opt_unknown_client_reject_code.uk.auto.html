Відповідь SMTP-сервера на невідомий клієнт відхиляє
unknown_client_reject_code
Цей параметр визначає код відповіді сервера SMTP, коли клієнт без відображення адреси імен порушує правило обмеження reject_unknown_clients яке є:
Заборонено доступ: Ім'я хоста SMTP-клієнта (без запису DNS A або MX)

Зауважте, що значення за замовчуванням постфіксу, яке використовується для unknown_client_reject_code є кодом 450, який також використовується для інших помилок. Цей код у "стандартному RFC 822" означає "Запрошена пошта не здійснена: поштова скринька зайнята або доступ заборонено", і попросіть відправника спробувати пізніше (сподіваючись, що DNS відповість правильно пізніше).

Не змінюйте це, якщо ви повністю не розумієте RFC 822.
