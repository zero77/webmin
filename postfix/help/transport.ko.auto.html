전송 매핑
선택적 전송 파일은 도메인 계층에서 메시지 배달 전송 및 / 또는 릴레이 호스트로의 매핑을 지정합니다. trivial-rewrite (8) 디먼이 맵핑을 사용합니다.

전송 테이블의 형식은 다음과 같습니다.

  • domain map .. transport:nexthop : 도메인 메일이 transport:nexthop 전송을 통해 전달됩니다.
  • .domain map .. transport:nexthop : 도메인의 모든 하위 도메인에 대한 메일이 전송을 통해 nexthop으로 전달됩니다.
참고 : 전송 맵 항목은 mydestination 매개 변수에 지정된 도메인보다 우선합니다. 선택적 전송 맵을 사용하는 경우 mydestination에 지정된 모든 도메인에 대해 명시적인 항목을 지정하는 것이 더 안전 할 수 있습니다. 예를 들면 다음과 같습니다.
  • hostname.my.domain ... local:
  • localhost.my.domain ... local:
nexthop 필드의 해석은 전송에 따라 다릅니다. SMTP의 경우 기본이 아닌 서버 포트에 host:service 를 지정하고 MX (메일 교환기) DNS 조회를 비활성화하려면 [host] 또는 [host:port] 를 사용하십시오. 호스트 이름 대신 IP 주소와 함께 [] 형식을 사용할 수도 있습니다.
