Respons pelayan SMTP pada penolakan nama host yang tidak diketahui
unknown_hostname_reject_code
Parameter ini menentukan kod tindak balas pelayan SMTP apabila pelanggan melanggar peraturan pembatasan reject_unknown_hostname iaitu:
Akses ditolak: Nama host HELO klien tidak diketahui (tanpa rekod DNS A atau MX)

Perhatikan bahawa nilai lalai postfix yang digunakan untuk unknown_hostname_reject_code tidak unknown_hostname_reject_code adalah kod 501, dan kod ini dalam 'standard RFC 822' bermaksud 'Kesalahan sintaks dalam parameter atau argumen'. Jadi dengan Postfix, kod status 501 tidak menggambarkan penjelasan RFC822 dengan tepat.

Untuk mengelakkan kekeliruan, anda boleh mengubahnya menjadi 577 misalnya (sebab lain adalah bahawa kod 501 juga merupakan nilai lalai postfix untuk jenis kesalahan lain), jika tidak, jangan ubah ini kecuali anda mempunyai pemahaman yang lengkap mengenai RFC 822.
