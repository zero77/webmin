Рэакцыя SMTP-сервера на парушэнне карты доступу
access_map_reject_code
Гэты параметр паказвае код адказу SMTP-сервера, калі пошта парушае абмежаванне access map доступу, гэта:
Адмоўлена ў доступе: парушэнне access_map (на SMTP-кліента або імя вузла HELO, адрас электроннай пошты адпраўніка або атрымальніка)

Звярніце ўвагу, што значэнне па змаўчанні Postfix, якое выкарыстоўваецца для access_map_reject_code з'яўляецца кодам 554, і гэты код у "стандартным RFC 822" павінен азначаць "Запытаная дзеянне пошты адхілена: доступ забаронены". Такім чынам, у Postfix парушэнне правілаў access map не адлюстроўвае дакладна тлумачэнне RFC822 паведамляемага кода.

Каб пазбегнуць блытаніны, вы можаце змяніць гэта на 570, напрыклад (іншая прычына заключаецца ў тым, што код 554 таксама з'яўляецца значэннем па змаўчанні постфікса для іншых відаў памылак), інакш не мяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
