Réponse du serveur SMTP au rejet du client
reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un client viole la règle smtpd_client_restrictions qui est:
Accès refusé: nom d'hôte du client SMTP rejeté

Notez que la valeur par défaut de postfixe utilisée pour reject_code est le code 554, et ce code dans la «norme RFC 822» devrait signifier «Action de messagerie demandée rejetée: accès refusé». Ainsi, avec Postfix, une violation de la règle smtpd_client_restrictions ne reflète pas exactement l'explication RFC822 du code signalé.

Pour éviter toute confusion, vous pouvez changer cela en 575 par exemple (une autre raison est que le code 554 est également la valeur par défaut de postfix pour d'autres types d'erreurs), sinon, ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
