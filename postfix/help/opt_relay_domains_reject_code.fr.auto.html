Réponse du serveur SMTP sur le relais interdit
relay_domains_reject_code
Ce paramètre spécifie le code de réponse du serveur SMTP lorsqu'un courrier viole la règle relay_domains_reject qui est:
Accès refusé: relais non autorisé ou hôte local non une passerelle

Notez que la valeur par défaut de postfix utilisée pour relay_domains_reject_code est le code 554, et ce code dans la «norme RFC 822» devrait signifier «Action de messagerie demandée rejetée: accès refusé». Ainsi, avec Postfix, une violation de la règle relay_domains_reject ne reflète pas exactement l'explication RFC822 du code signalé.

Pour éviter toute confusion, vous pouvez changer cela en 572 par exemple (une autre raison est que le code 554 est également la valeur par défaut de postfix pour d'autres types d'erreurs), sinon, ne changez pas cela à moins d'avoir une compréhension complète de la RFC 822.
