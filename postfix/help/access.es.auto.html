Asignación de tabla de acceso
La tabla de acceso opcional dirige al servidor SMTP Postfix para que rechace o acepte selectivamente el correo. Se puede permitir o denegar el acceso a nombres de host específicos, nombres de dominio, redes, direcciones de host o direcciones de correo.

Correlaciones de direcciones de correo electrónico

  • user@domain coincide con la dirección de correo especificada
  • domain.tld coincide con domain.tld como la parte del dominio de una dirección de correo electrónico
  • user@ coincide con todas las direcciones de correo con la parte de usuario especificada

Asignaciones de nombre de host / dirección IP

  • net.work.addr.ess
    net.work.addr
    net.work
    net     coincide con la dirección de host IPv4 especificada o el trabajo de subred
  • net:work:addr:ess
    net:work:addr
    net:work
    net     coincide con la dirección de host IPv6 especificada o el trabajo de subred

Acción en partidos

  • OK Aceptar el correo electrónico.
  • REJECT [message] Rechace el correo electrónico y responda con un [mensaje] o un error genérico.
  • DISCARD [message] Descarte silenciosamente el correo electrónico; opcionalmente, escriba un registro [mensaje].
  • REDIRECT spam@yourdomain.com reenvía el correo electrónico a otra dirección
