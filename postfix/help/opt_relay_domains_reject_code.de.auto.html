SMTP-Serverantwort bei verbotener Weiterleitung
relay_domains_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn eine E-Mail gegen die relay_domains_reject Regel verstößt.
Zugriff verweigert: Relay nicht autorisiert oder kein lokaler Host kein Gateway

Beachten Sie, dass der für relay_domains_reject_code verwendete Postfix-Standardwert Code 554 ist und dieser Code in 'Standard-RFC 822' 'Angeforderte Mail-Aktion abgelehnt: Zugriff verweigert' bedeuten sollte. Bei Postfix relay_domains_reject eine relay_domains_reject nicht genau die RFC822-Erklärung des gemeldeten Codes wider.

Um Verwirrung zu vermeiden, können Sie dies beispielsweise in 572 ändern (ein weiterer Grund ist, dass Code 554 auch der Postfix-Standardwert für andere Arten von Fehlern ist). Andernfalls ändern Sie dies nicht, es sei denn, Sie haben ein umfassendes Verständnis von RFC 822.
