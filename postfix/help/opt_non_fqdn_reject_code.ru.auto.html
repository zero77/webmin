Ответ SMTP-сервера на нарушение fqdn
non_fqdn_reject_code
Этот параметр указывает код ответа сервера SMTP, когда письмо нарушает reject_non_fqdn_hostname или reject_non_fqdn_sender или reject_non_fqdn_recipient , а именно:
Доступ запрещен: неверный домен для имени клиента HELO, адреса отправителя или получателя (не FQDN)

Обратите внимание, что значением по умолчанию для постфикса, используемым для non_fqdn_reject_code является код 504, и этот код в «стандарте RFC 822» должен означать «Параметр команды не реализован». Таким образом, с Postfix нарушение правила non_fqdn_reject_code не отражает в точности объяснение RFC822 сообщаемого кода.

Чтобы избежать путаницы, вы можете изменить это, например, на 578, в противном случае не изменяйте это, если у вас нет полного понимания RFC 822.
