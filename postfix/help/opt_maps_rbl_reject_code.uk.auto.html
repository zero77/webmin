Відповідь SMTP-сервера на порушення доменів RBL
maps_rbl_reject_code
Цей параметр вказує код відповіді сервера SMTP, коли пошта порушує правило maps_rbl_domains яке:
Заборонено доступ: клієнт SMTP вказаний у списку RBL

Зауважте, що значення за замовчуванням Postfix, яке використовується для maps_rbl_reject_code - це код 554, і цей код у "стандартному RFC 822" повинен означати "Запрошена поштова дія відхилена: доступ заборонено". Отже, з Postfix, порушення правил maps_rbl_domains не відображає саме пояснення RFC822 звітного коду.

Щоб уникнути плутанини, ви можете змінити це на 571, наприклад (інша причина полягає в тому, що код 554 є також значенням за замовчуванням постфікса для інших видів помилок), інакше не змінюйте це, якщо ви не маєте повного розуміння RFC 822.
