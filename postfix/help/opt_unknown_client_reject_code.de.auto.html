SMTP-Serverantwort auf unbekanntem Client ablehnen
unknown_client_reject_code
Dieser Parameter gibt den Antwortcode des SMTP-Servers an, wenn ein Client ohne Zuordnung von Adresse zu Name gegen die Einschränkungsregel " reject_unknown_clients verstößt:
Zugriff verweigert: Unbekannter SMTP-Client-Hostname (ohne DNS A- oder MX-Eintrag)

Beachten Sie, dass der Postfix-Standardwert für unknown_client_reject_code Code 450 ist, der auch für andere Fehler verwendet wird. Dieser Code in "Standard-RFC 822" bedeutet "Angeforderte E-Mail-Aktion nicht ausgeführt: Postfach belegt oder Zugriff verweigert" und fordert den Absender auf, es später erneut zu versuchen (in der Hoffnung, dass der DNS später korrekt antwortet).

Ändern Sie dies nur, wenn Sie mit RFC 822 vollständig vertraut sind.
