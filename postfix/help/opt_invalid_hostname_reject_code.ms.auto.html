Respons pelayan SMTP pada nama host ditolak
invalid_hostname_reject_code
Parameter ini menentukan kod respons pelayan SMTP apabila pelanggan melanggar peraturan reject_invalid_hostname anti-UCE reject_invalid_hostname yaitu:
Sintaks yang tidak baik untuk nama host HELO klien (Tidak mematuhi RFC)

Perhatikan bahawa nilai lalai postfix yang digunakan untuk invalid_hostname_reject_code adalah kod 501, dan kod ini dalam 'RFC 822 standard' bermaksud 'Kesalahan sintaks dalam parameter atau argumen'. Jadi dengan Postfix, kod status 501 tidak menggambarkan penjelasan RFC822 dengan tepat.

Untuk mengelakkan kekeliruan, anda boleh mengubahnya menjadi 574 misalnya (sebab lain adalah bahawa kod 501 juga merupakan nilai lalai postfix untuk jenis kesalahan lain), jika tidak, jangan ubah ini melainkan anda mempunyai pemahaman lengkap tentang RFC 822.
