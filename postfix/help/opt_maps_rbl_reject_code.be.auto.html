Рэакцыя SMTP-сервера на парушэнне даменаў RBL
maps_rbl_reject_code
Гэты параметр паказвае код адказу SMTP-сервера, калі пошта парушае правіла maps_rbl_domains , якое:
Доступ забаронены: кліент SMTP пазначаны ў RBL

Звярніце ўвагу, што значэнне па змаўчанні maps_rbl_reject_code якое выкарыстоўваецца для maps_rbl_reject_code гэта код 554, і гэты код у "стандартным RFC 822" павінен азначаць "Запытаная дзеянне пошты адхілена: доступ забаронены". Такім чынам, у Postfix, парушэнне правіл maps_rbl_domains не адлюстроўвае дакладна тлумачэнне maps_rbl_domains паведамляемага кода.

Каб пазбегнуць блытаніны, вы можаце змяніць гэта, напрыклад, на 571 (іншая прычына заключаецца ў тым, што код 554 таксама з'яўляецца значэннем па змаўчанні постфікса для іншых відаў памылак), інакш не мяняйце гэта, калі вы не маеце поўнага разумення RFC 822.
