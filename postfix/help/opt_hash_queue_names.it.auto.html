Nome delle directory della coda divise tra i sottodir
hash_queue_names
Questo parametro consente di specificare i nomi delle directory delle code che sono suddivise su più livelli di sottodirectory. L'hash non DEVE essere usato con una directory maildrop scrivibile in tutto il mondo. L'hash DEVE essere utilizzato per la directory di rinvio del file di registro, altrimenti le prestazioni del sistema di posta ne risentiranno.

Sfortunatamente, l'hashing della coda in entrata o differita può effettivamente rallentare il sistema di posta ( mailq con una coda vuota può richiedere diversi secondi), quindi dovrebbe essere fatto solo in caso di emergenza.
