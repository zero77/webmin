디스크 할당량

소개

할당량은 사용자 및 그룹이 사용할 수있는 디스크 공간을 제어하는 방법입니다. 각 파일 시스템마다 디스크 공간 및 각 사용자가 사용할 수있는 파일 수에 제한을 둘 수 있습니다. 일부 Unix 시스템에서는 일부 그룹의 모든 구성원이 사용하는 공간과 파일을 제한 할 수도 있습니다.

할당량은 로컬 Unix 파일 시스템에 대해서만 설정할 수 있습니다. 다른 시스템에서 마운트 된 NFS 파일 시스템은 서버의 할당량을 사용합니다 (있는 경우). MS-DOS 또는 ISO9660과 같은 비 유닉스 파일 시스템은 할당량을 전혀 지원할 수 없습니다.

각 사용자마다 네 가지 제한이 적용됩니다.

소프트 블록 제한
경고 메시지가 표시되기 전에 허용되는 디스크 블록 수.
하드 블록 제한
사용자 파일이 차지할 수있는 최대 블록 수입니다.
소프트 파일 제한
경고 메시지가 표시되기 전에 허용되는 파일 수입니다.
하드 파일 제한
사용자가 소유 할 수있는 최대 파일 수입니다.
디스크 블록의 크기는 일반적으로 운영 체제에 따라 512 또는 1024 바이트입니다. 할당량을 계산할 때 모든 파일 크기는 가장 가까운 블록으로 반올림됩니다. 따라서 빈 파일조차도 적어도 블록에서 사용합니다.

파일 할당량을 위해 모든 디렉토리, 장치, 심볼릭 링크 또는 일반 파일은 사용자가 소유 할 수있는 최대 파일 수를 계산합니다. 기술적으로 파일 할당량은 실제로 inode 의 할당량입니다.

디스크 할당량 모듈

이 모듈의 메인 페이지는 할당량을 지원할 수있는 로컬 파일 시스템 목록을 보여줍니다. 현재 파일 시스템에 할당량이 활성화되어있는 경우 해당 경로를 클릭하여 각 사용자에 대한 할당량을 표시 할 수 있습니다. 그렇지 않은 경우, 활성화 된 할당량을 클릭하면 해당 파일 시스템에 대한 할당량 이 활성화됩니다.

파일 시스템에 디스크 및 파일 시스템 모듈에서 할당량 사용 옵션이 설정되어 있지 않으면 Linux 운영 체제에서 할당량 을 설정할 수 없습니다. 파일 시스템에이 옵션이 설정되어 있지 않으면 기본 페이지에 아무것도 표시되지 않습니다.

파일 시스템 목록 아래에는 선택한 사용자의 모든 할당량을 표시하는 버튼이 있습니다. 적어도 하나의 파일 시스템에 사용자 할당량이 활성화되어있는 경우에만 표시됩니다. 마찬가지로, 하나 이상의 파일 시스템에 그룹 할당량이 활성화되어 있으면 선택한 그룹에 대한 모든 할당량을 표시하는 버튼이 있습니다.
