Υπηρεσίες Διαδικτύου

Αυτή η σελίδα βοήθειας είναι ελλιπής

Εισαγωγή

Αυτή η ενότητα σάς επιτρέπει να διαχειριστείτε και να ρυθμίσετε υπηρεσίες διαδικτύου όπως telnet , ftp και finger . Κάθε υπηρεσία Διαδικτύου έχει ένα όνομα, αριθμό θύρας και πρωτόκολλο (όπως telnet στη θύρα tcp 23). Πολλές υπηρεσίες διαθέτουν επίσης ένα πρόγραμμα διακομιστή για τη διαχείριση συνδέσεων δικτύου σε αυτές τις υπηρεσίες.

Οι υπηρεσίες με ένα σχετικό πρόγραμμα ελέγχονται από το Unix inetd daemon, ο οποίος ακούει συνδέσεις δικτύου για λογαριασμό όλων των υπηρεσιών του. Όταν λαμβάνεται μια σύνδεση, το πρόγραμμα διακομιστή ξεκινά να διαχειρίζεται δεδομένα από τη νέα σύνδεση.

Δεν διαχειρίζονται όλες οι υπηρεσίες Διαδικτύου από αυτήν την ενότητα. Διακομιστές που λαμβάνουν πολλές συνδέσεις όπως HTTPD, SMB ή NFS εκτελούνται ξεχωριστά για να αποφύγουν την έναρξη μιας νέας διαδικασίας για κάθε αίτημα. Αυτές οι υπηρεσίες μπορούν να ρυθμιστούν χρησιμοποιώντας άλλες λειτουργικές μονάδες Webmin, όπως Samba ή Apache .

Η κύρια σελίδα για αυτήν την ενότητα παραθέτει όλες τις υπηρεσίες διαδικτύου και RPC στο σύστημά σας. Οι υπηρεσίες με εκχωρημένο πρόγραμμα εμφανίζονται με έντονη γραφή. Η λίστα των υπηρεσιών Διαδικτύου, των υπηρεσιών RPC και των προγραμμάτων λαμβάνονται από τα αρχεία /etc/services , /etc/rpc και /etc/inetd.conf .

Δημιουργία νέας υπηρεσίας

Για να δημιουργήσετε μια νέα υπηρεσία Διαδικτύου, κάντε κλικ στο σύνδεσμο Δημιουργία νέας υπηρεσίας κάτω από τη λίστα των ενεργών υπηρεσιών. Αυτό θα σας μεταφέρει σε μια φόρμα στην οποία μπορούν να εισαχθούν οι λεπτομέρειες της νέας υπηρεσίας:
  • Όνομα Υπηρεσίας
    Κάθε υπηρεσία πρέπει να έχει ένα μοναδικό ζεύγος ονόματος και πρωτοκόλλου. Αυτό σημαίνει ότι δύο υπηρεσίες μπορούν να έχουν το ίδιο όνομα μόνο εάν χρησιμοποιούν διαφορετικά πρωτόκολλα.

  • Πρωτόκολλο
    Το πρωτόκολλο επικοινωνίας που χρησιμοποιείται από αυτήν την υπηρεσία. Σχεδόν όλες οι υπηρεσίες Διαδικτύου χρησιμοποιούν το πρωτόκολλο TCP και οι υπόλοιπες χρησιμοποιούν UDP .

  • Λιμάνι
    Κάθε υπηρεσία έχει έναν αριθμό θύρας που πρέπει να είναι μοναδικός στο πρωτόκολλο που χρησιμοποιείται από την υπηρεσία. Ο αριθμός θύρας πρέπει να είναι μεταξύ 0 και 65535 και για το TCP και το UDP .

  • Ψευδώνυμα
    Μια λίστα με εναλλακτικά ονόματα που χρησιμοποιούνται για την αναφορά αυτής της υπηρεσίας. Ένα ψευδώνυμο δεν μπορεί να χρησιμοποιηθεί ήδη ως το όνομα ή το ψευδώνυμο μιας υπάρχουσας υπηρεσίας με το ίδιο πρωτόκολλο.

Τα ονόματα υπηρεσιών Διαδικτύου μπορούν να χρησιμοποιηθούν αντί για ονόματα κεντρικών υπολογιστών σε εντολές όπως το telnet . Για παράδειγμα, μπορείτε να πληκτρολογήσετε telnet foo.bar.com smtp για να συνδεθείτε στη θύρα 25 στον κεντρικό υπολογιστή foo.bar.com , υποθέτοντας ότι η υπηρεσία διαδικτύου smtp έχει οριστεί σωστά.

Για υπηρεσίες με ένα σχετικό πρόγραμμα, πρέπει να επιλέξετε την επιλογή Χρήση του παρακάτω προγράμματος ;;;

  • Πρόγραμμα
    Ορισμένες σπάνιες υπηρεσίες αντιμετωπίζονται εσωτερικά από το inetd . Ωστόσο, για σχεδόν όλες τις νέες υπηρεσίες που δημιουργείτε την πλήρη διαδρομή προς το πρόγραμμα που θα εκτελεστεί για την υπηρεσία θα πρέπει να δοθεί εδώ.

  • Επιχειρήματα
    Ορίσματα γραμμής εντολών για μετάβαση στο παραπάνω πρόγραμμα. Το πρώτο όρισμα πρέπει να είναι το όνομα του προγράμματος και οι υπόλοιπες παράμετροι που πρέπει να περάσουν.

  • Περιμένετε
    Αυτό καθορίζει εάν το inetd θα περιμένει έως ότου ολοκληρωθεί το πρόγραμμα πριν ξεκινήσει κάποιο άλλο ή όχι. Για τις περισσότερες υπηρεσίες, αυτό δεν πρέπει να περιμένετε .

  • Εκτελέστε ως
    Το όνομα χρήστη του χρήστη το πρόγραμμα διακομιστή πρέπει να εκτελεστεί ως. Οι περισσότεροι διακομιστές θα πρέπει να εκτελούνται ως root .

Όταν προστίθεται μια νέα υπηρεσία, δεν θα τεθεί σε ισχύ αμέσως. Πρέπει να κάνετε κλικ στο κουμπί Επανεκκίνηση Inetd στην κύρια σελίδα για να εφαρμοστεί η προσθήκη.

Επεξεργασία υπάρχουσας υπηρεσίας

Για να επεξεργαστείτε μια υπηρεσία, κάντε κλικ στο όνομα της υπηρεσίας από τη λίστα στην κύρια σελίδα. Αυτό θα εμφανίσει την ίδια φόρμα που χρησιμοποιήθηκε για τη δημιουργία μιας υπηρεσίας που σας επιτρέπει να επεξεργαστείτε τις λεπτομέρειες της υπηρεσίας. Μπορείτε επίσης να κάνετε κλικ στο κουμπί Διαγραφή στο κάτω μέρος της σελίδας για να διαγράψετε την υπηρεσία.

Να είστε προσεκτικοί όταν επεξεργάζεστε ή διαγράφετε υπάρχουσες υπηρεσίες συστήματος, όπως telnet ή ftp . Η αλλαγή αυτών ενδέχεται να καθιστά αδύνατη τη σύνδεση στο μηχάνημά σας (ή χειρότερα).

Όταν μια υπηρεσία τροποποιηθεί, δεν θα τεθεί σε ισχύ αμέσως. Πρέπει να κάνετε κλικ στο κουμπί Επανεκκίνηση Inetd στην κύρια σελίδα για να εφαρμοστούν οι αλλαγές.

Δημιουργία και επεξεργασία προγραμμάτων RPC

Γενικά, δεν θα χρειαστεί ποτέ να δημιουργήσετε ή να επεξεργαστείτε προγράμματα RPC στο σύστημά σας. Ωστόσο, εάν το κάνετε, η διαδικασία είναι παρόμοια με τη δημιουργία και την επεξεργασία υπηρεσιών Διαδικτύου.

Η φόρμα για επεξεργασία και δημιουργία προγραμμάτων RPC θα περιέχει πάντα τουλάχιστον τα ακόλουθα πεδία:

  • Όνομα προγράμματος
    Ένα μοναδικό όνομα για αυτό το πρόγραμμα RPC (μέσα στο ίδιο πρωτόκολλο).

  • Αριθμός
    Κάθε πρόγραμμα RPC έχει έναν μοναδικό αριθμό προγράμματος, που καθορίζεται πότε γράφεται το πρόγραμμα.

  • Ψευδώνυμα
    Άλλα ονόματα που μπορούν να χρησιμοποιηθούν για αναφορά σε αυτό το πρόγραμμα RPC.

Ορισμένα λειτουργικά συστήματα (όπως το Solaris) επιτρέπουν τη σύνδεση ενός προγράμματος διακομιστή με μια υπηρεσία RPC. Αυτό το πρόγραμμα θα εκτελεστεί από το inetd όταν λάβει ένα αίτημα για το πρόγραμμα.
