Perkhidmatan Internet

Halaman bantuan ini tidak lengkap

Pengenalan

Modul ini membolehkan anda mengurus dan mengatur perkhidmatan internet seperti telnet , ftp dan jari . Setiap perkhidmatan internet mempunyai nama, nombor port dan protokol (seperti telnet pada tcp port 23). Banyak perkhidmatan juga mempunyai program pelayan untuk mengendalikan sambungan rangkaian ke perkhidmatan tersebut.

Perkhidmatan dengan program yang berkaitan dikendalikan oleh Unix inetd daemon, yang mendengar sambungan rangkaian atas nama semua perkhidmatannya. Apabila sambungan diterima, program pelayan dimulakan untuk menangani data dari sambungan baru.

Tidak semua perkhidmatan internet dikendalikan oleh modul ini. Server yang menerima banyak sambungan seperti HTTPD, SMB atau NFS dijalankan secara berasingan untuk mengelakkan daripada memulakan proses baru untuk setiap permintaan. Perkhidmatan ini dapat dikonfigurasi menggunakan modul Webmin lain, seperti Samba atau Apache .

Halaman utama modul ini menyenaraikan semua perkhidmatan internet dan RPC pada sistem anda. Perkhidmatan dengan program yang ditetapkan ditunjukkan dengan huruf tebal. Senarai perkhidmatan internet, perkhidmatan dan program RPC diambil dari fail /etc/services , /etc/rpc dan /etc/inetd.conf .

Membuat Perkhidmatan Baru

Untuk membuat perkhidmatan internet baru, klik pada pautan Buat Perkhidmatan Baru di bawah senarai perkhidmatan aktif. Ini akan membawa anda ke borang di mana perincian perkhidmatan baru dapat dimasukkan:
  • Nama Perkhidmatan
    Setiap perkhidmatan mesti mempunyai pasangan nama dan protokol yang unik. Ini bermaksud dua perkhidmatan boleh mempunyai nama yang sama hanya jika mereka menggunakan protokol yang berbeza.

  • Protokol
    Protokol komunikasi yang digunakan oleh perkhidmatan ini. Hampir semua perkhidmatan internet menggunakan protokol TCP , dan selebihnya menggunakan UDP .

  • Pelabuhan
    Setiap perkhidmatan mempunyai nombor port yang mesti unik dalam protokol yang digunakan oleh perkhidmatan tersebut. Nombor port mestilah antara 0 dan 65535 untuk kedua-dua TCP dan UDP .

  • Alias
    Senarai nama alternatif yang digunakan untuk merujuk kepada perkhidmatan ini. Alias tidak boleh digunakan sebagai nama atau nama perkhidmatan sedia ada dengan protokol yang sama.

Nama perkhidmatan internet boleh digunakan sebagai ganti nama host dalam perintah seperti telnet . Sebagai contoh, anda boleh menaip telnet foo.bar.com smtp untuk menyambung ke port 25 di host foo.bar.com , dengan andaian perkhidmatan internet smtp ditentukan dengan betul.

Untuk perkhidmatan dengan program yang berkaitan, anda mesti memilih pilihan Gunakan program di bawah ini ???

  • Program
    Beberapa perkhidmatan yang jarang berlaku dikendalikan secara dalaman oleh inetd . Walau bagaimanapun, untuk hampir semua perkhidmatan baru yang anda buat jalan penuh ke program yang akan dilaksanakan untuk perkhidmatan tersebut harus diberikan di sini.

  • Hujah
    Argumen baris perintah untuk diteruskan ke program di atas. Argumen pertama mestilah nama program, dan selebihnya parameter lain yang harus dilalui.

  • Tunggu mod
    Ini menentukan sama ada inetd akan menunggu sehingga program selesai sebelum memulakan yang lain atau tidak. Untuk kebanyakan perkhidmatan, ini mestilah Jangan tunggu .

  • Laksanakan sebagai
    Nama pengguna pengguna program pelayan harus dijalankan seperti. Sebilangan besar pelayan perlu dijalankan sebagai root .

Apabila perkhidmatan baru ditambahkan, perkhidmatan tersebut tidak akan langsung berlaku. Anda mesti mengklik butang Restart Inetd di halaman utama untuk penambahan yang akan digunakan.

Menyunting Perkhidmatan Sedia Ada

Untuk mengedit perkhidmatan, klik pada nama perkhidmatan dari senarai di halaman utama. Ini akan memaparkan bentuk yang sama yang digunakan untuk membuat perkhidmatan untuk membolehkan anda mengedit butiran perkhidmatan. Anda juga boleh mengklik butang Padam di bahagian bawah halaman untuk menghapus perkhidmatan.

Berhati-hatilah semasa mengedit atau menghapus perkhidmatan sistem yang ada seperti telnet atau ftp . Mengubahnya mungkin tidak mungkin masuk ke mesin anda (atau lebih teruk).

Apabila perkhidmatan diubah, perkhidmatan tersebut tidak akan berkuatkuasa serta-merta. Anda mesti mengklik butang Restart Inetd di halaman utama untuk perubahan yang akan dilaksanakan.

Membuat dan Menyunting Program RPC

Secara amnya, anda tidak perlu membuat atau menyunting program RPC pada sistem anda. Walau bagaimanapun, jika anda melakukan prosedur ini serupa dengan membuat dan menyunting perkhidmatan internet.

Borang untuk mengedit dan membuat program RPC akan selalu berisi sekurang-kurangnya bidang berikut:

  • Nama Program
    Nama unik untuk program RPC ini (dalam protokol yang sama).

  • Nombor
    Setiap program RPC mempunyai nombor program yang unik, ditentukan ketika program ditulis.

  • Alias
    Nama lain yang boleh digunakan untuk merujuk kepada program RPC ini.

Beberapa sistem operasi (seperti Solaris) membenarkan program pelayan dikaitkan dengan perkhidmatan RPC. Program ini akan dijalankan oleh inetd apabila menerima permintaan untuk program tersebut.
