OpenLDAP-Serverkonfiguration
Auf dieser Seite können Sie globale Einstellungen für Ihren OpenLDAP-Server konfigurieren, z. B. den Stamm-DN für die Datenbank, die Administratoranmeldung, die Cache-Größen und SSL-Zertifikate. Die am häufigsten geänderten Felder sind:
Root-DN für LDAP-Datenbank
In dieses Feld wird der DN (definierter Name) eingegeben, unter dem alle Objekte in Ihrer LDAP-Datenbank gespeichert werden müssen. Normalerweise sollte es in etwas geändert werden, das dem Namen Ihres Unternehmens oder Ihrer Organisation entspricht, z. B. dc = yourcompany, dc = com . Sie können jedoch alles eingeben, solange es korrekt formatiert ist.

Administrations-Login-DN
Dies ist effektiv der Root- Benutzer für Ihre LDAP-Datenbank. Es sollte ein DN unter Ihrem Stammsatz im obigen Feld sein, wie cn = Manager, dc = yourcompany, dc = com .

Administrationskennwort
Dieses Feld zeigt das aktuelle Passwort (möglicherweise verschlüsselt) und eine Option zur Eingabe eines neuen. Wenn Sie das Kennwort ändern, verwendet das Modul es automatisch, wenn eine Verbindung zur Datenbank hergestellt wird, um Änderungen vorzunehmen.

Alle auf dieser Seite vorgenommenen Änderungen werden erst wirksam, wenn Sie auf der Hauptseite des Moduls auf die Schaltfläche Konfiguration anwenden klicken .

Wenn Ihr LDAP-Server TLS-verschlüsselte Verbindungen akzeptieren soll, müssen Sie zuerst ein SSL-Zertifikat und einen öffentlichen Schlüssel generieren. Dies kann größtenteils automatisiert werden, indem Sie unten auf der Seite auf die Schaltfläche SSL-Zertifikat generieren klicken.
