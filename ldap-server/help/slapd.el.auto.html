Διαμόρφωση διακομιστή OpenLDAP
Αυτή η σελίδα σάς επιτρέπει να διαμορφώσετε καθολικές ρυθμίσεις για τον διακομιστή OpenLDAP, όπως το root DN για τη βάση δεδομένων, τα στοιχεία σύνδεσης διαχείρισης, τα μεγέθη προσωρινής μνήμης και τα πιστοποιητικά SSL. Τα πιο συχνά αλλαγμένα πεδία είναι:
Root DN για βάση δεδομένων LDAP
Αυτό το πεδίο προορίζεται για την εισαγωγή του DN (διακεκριμένο όνομα) στο οποίο πρέπει να αποθηκευτούν όλα τα αντικείμενα στη βάση δεδομένων LDAP. Συνήθως θα πρέπει να αλλάξει σε κάτι που να ταιριάζει με το όνομα της εταιρείας ή του οργανισμού σας, όπως dc = yourcompany, dc = com . Αλλά είστε ελεύθεροι να εισαγάγετε οτιδήποτε αρκεί να είναι σωστά μορφοποιημένο.

Σύνδεση διαχείρισης DN
Αυτός είναι ουσιαστικά ο ριζικός χρήστης της βάσης δεδομένων LDAP. Θα πρέπει να είναι DN κάτω από το root set σας στο παραπάνω πεδίο, όπως cn = Manager, dc = yourcompany, dc = com .

Κωδικός διαχείρισης
Αυτό το πεδίο εμφανίζει τον τρέχοντα κωδικό πρόσβασης (ενδεχομένως κρυπτογραφημένο) και μια επιλογή εισαγωγής νέου. Εάν αλλάξετε τον κωδικό πρόσβασης, η μονάδα θα τον χρησιμοποιήσει αυτόματα κατά τη σύνδεση στη βάση δεδομένων για να πραγματοποιήσει αλλαγές.

Τυχόν αλλαγές που γίνονται σε αυτήν τη σελίδα δεν θα τεθούν σε ισχύ έως ότου πατηθεί το κουμπί Εφαρμογή διαμόρφωσης στην κύρια σελίδα της μονάδας.

Εάν θέλετε ο διακομιστής LDAP να δέχεται κρυπτογραφημένες συνδέσεις TLS, πρέπει πρώτα να δημιουργήσετε ένα πιστοποιητικό SSL και ένα δημόσιο κλειδί. Αυτό μπορεί να αυτοματοποιηθεί κυρίως κάνοντας κλικ στο κουμπί Δημιουργία πιστοποιητικού SSL στο κάτω μέρος της σελίδας.
