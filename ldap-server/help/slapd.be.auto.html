Канфігурацыя сервера OpenLDAP
Гэтая старонка дазваляе наладзіць глабальныя налады для вашага сервера OpenLDAP, такія як каранёвая DN для базы дадзеных, адміністрацыйны ўваход, памеры кэша і сертыфікаты SSL. Самыя распаўсюджаныя палі:
Root DN для базы дадзеных LDAP
Гэта поле для ўводу DN (адрознівае імя), пад якім павінны захоўвацца ўсе аб'екты ў вашай базе дадзеных LDAP. Звычайна яго трэба змяніць на нешта, якое адпавядае назве вашай кампаніі ці арганізацыі, напрыклад, dc = yourcompany, dc = com . Але вы можаце ўвесці што заўгодна, пакуль ён правільна адфарматаваны.

Уваход адміністрацыі DN
Гэта эфектыўна для каранёвага карыстальніка вашай базы дадзеных LDAP. Гэта павінна быць DN пад вашым каранёвым наборам у вышэйзгаданым полі, як cn = Manager, dc = yourcompany, dc = com .

Пароль адміністрацыі
У гэтым полі паказаны бягучы пароль (магчыма, зашыфраваны) і магчымасць увесці новы. Пры змене пароля модуль будзе выкарыстоўваць яго аўтаматычна пры падключэнні да базы дадзеных для ўнясення змяненняў.

Любыя змены, зробленыя на гэтай старонцы, не ўступяць у сілу, пакуль на галоўнай старонцы модуля не будзе націснута кнопка Ужыць канфігурацыю .

Калі вы хочаце, каб ваш сервер LDAP прыняў зашыфраваныя злучэнні TLS, неабходна спачатку стварыць сертыфікат SSL і адкрыты ключ. У асноўным гэта можна аўтаматызаваць, націснуўшы кнопку Стварыць сертыфікат SSL унізе старонкі.
