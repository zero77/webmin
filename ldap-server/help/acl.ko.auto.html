LDAP 액세스 제어
기본적으로 LDAP 서버는 연결할 수있는 모든 클라이언트가 데이터베이스의 모든 객체와 속성을 읽도록 허용합니다. 그러나 관리 사용자 만 업데이트를 수행 할 수 있습니다. 그러나 신뢰 수준이 다른 사용자가있는 네트워크에는 적합하지 않을 수 있으므로 OpenLDAP을 사용하면 데이터베이스의 다른 부분에있는 다른 사용자에게 다양한 액세스 수준을 부여 할 수 있습니다.

이 페이지에는 현재 정의 된 모든 액세스 제어 규칙이 나열됩니다 (있는 경우). 새로 작성하려면 새 액세스 제어 규칙 추가 링크를 클릭하십시오. 여러 규칙을 한 번에 제거하려면 옆에있는 확인란을 선택하고 선택한 규칙 삭제 버튼을 누르십시오. 규칙 순서를 변경하려면 표 오른쪽에있는 위쪽 및 아래쪽 화살표를 사용하십시오.

대부분의 다른 LDAP 서버 구성 변경과 마찬가지로 모듈의 기본 페이지에서 Apply Configuration (구성 적용) 버튼을 클릭 할 때까지 액세스 제어 규칙이 적용 되지 않습니다.
