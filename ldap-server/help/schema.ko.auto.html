스키마 관리
LDAP 스키마는 LDAP 데이터베이스에 저장할 수있는 객체 클래스 및 속성을 결정합니다. 이 페이지에서는 각 스키마 파일 이름 옆에있는 확인란을 사용하여 서버에서 지원하는 스키마 유형을 선택할 수 있습니다. 변경 한 후에는 저장 을 클릭하여 OpenLDAP 구성 파일을 업데이트 한 다음 기본 페이지에서 구성 적용을 클릭하여 활성화하십시오.

그러나 기존 스키마 파일을 선택 취소하면 해당 파일에 정의 된 속성을 사용하여 데이터베이스에 오브젝트가 이미 존재하는 경우 LDAP 서버가 손상 될 수 있습니다. 가장 기본적인 LDAP 클래스를 포함하는 핵심 스키마는 선택 해제 할 수 없습니다.

이후 스키마는 이전에 정의 된 속성 만 참조 할 수 있으므로 스키마의 순서는 중요합니다. 이 페이지의 위쪽 및 아래쪽 화살표를 사용하여 목록의 순서를 다시 지정할 수 있지만 LDAP 데이터베이스가 손상되지 않도록 기존 스키마를주의하여 다시 수행해야합니다.

스키마가 정의한 클래스와 속성을 보려면 해당 보기 링크를 클릭하십시오. 스키마 형식에 익숙하고 파일을 편집하려면 대신 편집 을 클릭 하십시오 .
