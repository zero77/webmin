Configuración del servidor OpenLDAP
Esta página le permite configurar ajustes globales para su servidor OpenLDAP, como el DN raíz de la base de datos, inicio de sesión de administración, tamaños de caché y certificados SSL. Los campos más comúnmente modificados son:
DN raíz para la base de datos LDAP
Este campo es para ingresar el DN (nombre distinguido) bajo el cual deben almacenarse todos los objetos en su base de datos LDAP. Por lo general, debe cambiarse por algo que coincida con el nombre de su empresa u organización, como dc = yourcompany, dc = com . Pero puede ingresar cualquier cosa siempre que esté formateada correctamente.

DN de inicio de sesión de administración
Este es efectivamente el usuario raíz de su base de datos LDAP. Debe ser un DN debajo de su conjunto raíz en el campo anterior, como cn = Manager, dc = yourcompany, dc = com .

Contraseña de administración
Este campo muestra la contraseña actual (posiblemente cifrada) y una opción para ingresar una nueva. Si cambia la contraseña, el módulo la usará automáticamente cuando se conecte a la base de datos para realizar cambios.

Cualquier cambio realizado en esta página no tendrá efecto hasta que se haga clic en el botón Aplicar configuración en la página principal del módulo.

Si desea que su servidor LDAP acepte conexiones cifradas TLS, primero debe generar un certificado SSL y una clave pública. Esto se puede automatizar principalmente haciendo clic en el botón Generar certificado SSL en la parte inferior de la página.
