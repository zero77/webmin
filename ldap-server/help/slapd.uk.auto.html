Конфігурація сервера OpenLDAP
Ця сторінка дозволяє налаштувати глобальні налаштування для вашого сервера OpenLDAP, такі як кореневий DN для бази даних, вхід в систему адміністрації, розміри кешу та сертифікати SSL. Найпоширеніші поля:
Корінь DN для бази даних LDAP
Це поле призначене для введення DN (виділене ім'я), під яким повинні зберігатися всі об'єкти у вашій базі даних LDAP. Зазвичай його слід змінити на щось, що відповідає назві вашої компанії чи організації, наприклад, dc = ваша компанія , dc = com . Але ви можете ввести що завгодно, доки він правильно відформатований.

Логін адміністрації DN
Це фактично користувач root для вашої бази даних LDAP. Це має бути DN під вашим кореневим набором у вищевказаному полі, як cn = Manager, dc = yourcompany, dc = com .

Пароль адміністрації
Це поле показує поточний пароль (можливо, зашифрований) та можливість ввести новий. Якщо ви зміните пароль, модуль буде використовувати його автоматично під час підключення до бази даних для внесення змін.

Будь-які зміни, внесені на цій сторінці, не наберуть чинності, доки не буде натиснуто кнопку Застосувати конфігурацію на головній сторінці модуля.

Якщо ви хочете, щоб ваш сервер LDAP приймав зашифровані TLS з'єднання, спершу потрібно створити сертифікат SSL та відкритий ключ. Здебільшого це можна автоматизувати, натиснувши кнопку Генерувати сертифікат SSL внизу сторінки.
