管理架构
LDAP模式确定哪些对象类和属性可以存储在LDAP数据库中。该页面允许您使用每个模式文件名旁边的复选框来选择服务器支持的模式类型。进行更改后，单击“ 保存”以更新OpenLDAP配置文件，然后在主页上应用“应用配置”以将其激活。

但是请小心取消选择现有模式文件，因为如果使用这些文件中定义的属性在数据库中已经存在对象，这可能会破坏LDAP服务器。无法取消选择包含最基本的LDAP类的核心架构。

模式的顺序很重要，因为以后的模式只能引用先前定义的属性。您可以使用此页面上的向上和向下箭头对列表进行重新排序，但是同样，请谨慎使用现有模式，以免破坏LDAP数据库。

要查看架构定义的类和属性，请单击其“ 查看”链接。如果您熟悉架构格式，并且想要编辑文件，请单击“ 编辑” 。
