Configurazione del server OpenLDAP
Questa pagina consente di configurare le impostazioni globali per il server OpenLDAP, ad esempio il DN radice per il database, l'accesso di amministrazione, le dimensioni della cache e i certificati SSL. I campi più comunemente modificati sono:
DN radice per database LDAP
Questo campo serve per immettere il DN (nome distinto) in cui devono essere memorizzati tutti gli oggetti nel database LDAP. In genere, dovrebbe essere modificato in qualcosa corrispondente al nome della tua azienda o organizzazione, come dc = yourcompany, dc = com . Ma sei libero di inserire qualsiasi cosa purché sia formattato correttamente.

DN di accesso amministrazione
Questo è effettivamente l'utente root per il tuo database LDAP. Dovrebbe essere un DN sotto il set di root nel campo sopra, come cn = Manager, dc = yourcompany, dc = com .

Password di amministrazione
Questo campo mostra la password corrente (possibilmente crittografata) e un'opzione per inserirne una nuova. Se si modifica la password, il modulo la utilizzerà automaticamente durante la connessione al database per apportare modifiche.

Eventuali modifiche apportate a questa pagina non avranno effetto fino a quando non si fa clic sul pulsante Applica configurazione sulla pagina principale del modulo.

Se vuoi che il tuo server LDAP accetti connessioni crittografate TLS, devi prima generare un certificato SSL e una chiave pubblica. Ciò può essere per lo più automatizzato facendo clic sul pulsante Genera certificato SSL nella parte inferiore della pagina.
