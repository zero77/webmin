Configuração do servidor OpenLDAP
Esta página permite definir configurações globais para o servidor OpenLDAP, como o DN raiz do banco de dados, login de administração, tamanhos de cache e certificados SSL. Os campos alterados com mais frequência são:
DN raiz para banco de dados LDAP
Este campo é para inserir o DN (nome distinto) sob o qual todos os objetos em seu banco de dados LDAP devem ser armazenados. Normalmente, ele deve ser alterado para algo que corresponda ao nome da sua empresa ou organização, como dc = suaempresa, dc = com . Mas você é livre para inserir qualquer coisa, desde que esteja corretamente formatado.

DN de login da administração
Este é efetivamente o usuário raiz do seu banco de dados LDAP. Deve ser um DN em seu conjunto raiz no campo acima, como cn = Manager, dc = yourcompany, dc = com .

Senha de administração
Este campo mostra a senha atual (possivelmente criptografada) e uma opção para inserir uma nova. Se você alterar a senha, o módulo a utilizará automaticamente ao conectar-se ao banco de dados para fazer alterações.

Quaisquer alterações feitas nesta página não terão efeito até que o botão Aplicar configuração seja clicado na página principal do módulo.

Se você deseja que seu servidor LDAP aceite conexões criptografadas por TLS, primeiro você deve gerar um certificado SSL e uma chave pública. Isso pode ser automatizado principalmente ao clicar no botão Gerar certificado SSL na parte inferior da página.
