Konfigurasi Pelayan OpenLDAP
Halaman ini membolehkan anda mengkonfigurasi tetapan global untuk pelayan OpenLDAP anda, seperti root DN untuk pangkalan data, log masuk pentadbiran, ukuran cache dan sijil SSL. Medan yang paling biasa diubah adalah:
Root DN untuk pangkalan data LDAP
Medan ini adalah untuk memasukkan DN (nama yang dibezakan) di mana semua objek dalam pangkalan data LDAP anda mesti disimpan. Biasanya ia harus diubah menjadi sesuatu yang sepadan dengan nama syarikat atau organisasi anda, seperti dc = syarikat anda, dc = com . Tetapi anda bebas memasukkan apa sahaja asalkan diformat dengan betul.

Log masuk pentadbiran DN
Ini adalah pengguna root untuk pangkalan data LDAP anda dengan berkesan. Ini mestilah DN di bawah root root anda di bidang di atas, seperti cn = Manager, dc = yourcompany, dc = com .

Kata laluan pentadbiran
Medan ini menunjukkan kata laluan semasa (mungkin disulitkan), dan pilihan untuk memasukkan kata laluan baru. Sekiranya anda menukar kata laluan, modul akan menggunakannya secara automatik semasa menyambung ke pangkalan data untuk membuat perubahan.

Sebarang perubahan yang dibuat di halaman ini tidak akan berlaku sehingga butang Terapkan Konfigurasi diklik di halaman utama modul.

Sekiranya anda mahu pelayan LDAP anda menerima sambungan yang disulitkan TLS, anda mesti membuat sijil SSL dan kunci awam terlebih dahulu. Ini sebahagian besarnya dapat automatik dengan mengklik butang Buat Sijil SSL di bahagian bawah halaman.
