Programma di rotazione
Controlla la frequenza con cui il registro viene ruotato.

Dimensione massima prima della rotazione
Imposta un limite sulla dimensione del file di registro prima che venga ruotato. Questo campo può essere utilizzato al posto del programma di rotazione se si preferisce una dimensione piuttosto che un limite basato sul tempo.

Numero di vecchi registri da conservare
Determina quante vecchie generazioni del file di registro salvare. Questi avranno in genere lo stesso nome del registro originale, ma con .0, .1 e così via aggiunti. Se si immette 0 , non verranno conservati registri vecchi e quindi il contenuto del file di registro andrà perso quando viene ruotato.

Comprimi i vecchi file di registro?
Quando questa opzione è abilitata, i vecchi registri verranno compressi con gzip dopo la rotazione per ridurre la quantità di spazio su disco che consumano.

Ruota anche se il file di registro è vuoto?
Se abilitata, questa opzione indica a logrotate di ruotare ancora il file di registro anche se è vuoto, determinando il salvataggio di vecchie copie con dimensione zero.

Ricreare il file di registro dopo la rotazione?
Quando questa opzione è abilitata, il file di registro verrà ricreato dopo la rotazione invece di essere semplicemente lasciato al server che lo scrive per ricrearlo. Per impostazione predefinita, verrà creato con le autorizzazioni e il proprietario originali, ma è possibile utilizzare i campi accanto all'opzione Sì per specificare autorizzazioni file, utente e gruppo alternativi.

Archivia i vecchi log ruotati
Quando questo campo è impostato sulla stessa directory del file di registro , le vecchie copie ruotate del file di registro verranno archiviate nella stessa directory del registro originale. L'opzione Directory può essere utilizzata per specificare un percorso alternativo, tuttavia deve trovarsi sullo stesso filesystem del file di registro.

Comandi da eseguire dopo la rotazione
Quando questa casella di testo viene compilata, i comandi di shell immessi verranno eseguiti dopo che il registro è stato ruotato. Molti server devono essere riavviati dopo che il loro registro è stato rinominato, in modo da scrivere correttamente nel nuovo file. In questo caso, è possibile immettere un comando come killall -HUP httpd per attivare un riavvio dopo la rotazione.
