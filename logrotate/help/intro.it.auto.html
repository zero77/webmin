Rotazione del file di registro
La rotazione dei file di registro si riferisce al troncamento automatico, alla compressione e alla cancellazione dei file di registro in modo che non consumino troppo spazio su disco. La maggior parte dei server Unix (come Apache, Squid e Sendmail) generano file di registro e anche vari daemon di sistema creano registri tramite Syslog. Questo modulo può essere utilizzato per configurare il programma logrotate per gestire tutti quei registri.

In genere, un file di registro viene ruotato una volta ogni giorno, settimana o mese. Il file viene generalmente spostato in un nuovo nome file e compresso e al suo posto viene creato un nuovo file vuoto. Diverse generazioni di questi vecchi file di registro possono essere conservate, in modo da poter cercare o generare rapporti da esse anche dopo la rotazione. Una volta che il numero di vecchi registri supera un limite configurato, il più vecchio verrà eliminato.

Ogni file di registro che viene ruotato è elencato nella pagina principale del modulo. Ogni registro ha il proprio set di opzioni per controllare la frequenza con cui viene ruotato, quante copie precedenti vengono conservate e così via. È possibile modificare le opzioni per un registro facendo clic sul suo nome file o aggiungere un nuovo file di registro utilizzando il collegamento nella parte inferiore o superiore della tabella.

Molte distribuzioni Linux includono logrotate come standard e vengono fornite con configurazioni per la rotazione dei registri dei server inclusi come Apache e Squid. Quindi, anche se non hai mai usato questo modulo prima, molti file di registro potrebbero essere già elencati nella pagina principale. Spesso nella parte inferiore della pagina è presente un pulsante per la modifica della configurazione globale, che si applica a tutti i file di registro a meno che non venga sovrascritto. Sotto c'è un pulsante per impostare la pianificazione su cui logrotate viene eseguito da Cron, che è necessario per poter effettivamente ruotare i log. Sulla maggior parte dei sistemi operativi che includono il programma come standard, un processo Cron sarà già stato creato come parte del processo di installazione.
