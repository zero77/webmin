Horario de rotación
Controla con qué frecuencia se rota el registro.

Tamaño máximo antes de rotar
Establece un límite en el tamaño del archivo de registro antes de que se rote. Este campo se puede usar en lugar del programa de rotación si prefiere un tamaño en lugar del límite basado en el tiempo.

Número de registros antiguos para mantener
Determina cuántas generaciones antiguas del archivo de registro guardar. Por lo general, tendrán el mismo nombre que el registro original, pero con .0, .1 y así sucesivamente. Si ingresa 0 , no se guardarán registros antiguos y, por lo tanto, el contenido del archivo de registro se perderá cuando se gire.

Comprimir archivos de registro antiguos?
Cuando esta opción está habilitada, los registros antiguos se comprimirán con gzip después de la rotación para reducir la cantidad de espacio en disco que consumen.

Rotar incluso si el archivo de registro está vacío?
Si está habilitada, esta opción le dice a logrotate que todavía rote el archivo de registro incluso si está vacío, lo que da como resultado copias antiguas con tamaño cero que se guardan.

¿Volver a crear el archivo de registro después de la rotación?
Cuando esta opción está habilitada, el archivo de registro se volverá a crear después de la rotación en lugar de simplemente dejarse para que el servidor que escribe en él lo vuelva a crear. De forma predeterminada, se creará con los permisos y el propietario originales, pero puede usar los campos junto a la opción Sí para especificar permisos de archivo alternativo, usuario y grupo.

Almacenar registros rotados antiguos en
Cuando este campo se establece en el mismo directorio que el archivo de registro , las copias antiguas rotadas del archivo de registro se almacenarán en el mismo directorio que el registro original. La opción Directorio puede usarse para especificar una ubicación alternativa; sin embargo, debe estar en el mismo sistema de archivos que el archivo de registro.

Comandos para ejecutar después de la rotación
Cuando se completa este cuadro de texto, los comandos de shell ingresados se ejecutarán después de que se haya girado el registro. Muchos servidores deben reiniciarse después de que se haya cambiado el nombre de su registro, para que puedan escribir correctamente en el nuevo archivo. En este caso, puede ingresar un comando como killall -HUP httpd para activar un reinicio después de la rotación.
