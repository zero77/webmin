Rotationsplan
Steuert, wie oft das Protokoll gedreht wird.

Maximale Größe vor dem Drehen
Legt eine Begrenzung für die Größe der Protokolldatei fest, bevor diese gedreht wird. Dieses Feld kann anstelle des Rotationsplans verwendet werden, wenn Sie eine Größe anstelle eines zeitbasierten Limits bevorzugen.

Anzahl der alten Protokolle, die aufbewahrt werden sollen
Legt fest, wie viele alte Generationen der Protokolldatei gespeichert werden sollen. Diese haben normalerweise den gleichen Namen wie das ursprüngliche Protokoll, jedoch werden .0, .1 usw. angehängt. Wenn Sie 0 eingeben, werden keine alten Protokolle gespeichert, und der Inhalt der Protokolldatei geht verloren, wenn sie gedreht wird.

Alte Protokolldateien komprimieren?
Wenn diese Option aktiviert ist, werden alte Protokolle nach der Drehung mit gzip komprimiert , um den von ihnen belegten Speicherplatz zu reduzieren.

Drehen, auch wenn die Protokolldatei leer ist?
Wenn diese Option aktiviert ist, weist sie logrotate an, die Protokolldatei auch dann noch zu drehen, wenn sie leer ist. Dies führt dazu, dass alte Kopien mit der Größe Null gespeichert werden.

Protokolldatei nach Rotation neu erstellen?
Wenn diese Option aktiviert ist, wird die Protokolldatei nach der Drehung neu erstellt, anstatt nur dem Server, der darauf schreibt, eine Neuerstellung zu überlassen. Standardmäßig wird es mit den ursprünglichen Berechtigungen und dem ursprünglichen Eigentümer erstellt. Sie können jedoch die Felder neben der Option Ja verwenden , um alternative Dateiberechtigungen, Benutzer und Gruppen anzugeben.

Speichern Sie alte gedrehte Logs
Wenn dieses Feld auf das gleiche Verzeichnis wie die Protokolldatei festgelegt ist , werden die alten gedrehten Kopien der Protokolldatei im selben Verzeichnis wie das ursprüngliche Protokoll gespeichert. Mit der Option " Verzeichnis " können Sie einen alternativen Speicherort angeben. Sie muss sich jedoch im selben Dateisystem wie die Protokolldatei befinden.

Befehle, die nach der Drehung ausgeführt werden sollen
Wenn dieses Textfeld ausgefüllt ist, werden die eingegebenen Shell-Befehle ausgeführt, nachdem das Protokoll gedreht wurde. Viele Server müssen nach dem Umbenennen ihres Protokolls neu gestartet werden, damit sie ordnungsgemäß in die neue Datei schreiben können. In diesem Fall können Sie einen Befehl wie killall -HUP httpd eingeben , um nach der Drehung einen Neustart auszulösen.
