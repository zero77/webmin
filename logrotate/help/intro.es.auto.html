Rotación de archivo de registro
La rotación de archivos de registro se refiere al truncamiento automático, la compresión y la eliminación de archivos de registro para que no consuman demasiado espacio en disco. La mayoría de los servidores Unix (como Apache, Squid y Sendmail) generan archivos de registro, y varios demonios del sistema también crean registros a través de Syslog. Este módulo se puede usar para configurar el programa logrotate para administrar todos esos registros.

Normalmente, un archivo de registro se rotará una vez al día, semana o mes. El archivo generalmente se mueve a un nuevo nombre de archivo y se comprime, y se crea un nuevo archivo vacío en su lugar. Se pueden conservar varias generaciones de estos archivos de registro antiguos, para que pueda buscar o generar informes a partir de ellos incluso después de la rotación. Una vez que el número de registros antiguos excede un límite configurado, se eliminará el más antiguo.

Cada archivo de registro que se rota aparece en la página principal del módulo. Cada registro tiene su propio conjunto de opciones para controlar con qué frecuencia se rota, cuántas copias antiguas se conservan, etc. Puede editar las opciones para un registro haciendo clic en su nombre de archivo o agregar un nuevo archivo de registro utilizando el enlace en la parte inferior o superior de la tabla.

Muchas distribuciones de Linux incluyen logrotate como estándar y vienen con configuraciones para rotar los registros de los servidores incluidos como Apache y Squid. Entonces, incluso si nunca antes ha usado este módulo, es posible que muchos archivos de registro ya estén listados en la página principal. A menudo, cerca del final de la página hay un botón para editar la configuración global, que se aplica a todos los archivos de registro a menos que se anule. Debajo hay un botón para configurar la programación en la que Cron ejecuta logrotate , que es necesario para que realmente gire los registros. En la mayoría de los sistemas operativos que incluyen el programa como estándar, ya se habrá creado un trabajo Cron como parte del proceso de instalación.
