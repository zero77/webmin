График ротации
Управляет частотой вращения журнала.

Максимальный размер до вращения
Устанавливает ограничение на размер файла журнала до его поворота. Это поле можно использовать вместо графика ротации, если вы предпочитаете размер, а не ограничение по времени.

Количество старых журналов, чтобы сохранить
Определяет, сколько старых поколений файла журнала сохранить. Обычно они имеют то же имя, что и исходный журнал, но с добавлением .0, .1 и т. Д. Если вы введете 0 , старые журналы не будут сохраняться, и, следовательно, содержимое файла журнала будет потеряно при его вращении.

Сжатие старых файлов журнала?
Когда эта опция включена, старые журналы будут сжаты с помощью gzip после ротации, чтобы уменьшить объем занимаемого ими дискового пространства.

Поворот, даже если файл журнала пуст?
Если этот параметр включен, logrotate по-прежнему поворачивает файл журнала, даже если он пустой, в результате чего сохраняются старые копии с нулевым размером.

Повторно создать файл журнала после поворота?
Когда эта опция включена, файл журнала будет воссоздан после ротации, вместо того, чтобы просто быть оставленным для сервера, который пишет на него для повторного создания. По умолчанию он будет создан с исходными разрешениями и владельцем, но вы можете использовать поля рядом с параметром Да, чтобы указать альтернативные права доступа к файлу, пользователя и группы.

Хранить старые повернутые логи в
Если для этого поля установлено значение «Тот же каталог, что и у файла журнала» , старые повернутые копии файла журнала будут храниться в том же каталоге, что и исходный журнал. Параметр Каталог может использоваться для указания альтернативного расположения, однако он должен находиться в той же файловой системе, что и файл журнала.

Команды для запуска после поворота
Когда это текстовое поле заполнено, введенные команды оболочки будут выполняться после поворота журнала. Многие серверы необходимо перезапустить после переименования их журнала, чтобы они правильно записали в новый файл. В этом случае вы можете ввести команду типа killall -HUP httpd для запуска перезапуска после ротации.
