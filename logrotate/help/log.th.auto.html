กำหนดการหมุน
ควบคุมความถี่ในการหมุนของบันทึก

ขนาดสูงสุดก่อนหมุน
ตั้งค่าขีด จำกัด ขนาดของไฟล์บันทึกก่อนที่จะหมุน ฟิลด์นี้สามารถใช้แทนกำหนดการหมุนได้หากคุณต้องการขนาดแทนที่จะ จำกัด ตามเวลา

จำนวนบันทึกเก่าที่จะเก็บ
กำหนดว่าจะบันทึกไฟล์บันทึกอายุเท่าไหร่ โดยทั่วไปจะมีชื่อเหมือนกับบันทึกดั้งเดิม แต่มี. 0, .1 และต่อท้าย หากคุณป้อน 0 จะ ไม่มีการบันทึกเก่า ๆ ดังนั้นเนื้อหาของไฟล์บันทึกจะหายไปเมื่อมีการหมุน

บีบอัดไฟล์บันทึกเก่าหรือไม่
เมื่อเปิดใช้งานตัวเลือกนี้บันทึกเก่าจะถูกบีบอัดด้วย gzip หลังจากการหมุนเพื่อลดปริมาณพื้นที่ดิสก์ที่ใช้

หมุนแม้ว่าไฟล์บันทึกจะว่างเปล่า?
หากเปิดใช้งานตัวเลือกนี้จะบอกให้ logrotate ยังคงหมุนไฟล์บันทึกแม้ว่าจะว่างเปล่าส่งผลให้สำเนาเก่าที่มีขนาดศูนย์ถูกบันทึก

สร้างไฟล์บันทึกใหม่หลังจากการหมุนหรือไม่
เมื่อเปิดใช้งานตัวเลือกนี้ไฟล์บันทึกจะถูกสร้างขึ้นใหม่หลังจากการหมุนแทนที่จะปล่อยให้เซิร์ฟเวอร์ที่เขียนไปสร้างใหม่อีกครั้ง โดยค่าเริ่มต้นจะถูกสร้างขึ้นด้วยการอนุญาตดั้งเดิมและเจ้าของ แต่คุณสามารถใช้ฟิลด์ถัดจากตัวเลือก ใช่ เพื่อระบุการอนุญาตไฟล์ทางเลือกผู้ใช้และกลุ่ม

เก็บบันทึกเก่าที่หมุนเวียนใน
เมื่อฟิลด์นี้ถูกตั้งค่าเป็น ไดเร็กทอรีเดียวกันเป็นไฟล์บันทึก สำเนาที่หมุนเก่าของไฟล์บันทึกจะถูกเก็บไว้ในไดเรกทอรีเดียวกันกับไฟล์ต้นฉบับ ตัวเลือก Directory สามารถใช้เพื่อระบุตำแหน่งอื่นได้อย่างไรก็ตามจะต้องอยู่ในระบบไฟล์เดียวกับไฟล์บันทึก

คำสั่งให้เรียกใช้หลังจากการหมุน
เมื่อกรอกกล่องข้อความนี้คำสั่งเชลล์ที่ป้อนจะทำงานหลังจากบันทึกการหมุน เซิร์ฟเวอร์จำนวนมากจำเป็นต้องเริ่มต้นใหม่หลังจากที่มีการเปลี่ยนชื่อบันทึกเพื่อให้สามารถเขียนไฟล์ใหม่ได้อย่างถูกต้อง ในกรณีนี้คุณอาจป้อนคำสั่งเช่น killall -HUP httpd เพื่อทริกเกอร์การเริ่มต้นใหม่หลังจากการหมุน
