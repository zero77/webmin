Rotação do arquivo de log
A rotação do arquivo de log refere-se ao truncamento, compactação e exclusão automáticos dos arquivos de log, para que eles não consumam muito espaço em disco. A maioria dos servidores Unix (como Apache, Squid e Sendmail) gera arquivos de log, e vários daemons do sistema também criam logs através do Syslog. Este módulo pode ser usado para configurar o programa logrotate para gerenciar todos esses logs.

Normalmente, um arquivo de log será girado uma vez por dia, semana ou mês. O arquivo geralmente é movido para um novo nome de arquivo e compactado, e um novo arquivo vazio é criado em seu lugar. Várias gerações desses arquivos de log antigos podem ser mantidas, para que você possa pesquisar ou gerar relatórios a partir deles, mesmo após a rotação. Quando o número de logs antigos exceder um limite configurado, o mais antigo será excluído.

Cada arquivo de log que está sendo girado é listado na página principal do módulo. Cada log possui seu próprio conjunto de opções para controlar com que freqüência é girado, quantas cópias antigas são mantidas e assim por diante. Você pode editar as opções de um log clicando em seu nome de arquivo ou adicionar um novo arquivo de log usando o link na parte inferior ou superior da tabela.

Muitas distribuições Linux incluem logrotate como padrão e vêm com configurações para girar os logs dos servidores incluídos, como Apache e Squid. Portanto, mesmo que você nunca tenha usado esse módulo, muitos arquivos de log já podem estar listados na página principal. Frequentemente, na parte inferior da página, há um botão para editar a configuração global, que se aplica a todos os arquivos de log, a menos que substituídos. Abaixo é um botão para definir a agenda em que logrotate é executado por Cron, que é necessária para que os logs realmente girar. Na maioria dos sistemas operacionais que incluem o programa como padrão, um trabalho Cron já foi criado como parte do processo de instalação.
