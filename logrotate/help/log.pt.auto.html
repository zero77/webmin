Horário de rotação
Controla com que frequência o log é girado.

Tamanho máximo antes de girar
Define um limite para o tamanho do arquivo de log antes de ser girado. Este campo pode ser usado em vez da programação de rotação, se você preferir um tamanho em vez de um limite baseado no tempo.

Número de logs antigos a serem mantidos
Determina quantas gerações antigas do arquivo de log serão salvas. Eles normalmente terão o mesmo nome que o log original, mas com .0, .1 e assim por diante anexados. Se você digitar 0 , nenhum registro antigo será mantido e, portanto, o conteúdo do arquivo de registro será perdido quando for girado.

Compactar arquivos de log antigos?
Quando essa opção está ativada, os logs antigos serão compactados com gzip após a rotação para reduzir a quantidade de espaço em disco que eles consomem.

Girar mesmo se o arquivo de log estiver vazio?
Se ativada, essa opção informa ao logrotate para ainda girar o arquivo de log, mesmo que esteja vazio, resultando em cópias antigas com tamanho zero sendo salvas.

Recriar arquivo de log após a rotação?
Quando essa opção está ativada, o arquivo de log será recriado após a rotação, em vez de simplesmente ser deixado para o servidor que gravar nele para recriar. Por padrão, ele será criado com as permissões e o proprietário originais, mas você pode usar os campos ao lado da opção Sim para especificar permissões de arquivo, usuário e grupo alternativos.

Armazenar logs rotativos antigos
Quando esse campo é definido como Mesmo diretório que o arquivo de log , as cópias rotacionadas antigas do arquivo de log serão armazenadas no mesmo diretório que o log original. A opção Diretório pode ser usada para especificar um local alternativo - no entanto, ele deve estar no mesmo sistema de arquivos que o arquivo de log.

Comandos a serem executados após a rotação
Quando esta caixa de texto é preenchida, os comandos do shell inseridos serão executados após a rotação do log. Muitos servidores precisam ser reiniciados depois que o log for renomeado, para que gravem corretamente no novo arquivo. Nesse caso, você pode inserir um comando como killall -HUP httpd para acionar uma reinicialização após a rotação.
