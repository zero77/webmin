Сервер iSCSI
Гэты модуль дазваляе экспартаваць дыскавыя прылады ў іншыя сістэмы па сетцы з дапамогай пратакола iSCSI. Канфігурацыя ажыццяўляецца галоўным чынам шляхам стварэння трох тыпаў аб'ектаў - " Устройствы для сумеснага выкарыстання" , " Спалучэнне прылад" і " Мэты агульнага доступу" , апісаныя ніжэй. Вы таксама можаце рэдагаваць налады, якія прымяняюцца да ўсяго сервера iSCSI, выкарыстоўваючы старонку Параметры сервера iSCSI .

Прылады для сумеснага выкарыстання ( пашырэнні ў канфігурацыйным файле)
Кожны з іх - гэта раздзел, прылада RAID, лагічны том або файл, які можна поўнасцю або часткова экспартаваць праз iSCSI. Агульныя прылады не павінны выкарыстоўвацца ні для якіх іншых мэтаў у гэтай сістэме.

Камбінацыі прылад ( прылады ў канфігурацыйным файле)
Яны дазваляюць аб'яднаць некалькі прылад, каб падзяліцца на вялікую прыладу для экспарту. Прылады можна альбо злучыць у стылі RAID0, каб дадаць свае памеры разам, альбо люстраным стылем RAID1 для надмернасці і павышэння прадукцыйнасці чытання.

Агульныя мэты ( мэта ў файле канфігурацыі)
Кожная запіс у гэтым спісе экспартуе альбо прыладу, альбо камбінацыю прылад, ва ўсе сістэмы ў абранай сетцы. Акцыя можа быць альбо толькі для чытання, альбо для чытання / запісу.

У ніжняй частцы галоўнай старонкі модуля размешчаны кнопкі для прыпынку, запуску і перазапуску сервера iSCSI. Змены ў канфігурацыі не будуць прымяняцца, пакуль сервер не перазагрузіцца.
