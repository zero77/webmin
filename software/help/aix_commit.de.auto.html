COMMIT-Software-Updates?
Gibt an, ob Sie die von Ihnen installierten Softwareupdates festschreiben möchten. Wenn Sie Software festschreiben, verpflichten Sie sich, dass frühere Versionen dieses Produkts NICHT benötigt werden. ALLE VERSIONEN VOR DER EINE, DIE SIE VERPFLICHTEN, WERDEN GELÖSCHT. Das Festschreiben von Software hat zwei Auswirkungen. Dadurch wird Speicherplatz freigegeben, auf dem ältere Versionen dieser Software gespeichert wurden. Es entfällt die Möglichkeit, das Update abzulehnen und zur vorherigen Version zurückzukehren.

YES weist das System an, die von Ihnen installierten Softwareupdates festzuschreiben. Alle Versionen vor der von Ihnen festgeschriebenen Version werden gelöscht. Im Allgemeinen wird nicht empfohlen, Software-Updates festzuschreiben, die Sie zum ersten Mal installieren. Nachdem Sie beispielsweise eine neue Version verwendet haben, können Sie sich dafür entscheiden, eine frühere Version der Software zu verwenden. Sobald Sie die von Ihnen installierten Softwareupdates festgeschrieben haben, können Sie frühere Versionen nicht mehr abrufen. Sie müssen frühere Versionen neu installieren, um sie verwenden zu können.

NEIN weist das System an, die von Ihnen installierten Softwareupdates nicht festzuschreiben. Die Software, die Sie installieren, wird angewendet. Wenn Software auf das System angewendet wird, wird sie zur aktiven Version der Software. Wenn eine frühere Version der Software ersetzt wird, wird die vorherige Version in einem speziellen Verzeichnis auf der Festplatte gespeichert. Die vorherige Version kann bei Bedarf abgerufen werden, indem die aktuelle Version abgelehnt wird. Sobald Sie mit den Updates zufrieden sind, sollten Sie sie festschreiben, um Speicherplatz freizugeben, der von den gespeicherten Dateien verwendet wird.

Wenn Sie für "COMMIT-Software-Updates" NEIN wählen, müssen Sie JA wählen, um "ersetzte Dateien SPEICHERN" zu speichern.

Ob Sie Software-Updates festschreiben möchten oder nicht, hängt von den Umständen ab. Bei Ihrer Entscheidung sollten Sie Folgendes berücksichtigen.

o Wenn Sie einzelne Updates installieren, um ein gemeldetes Problem zu beheben, wird empfohlen, mit NEIN zu antworten. Sie können dann überprüfen, ob das Update Ihr Problem behebt und keine anderen Probleme verursacht, bevor Sie sich dazu verpflichten.

o Wenn Sie nur über einen sehr begrenzten Festplattenspeicher verfügen, müssen Sie möglicherweise mit JA antworten, um Software festzuschreiben, oder ein alternatives Speicherverzeichnis mit einem NFS-gemounteten Dateisystem verwenden.

o Wenn Sie beim Aktualisieren aller Ihrer Produkte auf eine neue Wartungsstufe die Option NEIN wählen, wird möglicherweise viel Speicherplatz beansprucht, und die Installation dauert länger. Sie müssen diese Faktoren abwägen, ob Sie die Änderungen zurücksetzen müssen, wenn Sie Probleme mit dem neuen Level haben.

o Wenn Sie eine Neuinstallation eines Produkts und seiner Updates durchführen, hat die Auswahl von NEIN möglicherweise keinen Vorteil für Sie.

Dies ist NICHT Ihre einzige Möglichkeit, Software zu verpflichten. Nachdem Sie die Software installiert und eine Weile verwendet haben, können Sie sie festschreiben. Gehen Sie auf SMIT zum Menü Softwareinstallation und -wartung, wählen Sie Installierte Software pflegen und dann Angewandte Software festschreiben (Gespeicherte Dateien entfernen).
