Aggiornamenti software COMMIT?
Indica se si desidera eseguire il commit degli aggiornamenti software che si stanno installando. Quando si esegue il commit del software, si sta impegnando che le versioni precedenti di questo prodotto NON sono necessarie. TUTTE LE VERSIONI PRECEDENTI A QUELLA CHE STAI IMPEGNANDO SONO CANCELLATE. Il commit del software ha due effetti. Libera lo spazio su disco utilizzato per archiviare le versioni precedenti di quel software. Elimina la possibilità di poter rifiutare l'aggiornamento e tornare alla versione precedente.

SÌ indica al sistema di eseguire il commit degli aggiornamenti software che si stanno installando. Tutte le versioni precedenti a quella che stai eseguendo vengono eliminate. In generale, non è consigliabile eseguire il commit degli aggiornamenti software che si stanno installando per la prima volta. Ad esempio, dopo aver utilizzato una nuova versione, è possibile decidere di preferire l'utilizzo di una versione precedente del software. Dopo aver eseguito il commit degli aggiornamenti software che stai installando, non avrai più modo di recuperare le versioni precedenti. È necessario reinstallare le versioni precedenti per poterle utilizzare.

NO indica al sistema di non eseguire il commit degli aggiornamenti software che si stanno installando. Verrà applicato il software che si sta installando. Quando il software viene applicato al sistema, diventa la versione attiva del software. Se sostituisce una versione precedente del software, la versione precedente viene salvata in una directory speciale sul disco. La versione precedente può essere recuperata, se necessario, rifiutando la versione corrente. Una volta che sei soddisfatto degli aggiornamenti, dovresti impegnarli a liberare spazio su disco utilizzato dai file salvati.

Se si sceglie NO per "COMMIT aggiornamenti software", è necessario selezionare SÌ su "SALVA file sostituiti".

Se è desiderabile che tu commetta aggiornamenti del software o meno dipende dalle circostanze. Dovresti considerare quanto segue nel prendere la tua decisione.

o Se si installano singoli aggiornamenti per risolvere un problema segnalato, si consiglia di rispondere NO. È quindi possibile verificare che la correzione risolva il problema e non causi altri problemi prima di impegnarsi.

o Se lo spazio sul disco rigido è molto limitato, potrebbe essere necessario rispondere SÌ per eseguire il commit del software o utilizzare una directory di salvataggio alternativa con un file system montato su NFS.

o La scelta di NO durante l'aggiornamento di tutti i prodotti a un nuovo livello di manutenzione può utilizzare una notevole quantità di spazio su disco e prolungare l'installazione. È necessario valutare questi fattori con la necessità di poter annullare le modifiche in caso di problemi con il nuovo livello.

o Se si sta effettuando una nuova installazione di un prodotto e dei suoi aggiornamenti, potrebbe non esserci alcun vantaggio nella scelta di NO.

Questa NON è la tua unica opportunità di impegnare software. Dopo aver installato il software e averlo utilizzato per un po ', è possibile eseguirne il commit. Vai al menu Installazione e manutenzione del software su SMIT, seleziona Mantieni software installato e quindi Salva software applicato (Rimuovi file salvati).
