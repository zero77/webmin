COMMITER les mises à jour logicielles?
Indique si vous souhaitez valider les mises à jour logicielles que vous installez. Lorsque vous validez un logiciel, vous vous engagez à ce que les versions précédentes de ce produit ne soient PAS nécessaires. TOUTES LES VERSIONS AVANT CELLE QUE VOUS ENGAGEZ SONT SUPPRIMÉES. La validation d'un logiciel a deux effets. Il libère de l'espace disque qui était utilisé pour stocker les anciennes versions de ce logiciel. Il élimine la possibilité de pouvoir rejeter la mise à jour et revenir à la version précédente.

OUI demande au système de valider les mises à jour logicielles que vous installez. Toutes les versions antérieures à celle que vous validez sont supprimées. En général, il n'est pas recommandé de valider les mises à jour logicielles que vous installez pour la première fois. Par exemple, après avoir utilisé une nouvelle version, vous pouvez décider que vous préférez utiliser une version antérieure du logiciel. Une fois que vous aurez validé les mises à jour logicielles que vous installez, vous n'aurez aucun moyen de récupérer les versions précédentes. Vous devez réinstaller les versions précédentes pour les utiliser.

NON indique au système de ne pas valider les mises à jour logicielles que vous installez. Le logiciel que vous installez sera appliqué. Lorsque le logiciel est appliqué au système, il devient la version active du logiciel. S'il remplace une version précédente du logiciel, la version précédente est enregistrée dans un répertoire spécial sur le disque. La version précédente peut être récupérée, si nécessaire, en rejetant la version actuelle. Une fois que vous êtes satisfait des mises à jour, vous devez les valider pour libérer de l'espace disque utilisé par les fichiers enregistrés.

Si vous choisissez NON pour "COMMITER les mises à jour logicielles", vous devez alors choisir OUI pour "ENREGISTRER les fichiers remplacés".

Qu'il soit souhaitable pour vous de valider ou non les mises à jour logicielles dépend des circonstances. Vous devez tenir compte des éléments suivants pour prendre votre décision.

o Si vous installez des mises à jour individuelles pour résoudre un problème signalé, il est recommandé de répondre NON. Vous pouvez ensuite vérifier que le correctif corrige votre problème et ne provoque pas d'autres problèmes avant de vous y engager.

o Si votre espace sur le disque dur est très limité, il peut être nécessaire de répondre OUI pour valider le logiciel ou d'utiliser un autre répertoire de sauvegarde avec un système de fichiers monté NFS.

o Choisir NON lors de la mise à jour de tous vos produits vers un nouveau niveau de maintenance peut utiliser une quantité importante d'espace disque et entraînera une installation plus longue. Vous devez évaluer ces facteurs pour savoir si vous avez besoin de pouvoir annuler les modifications si vous rencontrez des problèmes avec le nouveau niveau.

o Si vous effectuez une nouvelle installation d'un produit et de ses mises à jour, il peut ne vous être avantageux de choisir NON.

Ce n'est PAS votre seule occasion de valider un logiciel. Après avoir installé et utilisé le logiciel pendant un certain temps, vous pouvez le valider. Accédez au menu Installation et maintenance des logiciels sur SMIT, sélectionnez Gérer les logiciels installés, puis sélectionnez Valider les logiciels appliqués (Supprimer les fichiers enregistrés).
