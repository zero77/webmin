Bandbreddövervakning
Denna modul kan användas för att skapa enkla rapporter om bandbreddanvändning per port, värd, protokoll och tid för trafik som skickas från eller dirigeras via ditt system. Det är användbart för både fristående värdar och de som fungerar som en gateway (eventuellt med NAT) för ett nätverk.

Innan den kan användas måste modulen konfigurera flera brandväggsregler och en syslog-post för att fånga trafik som skickas och tas emot via ditt system. Vid installationstiden måste du välja systemets externa nätverksgränssnitt, på vilket trafiken övervakas. Detta kommer vanligtvis att vara PPP-gränssnittet som används för din uppringning eller ADSL-anslutning, eller Ethernet-gränssnittet anslutet till ditt kabelmodem eller uppströms router.

När minst en timmes trafik har tagits, kan modulen användas för att generera rapporter som sammanfattar trafiken med en av följande kategorier:

  • Timme
    Datum och timme då uppgifterna mottogs.
  • Dag
    Datum då uppgifterna mottogs.
  • Värd
    Värdet i ditt interna nätverk, eller brandväggsvärden, som skickade eller tog emot data.
  • Intern port
    Porten på din interna eller brandväggsvärd till vilken data skickades. Det här läget är användbart för att bestämma hur mycket trafik som genereras av anslutningar till var och en av dina servrar.
  • Extern port
    Porten på någon server utanför ditt nätverk till vilken data skickades. Detta kan användas för att avgöra vilka tjänster användare i ditt nätverk ska ha åtkomst till.
  • Hamn
    Både interna och externa portar. Användbart för att jämföra all nätverksanvändning efter servicetyp.
Du kan också välja att begränsa rapporten till en utvalda värd, port eller protokoll. Värden kan anges med IP-adress, värdnamn eller nätverksadress som 192.168.1.0. Porten kan anges med namn eller nummer och matchar både TCP och UDP.

Rapporten kan också begränsas till trafik som samlas in mellan utvalda timmar med fältet För trafik efter och för trafik före . Eftersom trafiken sammanfattas per timme kan du inte begränsa rapporten med större precision än timme.

Alternativet Serverportar bara? är användbart vid rapportering från inkommande, utgående eller alla portar. Det begränsar skärmen till portar som vanligtvis används av servrar (de under 1024 eller med namn), för att undvika rörig bildskärm med räkningar för klientsidanportar som inte vanligtvis är användbara.

Alternativet Lösa värdnamn? kan väljas vid rapportering av värd. Det kommer att göra att alla IP-adresser omvändes till värdnamn, där det är möjligt.
