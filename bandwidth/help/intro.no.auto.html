Båndbreddeovervåking
Denne modulen kan brukes til å lage enkle rapporter om båndbreddebruk per port, vert, protokoll og tid for trafikk sendt fra eller dirigert gjennom systemet ditt. Det er nyttig for både frittstående verter, og de som fungerer som en gateway (muligens med NAT) for et nettverk.

Før den kan brukes, må modulen konfigurere flere brannmurregler og en syslog-oppføring for å fange opp trafikk som er sendt og mottatt via systemet ditt. På installasjonstidspunktet må du velge systemets eksterne nettverksgrensesnitt, som trafikken skal overvåkes på. Dette vil vanligvis være PPP-grensesnittet som brukes til dialup- eller ADSL-tilkoblingen, eller Ethernet-grensesnittet som er koblet til kabelmodemet eller oppstrømsruteren.

Når det er fanget minst en times verdi av trafikken, kan modulen brukes til å generere rapporter som oppsummerer trafikken etter en av følgende kategorier:

  • Time
    Datoen og timen da dataene ble mottatt.
  • Dag
    Datoen da dataene ble mottatt.
  • Vert
    Verten i det interne nettverket ditt, eller brannmurverten, som sendte eller mottok dataene.
  • Intern port
    Porten på din interne eller brannmurverten som dataene ble sendt til. Denne modusen er nyttig for å bestemme hvor mye trafikk som genereres av tilkoblinger til hver av serverne dine.
  • Ekstern port
    Porten på noen server utenfor nettverket ditt som data ble sendt til. Dette kan brukes til å bestemme hvilke tjenester brukere av nettverket ditt får tilgang til.
  • Havn
    Både interne og eksterne porter. Nyttig for å sammenligne all nettverksbruk etter tjenestetype.
Du kan også velge å begrense rapporten til en valgt vert, port eller protokoll. Verten kan legges inn med IP-adresse, vertsnavn eller nettverksadresse som 192.168.1.0. Porten kan legges inn med navn eller nummer, og vil samsvare med både TCP og UDP.

Rapporten kan også begrenses til trafikk samlet mellom utvalgte timer ved å bruke For-trafikk etter og For trafikk før felt. Fordi trafikken er oppsummert etter time, kan du ikke begrense rapporten med større presisjon enn time.

Alternativet bare serverporter? er nyttig når du rapporterer fra innkommende, utgående eller alle porter. Det begrenser skjermen til porter som ofte brukes av servere (de under 1024 eller med navn), for å unngå rot på skjermen med tellinger for klientsiden-porter som ikke er nyttige.

Alternativet Løs vertsnavn? kan velges når du rapporterer av verten. Det vil føre til at alle IP-adresser blir omvendt løst til vertsnavn, der det er mulig.
