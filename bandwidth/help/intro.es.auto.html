Monitoreo de ancho de banda
Este módulo se puede usar para crear informes simples sobre el uso del ancho de banda por puerto, host, protocolo y tiempo para el tráfico enviado o enrutado a través de su sistema. Es útil tanto para hosts independientes como para aquellos que actúan como una puerta de enlace (posiblemente con NAT) para una red.

Antes de que pueda usarse, el módulo debe configurar varias reglas de firewall y una entrada de syslog para capturar el tráfico enviado y recibido a través de su sistema. En el momento de la configuración, debe seleccionar la interfaz de red externa del sistema, en la que se supervisará el tráfico. Por lo general, esta será la interfaz PPP utilizada para su conexión telefónica o ADSL, o la interfaz Ethernet conectada a su módem de cable o enrutador ascendente.

Una vez que se ha capturado al menos una hora de tráfico, el módulo puede usarse para generar informes que resumen el tráfico por una de las siguientes categorías:

  • Hora
    La fecha y hora en que se recibieron los datos.
  • Día
    La fecha en que se recibieron los datos.
  • Anfitrión
    El host en su red interna, o el host del firewall, que envió o recibió los datos.
  • Puerto interno
    El puerto en su host interno o firewall al que se enviaron los datos. Este modo es útil para determinar cuánto tráfico generan las conexiones a cada uno de sus servidores.
  • Puerto externo
    El puerto en algún servidor fuera de su red al que se enviaron los datos. Esto se puede usar para determinar a qué servicios acceden los usuarios de su red.
  • Puerto
    Puertos internos y externos. Útil para comparar todo el uso de la red por tipo de servicio.
También puede optar por limitar el informe a un host, puerto o protocolo seleccionado. El host se puede ingresar por dirección IP, nombre de host o dirección de red como 192.168.1.0. El puerto se puede ingresar por nombre o número, y coincidirá con TCP y UDP.

El informe también puede limitarse al tráfico recopilado entre las horas seleccionadas, utilizando los campos Para tráfico después y Para tráfico antes . Debido a que el tráfico se resume por hora, no puede limitar el informe con mayor precisión que la hora.

La opción ¿Solo puertos del servidor? es útil cuando se informa por entrada, salida o todos los puertos. Restringe la pantalla a los puertos comúnmente utilizados por los servidores (aquellos por debajo de 1024 o con nombres), para evitar saturar la pantalla con recuentos de puertos del lado del cliente que no son comúnmente útiles.

La opción Resolver nombres de host? se puede seleccionar cuando se informa por host. Hará que todas las direcciones IP se resuelvan inversamente a nombres de host, cuando sea posible.
