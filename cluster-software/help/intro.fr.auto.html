Packages logiciels de cluster
Ce module Webmin vous permet de gérer le logiciel installé sur plusieurs serveurs dans un cluster à partir d'une seule interface. Il est similaire au module Packages logiciels, mais avec la possibilité d'effectuer la plupart des fonctions (comme l'installation, l'interrogation et la suppression de packages) sur de nombreux serveurs Webmin en une seule opération.

La partie supérieure de la page principale sous l'en-tête Serveurs gérés répertorie les autres serveurs Webmin dont le logiciel est géré par ce module. Pour ajouter un serveur à cette liste, vous devez d'abord l'ajouter au module Serveurs Webmin, avec un nom d'utilisateur et un mot de passe spécifiés pour vous connecter à Webmin sur ce serveur. Vous pouvez ensuite sélectionner le serveur dans la liste à côté du bouton Ajouter un serveur.

Lorsqu'un serveur est ajouté, le module vérifie qu'il utilise le même système de gestion de packages que ce serveur (tel que RPM ou Debian), puis télécharge une liste de tous les packages installés pour la mise en cache locale.

Un serveur qui apparaîtra toujours dans la liste de ceux disponibles pour l'ajout est «ce serveur», afin que vous puissiez gérer les packages localement et sur plusieurs serveurs distants à partir d'une seule interface.

Une fois qu'il y a au moins une icône sous Serveurs gérés, vous pouvez utiliser le formulaire Packages installés pour rechercher des packages sur tous les serveurs, ou cliquer sur le bouton Actualiser les listes de packages pour récupérer à nouveau la liste des packages de tous les serveurs si des packages ont été ajoutés ou supprimés. via un autre module ou méthode.

Au bas de la page se trouve un formulaire pour installer de nouveaux packages sur tous les serveurs gérés. Vous pouvez choisir d'installer à partir d'un fichier sur ce serveur, d'un fichier téléchargé ou d'un fichier téléchargé via HTTP ou FTP à partir d'un autre site. Lors de l'installation à partir d'un fichier téléchargé, vous pouvez choisir si chaque serveur géré doit retélécharger le package à partir du site d'origine ou recevoir une copie de ce serveur. L'ancienne option sera plus rapide si le site sur lequel se trouve le package se trouve sur votre réseau local.
