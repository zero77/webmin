Auszeit
Der Wert in Zehntelsekunden vor dem Senden der ersten erneuten Übertragung nach einem RPC-Timeout.
Nach dem ersten Timeout wird das Timeout nach jedem aufeinanderfolgenden Timeout verdoppelt, bis ein maximales Timeout von 60 Sekunden erreicht ist.
Eine bessere Gesamtleistung kann erzielt werden, indem das Zeitlimit beim Mounten in einem ausgelasteten Netzwerk, auf einem langsamen Server oder über mehrere Router oder Gateways erhöht wird.

Mount-Option: timeo
Standard: 0,7s
