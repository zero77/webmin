Sendmail-Konfiguration

Einführung in Sendmail

Sendmail ist das Standard-Unix-Programm für den Transport und die Zustellung von E-Mails. Wenn ein Benutzer in Ihrem System E-Mails an jemanden auf einem anderen Host senden möchte, ist sendmail das Programm, das den eigentlichen Prozess der Zustellung der E-Mails ausführt. Wenn jemand auf einem anderen System E-Mails an einen Ihrer Benutzer sendet, empfängt sendmail die E-Mail und speichert sie in der E-Mail-Datei des Benutzers.

Sendmail ist nur ein MTA (Mail Transport Agent), dh, es wird keine Benutzeroberfläche für Personen angezeigt, die E-Mails senden möchten. Programme wie Eudora, elm oder /bin/mail, mit denen Benutzer tatsächlich mit sogenannten MUAs (Mail User Agents) interagieren. Ein MUA liefert keine E-Mails selbst, sondern leitet die E-Mails einfach an sendmail weiter, um die Zustellung durchzuführen.

Benutzer-MUAs können entweder direkt auf Ihrem System (MUAs wie Pine, Elm oder /bin/Mail) oder auf einem separaten Host ausgeführt werden, der ein Einzelbenutzer-PC sein kann (für MUAs wie Eudora, Exchange und Netscape Communicator). Im ersten Fall wird E-Mail gesendet, indem sendmail direkt aufgerufen und an die Nachricht weitergeleitet wird, und durch Lesen der Mail-Spooldatei des Benutzers empfangen wird. Bei MUAs, die auf anderen Hosts ausgeführt werden, werden E-Mails durch Herstellen einer Verbindung zum sendmail-Prozess auf Ihrem System mithilfe des SMTP-Protokolls gesendet und durch Herstellen einer Verbindung zu einem separaten POP3-Server empfangen.

Das Sendmail-Modul

Die Hauptseite dieses Moduls enthält eine Tabelle mit Symbolen, mit denen jeweils ein anderer Teil der Funktionen von sendmail konfiguriert werden kann. Die Funktionen hinter einigen Symbolen sind jedoch möglicherweise nicht verfügbar, wenn sie nicht in Ihrer sendmail.cf eingerichtet wurden . Bei der Installation von sendmail können Sie auswählen, welche Funktionen verfügbar sind, indem Sie eine .mc-Datei bearbeiten , die von m4 in Ihre sendmail.cf konvertiert wird. Viele Funktionen (wie Adresszuordnung, Domänenmaskierung und Spam-Kontrolle) sind möglicherweise nur verfügbar, wenn Sie sie bei der Installation aktiviert haben.
