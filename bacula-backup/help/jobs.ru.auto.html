Резервное копирование
Задание является наиболее важным настраиваемым объектом в Bacula, так как оно предоставляет клиенту, файлу и другим параметрам настройки для точного управления резервным копированием. По этой причине обычно лучше определить наборы файлов и клиентов перед созданием задания.

Каждая работа имеет следующие важные атрибуты:

Название работы
Уникальное имя для этой работы.
Тип вакансии
Это определяет, какое действие будет выполнять задание. Почти во всех случаях это должно быть установлено на Резервное копирование .
Уровень резервного копирования
Определяет, будет ли задание выполнять полное или частичное резервное копирование выбранных файлов.
Клиент для резервного копирования
Если в вашей системе Bacula есть несколько клиентов, этот параметр определяет, из какого файла будут считываться файлы этим заданием резервного копирования.
Файл установлен для резервного копирования
Выбранный набор файлов определяет, какие фактические файлы и каталоги включены в резервную копию.
Резервное копирование по расписанию
Этот необязательный параметр определяет, будет ли задание автоматически выполняться по расписанию, и если да, то когда.
Устройство хранения назначения
Если в вашей конфигурации Bacula настроено более одного демона хранилища, этот параметр определяет, в какой из резервных копий будет записана.

Чтобы упростить процесс создания задания, Bacula позволяет создавать задания определения по умолчанию, которые на самом деле не запускаются сами, а скорее задают параметры, которые должны наследоваться реальными заданиями. При редактировании задания поле Тип по умолчанию определяет, является ли оно определением по умолчанию, наследует ли оно настройки от какого-либо значения по умолчанию или ни того, ни другого.
