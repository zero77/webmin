Processi di backup
Un lavoro è l'oggetto configurabile più importante in Bacula, poiché riunisce un client, un set di file e altre impostazioni per controllare esattamente ciò di cui è stato eseguito il backup. Per questo motivo, in genere è preferibile definire set di file e client prima di creare un lavoro.

Ogni lavoro ha i seguenti attributi importanti:

Nome del lavoro
Un nome univoco per questo lavoro.
Tipo di lavoro
Ciò determina il tipo di azione che verrà eseguita dal lavoro. In quasi tutti i casi questo dovrebbe essere impostato su Backup .
Livello di backup
Determina se il lavoro eseguirà un backup completo o parziale dei file selezionati.
Client per il backup
Se il sistema Bacula ha più client, questa opzione determina da quale processo vengono letti i file da questo processo di backup.
File impostato su backup
Il set di file selezionato determina quali file e directory effettivi sono inclusi nel backup.
Backup nei tempi previsti
Questa opzione facoltativa determina se il lavoro viene eseguito automaticamente nei tempi previsti e, in tal caso, quando.
Dispositivo di archiviazione di destinazione
Se la configurazione di Bacula è impostata con più di un demone di archiviazione, questa opzione determina su quale verrà scritto il backup.

Per semplificare il processo di creazione di posti di lavoro, Bacula consente di creare lavori a definizione predefinita che non vengono eseguiti da soli, ma piuttosto specificare le impostazioni da ereditare dai lavori reali. Quando si modifica un lavoro, il campo Tipo predefinito determina se si tratta di una definizione predefinita, se eredita le impostazioni da alcuni valori predefiniti o nessuno dei due.
