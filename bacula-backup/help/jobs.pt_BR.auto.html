Trabalhos de backup
Um trabalho é o objeto configurável mais importante no Bacula, pois reúne um cliente, conjunto de arquivos e outras configurações para controlar exatamente o que é feito o backup. Por esse motivo, geralmente é melhor definir seus conjuntos de arquivos e clientes antes de criar um trabalho.

Cada trabalho possui os seguintes atributos importantes:

Nome do trabalho
Um nome exclusivo para este trabalho.
Tipo de emprego
Isso determina que tipo de ação o trabalho executará. Em quase todos os casos, isso deve ser definido como Backup .
Nível de backup
Determina se o trabalho fará um backup completo ou parcial dos arquivos selecionados.
Cliente para backup
Se o seu sistema Bacula tiver vários clientes, essa opção determinará em qual dos arquivos serão lidos por esta tarefa de backup.
Arquivo definido para backup
O conjunto de arquivos selecionado determina quais arquivos e diretórios reais estão incluídos no backup.
Backup agendado
Esta opção opcional determina se o trabalho é executado automaticamente dentro do cronograma e, se sim, quando.
Dispositivo de armazenamento de destino
Se sua configuração do Bacula estiver configurada com mais de um daemon de armazenamento, essa opção determinará em qual backup o gravador será gravado.

Para simplificar o processo de criação de trabalho, o Bacula permite criar trabalhos de definição padrão que na verdade não são executados, mas especificar configurações a serem herdadas por trabalhos reais. Ao editar um trabalho, o campo Tipo padrão determina se é uma definição padrão, se herda configurações de algum padrão ou nenhuma.
