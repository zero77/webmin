Demonios de almacenamiento
Cada entrada en esta página define una conexión a un dispositivo particular en un demonio de almacenamiento Bacula. Normalmente, el daemon se ejecuta en el mismo sistema, pero es posible definir más de un daemon, algunos de los cuales están en hosts remotos con diferentes tipos de unidades de cinta o más espacio en disco.

Los atributos importantes para cada demonio son:

Nombre de almacenamiento
Un nombre único utilizado en este sistema para identificar al demonio.
Contraseña SD Bacula
Este campo debe contener la contraseña utilizada por el demonio de almacenamiento en el sistema remoto, que se puede encontrar en el archivo /etc/bacula/bacula-sd.conf .
Nombre de host o la dirección IP
El nombre de host o IP totalmente calificado del sistema que ejecuta el daemon de almacenamiento.
Nombre del dispositivo de almacenamiento
Cada daemon de almacenamiento puede definir uno o más dispositivos, cada uno de los cuales corresponde a un directorio o unidad de cinta diferente. Este campo determina qué dispositivo utiliza esta conexión de daemon en particular.
Nombre del tipo de medio
Este campo debe contener un nombre corto para el tipo de medio utilizado por el dispositivo de almacenamiento, como Archivo o DDS-2 . Debe ser el mismo tipo de medio en la sección Dispositivos de almacenamiento en el servidor de almacenamiento.
A menudo es útil tener más de una entrada para el mismo host en esta página, cada una de las cuales tiene un dispositivo de almacenamiento diferente. Esto le permite crear copias de seguridad que escriben en diferentes directorios o unidades de cinta.

Si las opciones de TLS se muestran en el formulario del demonio de almacenamiento, se pueden usar para permitir una comunicación segura entre el director y el servidor de almacenamiento remoto. Esto normalmente no es necesario en una configuración simple de Bacula, ya que los demonios de almacenamiento y director generalmente se ejecutan en el mismo sistema.
