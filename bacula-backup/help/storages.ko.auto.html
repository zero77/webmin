스토리지 데몬
이 페이지의 각 항목은 Bacula 스토리지 데몬의 특정 장치에 대한 연결을 정의합니다. 일반적으로 디먼은 동일한 시스템에서 실행되지만 둘 이상의 디먼을 정의 할 수 있습니다.이 중 일부는 테이프 드라이브 유형이 다르거 나 디스크 공간이 다른 원격 호스트에 있습니다.

각 데몬의 중요한 속성은 다음과 같습니다.

저장소 이름
이 시스템에서 데몬을 식별하는 데 사용되는 고유 이름입니다.
Bacula SD 비밀번호
이 필드에는 원격 시스템의 스토리지 데몬이 사용하는 비밀번호가 포함되어 있어야합니다.이 비밀번호는 /etc/bacula/bacula-sd.conf 파일에 있습니다.
호스트 이름 또는 IP 주소
스토리지 디먼을 실행중인 시스템의 완전한 호스트 이름 또는 IP
저장 장치 이름
각 스토리지 디먼은 하나 이상의 장치를 정의 할 수 있으며 각 장치는 다른 디렉토리 또는 테이프 드라이브에 해당합니다. 이 필드는이 특정 데몬 연결이 사용하는 장치를 결정합니다.
미디어 타입 이름
이 필드에는 저장 장치에서 사용하는 미디어 유형 (예 : File 또는 DDS-2) 의 짧은 이름이 포함되어야합니다. 스토리지 서버의 스토리지 장치 섹션에있는 미디어 유형과 같아야합니다.
이 페이지에서 동일한 호스트에 대해 둘 이상의 항목을 갖는 것이 유용하며 각 항목마다 다른 저장 장치가 있습니다. 이를 통해 다른 디렉토리 나 테이프 드라이브에 쓰는 백업을 만들 수 있습니다.

TLS 옵션이 스토리지 디먼 양식에 표시되면 디렉터와 원격 스토리지 서버 간의 보안 통신을 가능하게하는 데 사용할 수 있습니다. 일반적으로 스토리지 및 디렉터 디먼이 동일한 시스템에서 실행되므로 간단한 Bacula 구성에서는 필요하지 않습니다.
