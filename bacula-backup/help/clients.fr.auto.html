Clients de sauvegarde
Un client Bacula est un système dont les fichiers peuvent être sauvegardés. Tous les clients sur lesquels vous souhaitez sauvegarder des fichiers doivent être répertoriés sur cette page, et chacun doit avoir le démon de fichier Bacula en cours d'exécution. Dans une configuration typique à système unique, ce serveur doit être répertorié.

Lors de l'ajout de clients distants, en plus du nom d'hôte, vous devez également connaître le nom de son démon de fichier Bacula et le mot de passe du démon. Ceux-ci sont définis dans le fichier /etc/bacula/bacula-fd.conf sur le système client.

Si elles sont affichées, les options TLS de la page client déterminent si le chiffrement est utilisé lorsque ce directeur communique avec le client. Avant que TLS puisse être activé, vous devez générer un certificat et une clé SSL et saisir leurs chemins d'accès dans les champs appropriés de ce formulaire.
