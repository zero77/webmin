Bacula Backup
Bacula es un conjunto de programas informáticos que le permite a usted (o al administrador del sistema) administrar copias de seguridad, recuperación y verificación de datos informáticos en una red de computadoras de diferentes tipos. Bacula también puede ejecutarse por completo en una sola computadora y puede realizar copias de seguridad en varios tipos de medios, incluidas cintas y discos.

En términos técnicos, es un programa de respaldo basado en cliente / servidor de red. Bacula es relativamente fácil de usar y eficiente, al tiempo que ofrece muchas funciones avanzadas de administración de almacenamiento que facilitan la búsqueda y recuperación de archivos perdidos o dañados. Debido a su diseño modular, Bacula es escalable desde pequeños sistemas de computadoras individuales hasta sistemas que consisten en cientos de computadoras ubicadas en una red grande.

El sistema Bacula se divide en tres demonios separados, que teóricamente pueden ejecutarse en diferentes hosts. Estos son:

Director
El director almacena trabajos, clientes y la mayoría de los otros ajustes de configuración, y es responsable de iniciar los trabajos de respaldo. Este módulo de Webmin debe ejecutarse en el sistema que tiene instalado el director.
File Daemon
Este demonio es responsable de leer los archivos reales de los que se realizará una copia de seguridad. Una configuración de Bacula puede tener más de un demonio de archivo, cada uno ejecutado en un sistema para hacer una copia de seguridad.
Demonio de almacenamiento
El daemon de almacenamiento es responsable de escribir datos en el medio de copia de seguridad final, como una unidad de cinta o un archivo. Por lo general, solo necesitará uno de esos demonios (generalmente se ejecuta en el mismo host que el director), pero una configuración de Bacula más compleja puede tener múltiples sistemas con diferentes unidades de cinta, para distribuir la carga de respaldo.
Este módulo de Webmin puede administrar un sistema que tenga uno o más de los demonios de Bacula instalados.
