Demoni di archiviazione
Ogni voce in questa pagina definisce una connessione a un dispositivo particolare su un demone di archiviazione Bacula. In genere il demone viene eseguito sullo stesso sistema, ma è possibile definire più di un demone, alcuni dei quali si trovano su host remoti con diversi tipi di unità nastro o più spazio su disco.

Gli attributi importanti per ciascun demone sono:

Nome di archiviazione
Un nome univoco utilizzato su questo sistema per identificare il demone.
Password SD Bacula
Questo campo deve contenere la password utilizzata dal demone di archiviazione sul sistema remoto, che si trova nel file /etc/bacula/bacula-sd.conf .
Nome host o indirizzo IP
Il nome host o IP completo del sistema che esegue il daemon di archiviazione.
Nome del dispositivo di archiviazione
Ciascun demone di archiviazione può definire uno o più dispositivi, ognuno dei quali corrisponde a una directory o unità nastro diversa. Questo campo determina quale dispositivo utilizza questa particolare connessione daemon.
Nome del tipo di supporto
Questo campo deve contenere un nome breve per il tipo di supporto utilizzato dal dispositivo di archiviazione, ad esempio File o DDS-2 . Deve corrispondere al tipo di supporto nella sezione Dispositivi di archiviazione sul server di archiviazione.
Spesso è utile avere più di una voce per lo stesso host in questa pagina, ognuna delle quali ha un dispositivo di archiviazione diverso. Ciò consente di creare backup che scrivono in directory o unità nastro diverse.

Se le opzioni TLS sono visualizzate nel modulo del demone di archiviazione, possono essere utilizzate per abilitare la comunicazione sicura tra il direttore e il server di archiviazione remoto. Questo in genere non è necessario in una semplice configurazione di Bacula, poiché i demoni di archiviazione e director di solito vengono eseguiti sullo stesso sistema.
