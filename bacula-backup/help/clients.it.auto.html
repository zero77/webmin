Client di backup
Un client Bacula è un sistema di cui è possibile eseguire il backup dei file. Tutti i client su cui si desidera eseguire il backup dei file devono essere elencati in questa pagina e ognuno deve avere il demone del file Bacula in esecuzione su di esso. In una tipica configurazione a sistema singolo, su questo server deve essere elencato.

Quando si aggiungono client remoti, oltre al nome host è necessario conoscere anche il nome del suo demone del file Bacula e la password del demone. Questi sono impostati nel file /etc/bacula/bacula-fd.conf sul sistema client.

Se visualizzato, le opzioni TLS nella pagina client determinano se la crittografia viene utilizzata quando questo director comunica con il client. Prima di poter abilitare TLS, è necessario generare un certificato e una chiave SSL e immettere i relativi percorsi nei campi appropriati in questo modulo.
