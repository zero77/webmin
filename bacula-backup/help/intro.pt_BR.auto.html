Bacula Backup
Bacula é um conjunto de programas de computador que permite que você (ou o administrador do sistema) gerencie o backup, a recuperação e a verificação dos dados do computador em uma rede de computadores de diferentes tipos. O Bacula também pode ser executado inteiramente em um único computador e pode fazer backup em vários tipos de mídia, incluindo fita e disco.

Em termos técnicos, é um programa de backup em rede baseado em cliente / servidor. O Bacula é relativamente fácil de usar e eficiente, oferecendo muitos recursos avançados de gerenciamento de armazenamento que facilitam a localização e recuperação de arquivos perdidos ou danificados. Devido ao seu design modular, o Bacula é escalável de pequenos sistemas de computador único para sistemas constituídos por centenas de computadores localizados em uma grande rede.

O sistema Bacula é dividido em três daemons separados, que teoricamente podem ser executados em diferentes hosts. Esses são:

Diretor
O diretor armazena tarefas, clientes e a maioria das outras configurações e é responsável por iniciar tarefas de backup. Este módulo Webmin deve ser executado no sistema em que o diretor está instalado.
Daemon de arquivo
Este daemon é responsável pela leitura dos arquivos reais a serem copiados. Uma configuração do Bacula pode ter mais de um daemon de arquivo, cada um executado em um sistema para backup.
Daemon de armazenamento
O daemon de armazenamento é responsável por gravar dados na mídia de backup final, como uma unidade ou arquivo de fita. Normalmente, você precisará apenas de um daemon (geralmente executado no mesmo host que o diretor), mas uma configuração mais complexa do Bacula pode ter vários sistemas com unidades de fita diferentes, para espalhar a carga de backup.
Este módulo Webmin pode gerenciar um sistema que possui um ou mais dos daemons Bacula instalados.
