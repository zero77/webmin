Travaux de sauvegarde
Un travail est l'objet configurable le plus important dans Bacula, car il rassemble un client, un ensemble de fichiers et d'autres paramètres pour contrôler exactement ce qui est sauvegardé. Pour cette raison, il est généralement préférable de définir vos ensembles de fichiers et vos clients avant de créer un travail.

Chaque travail a les attributs importants suivants:

Nom du travail
Un nom unique pour ce travail.
Type d'emploi
Cela détermine le type d'action que le travail effectuera. Dans presque tous les cas, cela doit être défini sur Sauvegarde .
Niveau de sauvegarde
Détermine si le travail effectuera une sauvegarde complète ou partielle des fichiers sélectionnés.
Client à sauvegarder
Si votre système Bacula possède plusieurs clients, cette option détermine à partir de quel fichier les fichiers sont lus par ce travail de sauvegarde.
Jeu de fichiers à sauvegarder
L'ensemble de fichiers sélectionné détermine quels fichiers et répertoires réels sont inclus dans la sauvegarde.
Sauvegarde dans les délais
Cette option facultative détermine si le travail s'exécute automatiquement dans les délais et, le cas échéant, quand.
Périphérique de stockage de destination
Si votre configuration Bacula est configurée avec plusieurs démons de stockage, cette option détermine celui vers lequel la sauvegarde est écrite.

Pour simplifier le processus de création d'emplois, Bacula vous permet de créer des travaux de définition par défaut qui ne s'exécutent pas eux-mêmes, mais spécifient plutôt les paramètres à hériter des travaux réels. Lors de la modification d'un travail, le champ Type par défaut détermine s'il s'agit d'une définition par défaut, s'il hérite des paramètres d'une valeur par défaut, ou ni l'un ni l'autre.
