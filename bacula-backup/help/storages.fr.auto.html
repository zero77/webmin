Démons de stockage
Chaque entrée de cette page définit une connexion à un périphérique particulier sur un démon de stockage Bacula. En règle générale, le démon s'exécute sur le même système, mais il est possible de définir plusieurs démons, dont certains se trouvent sur des hôtes distants avec différents types de lecteur de bande ou plus d'espace disque.

Les attributs importants pour chaque démon sont:

Nom de stockage
Un nom unique utilisé sur ce système pour identifier le démon.
Mot de passe Bacula SD
Ce champ doit contenir le mot de passe utilisé par le démon de stockage sur le système distant, qui se trouve dans le fichier /etc/bacula/bacula-sd.conf .
Nom d'hôte ou adresse IP
Nom d'hôte ou IP complet du système exécutant le démon de stockage.
Nom du périphérique de stockage
Chaque démon de stockage peut définir un ou plusieurs périphériques, chacun correspondant à un répertoire ou un lecteur de bande différent. Ce champ détermine le périphérique utilisé par cette connexion démon particulière.
Nom du type de média
Ce champ doit contenir un nom court pour le type de support utilisé par le périphérique de stockage, tel que Fichier ou DDS-2 . Il doit être identique au type de support dans la section Périphériques de stockage sur le serveur de stockage.
Il est souvent utile d'avoir plusieurs entrées pour le même hôte sur cette page, chacune ayant un périphérique de stockage différent. Cela vous permet de créer des sauvegardes qui écrivent dans différents répertoires ou lecteurs de bande.

Si les options TLS sont affichées sur le formulaire du démon de stockage, elles peuvent être utilisées pour permettre une communication sécurisée entre le directeur et le serveur de stockage distant. Cela n'est généralement pas nécessaire dans une configuration Bacula simple, car les démons de stockage et de directeur s'exécutent généralement sur le même système.
