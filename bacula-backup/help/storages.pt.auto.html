Daemons de armazenamento
Cada entrada nesta página define uma conexão com um dispositivo específico em um daemon de armazenamento Bacula. Normalmente, o daemon é executado no mesmo sistema, mas é possível definir mais de um daemon, alguns dos quais em hosts remotos com diferentes tipos de unidades de fita ou mais espaço em disco.

Os atributos importantes para cada daemon são:

Nome de armazenamento
Um nome exclusivo usado neste sistema para identificar o daemon.
Senha do Bacula SD
Este campo deve conter a senha usada pelo daemon de armazenamento no sistema remoto, que pode ser encontrado no arquivo /etc/bacula/bacula-sd.conf .
Nome do host ou endereço IP
O nome completo do host ou IP do sistema executando o daemon de armazenamento.
Nome do dispositivo de armazenamento
Cada daemon de armazenamento pode definir um ou mais dispositivos, cada um dos quais corresponde a um diretório ou unidade de fita diferente. Este campo determina qual dispositivo esta conexão daemon específica usa.
Nome do tipo de mídia
Este campo deve conter um nome abreviado para o tipo de mídia usado pelo dispositivo de armazenamento, como Arquivo ou DDS-2 . Ele deve ser o mesmo que o tipo de mídia na seção Dispositivos de armazenamento no servidor de armazenamento.
Muitas vezes, é útil ter mais de uma entrada para o mesmo host nesta página, cada uma com um dispositivo de armazenamento diferente. Isso permite criar backups que gravam em diferentes diretórios ou unidades de fita.

Se as opções TLS forem mostradas no formulário do daemon de armazenamento, elas poderão ser usadas para ativar a comunicação segura entre o diretor e o servidor de armazenamento remoto. Isso geralmente não é necessário em uma configuração simples do Bacula, pois os daemons de armazenamento e diretor geralmente são executados no mesmo sistema.
