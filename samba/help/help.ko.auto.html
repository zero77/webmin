삼바 관리자

이 도움말 페이지가 불완전합니다

소개

Samba는 Windows 95 및 NT를 실행하는 PC에서 Windows 파일 / 인쇄 서버 인 것처럼 Unix 시스템의 파일 및 프린터에 액세스 할 수있는 프리웨어 프로그램입니다. Redhat Linux와 같은 일부 Unix 시스템에는 표준 설치의 일부로 Samba가 포함되어 있으며 다른 시스템에서는 별도로 다운로드하여 설치해야합니다.

시스템에 Samba가 설치되어 있지 않으면 모듈의 기본 페이지에서 Samba 구성 파일을 찾을 수없고 파일이나 디렉토리를 공유 할 수 없다는 불만이 표시됩니다. 당신은 삼바가 설치되어 할 수 있지만,이 오류 메시지가 표시되는 경우, 확인 모듈 구성을 확인 Webmin은 구성 파일 및 삼바 프로그램에 대한 올바른 장소에보고되었는지 확인 할 수 있습니다.

Samba가 올바르게 설치되었다고 가정하면 모듈의 메인 페이지에 시스템에 정의 된 모든 파일 및 프린터 공유 목록이 표시됩니다. Webmin이 Samba 서버가 실행되고 있지 않다고 판단하면 서버 시작 시도 옵션이 제공됩니다. 부팅시 Samba가 시작되지 않은 경우에 필요할 수 있습니다.

파일 공유 생성

Samba 파일 공유는 클라이언트 PC가 마운트 할 수있는 디렉토리입니다. 모든 공유에는 docs 또는 www 와 같은 짧은 이름이 있으며 이는 Unix 시스템의 일부 디렉토리에 맵핑됩니다. 공유는 일반적으로 클라이언트가 \\server\share 표기법을 사용하여 참조합니다.

공유를 작성하려면 기본 페이지에서 기존 공유 목록 아래의 Create File Share 링크를 클릭하십시오. 새 공유의 세부 사항을 입력 할 수있는 양식이 표시됩니다. Samba는 각 공유에 대해 많은 수의 매개 변수를 지원하지만 가장 중요한 것은 다음과 같습니다.

공유 이름 모든 공유에는 클라이언트가 참조 할 때 사용하는 이름이 있어야합니다. 하나의 특별한 경우는 All home directories 공유이며, 나중에 설명합니다.
통로 공유 할 디렉토리입니다. 이 공유를 사용하는 클라이언트는이 디렉토리 내부 또는 아래의 파일에만 액세스 할 수 있습니다.
쓰기 가능 클라이언트가이 공유의 파일 및 디렉토리에 쓸 수 있는지 여부를 결정합니다.
손님 액세스 공유가 게스트 사용자를 허용하면 클라이언트는 유효한 이름과 암호를 제공하지 않고도 액세스 할 수 있습니다. 공유는 게스트 사용자를 허용하지 않거나 게스트 및 일반 사용자를 허용하거나 게스트 만 허용 할 수 있습니다.
다른 공유 옵션에 대한 자세한 내용은 아래 고급 옵션 섹션을 참조하십시오.

새 공유의 세부 사항을 입력하고 Create 단추를 클릭하면 클라이언트 PC에서 즉시 사용할 수 있습니다. 모든 것이 네트워크에서 올바르게 설정되었다고 가정하면 사용자는 Windows에서 Network Neighbourhood 기능을 사용하여 서버와 모든 공유를 볼 수 있습니다. 그런 다음 새로 만든 공유를 PC의 드라이브 문자에 매핑 할 수 있습니다.

프린터 공유 생성

Samba 프린터 공유는 PC 클라이언트가 사용할 수있는 로컬 또는 원격 Unix 프린터입니다. 파일 공유와 마찬가지로 모든 프린터 공유는 짧은 이름 (예 : hp5si )을 가지며 \\server\share 표기법을 사용합니다.

새 프린터 공유를 만들려면 기본 페이지에서와 같은 Create Printer Share 클릭하십시오. 파일 공유 작성에 사용 된 것과 유사한 양식이 표시되며, 새 공유의 세부 사항을 입력 할 수 있습니다. 새로운 프린터 공유를위한 가장 중요한 옵션은 다음과 같습니다.

공유 이름 모든 공유에는 클라이언트가 참조 할 때 사용하는 이름이 있어야합니다. 특별한 경우 중 하나는 All printers 공유이며 나중에 설명합니다.
인쇄기 공유 할 Unix 프린터 lpr 명령을 사용하여 인쇄 할 프린터가 될 수 있습니다.
통로 임시 인쇄 파일이있는 디렉토리
손님 액세스 공유가 게스트 사용자를 허용하면 클라이언트는 유효한 이름과 암호를 제공하지 않고도 액세스 할 수 있습니다. 공유는 게스트 사용자를 허용하지 않거나 게스트 및 일반 사용자를 허용하거나 게스트 만 허용 할 수 있습니다.
프린터 유형 이 프린터에 사용해야하는 프린터 드라이버의 이름입니다. 이 옵션에 값을 입력하고 클라이언트 PC의 프린터 드라이브와 일치하면이 프린터를 추가 할 때 프린터 드라이버를 선택할 필요가 없습니다.

파일 공유와 마찬가지로 인쇄 공유에 더 많은 옵션을 사용할 수 있습니다. 프린터 공유가 추가 되 자마자 클라이언트가 사용할 수 있습니다. Windows 95 및 NT에서는 프린터 제어판의 프린터 Add Printer 옵션을 사용하여 프린터를 추가 할 수 있습니다. 추가되면 사용자는 로컬 또는 네트워크 프린터처럼 인쇄 할 수 있습니다.

공유 편집

기존 공유를 편집하려면 기본 페이지의 공유 목록에서 해당 공유 이름을 클릭하십시오. 공유 작성에 사용 된 것과 동일한 양식이 표시되어 모든 옵션과 매개 변수를 수정할 수 있습니다. 변경 사항을 저장하려면 페이지 왼쪽 하단에있는 Apply 버튼을 클릭하십시오.

공유를 삭제하려면 페이지 오른쪽 하단에있는 Delete 버튼을 클릭하십시오. 기존 사용자와의 연결이 즉시 끊어지지는 않지만 새로운 사용자는 공유에 연결할 수 없습니다.

사용자 및 보안

게스트 액세스를 허용하지 않는 공유의 경우 Samba는 클라이언트가 사용자 이름과 비밀번호를 제공 할 것을 기대합니다. 일반적으로 User Manager 모듈에서 업데이트 할 수있는 Unix 사용자 목록과 비교하여 점검합니다. Samba는 자체 사용자 이름 및 비밀번호 목록을 유지할 수도 있습니다. 자세한 내용은 암호화 된 비밀번호 섹션을 참조하십시오.

일부 Windows 버전에서는 사용자가 공유에 액세스 할 때 사용자 이름을 제공 할 수 없습니다. 대신 Windows에 '로그인'할 때 제공된 사용자 이름이 사용됩니다. 모든 클라이언트 사용자가 Unix에 사용하는 것과 동일한 사용자 이름을 사용하여 Windows에 로그인하면 이는 문제가되지 않습니다. 그렇지 않은 경우 두 가지 옵션이 있습니다.

  1. Windows 95 및 NT는 명령 프롬프트에서 net use drive: share 를 입력하거나 파일 창에서 네트워크 드라이브 연결 단추를 클릭하고 공유 경로를 입력하여 공유를 수동으로 마운트 할 수 있습니다. 두 경우 모두 공유는 \\server\share%user 형식 일 수 있습니다. 여기서 user 는 서버에 로그인하기위한 사용자 이름입니다.

  2. Samba는 올바른 사용자 이름을 찾기 위해 사용자 목록에 제공된 비밀번호와 일치하도록 구성 할 수 있습니다. Unix users 공유 옵션을 사용하면 확인할 사용자 또는 사용자 그룹을 지정할 수 있습니다. 두 사람이 동일한 암호를 가질 수 있기 때문에 많은 수의 사용자와 함께이 옵션을 사용하는 것은 안전하지 않습니다.

    암호화 된 비밀번호

    일반적으로 Samba 서버와 PC 클라이언트간에 네트워크를 통해 전송 된 비밀번호는 암호화되지 않습니다. 이것은 두 가지 이유로 나쁩니다.
    1. 누구나 네트워크에서들을 수 있고 다른 사람의 암호를 캡처 할 수 있습니다.
    2. Windows 9x 및 NT의 최신 버전은 암호화되지 않은 비밀번호 전송을 거부합니다. 이러한 시스템에서 Samba 서버에 액세스하려는 사용자는 ??? 와 같은 오류 메시지를받습니다. . 이 변경 내용은 NT 서비스 팩 3에서 처음 나타났습니다.

    다행히 Samba는 암호화 된 비밀번호를 사용할 수 있습니다. 그러나 사용 된 암호화 형식이 다르기 때문에 클라이언트가 보낸 암호를 일반 Unix 암호 파일과 일치시킬 수 없습니다. 이러한 이유로 Samba는 자체 사용자 및 비밀번호 목록을 유지해야합니다.

    Samba를 암호화 된 비밀번호 모드로 전환하려면 다음을 수행해야합니다.

    1. Samba 버전이 비밀번호 암호화를 지원하는지 확인하십시오. Password Options 페이지에서 Encrypted passwords 옵션을 Encrypted passwords 설정하십시오. 이것이 실패하면, 암호화 된 비밀번호를 지원하는 버전을 다운로드하거나 컴파일해야합니다.

    2. 메인 페이지 하단에서 Convert Unix users to Samba users 링크를 클릭하십시오. 기존 Unix 사용자의 일부 또는 전부를 Samba 사용자로 변환 할 수있는 페이지로 이동합니다. 사용자 이름, 실명, UID 및 쉘은 변환되지만 암호는 변환되지 않습니다.

    3. 사용자에게 비밀번호를 지정하려면 기본 페이지에서 Edit Samba user list 옵션을 사용하십시오. 편집 할 사용자를 선택할 수있는 모든 Samba 사용자 목록이 표시됩니다. 각 사용자의 비밀번호는 Locked (로그인 허용 안 함), No Password (비밀번호 필요 No Password ) 또는 입력 한 일부 비밀번호 일 수 있습니다.
