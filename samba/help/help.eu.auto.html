Laguntza orri hau osatu gabe dago

Sarrera

Samba doako programa bat da, PCak Windows 95 eta NT exekutatzen dituen Unix sisteman fitxategiak eta inprimagailuak sartzeko aukera ematen duena, Windows fitxategi / inprimagailu zerbitzaria izango balitz bezala. Redhat Linux bezalako Unix sistema batzuek Samba instalazio estandarraren baitan hartzen dute; beste gehienek, berriz, deskargatu eta instalatu behar dituzu.

Samba zure sisteman instalatuta ez baduzu, moduluaren orrialde nagusiak Samba konfigurazio fitxategia ezin duela aurkitu salatu du eta ezin izango duzu fitxategi edo direktoriorik partekatu. Samba instalatuta baduzu eta akats mezu hau jasotzen baduzu, egiaztatu moduluaren konfigurazioa ziurtatu Webmin konfigurazio fitxategia eta Samba programak leku egokian bilatzen ari dela.

Samba behar bezala instalatuta dagoela onartuta, moduluaren orrialde nagusiak zure sisteman zehaztutako fitxategi eta inprimagailu akzio guztien zerrenda bistaratuko du. Webmin-ek Samba zerbitzaria ez dagoela exekutatzen zehazten badu, zerbitzaria hasten saiatzeko aukera emango zaio. Baliteke Samba abiarazteko orduan hasi ez bada.

Fitxategi partekatzea sortzen

Samba fitxategi partekatzea bezeroaren ordenagailuak muntatu dezakeen direktorio bat da. Partekatu bakoitzak izen laburra du, hala nola docs edo www , zure Unix sistemako direktorio batzuetara mapatzen duena. Partekatu ohi du bezero batek \\server\share notazioa erabiliz.

Partekatzeko, egin klik Create File Share estekan, orrialde nagusian dauden akzioen zerrendaren azpian. Honek partekatutako berriaren xehetasunak sartzeko aukera ematen duen inprimakia bistaratuko da. Sambak partaidetza bakoitzerako parametro ugari onartzen dituen arren, garrantzitsuenak hauek dira:

Partekatu izena Akzio bakoitzak izen bat izan behar du eta bezeroek aipatzeko erabiltzen dute. Kasu berezi bat All home directories partekatzea da, gero azalduko dena.
Path Partekatu beharreko direktorioa. Partekatzen duten bezeroek direktorio horretako edo azpitik dauden fitxategietara sar dezakete.
Writable Partekatu honetako bezeroek fitxategi eta direktorioetan idatz ditzaketen zehazten du.
Gonbidatuen sarbidea Partekatu batek erabiltzaile gonbidatuak ahalbidetzen baditu, bezeroak bertara sar daitezke, baliozko izen eta pasahitz bat eman beharrik izan gabe. Partekatu batek gonbidapenak erabiltzaile baztertu ditzake, gonbidatuak eta erabiltzaile arruntak edo gonbidatuak soilik baimendu ditzake.
Partekatzeko beste aukerei buruzko informazioa lortzeko, ikus Aukera aurreratuak atala.

Partekatu berri baten xehetasunak sartu eta Create botoia sakatu ondoren, bezeroaren ordenagailuen eskuragarri egongo da berehala. Zure sarean dena ondo konfiguratuta dagoela uste baduzu, erabiltzaileek Network Neighbourhood funtzioa erabili ahal izango dute Windows azpian zure zerbitzaria eta bere akzio guztiak ikusteko. Sortu berri den parteka ordenagailuan unitateko gutun batera maita daiteke.

Inprimagailu partekatua sortzen

Samba inprimagailu partekatua PC bezeroen eskura jarri nahi duzun Unix inprimagailu lokal edo urruna da. Fitxategien akzioak bezala, inprimagailu partekatu bakoitzak izen laburra du ( hp5si bezala), eta \\server\share notazioa erabiliz aipatzen da.

Inprimagailu partekatu berria sortzeko, egin klik Create Printer Share orri nagusian bezala. Honek fitxategien akzioak sortzeko erabilitako antzeko formularioa bistaratuko du eta bertan partekatu berriaren xehetasunak sar ditzakezu. Inprimagailu berria partekatzeko aukerarik garrantzitsuenak hauek dira:

Partekatu izena Akzio bakoitzak izen bat izan behar du eta bezeroek aipatzeko erabiltzen dute. Kasu berezi bat All printers partekatzea da, gero azalduko dena.
Printer Partekatu beharreko Unix inprimagailua. lpr agindua erabiliz inprimatuko zenuen edozein inprimagailu izan daiteke.
Path Behin-behineko fitxategiak inprimatzeko direktorioa
Gonbidatuen sarbidea Partekatu batek erabiltzaile gonbidatuak ahalbidetzen baditu, bezeroak bertara sar daitezke, baliozko izen eta pasahitz bat eman beharrik izan gabe. Partekatu batek gonbidapenak erabiltzaile baztertu ditzake, gonbidatuak eta erabiltzaile arruntak edo gonbidatuak soilik baimendu ditzake.
Inprimagailu mota Inprimagailu honetarako erabili behar den inprimagailuaren izena. Aukera honen balioa sartzen baduzu eta inprimagailu unitatearekin bat dator bezero bezeroetan, orduan erabiltzaileek ez dute inprimagailu gidaririk aukeratu behar inprimagailu hau gehitzean.

Fitxategien akzioekin gertatzen den bezala, aukera gehiago daude inprimatzeko akzioetarako. Inprimagailu partekatu bat gehitu bezain laster, bezeroen eskura egongo da. Windows 95 eta NT azpian, inprimagailu bat gehitu daiteke inprimagailuak kontrol panelean Add Printer erabiliz. Gehituta, erabiltzaileak edozein tokiko edo sareko inprimagailu bezala inprimatu dezake.

Kopia sortzen

Partekazioa editatzen

Lehendik dagoen akzio bat editatzeko, egin klik haren izenean orri nagusiko akzioen zerrendatik. Honek partekatzea sortzeko erabilitako formulario bera bistaratuko da, aukera eta parametro guztiak aldatzeko aukera emanez. Aldaketak gordetzeko, egin klik orriaren beheko ezkerraldean Apply botoian.

Partekazio bat ezabatzeko, egin klik orrialdearen beheko eskuinean Delete botoian. Lehendik dauden erabiltzaileak ez dira berehala deskonektatuko. Hala ere, erabiltzaile berririk ezin izango da partekatu partekatzera.

Erabiltzaileak eta Segurtasuna

Gonbidatuak sartzea baimentzen ez duten akzioetarako, Sambak espero du bezeroak erabiltzaile izena eta pasahitza ematea. Normalean, Erabiltzaile Kudeaketa moduluak eguneratu ditzakeen Unix erabiltzaile zerrendaren aurka egiaztatzen dira. Sambak erabiltzaile izen propioa eta pasahitzen zerrenda ere gorde dezake - ikus Zifratutako pasahitzak atalean xehetasun gehiago lortzeko.

Windows-eko bertsio batzuek ez diote erabiltzaileari erabiltzaileak parteka sartzeko unean erabiltzaile izenik ematen ematen. Horren ordez, Windows-en 'saioa hasterakoan emandako izena' erabiltzen da. Bezero erabiltzaile guztiek Windows-en saioa egiten badute Unix erabiltzen duten erabiltzaile izen bera erabilita, orduan ez da arazorik. Bestela, 2 aukera dituzu:

  1. Windows 95 eta NT-k parteka eskuz muntatu dezakete, bai net use drive: share idatziz net use drive: share komando gonbidapenean edo mapako sareko unitatearen botoian klik eginez fitxategi leihoan eta sartu partekatzeko bidea. Bi kasuetan, parteka \\server\share%user formatuan egon daiteke, non erabiltzaileak zerbitzarian sartzeko erabiltzaile izena duen.

  2. Samba erabiltzaile-zerrenda batekin emandako pasahitzarekin bat egiten saiatzeko konfiguratu daiteke, erabiltzaile-izena egokia aurkitzeko. Unix users partekatzeko aukera, erabiltzaileek edo erabiltzaileek taldeek zer aurka egin behar duten zehazteko aukera ematen du. Kontuz erabiltzaile honen aukera asko erabiltzea ez dela oso segurua, izan ere, 2 pertsonek pasahitz bera izan dezakete.

    Zifratutako pasahitzak

    Normalean, Samba zerbitzariaren eta PC bezeroen artean sarean bidalitako pasahitzak ez dira enkriptatzen. Hau txarra da 2 arrazoirengatik:
    1. Edozeinek zure sarean entzun eta besteen pasahitzak atera ditzake.
    2. Windows 9x eta NT bertsio berrienek ukatu gabeko pasahitzak bidaltzeari uko egiten diote. Sistema horietatik zure Samba zerbitzaria sartu nahian dabiltzan erabiltzaileek akats mezu bat jasoko dute ??? . Aldaketa hau NT service pack 3-an agertu zen lehenengo aldiz.

    Zorionez, Sambak enkriptatutako pasahitzak erabil ditzake. Hala ere, ezin da bezeroek bidalitako pasahitzak Unix pasahitz arruntaren fitxategiaren aurka lotu, erabiltzen diren kodeketa mota desberdinak direla eta. Hori dela eta, Sambak erabiltzaileen eta pasahitzen zerrenda propioa mantendu behar du.

    Samba enkriptatutako pasahitz moduan aldatzeko, hau egin behar duzu:

    1. Ziurtatu zure Samba bertsioak pasahitza zifratzea onartzen duela. Password Options orrialdean, ezarri Encrypted passwords aukera bai. Huts egiten badu, bertsio bat deskargatu edo konpilatu beharko duzu pasahitz zifratutako laguntzarekin.

    2. Egin klik Convert Unix users to Samba users esteka orrialde nagusian. Honek Lehendik dauden Unix erabiltzaile batzuk Samba erabiltzaileetara bihur ditzakezun orri batera eramango zaitu. Erabiltzaile-izenak, benetako izenak, UIDak eta maskorrak bihurtuko dira, baina pasahitzak ez dira izango.

    3. Erabiltzaileak pasahitzak esleitzeko, erabili Edit Samba user list aukera orri nagusian. Honek Samba erabiltzaile guztien zerrenda bistaratuko du, eta horietatik erabiltzaile bat editatu ahal izango duzu. Erabiltzaile bakoitzeko pasahitza Locked egon daiteke (ez da saioa onartzen), No Password ez (pasahitzik behar) edo sartutako pasahitzren bat.
