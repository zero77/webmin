Einführung

Samba ist ein Freeware-Programm, mit dem PCs unter Windows 95 und NT auf Dateien und Drucker auf Ihrem Unix-System zugreifen können, als wäre es ein Windows-Datei- / Druckserver. Einige Unix-Systeme wie Redhat Linux enthalten Samba als Teil der Standardinstallation, während die meisten anderen erfordern, dass Sie es separat herunterladen und installieren.

Wenn Sie Samba nicht auf Ihrem System installiert haben, wird auf der Hauptseite des Moduls beanstandet, dass die Samba-Konfigurationsdatei nicht gefunden werden kann und Sie keine Dateien oder Verzeichnisse freigeben können. Wenn Sie Samba installiert haben, aber diese Fehlermeldung erhalten, überprüfen Sie die Modulkonfiguration , um sicherzustellen, dass Webmin an der richtigen Stelle nach der Konfigurationsdatei und den Samba-Programmen sucht.

Vorausgesetzt, Samba ist ordnungsgemäß installiert, wird auf der Hauptseite des Moduls eine Liste aller auf Ihrem System definierten Datei- und Druckerfreigaben angezeigt. Wenn Webmin feststellt, dass der Samba-Server nicht ausgeführt wird, können Sie versuchen, den Server zu starten. Dies kann erforderlich sein, wenn Samba beim Booten nicht gestartet wird.

Erstellen einer Dateifreigabe

Eine Samba-Dateifreigabe ist ein Verzeichnis, das von Client-PCs bereitgestellt werden kann. Jede Freigabe hat einen Kurznamen wie docs oder www , der einem Verzeichnis auf Ihrem Unix-System zugeordnet ist. Eine Freigabe wird normalerweise von einem Client unter Verwendung der Notation \\server\share .

Um eine Freigabe zu erstellen, klicken Create File Share auf der Hauptseite unter der Liste der vorhandenen Freigaben auf den Link Create File Share . Daraufhin wird ein Formular angezeigt, in das Sie die Details der neuen Freigabe eingeben können. Obwohl Samba eine große Anzahl von Parametern für jede Freigabe unterstützt, sind die wichtigsten:

Teilename Jede Freigabe muss einen Namen haben, mit dem Kunden darauf verweisen. Ein Sonderfall ist die Freigabe All home directories , die später erläutert wird.
Pfad Das Verzeichnis, das freigegeben werden soll. Clients, die diese Freigabe verwenden, können nur auf Dateien in oder unter diesem Verzeichnis zugreifen.
Schreibbar Legt fest, ob Clients in Dateien und Verzeichnisse in dieser Freigabe schreiben können.
Gastzugang Wenn eine Freigabe Gastbenutzer zulässt, können Clients darauf zugreifen, ohne einen gültigen Namen und ein gültiges Kennwort angeben zu müssen. Eine Freigabe kann entweder Gastbenutzer verbieten, Gäste und normale Benutzer zulassen oder nur Gäste zulassen.
Informationen zu anderen Freigabeoptionen finden Sie im Abschnitt Erweiterte Optionen weiter unten.

Sobald Sie die Details einer neuen Freigabe eingegeben und auf die Schaltfläche Create geklickt haben, steht diese sofort den Client-PCs zur Verfügung. Vorausgesetzt, in Ihrem Netzwerk ist alles korrekt eingerichtet, können Benutzer die Funktion " Network Neighbourhood unter Windows verwenden, um Ihren Server und alle seine Freigaben anzuzeigen. Die neu erstellte Freigabe kann dann einem Laufwerksbuchstaben auf dem PC zugeordnet werden.

Erstellen einer Druckerfreigabe

Eine Samba-Druckerfreigabe ist ein lokaler oder entfernter Unix-Drucker, den Sie PC-Clients zur Verfügung stellen möchten. Wie Dateifreigaben hat jede Druckerfreigabe einen Kurznamen (wie hp5si ) und wird mit der Notation \\server\share .

Um eine neue Druckerfreigabe zu erstellen, klicken Sie auf der Hauptseite auf Create Printer Share wie. Daraufhin wird ein ähnliches Formular wie beim Erstellen von Dateifreigaben angezeigt, in das Sie die Details der neuen Freigabe eingeben können. Die wichtigsten Optionen für eine neue Druckerfreigabe sind:

Teilename Jede Freigabe muss einen Namen haben, mit dem Kunden darauf verweisen. Ein Sonderfall ist die Freigabe All printers , die später erläutert wird.
Drucker Der freizugebende Unix-Drucker. Dies kann jeder Drucker sein, auf dem Sie mit dem Befehl lpr drucken würden.
Pfad Das Verzeichnis, in dem temporäre Dateien gedruckt werden
Gastzugang Wenn eine Freigabe Gastbenutzer zulässt, können Clients darauf zugreifen, ohne einen gültigen Namen und ein gültiges Kennwort angeben zu müssen. Eine Freigabe kann entweder Gastbenutzer verbieten, Gäste und normale Benutzer zulassen oder nur Gäste zulassen.
Druckertyp Der Name des Druckertreibers, der für diesen Drucker verwendet werden soll. Wenn Sie für diese Option einen Wert eingeben, der mit einem Druckerlaufwerk auf Client-PCs übereinstimmt, müssen Benutzer beim Hinzufügen dieses Druckers keinen Druckertreiber auswählen.

Wie bei Dateifreigaben stehen für Druckfreigaben viel mehr Optionen zur Verfügung. Sobald eine Druckerfreigabe hinzugefügt wird, steht sie den Clients zur Verfügung. Unter Windows 95 und NT kann ein Drucker mithilfe der Option Drucker hinzufügen in der Add Printer hinzugefügt werden. Nach dem Hinzufügen kann der Benutzer wie jeder lokale oder Netzwerkdrucker darauf drucken.

Bearbeiten einer Freigabe

Um eine vorhandene Freigabe zu bearbeiten, klicken Sie auf den Namen in der Liste der Freigaben auf der Hauptseite. Daraufhin wird dasselbe Formular angezeigt, das zum Erstellen einer Freigabe verwendet wurde, sodass Sie alle Optionen und Parameter ändern können. Um Ihre Änderungen zu speichern, klicken Apply unten links auf der Seite auf die Schaltfläche Apply .

Um eine Freigabe zu löschen, klicken Sie unten rechts auf der Seite auf die Schaltfläche Delete . Bestehende Benutzer werden nicht sofort getrennt. Es können jedoch keine neuen Benutzer eine Verbindung zur Freigabe herstellen.

Benutzer und Sicherheit

Für Freigaben, die keinen Gastzugriff zulassen, erwartet Samba, dass der Client einen Benutzernamen und ein Kennwort angibt. In der Regel werden diese mit der Unix-Benutzerliste verglichen, die vom Benutzermanager- Modul aktualisiert werden kann. Samba kann auch eine eigene Liste mit Benutzernamen und Kennwörtern führen. Weitere Informationen finden Sie im Abschnitt Verschlüsselte Kennwörter .

In einigen Windows-Versionen kann der Benutzer beim Zugriff auf eine Freigabe keinen Benutzernamen angeben. Stattdessen wird der Benutzername verwendet, der beim Anmelden bei Windows angegeben wird. Wenn sich alle Clientbenutzer mit demselben Benutzernamen bei Windows anmelden, den sie für Unix verwenden, ist dies kein Problem. Wenn nicht, haben Sie zwei Möglichkeiten:

  1. Windows 95 und NT können eine Freigabe manuell bereitstellen, indem Sie entweder an der Eingabeaufforderung net use drive: share eingeben oder in einem Dateifenster auf die Schaltfläche Netzwerklaufwerk zuordnen klicken und den Freigabepfad eingeben. In beiden Fällen kann die Freigabe das Format \\server\share%user , wobei Benutzer der Benutzername ist, mit dem Sie sich beim Server anmelden möchten.

  2. Samba kann so konfiguriert werden, dass versucht wird, das angegebene Kennwort mit einer Liste von Benutzern abzugleichen, um den richtigen Benutzernamen zu finden. Mit der Unix users können Sie die Benutzer oder Benutzergruppen angeben, gegen die geprüft werden soll. Beachten Sie, dass die Verwendung dieser Option bei einer großen Anzahl von Benutzern nicht sehr sicher ist, da möglicherweise zwei Personen dasselbe Kennwort haben.

    Verschlüsselte Passwörter

    Normalerweise werden über das Netzwerk zwischen einem Samba-Server und PC-Clients gesendete Kennwörter nicht verschlüsselt. Das ist aus zwei Gründen schlecht:
    1. Jeder kann in Ihrem Netzwerk mithören und die Passwörter anderer Personen erfassen.
    2. Die neuesten Versionen unter Windows 9x und NT lehnen es ab, Kennwörter unverschlüsselt zu senden. Benutzer, die versuchen, von diesen Systemen aus auf Ihren Samba-Server zuzugreifen, erhalten eine Fehlermeldung wie ??? . Diese Änderung wurde erstmals in NT Service Pack 3 angezeigt.

    Glücklicherweise kann Samba verschlüsselte Passwörter verwenden. Aufgrund der unterschiedlichen Verschlüsselungsformen ist es jedoch nicht möglich, von Clients gesendete Kennwörter mit der normalen Unix-Kennwortdatei abzugleichen. Aus diesem Grund muss Samba eine eigene Liste von Benutzern und Passwörtern führen.

    Um Samba in den verschlüsselten Passwortmodus zu versetzen, müssen Sie:

    1. Stellen Sie sicher, dass Ihre Samba-Version die Kennwortverschlüsselung unterstützt. In der Password Options Seite, setzen Sie die Encrypted passwords von Encrypted passwords Option auf yes. Wenn dies fehlschlägt, müssen Sie eine Version mit Unterstützung für verschlüsselte Kennwörter herunterladen oder kompilieren.

    2. Klicken Sie unten auf der Hauptseite auf den Link Convert Unix users to Samba users . Dadurch gelangen Sie zu einer Seite, auf der Sie einige oder alle Ihrer vorhandenen Unix-Benutzer in Samba-Benutzer konvertieren können. Benutzernamen, echte Namen, UIDs und Shells werden konvertiert - Passwörter jedoch nicht.

    3. Verwenden Sie zum Zuweisen von Kennwörtern zu Benutzern die Option Edit Samba user list auf der Hauptseite. Daraufhin wird eine Liste aller Samba-Benutzer angezeigt, aus der Sie einen Benutzer zum Bearbeiten auswählen können. Das Passwort für jeden Benutzer kann entweder Locked (kein Login erlaubt), No Password (kein Passwort erforderlich) oder ein von Ihnen eingegebenes Passwort sein.
