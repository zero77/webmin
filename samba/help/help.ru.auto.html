Введение

Samba - это бесплатная программа, которая позволяет ПК под управлением Windows 95 и NT получать доступ к файлам и принтерам в вашей системе Unix, как если бы это был сервер файлов / печати Windows. Некоторые системы Unix, такие как Redhat Linux, включают Samba как часть стандартной установки, в то время как для большинства других требуется, чтобы вы загружали и устанавливали его отдельно.

Если в вашей системе не установлена Samba, главная страница модуля будет жаловаться на то, что он не может найти файл конфигурации Samba, и вы не сможете поделиться какими-либо файлами или каталогами. Если у вас установлена Samba, но вы получаете это сообщение об ошибке, проверьте конфигурацию модуля, чтобы убедиться, что Webmin ищет в нужном месте файл конфигурации и программы Samba.

При условии, что Samba установлена правильно, на главной странице модуля отобразится список всех общих файловых и принтерных ресурсов, определенных в вашей системе. Если Webmin определит, что сервер Samba не запущен, вам будет предложено попытаться запустить сервер. Это может быть необходимо, если Samba не запускается во время загрузки.

Создание общего файлового ресурса

Общий файловый ресурс Samba - это каталог, который может быть смонтирован на клиентских ПК. Каждый ресурс имеет короткое имя, такое как docs или www , которое соответствует некоторому каталогу в вашей системе Unix. К общему ресурсу обычно обращается клиент с помощью нотации \\server\share .

Чтобы создать общий ресурс, нажмите ссылку « Create File Share ресурс» под списком существующих общих ресурсов на главной странице. Появится форма, позволяющая вам ввести детали нового ресурса. Хотя Samba поддерживает большое количество параметров для каждого общего ресурса, наиболее важными из них являются:

Поделиться Имя Каждая акция должна иметь имя, используемое клиентами для ссылки на него. Один особый случай - это общий All home directories , поясняемый ниже.
Дорожка Каталог для совместного использования. Клиенты, использующие этот общий ресурс, могут получать доступ только к файлам в этом каталоге или ниже.
Writable Определяет, могут ли клиенты выполнять запись в файлы и каталоги в этой общей папке.
Гостевой доступ Если общий ресурс разрешает использование гостевых пользователей, клиенты могут получить к нему доступ без указания правильного имени и пароля. Общий ресурс может либо запрещать гостевых пользователей, разрешать гостям и обычным пользователям, либо разрешать только гостям.
Сведения о других параметрах общего доступа см. В разделе «Дополнительные параметры» ниже.

После того, как вы ввели данные о новом общем ресурсе и нажали кнопку « Create , он будет сразу же доступен для клиентских ПК. Предполагая, что в вашей сети все настроено правильно, пользователи смогут использовать функцию Network Neighbourhood в Windows, чтобы увидеть ваш сервер и все его общие ресурсы. Вновь созданный общий ресурс может быть сопоставлен с буквой диска на ПК.

Создание общего ресурса принтера

Общий ресурс принтера Samba - это локальный или удаленный принтер Unix, который вы хотите сделать доступным для клиентов ПК. Как и в случае общих файловых ресурсов, каждый общий ресурс принтера имеет короткое имя (например, hp5si ), и на него ссылается нотация \\server\share .

Чтобы создать новый общий ресурс принтера, нажмите « Create Printer Share как на главной странице. Появится форма, аналогичная той, которая используется для создания общих файловых ресурсов, в которой вы можете ввести информацию о новой общей папке. Наиболее важные параметры для нового общего ресурса принтера:

Поделиться Имя Каждая акция должна иметь имя, используемое клиентами для ссылки на него. Один особый случай - это общий All printers , объясненный ниже.
принтер Принтер Unix для общего доступа. Это может быть любой принтер, на котором вы будете печатать, используя команду lpr .
Дорожка Каталог, в котором находятся временные файлы печати
Гостевой доступ Если общий ресурс разрешает использование гостевых пользователей, клиенты могут получить к нему доступ без указания правильного имени и пароля. Общий ресурс может либо запрещать гостевых пользователей, разрешать гостям и обычным пользователям, либо разрешать только гостям.
Тип принтера Имя драйвера принтера, который должен использоваться для этого принтера. Если вы введете значение для этой опции, и оно будет совпадать с диском принтера на клиентских ПК, пользователям не придется выбирать драйвер принтера при добавлении этого принтера.

Как и в случае с общими файлами, для печати можно использовать гораздо больше параметров. Как только общий ресурс принтера будет добавлен, он будет доступен для клиентов. В Windows 95 и NT принтер можно добавить с помощью параметра « Add Printer на панели управления принтерами. После добавления пользователь может печатать на нем, как на любом локальном или сетевом принтере.

Редактирование доли

Чтобы отредактировать существующий общий ресурс, нажмите на его название в списке общих ресурсов на главной странице. Появится та же форма, что и для создания общего ресурса, что позволит вам изменить все параметры и параметры. Чтобы сохранить изменения, нажмите кнопку « Apply в левом нижнем углу страницы.

Чтобы удалить общий ресурс, нажмите кнопку « Delete в правом нижнем углу страницы. Существующие пользователи не будут немедленно отключены, однако новые пользователи не смогут подключиться к общему ресурсу.

Пользователи и безопасность

Для общих ресурсов, которые не разрешают гостевой доступ, Samba ожидает, что клиент предоставит имя пользователя и пароль. Как правило, они проверяются по списку пользователей Unix, который может обновляться модулем User Manager . Samba также может хранить свои собственные имена пользователей и пароли - см. Раздел «Зашифрованные пароли» для получения более подробной информации.

Некоторые версии Windows не позволяют пользователю указывать имя пользователя при доступе к общему ресурсу - вместо этого используется имя пользователя, указанное при входе в Windows. Если все пользователи клиента входят в Windows под тем же именем, которое они используют для Unix, то это не проблема. Если нет, у вас есть 2 варианта:

  1. Windows 95 и NT могут смонтировать общий ресурс вручную, введя команду net use drive: share в командной строке или нажав кнопку сетевого диска карты в окне файла и введя путь общего ресурса. В обоих случаях общий ресурс может быть в формате \\server\share%user , где user - это имя пользователя для входа на сервер.

  2. Samba может быть настроен на попытку сопоставления пароля, предоставленного списку пользователей, чтобы найти правильное имя пользователя. Опция Unix users позволяет вам указать пользователей или группы пользователей для проверки. Помните, что использование этой опции с большим количеством пользователей не очень безопасно, потому что 2 человека могут иметь один и тот же пароль.

    Зашифрованные пароли

    Обычно пароли, отправляемые по сети между сервером Samba и клиентами ПК, не шифруются. Это плохо по 2 причинам:
    1. Любой может прослушивать в вашей сети и захватывать пароли других людей.
    2. Последние версии Windows 9x и NT отказываются отправлять пароли в незашифрованном виде. Пользователи, пытающиеся получить доступ к вашему серверу Samba из этих систем, получат сообщение об ошибке, подобное ??? , Это изменение впервые появилось в пакете обновления 3 для NT.

    К счастью, Samba может использовать зашифрованные пароли. Однако невозможно сопоставить пароли, отправленные клиентами, с обычным файлом паролей Unix из-за различных используемых форм шифрования. По этой причине Samba должна вести собственный список пользователей и паролей.

    Чтобы переключить Samba в режим зашифрованного пароля, вы должны:

    1. Убедитесь, что ваша версия Samba поддерживает шифрование пароля. На странице « Password Options установите для параметра « Encrypted passwords значение «Да». Если это не поможет, вам нужно будет скачать или скомпилировать версию с поддержкой зашифрованных паролей.

    2. Нажмите на ссылку Convert Unix users to Samba users внизу главной страницы. Это приведет вас к странице, на которой вы сможете преобразовать некоторых или всех существующих пользователей Unix в пользователей Samba. Имена пользователей, настоящие имена, UID и оболочки будут преобразованы, но пароли не будут.

    3. Чтобы назначить пароли пользователям, используйте параметр Edit Samba user list на главной странице. Появится список всех пользователей Samba, из которого вы можете выбрать пользователя для редактирования. Пароль для каждого пользователя может быть Locked (без входа в систему), No Password (без пароля) или с помощью введенного вами пароля.
