Διαμόρφωση διακομιστή LDAP
Αυτή η σελίδα σάς επιτρέπει να επιλέξετε το διακομιστή LDAP στον οποίο θα επικοινωνήσει αυτό το σύστημα πελάτη για να πάρει πληροφορίες χρήστη και ομάδας. Το πιο σημαντικό πεδίο είναι τα ονόματα κεντρικών υπολογιστών διακομιστή LDAP , στα οποία πρέπει να εισαγάγετε το όνομα κεντρικού υπολογιστή ή τη διεύθυνση IP του διακομιστή LDAP στο δίκτυό σας. Εάν έχετε περισσότερους από έναν επαναληπτικούς διακομιστές, μπορούν όλοι να εισαχθούν εδώ.

Τα πεδία Σύνδεση για μη ριζικούς χρήστες και Κωδικός πρόσβασης για μη ριζικούς χρήστες πρέπει να συμπληρωθούν με όνομα χρήστη και κωδικό πρόσβασης αποδεκτό από τον διακομιστή LDAP, αντίστοιχα. Τα στοιχεία σύνδεσης θα είναι συνήθως ένα διακεκριμένο όνομα LDAP, όπως cn = Manager, dc = my-domain, dc = com .

Επειδή αυτή η σύνδεση και ο κωδικός πρόσβασης είναι ορατές σε όλους τους χρήστες του Unix στο σύστημά σας, συνήθως ορίζεται σε αυτόν ενός χρήστη LDAP με περιορισμένα δικαιώματα (όπως πρόσβαση μόνο για ανάγνωση και χωρίς δυνατότητα προβολής κωδικών πρόσβασης). Επειδή οι κωδικοί πρόσβασης πρέπει να ελέγχονται σε ορισμένες περιπτώσεις, τα πεδία Σύνδεση για χρήστη ρίζας και Κωδικός πρόσβασης για root χρήστες πρέπει να οριστούν σε όνομα χρήστη και κωδικό πρόσβασης LDAP που έχει πρόσβαση ανάγνωσης σε ολόκληρο τον διακομιστή LDAP. Αυτός ο δεύτερος κωδικός πρόσβασης αποθηκεύεται σε ένα αρχείο που μπορεί να διαβαστεί μόνο από τον χρήστη ρίζας και επομένως είναι ασφαλής από τους κανονικούς μη αξιόπιστους χρήστες του Unix.
