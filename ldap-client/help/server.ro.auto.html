Configurare server LDAP
Această pagină vă permite să selectați serverul LDAP cu care va contacta acest sistem client pentru a obține informații despre utilizator și grup. Cel mai important câmp este numele de host al serverului LDAP , în care trebuie să introduceți numele de gazdă sau adresa IP a serverului LDAP din rețeaua dvs. Dacă aveți mai multe servere replicate, toate pot fi introduse aici.

Câmpurile de conectare pentru utilizatorii non-root și Parola pentru utilizatorii non-root trebuie completate cu un nume de utilizator și o parolă acceptate de serverul LDAP, respectiv. Autentificarea va fi de obicei un nume distins LDAP, cum ar fi cn = Manager, dc = my-domain, dc = com .

Deoarece această conectare și parolă sunt vizibile pentru toți utilizatorii Unix din sistemul dvs., este de obicei setată pe cea a unui utilizator LDAP cu privilegii limitate (cum ar fi accesul numai în citire și fără posibilitatea de a vizualiza parolele). Deoarece parolele trebuie să fie verificate în unele situații, conectarea pentru utilizator root și Parola pentru câmpurile de utilizator root ar trebui să fie setate la un nume de utilizator și o parolă LDAP care au citit acces la întregul server LDAP. Această a doua parolă este stocată într-un fișier care poate fi citit doar de către utilizatorul rădăcină și, astfel, este protejat de utilizatorii obișnuiți de încredere Unix.
