Configurazione del server LDAP
Questa pagina consente di selezionare il server LDAP a cui questo sistema client contatterà per recuperare informazioni su utenti e gruppi. Il campo più importante sono i nomi host del server LDAP , in cui è necessario immettere il nome host o l'indirizzo IP del server LDAP sulla rete. Se hai più di un server replicato, possono essere inseriti tutti qui.

I campi Accesso per utenti non root e Password per utenti non root devono essere compilati rispettivamente con un nome utente e una password accettati dal server LDAP. Il login sarà in genere un nome distinto LDAP, come cn = Manager, dc = my-domain, dc = com .

Poiché questo login e password sono visibili a tutti gli utenti Unix sul sistema, in genere è impostato su quello di un utente LDAP con privilegi limitati (come l'accesso in sola lettura e nessuna possibilità di visualizzare le password). Poiché le password devono essere verificate in alcune situazioni, i campi Accesso per utente root e Password per utente root devono essere impostati su un nome utente e una password LDAP che hanno accesso in lettura all'intero server LDAP. Questa seconda password è memorizzata in un file che è leggibile solo dall'utente root e quindi è sicura da utenti Unix non affidabili.
