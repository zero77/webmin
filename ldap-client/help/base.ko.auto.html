LDAP 검색 기반
LDAP 서버의 데이터베이스는 계층 적으로 구성되며 일반적으로 사용자, 그룹 및 기타 서비스를 위해 트리에 서로 다른 분기가 있습니다. 이 페이지에서는 LDAP 데이터베이스에서 다양한 유형의 정보를 검색 할 위치와 각 정보를 검색 할 깊이를 선택할 수 있습니다.

기본적으로 모든 검색은 전체 검색 기반 에서 수행됩니다. 그러나 대부분의 서버는 사용자와 그룹을 서로 다른 하위 트리에 배치하기 때문에 Unix 사용자 용 Base와 Unix 용 Base 그룹 을 해당 하위 트리의 고유 이름으로 설정해야합니다. 암호 정보는 거의 항상 다른 사용자 속성에 저장 될 때 유닉스 암호에 대한 자료는, 유닉스 사용자를위한 자료와 동일한 DN으로 설정해야합니다.
