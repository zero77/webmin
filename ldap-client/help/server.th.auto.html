การกำหนดค่าเซิร์ฟเวอร์ LDAP
หน้านี้ให้คุณเลือกเซิร์ฟเวอร์ LDAP ที่ระบบไคลเอ็นต์นี้จะติดต่อเพื่อดึงข้อมูลผู้ใช้และกลุ่ม ฟิลด์ที่สำคัญที่สุดคือ ชื่อโฮสต์เซิร์ฟเวอร์ LDAP ซึ่งคุณจะต้องป้อนชื่อโฮสต์หรือที่อยู่ IP ของเซิร์ฟเวอร์ LDAP ในเครือข่ายของคุณ หากคุณมีเซิร์ฟเวอร์ที่จำลองแบบมากกว่าหนึ่งเซิร์ฟเวอร์พวกเขาทั้งหมดสามารถเข้ามาที่นี่

ฟิลด์ ล็อกอินสำหรับผู้ใช้ที่ไม่ใช่รูท และ รหัสผ่านสำหรับ ผู้ใช้ที่ ไม่ใช่รูท ต้องกรอกข้อมูลด้วยชื่อผู้ใช้และรหัสผ่านที่เซิร์ฟเวอร์ LDAP ยอมรับตามลำดับ โดยทั่วไปการเข้าสู่ระบบจะเป็นชื่อจำเพาะของ LDAP เช่น cn = Manager, dc = my-domain, dc = com

เนื่องจากการเข้าสู่ระบบและรหัสผ่านนี้จะปรากฏแก่ผู้ใช้ Unix ทั้งหมดในระบบของคุณโดยทั่วไปจะถูกตั้งเป็นผู้ใช้ LDAP ที่มีสิทธิ์ จำกัด (เช่นการเข้าถึงแบบอ่านอย่างเดียวและไม่มีความสามารถในการดูรหัสผ่าน) เนื่องจากจำเป็นต้องตรวจสอบรหัสผ่านในบางสถานการณ์ฟิลด์ ล็อกอินสำหรับผู้ใช้รูท และ รหัสผ่านสำหรับผู้ใช้รูท ควรตั้งค่าเป็นชื่อผู้ใช้และรหัสผ่าน LDAP ที่เข้าถึงการอ่านไปยังเซิร์ฟเวอร์ LDAP ทั้งหมด รหัสผ่านที่สองนี้ถูกเก็บไว้ในไฟล์ที่ผู้ใช้ รูท สามารถอ่านได้เท่านั้นและจะปลอดภัยจากผู้ใช้ Unix ที่ไม่น่าเชื่อถือปกติ
