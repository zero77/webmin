Конфігурація сервера LDAP
Ця сторінка дозволяє вибрати сервер LDAP, до якого ця клієнтська система звернеться, щоб отримати інформацію про користувачів та групи. Найважливішим полем є імена хостів сервера LDAP , в яке потрібно ввести ім'я хоста або IP-адресу сервера LDAP у вашій мережі. Якщо у вас є декілька реплікуваних серверів, їх можна ввести сюди.

Поля Вхід для некорінних користувачів та Пароль для некорінних користувачів повинні бути заповнені відповідно ім'ям користувача та паролем, прийнятими сервером LDAP. Як правило, логін має бути відомим іменем LDAP, наприклад, cn = Manager, dc = мій домен, dc = com .

Оскільки цей логін та пароль видно всім користувачам Unix у вашій системі, він зазвичай встановлюється для користувача LDAP з обмеженими привілеями (наприклад, доступ лише для читання та відсутність можливості перегляду паролів). Оскільки паролі потрібно перевіряти в деяких ситуаціях, поля Увійти для кореневого користувача та Пароль для кореневих користувачів повинні бути встановлені на ім'я користувача та пароль LDAP, які мають доступ до доступу до всього сервера LDAP. Цей другий пароль зберігається у файлі, який читається тільки кореневим користувачем, і таким чином захищений від звичайних непідтверджених користувачів Unix.
