Asynchroon schrijft
"Ja" - Schrijft naar de server asynchroon. Wanneer uw NFS-client een schrijfverzoek naar een NFS-server verzendt, retourneert de server een positief antwoord aan uw client voordat er gegevens naar de schijf worden geschreven. Uw cliÃ«nt kan dan nog een schrijfverzoek verzenden zonder te wachten totdat de server het vorige schrijven heeft voltooid. De schrijfprestaties zijn beter bij asynchrone schrijfbewerkingen dan bij synchrone schrijfbewerkingen. Als de server echter crasht nadat de positieve reactie naar de client is verzonden, maar voordat de gegevens naar de schijf zijn geschreven, gaan de gegevens verloren en wordt de client niet op de hoogte gesteld.

"Nee" - Schrijft naar de server gebeurt synchroon. Wanneer uw NFS-client een schrijfverzoek naar een NFS-server verzendt, schrijft de server de gegevens naar zijn schijf voordat hij een positief antwoord aan uw client retourneert. Uw client moet wachten totdat de server de gegevens naar de schijf schrijft voordat hij nog een schrijfverzoek kan verzenden. De schrijfprestaties zijn niet zo goed bij synchrone schrijfbewerkingen als bij asynchrone schrijfbewerkingen.

De standaard is "Nee".
