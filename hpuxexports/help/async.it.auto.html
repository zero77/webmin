Scritture asincrone
"Sì": le scritture sul server vengono eseguite in modo asincrono. Quando il client NFS invia una richiesta di scrittura a un server NFS, il server restituisce una risposta positiva al client prima di scrivere qualsiasi dato sul suo disco. Il client può quindi inviare un'altra richiesta di scrittura senza attendere che il server completi la scrittura precedente. Le prestazioni di scrittura sono migliori con le scritture asincrone che con le scritture sincrone. Tuttavia, se il server si arresta in modo anomalo dopo aver inviato la risposta positiva al client ma prima di scrivere i dati sul disco, i dati vengono persi e il client non viene avvisato.

"No": le scritture sul server vengono eseguite in modo sincrono. Quando il client NFS invia una richiesta di scrittura a un server NFS, il server scrive i dati sul proprio disco prima di restituire una risposta positiva al client. Il client deve attendere fino a quando il server non scrive i dati sul disco prima di poter inviare un'altra richiesta di scrittura. Le prestazioni di scrittura non sono buone con le scritture sincrone come con le scritture asincrone.

L'impostazione predefinita è "No".
