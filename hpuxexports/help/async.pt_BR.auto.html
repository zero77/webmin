Gravações assíncronas
"Sim" - as gravações no servidor são feitas de forma assíncrona. Quando o cliente NFS envia uma solicitação de gravação para um servidor NFS, o servidor retorna uma resposta positiva ao cliente antes de gravar qualquer dado no disco. Seu cliente pode enviar outra solicitação de gravação sem aguardar que o servidor conclua a gravação anterior. O desempenho de gravação é melhor com gravações assíncronas do que com gravações síncronas. No entanto, se o servidor travar após o envio da resposta positiva ao cliente, mas antes de gravar os dados no disco, os dados serão perdidos e o cliente não será notificado.

"Não" - as gravações no servidor são feitas de forma síncrona. Quando seu cliente NFS envia uma solicitação de gravação para um servidor NFS, ele grava os dados em seu disco antes de retornar uma resposta positiva ao seu cliente. Seu cliente deve esperar até que o servidor grave os dados no disco para poder enviar outra solicitação de gravação. O desempenho de gravação não é tão bom com gravações síncronas quanto com gravações assíncronas.

O padrão é "Não".
