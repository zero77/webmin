Scrie asincronă
„Da” - Scrierile pe server se fac asincron. Când clientul dvs. NFS trimite o solicitare de scriere către un server NFS, serverul returnează un răspuns pozitiv clientului dvs. înainte de a scrie date pe discul său. Clientul dvs. poate apoi trimite o altă solicitare de scriere fără a aștepta ca serverul să completeze scrierea anterioară. Performanța de scriere este mai bună cu scrieri asincrone decât cu scrieri sincrone. Cu toate acestea, dacă serverul se prăbușește după ce a trimis răspunsul pozitiv către client, dar înainte de a scrie datele pe disc, datele se pierd, iar clientul nu este notificat.

„Nu” - Scrierile pe server se realizează sincron. Când clientul dvs. NFS trimite o solicitare de scriere către un server NFS, serverul scrie datele pe discul său înainte de a returna un răspuns pozitiv clientului dumneavoastră. Clientul dvs. trebuie să aștepte până când serverul scrie datele pe disc înainte de a putea trimite o altă solicitare de scriere. Performanța de scriere nu este la fel de bună cu scrierea sincronă ca și cu cele asincrone.

Valoarea implicită este „Nu”.
