Асинхронные записи
«Да» - запись на сервер производится асинхронно. Когда ваш клиент NFS отправляет запрос на запись на сервер NFS, сервер возвращает положительный ответ вашему клиенту перед записью каких-либо данных на его диск. Ваш клиент может затем отправить еще один запрос на запись, не дожидаясь, пока сервер завершит предыдущую запись. Производительность записи лучше при асинхронной записи, чем при синхронной записи. Однако, если происходит сбой сервера после отправки положительного ответа клиенту, но перед записью данных на диск, данные теряются, и клиент не уведомляется.

«Нет» - запись на сервер производится синхронно. Когда ваш NFS-клиент отправляет запрос на запись на NFS-сервер, сервер записывает данные на свой диск, прежде чем вернуть положительный ответ вашему клиенту. Ваш клиент должен дождаться, пока сервер записывает данные на диск, прежде чем он сможет отправить еще один запрос на запись. Производительность записи не так хороша при синхронной записи, как при асинхронной записи.

По умолчанию «Нет».
