Asynchrone Schreibvorgänge
"Ja" - Schreibvorgänge auf den Server erfolgen asynchron. Wenn Ihr NFS-Client eine Schreibanforderung an einen NFS-Server sendet, gibt der Server eine positive Antwort an Ihren Client zurück, bevor Daten auf seine Festplatte geschrieben werden. Ihr Client kann dann eine weitere Schreibanforderung senden, ohne darauf zu warten, dass der Server den vorherigen Schreibvorgang abschließt. Die Schreibleistung ist bei asynchronen Schreibvorgängen besser als bei synchronen Schreibvorgängen. Wenn der Server jedoch nach dem Senden der positiven Antwort an den Client, aber vor dem Schreiben der Daten auf die Festplatte abstürzt, gehen die Daten verloren und der Client wird nicht benachrichtigt.

"Nein" - Schreibvorgänge auf den Server erfolgen synchron. Wenn Ihr NFS-Client eine Schreibanforderung an einen NFS-Server sendet, schreibt der Server die Daten auf seine Festplatte, bevor er eine positive Antwort an Ihren Client zurückgibt. Ihr Client muss warten, bis der Server die Daten auf die Festplatte schreibt, bevor er eine weitere Schreibanforderung senden kann. Die Schreibleistung ist bei synchronen Schreibvorgängen nicht so gut wie bei asynchronen Schreibvorgängen.

Der Standardwert ist "Nein".
