Écritures asynchrones
"Oui" - Les écritures sur le serveur sont effectuées de manière asynchrone. Lorsque votre client NFS envoie une demande d'écriture à un serveur NFS, le serveur renvoie une réponse positive à votre client avant d'écrire des données sur son disque. Votre client peut alors envoyer une autre demande d'écriture sans attendre que le serveur termine l'écriture précédente. Les performances d'écriture sont meilleures avec les écritures asynchrones qu'avec les écritures synchrones. Toutefois, si le serveur se bloque après l'envoi de la réponse positive au client mais avant l'écriture des données sur le disque, les données sont perdues et le client n'est pas notifié.

"Non" - Les écritures sur le serveur sont effectuées de manière synchrone. Lorsque votre client NFS envoie une demande d'écriture à un serveur NFS, le serveur écrit les données sur son disque avant de renvoyer une réponse positive à votre client. Votre client doit attendre que le serveur écrive les données sur le disque avant de pouvoir envoyer une autre demande d'écriture. Les performances d'écriture ne sont pas aussi bonnes avec les écritures synchrones qu'avec les écritures asynchrones.

La valeur par défaut est "Non".
