Conteggio settoriale
Imposta il conteggio dei settori per il filesystem read-ahead.

Viene utilizzato per migliorare le prestazioni nelle letture sequenziali di file di grandi dimensioni, precaricando blocchi aggiuntivi in previsione della necessità di essere eseguiti dall'attività in esecuzione. Nell'attuale versione del kernel (2.0.10) ha un'impostazione predefinita di 8 settori (4 kB). Questo valore sembra buono per la maggior parte degli scopi, ma in un sistema in cui la maggior parte degli accessi ai file sono ricerche casuali, un'impostazione più piccola potrebbe fornire prestazioni migliori. Inoltre, molte unità IDE hanno anche una funzione di read-ahead integrata separata, che allevia la necessità di un read-ahead del filesystem in molte situazioni.
