연결 편집 또는 생성
이 페이지에서 새로운 IPsec 연결을 만들거나 기존 연결의 세부 정보를 편집 할 수 있습니다. 만들려는 VPN 터널의 양쪽 끝에있는 시스템에서 동일한 설정으로 연결을 만들어야합니다.

각 연결에 제공해야하는 IPsec VPN 연결 세부 정보 섹션의 필드와 권장 설정은 다음과 같습니다.

연결 이름
공백이없는이 연결의 짧은 이름입니다.
IPsec 시작시
연결 추가 를 선택하면이 연결은 명시 적으로 시작된 경우에만 설정됩니다. 연결 시작을 선택하면 IPsec 서버가 시작 되 자마자 시작됩니다. 전자는 주기적으로 만 활성화되는 원격 시스템으로의 터널에 가장 적합한 반면 후자는 영구 VPN 링크에 적합합니다.
데이터를 압축 하시겠습니까?
기본값으로 설정되어 있어야합니다.
연결 타입
또한 Default로 설정되어 있어야합니다.
연결의 양쪽 끝에있는 시스템에 대한 추가 정보를 제공해야합니다. 일반적으로이 호스트의 설정이 항상 로컬 또는 왼쪽 시스템의 설정 섹션에 있도록 로컬 및 원격 세부 사항이 서로 바뀌더라도 두 시스템에서 동일해야합니다. 각 섹션의 필드와 제안 된 설정은 다음과 같습니다.
퍼블릭 IP 주소
이 필드는 시스템의 고정 IP 주소 또는 IP가 동적으로 할당 된 호스트의 경우 자동 으로 설정해야합니다. 그러나 로컬 및 원격 섹션 모두이 필드를 동시에 자동으로 설정할 수는 없습니다.
시스템 식별자
호스트 이름 옵션을 선택하고 시스템의 호스트 이름을 텍스트 필드에 입력해야합니다. FreeSWAN은이 설정을 사용하여 연결의 어느 섹션이 어떤 호스트에 적용되는지 결정합니다.
시스템 뒤의 프라이빗 서브넷
이 시스템에 다른 호스트에 대한 액세스 권한이 부여 된 내부 네트워크가 연결되어있는 경우이 필드에 네트워크 주소와 접두사 길이 (예 : 192.168.1.0/24 )를 입력하십시오.
시스템의 공개 키
일반적으로 아래 에서 입력됨을 선택하고 텍스트 상자에 시스템의 RSA 공개 키를 입력해야합니다. 해당 시스템의 공개 키 표시 페이지에서 확인할 수 있습니다.
다른 시스템으로의 다음 홉
비정상적인 네트워크 설정이 없으면이 필드는 기본 경로 로 설정되어야 합니다 .
연결을 생성하거나 편집 한 후 모듈의 메인 페이지에있는 Apply Configuration (구성 적용) 버튼을 사용하여 변경 사항을 활성화해야합니다.
