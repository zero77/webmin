Programmi di autenticazione

Programma di autenticazione personalizzato : il valore predefinito è nessuno. Specificare il comando per l'autenticatore esterno. Tale programma legge una riga contenente "password nome utente" e risponde "OK" o "ERR" in un ciclo infinito. Se si utilizza un autenticatore, assicurarsi di disporre di 1 acl di tipo proxy_auth.

Se si desidera utilizzare l'autenticazione proxy tradizionale, impostare questa riga su qualcosa come /usr/bin/ncsa_auth /usr/etc/passwd. Supponendo che ncsa_auth sia stato compilato e installato sul tuo sistema.

Numero di programmi di autenticazione : il valore predefinito è 5 se è stato abilitato un autenticatore.

Tempo per memorizzare nella cache le password per : per quanto tempo Squid memorizzerà nella cache un login riuscito prima di interrogare nuovamente il programma di autenticazione.

È tempo di associare l'utente a un indirizzo IP per : Se lo stesso utente tenta di accedere due volte da due indirizzi IP diversi durante questo periodo, verrà annullato. Questo può essere usato per impedire la condivisione di password proxy tra più utenti.

Se si esegue Squid versione 2.5 e successive, è possibile impostare separatamente le opzioni precedenti e altre per diversi tipi di autenticazione come Digest e NTLM. Tuttavia, se si desidera utilizzare Webmin per modificare l'elenco degli utenti proxy, è necessario impostare l'autenticazione di base.
