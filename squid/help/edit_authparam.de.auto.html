Authentifizierungsprogramme

Benutzerdefiniertes Authentifizierungsprogramm : Standardmäßig keine. Geben Sie den Befehl für den externen Authentifikator an. Ein solches Programm liest eine Zeile mit "Benutzername Passwort" und antwortet "OK" oder "ERR" in einer Endlosschleife. Wenn Sie einen Authentifikator verwenden, stellen Sie sicher, dass Sie 1 acl vom Typ proxy_auth haben.

Wenn Sie die herkömmliche Proxy-Authentifizierung verwenden möchten, setzen Sie diese Zeile auf /usr/bin/ncsa_auth /usr/etc/passwd. Angenommen, ncsa_auth wurde kompiliert und auf Ihrem System installiert.

Anzahl der Authentifizierungsprogramme : Der Standardwert ist 5, wenn ein Authentifikator aktiviert wurde.

Zeit zum Zwischenspeichern von Passwörtern für : Wie lange Squid eine erfolgreiche Anmeldung zwischenspeichert, bevor das Authentifizierungsprogramm erneut abgefragt wird.

Zeit, um den Benutzer an eine IP-Adresse zu binden für : Wenn derselbe Benutzer während dieses Zeitraums zweimal versucht, sich von zwei verschiedenen IP-Adressen aus anzumelden, wird er davon profitieren. Dies kann verwendet werden, um die gemeinsame Nutzung von Proxy-Passwörtern zwischen mehreren Benutzern zu verhindern.

Wenn Sie Squid-Versionen 2.5 und höher ausführen, können zusätzlich die oben genannten und andere Optionen für verschiedene Authentifizierungstypen wie Digest und NTLM separat festgelegt werden. Wenn Sie jedoch Webmin zum Bearbeiten der Proxy-Benutzerliste verwenden möchten, müssen Sie die Standardauthentifizierung einrichten.
