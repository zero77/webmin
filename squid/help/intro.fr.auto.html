Configuration de Squid

Introduction à Squid

Squid est un démon de serveur proxy de mise en cache Web complet. Squid permet à votre serveur d'enregistrer des copies locales d'objets Web distants. Lorsque ces objets sont accessibles à nouveau par n'importe qui sur le réseau local, les objets se chargeront plus rapidement et ne consommeront pas plus de bande passante pour récupérer à nouveau l'objet à partir du serveur d'origine. Cela peut réduire la pression sur les réseaux encombrés et fournir un environnement réseau plus réactif dans l'ensemble.

Un autre avantage de l'utilisation de Squid est celui des contrôles d'accès. Squid offre des capacités étendues pour bloquer l'accès à certains sites, utilisateurs, réseaux, types de contenu de données, etc. Squid peut fonctionner en mode transparent, en mode proxy traditionnel et en mode proxy inverse (accélérateur httpd).

Le module Squid

La page principale de ce module affiche un tableau d'icônes, chacune pour configurer des aspects spécifiques de Squid. Lors de la première configuration de Squid, toutes les valeurs seront définies sur leurs valeurs par défaut. Pour voir quelles sont ces valeurs par défaut, reportez-vous simplement à la page d'aide de chaque page de configuration.

La majorité des pages d'aide de ce module ont été extraites du fichier squid.conf par défaut inclus dans chaque installation de Squid. Plus d'informations peuvent être trouvées sur la page d'accueil de Squid .
