بنادر و شبکه
پورت پروکسی : پیش فرض 3128 است. این مقدار تعیین می کند که سرور Squid شما برای درخواست های http به چه پستی گوش می دهد. برای اتصال به پروکسی در این پورت ، باید مرورگرهای مشتری پیکربندی شوند ، یا نیاز به تنظیم شفافیت نیز دارد.

درگاه ICP : شماره پورت که Squid درخواست های ICP را از طریق انبارهای همسایه ارسال و دریافت می کند. پیش فرض 3130 است. برای غیرفعال کردن استفاده از "0". ممکن است با خط بر روی خط فرمان شود.

آدرس TCP ورودی : پیش فرض برای هیچ کس. سوکت HTTP را تعریف می کند که اتصالات مشتری و انبارهای دیگر را می پذیرد.

آدرس TCP خروجی : پیش فرض هیچکدام. سوکت HTTP را برای اتصالات ایجاد شده به سرورهای راه دور و حافظه های دیگر ذخیره می کند.

آدرس UDP خروجی : پیش فرض برای هیچ کس. بسته های دریافت کننده سوکت ICP را از سایر انبارها تعریف می کند.

آدرس UDP ورودی : پیش فرض برای هیچ کس. سوکت بسته های ICP را که به انبارهای دیگر ارسال می شود ، تعریف می کند.

TCP دریافت بافر : اندازه بافر TCP. پیش فرض های سیستم عامل بافر می گیرند.
