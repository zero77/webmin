도우미 프로그램
FTP 열 너비 : 기본값은 32입니다. ftp 목록의 너비를 설정합니다. 표준 브라우저 너비에 맞게 설정해야합니다. 이 값을 너무 작게 설정하면 ftp 사이트를 탐색 할 때 긴 파일 이름이 차단 될 수 있습니다.

Anon FTP 로그인 : 기본값은 'Squid @'입니다. 익명의 로그인 암호를보다 유익하게 만들고 (picky ftp 서버를 사용하도록 설정하려면) wwwuser@somewhere.net과 같이 도메인에 적합한 것으로 설정하십시오.

이것이 기본적으로 도메인이없는 이유는 캐시 사용 방법에 따라 모든 도메인의 사용자를 대신하여 요청을 할 수 있기 때문입니다. 일부 ftp 서버는 이메일 주소가 유효한지 확인합니다 (예 : perl.com).

오징어 DNS 프로그램 : 기본값은 dnsserver입니다. dnslookup 프로세스의 실행 파일 위치를 지정하십시오. 기본값은 비 차단 캐싱 네임 서버이며 기본적으로 Squid와 함께 설치됩니다.

DNS 프로그램 수 : 기본값은 5입니다. DNS 이름 조회 서비스를 위해 생성 된 프로세스 수입니다. 대형 서버에서로드가 많은 캐시의 경우이 값을 10 이상으로 늘려야합니다. 최대 값은 32입니다.

요청에 도메인 추가 : 기본값은 해제입니다. 일반적으로 'dnsserver'는 RES_DEFNAMES 리졸버 옵션을 비활성화합니다 (res_init (3) 참조). 이렇게하면 계층의 캐시가 단일 구성 요소 호스트 이름을 로컬로 해석하지 못합니다. dnsserver가 단일 구성 요소 이름을 처리 할 수있게하려면이 옵션을 활성화하십시오.

DNS 서버 주소 : 기본값은 resolv.conf 파일을 사용하는 것입니다. /etc/resolv.conf 파일에 지정된 것 대신 사용할 DNS 이름 서버 (IP 주소) 목록을 지정하려면이 옵션을 사용하십시오.

캐시 삭제 프로그램 : 기본값은 '링크 해제'입니다. 파일 삭제 프로세스를위한 실행 파일의 위치를 지정하십시오. async-io를 사용하는 경우 일부 버전의 스레드에서 처리되므로 필요하지 않을 수 있습니다.

오징어 핑 프로그램 : 기본값은 '핑거'입니다. pinger 프로세스의 실행 파일 위치를 지정하십시오. '--enable-icmp'옵션을 사용하여 Squid (보완 중)를 구성한 경우에만 유용합니다.

사용자 지정 리디렉션 프로그램 : 기본값은 none입니다. URL 리디렉터의 실행 파일 위치를 지정하십시오. 그들은 거의 모든 기능을 수행 할 수 있기 때문에 포함되어 있지 않습니다. 작성 방법에 대한 정보는 릴리스 노트를 참조하십시오.

리디렉터 프로그램 수 : 리디렉터가 구성된 경우 기본값은 5입니다. 스폰 할 리디렉터 프로세스 수입니다. 너무 적게 시작하면 Squid는 URL의 백 로그를 처리 할 때까지 기다려야 속도가 느려집니다. 너무 많이 시작하면 RAM 및 기타 시스템 리소스를 사용합니다.
