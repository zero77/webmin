Нарыхтоўка лесу
Файл часопіса доступу : па змаўчанні залежыць сістэма, звычайна дзесьці ў / var . Улічвае дзейнасць запыту кліента. Змяшчае запіс для кожнага атрыманага запыту HTTP і ICP.

Файл часопіса адладкі : па змаўчанні залежыць сістэма. Файл рэгістрацыі кэша. Тут ідзе агульная інфармацыя пра паводзіны вашага кэша. Вы можаце павялічыць колькасць дадзеных, якія ўвайшлі ў гэты файл з дапамогай параметраў адладкі .

Файл часопіса захоўвання : па змаўчанні залежыць сістэма. Запісвае дзейнасць кіраўніка сховішча. Паказвае, якія аб'екты выкідваюцца з кэша, а якія аб'екты захоўваюцца і на які тэрмін. Каб адключыць, увядзіце "не". На самай справе не існуе ўтылітаў для аналізу гэтых дадзеных, таму вы можаце смела іх адключыць.

Файл метададзеных кэша : па змаўчанні знаходзіцца swap.log у кожным каталогу кэша. Гэты файл часопіса змяшчае метададзеныя аб'ектаў, захаваных на дыску. Ён выкарыстоўваецца для аднаўлення кэша падчас запуску. Звычайна гэты файл знаходзіцца ў кожным каталогу кэша, але вы можаце паказаць альтэрнатыўнае імя шляху тут. Звярніце ўвагу, што вы павінны даць поўнае імя файла, а не толькі каталог. Паколькі гэта індэкс для ўсяго спісу аб'ектаў, вы НЕ МОЖНА перыядычна паварочваць яго!

Калі ў імя файла можа выкарыстоўвацца% s, то ён будзе заменены прадстаўленнем імя каталога кэша, дзе кожны / замяняецца на '.' Гэта неабходна, каб дазволіць даданне / выдаленне радкоў каталога кэша, калі выкарыстоўваецца альтэрнатыўны часопіс падпампоўкі.

Калі ў імёне больш за адзін каталог кэша, і% s не выкарыстоўваецца, то гэтыя часопісы свопаў будуць мець такія назвы, як:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

Пранумараванае пашырэнне (якое дадаецца аўтаматычна) адпавядае парадку радкоў каталога кэша ў канфігурацыі. Калі вы змяніце парадак радкоў каталога кэша ў гэтым файле, то гэтыя файлы часопісаў НЕ будуць адпавядаць правільнаму запісу каталога кэша (калі вы не перайменаваеце іх уручную). Мы рэкамендуем НЕ выкарыстоўваць гэтую опцыю. Лепш захоўваць гэтыя файлы часопісаў у кожным каталогу кэша.

Выкарыстоўвайце фармат часопіса HTTPD : па змаўчанні няма. Кэш можа эмуляваць фармат файла часопіса, які выкарыстоўваюць многія праграмы "httpd". Каб адключыць / ўключыць гэтую эмуляцыю, усталюйце гэтае значэнне на выключэнне або ўключэнне. Па змаўчанні выкарыстоўваецца натыўны фармат часопіса, бо ён утрымлівае карысную інфармацыю, якую выкарыстоўваюць аналізатары часопісаў для кальмараў.

Увайдзіце загалоўкі MIME : па змаўчанні няма. Кэш можа запісваць як запыт, так і загалоўкі MIME адказаў для кожнай аперацыі HTTP. Загалоўкі кадуюцца бяспечна і ў канцы часопіса доступу будуць выглядаць як два ўкручаныя ў поле (як для роднага, так і для httpd-эмуляванага фарматаў часопіса). Для ўключэння гэтай рэгістрацыі ўключыце гэта.

Файл часопіса карыстальніка агента : па змаўчанні няма. Калі наладжана опцыя наладкі "--enable-useragent-log", Squid запіша поле User-Agent з HTTP-запытаў на названае імя файла.

PID-файл : па змаўчанні залежыць сістэма. Імя файла, у якое трэба запісаць ідэнтыфікатар працэсу. Каб адключыць, увядзіце "не".

Выканайце пошукавыя ідэнтыфікацыі RFC931 для ACLs : па змаўчанні няма. Выканайце ідэнтыфікацыю пошуку, каб знайсці імя карыстальніка кліентаў, якія звяртаюцца да чаго-небудзь у гэтых ACL.

Час чакання ідэнтыфікацыі RFC931 : па змаўчанні 30 секунд. Максімальны час для чакання IDENT-запытаў. Калі гэта занадта высока, і вы ўключылі любую праверку ідэнтыфікатараў, вы можаце падвергнуць адмове ў абслугоўванні, калі адначасова будзе задзейнічана шмат запытаў ідэнтыфікатараў.

Увядзіце поўныя імёны хастоў : па змаўчанні няма. Улічвае поўнае імя кліентаў.

Сеткавая маска для вядзення часопісаў : па змаўчанні 255.255.255.255. Сеткавая маска для адрасоў кліента ў лагістычных файлах і выхадзе cachemgr. Змяніце гэта, каб абараніць прыватнасць вашых кліентаў кэша. Сеткавая маска 255.255.255.0 будзе рэгістраваць усе IP-адрасы ў гэтым дыяпазоне з апошняй лічбай, усталяванай на '0'.

Параметры адладкі : па змаўчанні "УСЕ, 1". Параметры рэгістрацыі ўсталёўваюцца як раздзел, узровень, дзе кожнаму зыходнаму файлу прызначаецца унікальны раздзел. Ніжнія ўзроўні прыводзяць да менш поўнай адладкі (узровень 9) можа прывесці да вельмі вялікага файла часопісаў, таму будзьце ўважлівыя. Чароўнае слова "УСЕ" устанаўлівае ўзроўні адладкі для ўсіх раздзелаў. Мы рэкамендуем нармальна працаваць з "УСЕ, 1".

Табліца загалоўкаў MIME : Pathname да табліцы MIME Squid. Вам не трэба мяняць, але файл па змаўчанні змяшчае прыклады і інфармацыю пра фарматаванне.
