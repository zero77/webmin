Saio
Sarbide-erregistroko fitxategia : sistema lehenetsia sistemaren menpe dago, normalean nonbait / var . Bezeroaren eskaeraren jarduera erregistratzen du. Jasotako HTTP eta ICP eskaera guztientzako sarrera bat dauka.

Arazketa erregistro fitxategia : lehenetsia sistemaren menpe dago. Cache erregistroko fitxategia. Hemen gertatzen da zure cachearen portaerari buruzko informazio orokorra. Fitxategi honetan erregistratutako datu kopurua handitu dezakezu arazketa aukerak .

Biltegiratze erregistroaren fitxategia : lehenetsia sistemaren menpe dago. Biltegiratze kudeatzailearen jarduerak erregistratzen ditu. Cache-tik zein objektu ateratzen diren erakusten du, eta zein objektu gordetzen diren eta zenbat denboraz. Ezgaitzeko, idatzi "bat ere ez". Ez dago datu hauek aztertzeko utilitaterik, beraz, segurtasunez ezgaitu dezakezu.

Cacheko metadatuen fitxategia : cache direktorio bakoitza swap.log da. Log fitxategi honek diskoan gordetako objektuen metadatuak gordetzen ditu. Abiapuntuan cachea berreraikitzeko erabiltzen da. Normalean fitxategi hau cache direktorio bakoitzean dago, baina hemen beste izen bat zehaztu dezakezu. Kontuan izan fitxategi-izen osoa eman behar duzula, eta ez bakarrik direktorio bat. Objektuen zerrenda osoko aurkibidea denez ezin duzu aldian-aldian biratu.

% S fitxategiaren izenean erabil badaiteke, cache direktorioaren izenaren ordezkaritza batekin ordezkatuko da / '.' '. Beharrezkoa da cache direktorioen lerroak gehitzea / kentzea ahalbidetzeko beste cache swap erregistroa erabiltzen ari denean.

Cache direktorio bat baino gehiago badituzu eta% s izenean erabiltzen ez bada, swap erregistro hauek izenak izango dituzte:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

Zenbakiaren luzapena (automatikoki gehitzen dena) konfigurazioko cache direktorioen lerroen hurrenkerari dagokio. Fitxategi honetako cache direktorioen lerroen ordena aldatzen baduzu, orduan fitxategi log hauek ez dira cache direktorioaren sarrera egokiari lotuko (eskuz aldatu itzazu behintzat). Aukera hau EZ erabiltzea gomendatzen dizugu. Hobe da erregistro fitxategi hauek gordetzeko cache direktorio bakoitzean.

Erabili HTTPD erregistro formatua : Lehenetsitakoak no. Cache-k “httpd” programek erabiltzen dituzten erregistroko formatuen imita dezakete. Emulazioa desgaitzeko / gaitzeko, ezarri balio hau desaktibatuta edo aktibatuta. Lehenespenez, jatorrizko erregistro formatua erabiltzea da, Squid-en berariazko erregistro analizatzaileek erabiltzen duten informazio erabilgarria biltzen baitu.

Log MIME goiburuak : Lehenetsitakoak no. Cache-k HTTP transakzio bakoitzeko eskaera eta erantzun MIME goiburuak grabatu ditzake. Goiburuak modu seguruan kodetzen dira eta parentesi bi eremu gisa agertuko dira sarbide erregistroaren amaieran (jatorrizko edo httpd-en emulatutako log formatuetarako). Logging hau gaitzeko, hau aktibatu behar da.

Erabiltzaileen erregistro fitxategia : lehenetsia ez da. "--Enable-useragent-log" konfiguratzeko aukerarekin konfiguratuta badago, Squid-ek User-Agent eremua HTTP eskaeretatik hemen zehaztutako fitxategi-izenera idatziko du.

PID fitxategia : lehenetsia sistemaren menpe dago. Prozesuaren ID-a idazteko fitxategi-izena. Ezgaitzeko, idatzi "bat ere ez".

Egin RFC931 identitate bilaketa ACLetarako : Lehenetsia ez da. Egin identifikazio bilaketa ACL horietan edozer sarbidea duten bezeroen erabiltzaile izena aurkitzeko.

RFC931 ident timeout : lehenetsia 30 segundokoa da. IDENT eskaerak itxaroteko gehienezko denbora. Hori oso altua bada, eta identifikazio bilaketa guztiak gaitu dituzu, orduan ukatzeko zerbitzua jasan lezakezu, identifikazio eskaera asko aldi berean joanez gero.

Izena eman ostalari izenak : Lehenetsia ez da. Bezeroen ostalari izen osoa erregistratzen du.

Sarearen sareko erregistroa : lehenetsia 255.255.255.255 da. Bezeroen helbideetarako sare-sare bat. Fitxategi eta cachemgr irteeretan. Aldatu hau zure cache bezeroen pribatutasuna babesteko. 255.255.255.0 sareko sareak IPko barruti guztiak erregistratuko ditu '0' ezarrita duen azken digituarekin.

Arazketa aukerak : lehenetsia "GUZTIRA, 1" da. Eguneratzeko aukerak atal gisa finkatzen dira, iturri fitxategi bakoitzari atal bakarra esleitzen zaion maila. Maila baxuagoak gutxiagotan arazketa osoa (9. maila) erregistroko fitxategi oso handia izan daiteke, beraz, kontuz ibili. "GUZTIA" hitz magikoak atal guztietako arazketa maila ezartzen du. Normalean "ALL, 1" ekin exekutatzea gomendatzen dugu.

MIME goiburuko taula : Squid-en MIME taulara bideratutako izena. Ez zenuke aldatu behar, baina fitxategi lehenetsiak adibideak eta formatua emateko informazioa ditu.
