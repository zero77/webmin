Контроль доступа
Списки контроля доступа : списки ACL - это списки терминов, которые должны соответствовать регулярным выражениям литеральных выражений. ACL также может быть файлом, который содержит один элемент или регулярное выражение на строку. Поле ACL на этой странице имеет несколько полей, которые можно редактировать. Новые ACL также могут быть созданы.

Представленные поля:

Имя - это имя, по которому можно идентифицировать ACL. При создании прокси-ограничения это имя используется для определения того, что ограничивается.

Тип - это тип информации, с которой должен сопоставляться ACL.

Соответствие .. - это адрес, порт, URL, пользователь и т. Д., Которым будет соответствовать ACL.

Ограничения прокси. Ограничения прокси - это правила, которые разрешают или отклоняют данный запрос в зависимости от того, соответствует ли его ACL специфике запроса. В этом поле есть три поля.

Действие либо разрешить, либо запретить .

ACL - это ACL, с которыми будут сопоставляться, чтобы решить, разрешен или отклонен запрос. Если перед восклицательным знаком добавлен ACL, он будет отменен, иными словами, все, кроме членов этого ACL, будет разрешено или запрещено.

Перемещение позволяет порядок ограничений в списке. Порядок, в котором они появляются, важен, потому что Squid читает список только до тех пор, пока не найдет первое совпадение.

Ограничения ICP . Это поле предназначено для ограничения запросов ICP. Запросы ICP - это запросы от других соседних кэшей. Этот раздел работает так же, как ограничения прокси .
