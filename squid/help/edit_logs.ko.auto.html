벌채 반출
액세스 로그 파일 : 기본값은 시스템에 따라 다르며 일반적으로 / var에 있습니다. 클라이언트 요청 활동을 기록합니다. 수신 된 모든 HTTP 및 ICP 요청에 대한 항목을 포함합니다.

디버그 로그 파일 : 기본값은 시스템에 따라 다릅니다. 캐시 로깅 파일. 캐시 동작에 대한 일반적인 정보가 여기에 있습니다. 디버그 옵션 을 사용하여이 파일에 기록되는 데이터의 양을 늘릴 수 있습니다.

저장소 로그 파일 : 기본값은 시스템에 따라 다릅니다. 스토리지 관리자의 활동을 기록합니다. 캐시에서 배출되는 오브젝트와 저장되는 오브젝트 및 기간을 표시합니다. 비활성화하려면 "없음"을 입력하십시오. 실제로이 데이터를 분석하는 유틸리티는 없으므로 안전하게 비활성화 할 수 있습니다.

캐시 메타 데이터 파일 : 기본값은 각 캐시 디렉토리의 swap.log입니다. 이 로그 파일은 디스크에 저장된 객체의 메타 데이터를 보유합니다. 시작하는 동안 캐시를 다시 작성하는 데 사용됩니다. 일반적으로이 파일은 각 캐시 디렉토리에 상주하지만 여기서 대체 경로 이름을 지정할 수 있습니다. 디렉토리뿐만 아니라 전체 파일 이름을 제공해야합니다. 이것은 전체 객체 목록에 대한 색인이므로 주기적으로 회전시킬 수 없습니다!

파일 이름에 % s를 사용할 수 있으면 각 /가 '.'으로 바뀌는 캐시 디렉토리 이름으로 대체됩니다. 대체 캐시 스왑 로그를 사용할 때 캐시 디렉토리 행을 추가 / 제거 할 수 있어야합니다.

캐시 디렉토리가 둘 이상 있고 이름에 % s가 사용되지 않으면 이러한 스왑 로그의 이름은 다음과 같습니다.

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

번호가 매겨진 확장자 (자동으로 추가됨)는 구성에서 캐시 디렉토리 행의 순서에 해당합니다. 이 파일에서 캐시 디렉토리 행의 순서를 변경하면 수동으로 이름을 바꾸지 않는 한 이러한 로그 파일이 올바른 캐시 디렉토리 항목에 해당하지 않습니다. 이 옵션을 사용하지 않는 것이 좋습니다. 이러한 로그 파일을 각 캐시 디렉토리에 보관하는 것이 좋습니다.

HTTPD 로그 형식 사용 : 기본값은 no입니다. 캐시는 많은 'httpd'프로그램이 사용하는 로그 파일 형식을 에뮬레이션 할 수 있습니다. 이 에뮬레이션을 비활성화 / 활성화하려면이 값을 off 또는 on으로 설정하십시오. 기본적으로 오징어 별 로그 분석기가 사용하는 유용한 정보가 포함되어 있으므로 기본 로그 형식을 사용합니다.

MIME 헤더 로그 : 기본값은 no입니다. 캐시는 각 HTTP 트랜잭션에 대한 요청 및 응답 MIME 헤더를 모두 기록 할 수 있습니다. 헤더는 안전하게 인코딩되며 액세스 로그 끝에 두 개의 괄호 필드로 나타납니다 (기본 또는 httpd 에뮬레이트 된 로그 형식). 이 로깅을 활성화하려면 이것을 on으로 설정하십시오.

사용자 에이전트 로그 파일 : 기본값은 none입니다. "--enable-useragent-log"구성 옵션으로 구성된 경우 Squid는 HTTP 요청의 User-Agent 필드를 여기에 지정된 파일 이름으로 씁니다.

PID 파일 : 기본값은 시스템에 따라 다릅니다. 프로세스 ID를 쓸 파일 이름입니다. 비활성화하려면 "없음"을 입력하십시오.

ACL에 대한 RFC931 ID 검색 수행 : 기본값은 없음입니다. 이러한 ACL에서 액세스하는 클라이언트의 사용자 이름을 찾으려면 ID 조회를 수행하십시오.

RFC931 ident timeout : 기본값은 30 초입니다. IDENT 요청을 기다리는 최대 시간입니다. 이 값이 너무 높고 아이디 조회를 사용하도록 설정 한 경우 많은 아이디 요청을 한 번에 수행하면 서비스 거부가 발생할 수 있습니다.

전체 호스트 이름 기록 : 기본값은 no입니다. 클라이언트의 전체 호스트 이름을 기록합니다.

넷 마스크 로깅 : 기본값은 255.255.255.255입니다. 로그 파일 및 cachemgr 출력의 클라이언트 주소에 대한 넷 마스크. 캐시 클라이언트의 개인 정보를 보호하려면이를 변경하십시오. 255.255.255.0의 넷 마스크는 마지막 숫자가 '0'으로 설정된 해당 범위의 모든 IP를 기록합니다.

디버그 옵션 : 기본값은 "ALL, 1"입니다. 로깅 옵션은 각 소스 파일에 고유 한 섹션이 할당 된 섹션 레벨로 설정됩니다. 수준이 낮 으면 전체 디버깅 (수준 9)으로 인해 로그 파일이 매우 커질 수 있으므로주의하십시오. 매직 단어 "ALL"은 모든 섹션에 대한 디버깅 수준을 설정합니다. 일반적으로 "ALL, 1"로 실행하는 것이 좋습니다.

MIME 헤더 테이블 : 오징어 MIME 테이블의 경로 이름. 변경할 필요는 없지만 기본 파일에는 예제와 서식 정보가 포함되어 있습니다.
