Opções de cache
Diretórios de cache : você pode especificar várias linhas de diretório de cache para distribuir o cache entre diferentes partições de disco.

Diretório é um diretório de nível superior onde os arquivos de troca de cache serão armazenados. Se você deseja usar um disco inteiro para armazenamento em cache, este pode ser o diretório do ponto de montagem. O diretório deve existir e ser gravável pelo processo Squid. O Squid NÃO criará este diretório para você.

Tamanho é a quantidade de espaço em disco (MB) a ser usada nesse diretório. O padrão é 100 MB. Altere isso para se adequar à sua configuração.

Dirs de 1º nível é o número de subdiretórios de primeiro nível que serão criados no Directoryi . O padrão é 16.

Dirs de segundo nível é o número de subdiretórios de segundo nível que serão criados em cada diretório de primeiro nível. O padrão é 256.

Tamanho médio do objeto : O padrão é 13 kB. Tamanho médio do objeto, usado para estimar o número de objetos que seu cache pode conter.

Objetos por bucket : o padrão é 50. O número alvo de objetos por bucket na tabela de hash da loja. A redução desse valor aumenta o número total de caçambas e também a taxa de manutenção de armazenamento.

Não armazenar em cache URLs para ACLs : o padrão é nenhum. Selecione todas as ACLs para as quais você não deseja que o Squid armazene dados em cache. Útil para definir um servidor da web interno que não deve ser armazenado em cache.

Tempo máximo de cache : o padrão é 1 ano. Este parâmetro define a idade máxima da LRU. Por exemplo, defini-lo como '1 semana' fará com que os objetos sejam removidos se não forem acessados por uma semana ou mais.

Tempo de cache de solicitação com falha O padrão é 5 minutos. Tempo de vida (TTL) para solicitações com falha. Certos tipos de falhas (como "conexão recusada" e "404 não encontrado") são armazenados em cache negativamente por um período configurável. Observe que isso é diferente do cache negativo das pesquisas de DNS.

Tempo de cache da pesquisa de DNS : o padrão é 6 horas (360 minutos). Tempo de vida (TTL) para armazenamento em cache positivo de pesquisas de DNS bem-sucedidas. Se você deseja minimizar o uso do ipcache do Squid, defina como 1, não 0.

Tempo de cache do DNS com falha : o padrão é 5 minutos. Tempo de vida (TTL) para armazenamento em cache negativo de pesquisas de DNS com falha.

Tempo limite de conexão : o padrão é 120 segundos. Alguns sistemas (principalmente o Linux) não podem ser confiáveis para atingir o tempo limite de solicitações de conexão (2). Portanto, o processo do Squid impõe seu próprio tempo limite nas conexões do servidor. Este parâmetro especifica quanto tempo aguardar a conclusão da conexão.

Tempo limite de leitura : o padrão é 15 minutos. O read_timeout é aplicado nas conexões do servidor. Após cada leitura bem-sucedida (), o tempo limite será estendido por esse valor. Se nenhum dado for lido novamente após esse período, a solicitação será abortada e registrada com ERR_READ_TIMEOUT.

Tempo limite da seleção de sites : o padrão é 4 segundos. Para URN para seleção de URL de vários URLs.

Tempo limite da solicitação do cliente : o padrão é 15 minutos. O read_timeout é aplicado nas conexões do servidor. Após cada leitura bem-sucedida (), o tempo limite será estendido por esse valor. Se nenhum dado for lido novamente após esse período, a solicitação será abortada e registrada com ERR_READ_TIMEOUT.

Tempo máximo de conexão do cliente : o padrão é 1 dia. A quantidade máxima de tempo que um cliente (navegador) tem permissão para permanecer conectado ao processo de cache. Isso protege o cache de ter muitos soquetes (e, portanto, descritores de arquivo) amarrados em um estado CLOSE_WAIT de clientes remotos que desaparecem sem desligar corretamente (devido a uma falha na rede ou devido a uma má implementação do cliente).

Tempo máximo de desligamento : o padrão é 30 segundos. Quando SIGTERM ou SIGHUP é recebido, o cache é colocado no modo "desligamento pendente" até que todos os soquetes ativos sejam fechados. Esse valor é a vida útil a ser definida para todos os descritores abertos durante o modo de desligamento. Qualquer cliente ativo após esse período de segundos receberá uma mensagem de 'tempo limite'.

Clientes semi-fechados : o padrão é 'on'. Alguns clientes podem desligar o lado de envio de suas conexões TCP, deixando seus lados de recebimento abertos. Às vezes, o Squid não pode dizer a diferença entre uma conexão TCP semi-fechada e totalmente fechada. Por padrão, as conexões de clientes semi-fechadas são mantidas abertas até que uma leitura (2) ou gravação (2) no soquete retorne um erro. Mude esta opção para 'off' e o Squid fechará imediatamente as conexões do cliente quando read (2) retornar "sem mais dados para ler".

Tempo limite persistente : o padrão é 120 segundos. Tempo limite para conexões persistentes ociosas com clientes, servidores e outros proxies. As conexões persistentes serão desativadas se isso for inferior a 10 segundos.

Host de retransmissão WAIS : O padrão é nenhum. Nome do host ou endereço para retransmitir solicitações do WAIS.
