Разные варианты
Тестовые адреса DNS для запуска : DNS-серверы, которые Squid будет запрашивать для проверки правильности поиска DNS.

Вращения лог-файла SIGUSR1 : по умолчанию 10 или 0, в зависимости от системы. Определяет количество вращений файла журнала, которые нужно выполнить при вводе 'squid -k rotate'. По умолчанию используется значение 10, которое будет вращаться с расширениями от 0 до 9. Если для logfile_rotate задано значение 0, вращение будет отключено, но файлы журналов по-прежнему закрыты и открываются заново. Это позволит вам переименовать файлы журналов непосредственно перед отправкой сигнала поворота.

Обратите внимание, что команда 'squid -k rotate' обычно отправляет сигнал USR1 запущенному процессу squid. В определенных ситуациях (например, в Linux с Async I / O), USR1 используется для других целей, поэтому -k rotate использует другой сигнал. Лучше всего привыкнуть использовать 'squid -k rotate' вместо 'kill -USR1 ».

Текст сообщений об ошибках : HTML-текст для включения в сообщения об ошибках. Сделайте это "mailto" URL-адресом для вашего админ-адреса или просто ссылкой на веб-страницу вашей организации.

Чтобы включить это в ваши сообщения об ошибках, вы должны переписать файлы шаблонов ошибок (находятся в каталоге «errors»). Везде, где вы хотите, чтобы появилась строка «err_html_text», вставьте тег% L в файл шаблона ошибки.

Статистика для каждого клиента : если вы хотите отключить сбор статистики для каждого клиента, отключите client_db здесь.

Заголовок X-Forwarded-For : по умолчанию установлено значение «on». Если установлено, Squid будет включать IP-адрес или имя вашей системы в запросы HTTP, которые он пересылает. По умолчанию это выглядит так, как в HTTP-запросах. По умолчанию это выглядит так:

X-Forwarded-For: 192.1.2.3

Если вы отключите это, оно будет выглядеть так:

X-Forwarded-For: неизвестно

Журнал ICP-запросов : по умолчанию «включено». Если установлено, запросы ICP регистрируются в access.log. Вы можете отключить эту опцию, если ваша нагрузка ICP ОЧЕНЬ высока, чтобы ускорить процесс или упростить анализ журнала.

Минимальное количество прямых прыжков : по умолчанию 4. Если вы используете функцию проверки связи ICMP, выполняйте прямые выборки для сайтов, которые находятся на расстоянии не более этого большого количества прыжков.

Сохранить память для будущего использования : по умолчанию «да». Если установлено, Squid будет сохранять пулы выделенной (но неиспользованной) памяти доступной для будущего использования. Если память в вашей системе имеет первостепенное значение и вы считаете, что ваша библиотека malloc превосходит процедуры Squid, отключите это.

Объем памяти для хранения : по умолчанию «Неограниченно». Если установлено ненулевое значение, Squid будет хранить не более указанного предела выделенной (но неиспользованной) памяти в пулах памяти. Все запросы free (), превышающие этот лимит, будут обрабатываться вашей библиотекой malloc. Squid предварительно не выделяет память, а просто хранит объекты, которые в противном случае были бы свободны () d. Таким образом, безопасно установить для memory_pools_limit достаточно высокое значение, даже если ваша конфигурация будет использовать меньше памяти.

Если не установлено (по умолчанию) или установлено в ноль, Squid сохранит всю память, какую только может. То есть не будет ограничений на общий объем памяти, используемой для хранения.

Заголовки для прохождения : по умолчанию «Все заголовки». Эта опция заменяет старую опцию http_anonymizer чем-то гораздо более настраиваемым. Теперь вы можете точно указать, какие заголовки должны быть разрешены или которые должны быть удалены из исходящих запросов.

Есть два способа использования этой опции. Вы можете либо разрешить определенные заголовки (тем самым запретив все остальные), либо вы можете запретить определенные заголовки (тем самым разрешив все остальные).

Поддельный User-Agent : по умолчанию 'none'. Если вы фильтруете заголовок User-Agent с помощью опции Заголовки для прохождения, это может привести к тому, что некоторые веб-серверы отклонят ваш запрос. Используйте это, чтобы подделать один.

Адрес маршрутизатора WCCP : используйте эту опцию, чтобы определить ваш "домашний" маршрутизатор WCCP для Squid.

Входящий адрес WCCP : используйте этот параметр, если требуется, чтобы сообщения WCCP принимались только на одном интерфейсе. НЕ используйте эту опцию, если вы не уверены, сколько интерфейсов у вас есть, или если вы знаете, что у вас только один интерфейс.

Исходящий адрес WCCP : используйте этот параметр, если требуется, чтобы сообщения WCCP отправлялись только на одном интерфейсе. НЕ используйте эту опцию, если вы не уверены, сколько интерфейсов у вас есть, или если вы знаете, что у вас только один интерфейс.
