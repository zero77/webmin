Programmes d'authentification

Programme d'authentification personnalisé : par défaut, aucun. Spécifiez la commande pour l'authentificateur externe. Un tel programme lit une ligne contenant "mot de passe de nom d'utilisateur" et répond "OK" ou "ERR" dans une boucle sans fin. Si vous utilisez un authentificateur, assurez-vous que vous avez 1 acl de type proxy_auth.

Si vous souhaitez utiliser l'authentification proxy traditionnelle, définissez cette ligne sur quelque chose comme /usr/bin/ncsa_auth /usr/etc/passwd. En supposant que ncsa_auth a été compilé et installé sur votre système.

Nombre de programmes d'authentification : 5 par défaut si un authentificateur a été activé.

Temps de mise en cache des mots de passe pendant : combien de temps Squid mettra en cache une connexion réussie avant de réinterroger le programme d'authentification.

Il est temps de lier l'utilisateur à une adresse IP pour : Si le même utilisateur essaie de se connecter deux fois à partir de deux adresses IP différentes pendant cette période, il sera béni. Cela peut être utilisé pour empêcher le partage de mots de passe proxy entre plusieurs utilisateurs.

Si vous exécutez Squid versions 2.5 et supérieures, les options ci-dessus et d'autres peuvent être définies séparément pour différents types d'authentification tels que Digest et NTLM. Cependant, si vous souhaitez utiliser Webmin pour modifier la liste des utilisateurs proxy, vous devez configurer l'authentification de base.
