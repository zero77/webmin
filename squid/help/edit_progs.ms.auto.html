Program Pembantu
Lebar lajur FTP : Lalai adalah 32. Menetapkan lebar senarai ftp. Ini harus diatur agar sesuai dengan lebar penyemak imbas standard. Menetapkan ini terlalu kecil dapat memotong nama fail yang panjang ketika melayari laman ftp.

Log masuk FTP Anon : Lalai adalah 'Squid @'. Sekiranya anda mahu kata laluan masuk tanpa nama lebih bermaklumat (dan membolehkan penggunaan pelayan ftp pilih), tetapkan ini ke sesuatu yang sesuai untuk domain anda, seperti wwwuser@somwhere.net.

Sebab mengapa ini tanpa domain secara lalai adalah bahawa permintaan dapat dibuat atas nama pengguna dalam domain apa pun, bergantung pada bagaimana cache digunakan. Beberapa pelayan ftp juga mengesahkan bahawa alamat e-mel adalah sah (contohnya perl.com).

Program DNS Squid : Lalai adalah dnsserver. Tentukan lokasi yang dapat dilaksanakan untuk proses dnslookup. Lalai adalah pelayan nama caching yang tidak menyekat yang dipasang dengan Squid secara lalai.

Bilangan program DNS : Lalai adalah 5. Jumlah proses yang dihasilkan untuk mencari nama DNS perkhidmatan. Untuk cache yang banyak dimuat pada pelayan besar, anda mungkin harus meningkatkan nilai ini kepada sekurang-kurangnya 10. Maksimum adalah 32.

Tambahkan domain ke permintaan : Lalai dimatikan. Biasanya 'dnsserver' melumpuhkan pilihan penyelesai RES_DEFNAMES (lihat res_init (3)). Ini menghalang cache dalam hierarki untuk mentafsirkan nama host komponen tunggal secara tempatan. Untuk membolehkan dnsserver mengendalikan nama komponen tunggal, aktifkan pilihan ini.

Alamat pelayan DNS : Lalai adalah menggunakan file resolv.conf. Gunakan ini jika anda ingin menentukan senarai pelayan nama DNS (alamat IP) untuk digunakan dan bukan yang diberikan dalam fail /etc/resolv.conf anda.

Program pembersihan cache : Lalai adalah 'unlinkd'. Tentukan lokasi yang dapat dilaksanakan untuk proses penghapusan fail. Ini mungkin tidak diperlukan jika anda menggunakan async-io kerana ia dikendalikan oleh utas dalam beberapa versi.

Program ping sotong : Lalai untuk 'pinger'. Tentukan lokasi yang dapat dilaksanakan untuk proses pinger. Ini hanya berguna jika anda mengkonfigurasi Squid (semasa pematuhan) dengan pilihan '--enable-icmp'.

Program pengalihan tersuai : Lalai untuk tidak ada. Tentukan lokasi yang dapat dilaksanakan untuk pengalihan URL. Oleh kerana mereka dapat melakukan hampir semua fungsi, tidak ada yang disertakan. Lihat Catatan Catatan untuk maklumat mengenai cara menulisnya.

Bilangan program pengalihan : Lalai ke 5 jika pengalihan dikonfigurasi. Bilangan proses pengalihan untuk bertelur. Sekiranya anda memulakan terlalu sedikit, Squid perlu menunggu mereka memproses tunggakan URL, memperlahankannya. Sekiranya anda memulakan terlalu banyak, mereka akan menggunakan RAM dan sumber sistem lain.
