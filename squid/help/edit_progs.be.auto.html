Праграмы дапаможнікаў
Шырыня слупка FTP : па змаўчанні 32. Устанавіць шырыню ftp спісаў. Гэта павінна быць усталявана ў шырыню стандартнага браўзэра. Усталяванне гэтага занадта мала можа адрэзаць доўгія імёны файлаў пры праглядзе ftp-сайтаў.

Уваход Anon FTP : па змаўчанні "Squid @". Калі вы хочаце, каб пароль ананімнага ўваходу быў больш інфарматыўным (і дазволілі выкарыстоўваць пераборлівыя ftp-серверы), усталюйце гэта на нешта рэзананснае для вашага дамена, напрыклад wwwuser@somewhere.net.

Прычына, па якой па змаўчанні гэта бязмежжа, заключаецца ў тым, што запыт можна зрабіць ад імя карыстальніка ў любым дамене, у залежнасці ад таго, як выкарыстоўваецца кэш. Некаторыя ftp-сервера таксама правяраюць, што адрас электроннай пошты сапраўдны (напрыклад, perl.com).

Праграма кальмараў DNS : па змаўчанні знаходзіцца dnsserver. Укажыце месца выкананага файла для працэсу dnslookup. Па змаўчанні выкарыстоўваецца неблакіруючы сервер імёнаў, які ўсталёўваецца па змаўчанні з Squid.

Колькасць DNS-праграм : па змаўчанні роўная 5. Колькасць працэсаў, спароджаных пошукамі DNS-пошуку службовых імёнаў. Для моцна загружаных кэшаў на вялікіх серверах вы, верагодна, павінны павялічыць гэта значэнне як мінімум на 10. Максімум - 32.

Дадаваць дамен да запытаў : па змаўчанні выключана. Звычайна "dnsserver" адключае варыянт дазволу RES_DEFNAMES (гл. Res_init (3)). Гэта прадухіляе кэшы ў іерархіі ад інтэрпрэтацыі аднакампанентных імёнаў хастоў на мясцовым узроўні. Каб дазволіць dnsserver апрацоўваць аднакампанентныя імёны, уключыце гэтую опцыю.

Адрасы сервера DNS : па змаўчанні выкарыстоўваецца файл rezlv.conf. Выкарыстоўвайце гэта, калі вы хочаце пазначыць спіс сервераў імёнаў DNS (IP-адрасоў), якія трэба выкарыстоўваць замест тых, якія паказаны ў вашым файле /etc/resolv.conf.

Праграма ачысткі кэша : па змаўчанні 'unlinkd'. Укажыце месца выкананага файла для працэсу выдалення файла. Гэта можа не спатрэбіцца, калі вы выкарыстоўваеце async-io, бо ў некаторых версіях ён апрацоўваецца ніткай.

Праграма пінг для кальмараў : па змаўчанні "пінгер". Укажыце месца выкананага файла для працэсу pinger. Гэта карысна толькі ў тым выпадку, калі вы наладжвалі Squid (падчас сумяшчання) з опцыяй '--enable-icmp'.

Спецыяльная праграма перанакіравання : па змаўчанні няма. Укажыце месцазнаходжанне выкананага файла для перанакіравальніка URL. Паколькі яны могуць выконваць практычна любыя функцыі, у іх не ўваходзіць ні адзін. Інфармацыю пра тое, як напісаць яго, можна даведацца пра заўвагі да выпуску.

Колькасць праграм для перанакіравання : па змаўчанні 5, калі перанакіраваны. Колькасць працэсаў перанакіравання на нераст. Калі вы запусціце занадта мала Кальмараў, вам прыйдзецца чакаць, пакуль яны апрацуюць адставанне URL, запавольваючы яго. Калі вы запусціце занадта шмат, яны будуць выкарыстоўваць аператыўную памяць і іншыя сістэмныя рэсурсы.
