캐시 옵션
캐시 디렉토리 : 여러 디스크 디렉토리간에 캐시를 분산시키기 위해 여러 캐시 디렉토리 행을 지정할 수 있습니다.

디렉토리 는 캐시 스왑 파일이 저장되는 최상위 디렉토리입니다. 캐싱에 전체 디스크를 사용하려면 마운트 지점 디렉토리가 될 수 있습니다. 디렉토리는 반드시 존재해야하며 오징어 프로세스에 의해 쓰기 가능해야합니다. Squid는이 디렉토리를 만들지 않습니다.

크기 는이 디렉토리에서 사용할 디스크 공간 (MB)입니다. 기본값은 100MB입니다. 구성에 맞게이를 변경하십시오.

1 단계 디렉토리 는 Directoryi 아래에 작성 될 1 단계 서브 디렉토리의 수입니다. 기본값은 16입니다.

2 단계 디렉토리 는 각 1 단계 디렉토리 아래에 작성 될 2 단계 서브 디렉토리의 수입니다. 기본값은 256입니다.

평균 객체 크기 : 기본값은 13 kB입니다. 캐시가 보유 할 수있는 오브젝트 수를 추정하는 데 사용되는 평균 오브젝트 크기.

버킷 당 오브젝트 : 기본값은 50입니다. 상점 해시 테이블에서 버킷 당 오브젝트의 대상 수입니다. 이 값을 낮추면 총 버킷 수와 스토리지 유지 관리 속도가 증가합니다.

ACL의 URL을 캐시하지 않음 : 기본값은 none입니다. Squid가 데이터를 캐시하지 못하게하려는 ACL을 선택하십시오. 캐시해서는 안되는 내부 웹 서버를 정의하는 데 도움이됩니다.

최대 캐시 시간 : 기본값은 1 년입니다. 이 매개 변수는 최대 LRU 수명을 정의합니다. 예를 들어이 항목을 '1 주일'로 설정하면 일주일 이상 액세스하지 않은 개체가 제거됩니다.

실패한 요청 캐시 시간 기본값은 5 분입니다. 실패한 요청에 대한 TTL (Time-to-Live). 특정 유형의 장애 (예 : "연결 거부"및 "404를 찾을 수 없음")는 구성 가능한 시간 동안 네거티브 캐시됩니다. 이는 DNS 조회의 네거티브 캐싱과 다릅니다.

DNS 조회 캐시 시간 : 기본값은 6 시간 (360 분)입니다. 성공적인 DNS 조회의 긍정적 인 캐싱을위한 TTL (Time-to-Live). Squid의 ipcache 사용을 최소화하려면 이것을 0이 아닌 1로 설정하십시오.

실패한 DNS 캐시 시간 : 기본값은 5 분입니다. 실패한 DNS 조회의 네거티브 캐싱을위한 TTL (Time-to-Live).

연결 시간 초과 : 기본값은 120 초입니다. connect (2) 요청을 올바르게 시간 종료하기 위해 일부 시스템 (특히 Linux)에 의존 할 수 없습니다. 따라서 Squid 프로세스는 서버 연결시 자체 시간 초과를 적용합니다. 이 매개 변수는 연결이 완료 될 때까지 대기하는 시간을 지정합니다.

읽기 시간 초과 : 기본값은 15 분입니다. read_timeout은 서버 측 연결에 적용됩니다. 각 성공적인 read () 후에 시간 초과가이 양만큼 연장됩니다. 이 시간이 지난 후 데이터를 다시 읽지 않으면 요청이 중단되고 ERR_READ_TIMEOUT으로 기록됩니다.

사이트 선택 시간 초과 : 기본값은 4 초입니다. URN에서 여러 URL의 URL을 선택합니다.

클라이언트 요청 시간 초과 : 기본값은 15 분입니다. read_timeout은 서버 측 연결에 적용됩니다. 각 성공적인 read () 후에 시간 초과가이 양만큼 연장됩니다. 이 시간이 지난 후 데이터를 다시 읽지 않으면 요청이 중단되고 ERR_READ_TIMEOUT으로 기록됩니다.

최대 클라이언트 연결 시간 : 기본값은 1 일입니다. 클라이언트 (브라우저)가 캐시 프로세스에 연결된 상태를 유지할 수있는 최대 시간입니다. 이를 통해 캐시가 제대로 종료되지 않고 (네트워크 장애 또는 클라이언트 구현 불량으로 인해) 원격 클라이언트로부터 CLOSE_WAIT 상태로 많은 소켓 (및 파일 디스크립터)이 묶이지 않도록 보호합니다.

최대 종료 시간 : 기본값은 30 초입니다. SIGTERM 또는 SIGHUP이 수신되면 모든 활성 소켓이 닫힐 때까지 캐시가 "종료 보류"모드로 전환됩니다. 이 값은 종료 모드 동안 열려있는 모든 디스크립터에 대해 설정된 수명입니다. 이 몇 초 후에 활성화 된 클라이언트는 'timeout'메시지를받습니다.

반 폐쇄 클라이언트 : 기본값은 'on'입니다. 일부 클라이언트는 TCP 연결의 송신 측을 종료하면서 수신 측은 열린 상태로 둘 수 있습니다. 때때로, Squid는 반 폐쇄와 완전 폐쇄 TCP 연결의 차이점을 알 수 없습니다. 기본적으로 절반 닫힌 클라이언트 연결은 소켓의 read (2) 또는 write (2)가 오류를 반환 할 때까지 열린 상태로 유지됩니다. 이 옵션을 'off'로 변경하면 read (2)가 "no more data to read"를 반환하면 Squid는 즉시 클라이언트 연결을 닫습니다.

영구 시간 초과 : 기본값은 120 초입니다. 클라이언트, 서버 및 기타 프록시에 대한 유휴 영구 연결 시간 초과. 이 연결이 10 초 미만이면 영구 연결이 비활성화됩니다.

WAIS 릴레이 호스트 : 기본값은 없습니다. WAIS 요청을 릴레이 할 호스트 이름 또는 주소입니다.

WAIS 릴레이 포트 : 기본값은 none입니다. 릴레이 WAIS 요청을 전달하는 포트입니다.
