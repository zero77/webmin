Административные параметры
Запуск от имени пользователя Unix : по умолчанию установлено «nobody» или «squid» в зависимости от системы. Если кеш запускается как root, он изменит свой действительный / реальный идентификатор на это значение.

Запуск от имени группы Unix : по умолчанию «nogroup» или «squid» в зависимости от системы. Если кеш запускается от имени пользователя root, он изменит действующий / реальный идентификатор группы на это значение.

Область проверки подлинности прокси : Указывает имя области, которое должно быть сообщено клиенту для проверки подлинности прокси (часть текста, которую пользователь увидит при запросе имени пользователя и пароля).

Адрес электронной почты менеджера кэша : по умолчанию «webmaster» или «root» в зависимости от системы. E-mail адрес локального менеджера кеша, который будет получать почту, если кеш умирает.

Видимое имя хоста : если вы хотите представить специальное имя хоста в сообщениях об ошибках и т. Д., Определите это. В противном случае будет использоваться возвращаемое значение gethostname (). Если у вас есть несколько кэшей в кластере и вы получаете ошибки об IP-пересылке, вы должны установить для них индивидуальные имена с этим параметром.

Уникальное имя хоста : если вы хотите иметь несколько машин с одним и тем же видимым именем хоста, вы должны назначить каждому компьютеру свое уникальное имя хоста, чтобы можно было обнаружить петли пересылки.

Хост объявления кеша : Имя хоста, на который будут отправляться объявления. Узнайте больше об иерархиях кеша в FAQ по Squid . По умолчанию используется «tracker.ircache.net», когда объявление включено путем установки периода объявления.

Порт кэширования объявлений : порт хоста, на который будут отправляться объявления. По умолчанию 3131, когда объявление включено.

Период объявления : частота объявлений. Включите объявления в иерархии кэша, установив для этого значение «1 день». По умолчанию установлено значение «0», что отключает объявление.
