Pembalakan
Fail log akses : Lalai bergantung pada sistem, biasanya di suatu tempat di / var . Mencatat aktiviti permintaan pelanggan. Mengandungi entri untuk setiap permintaan HTTP dan ICP yang diterima.

Fail log debug : Lalai bergantung pada sistem. Fail pembalakan cache. Di sinilah maklumat umum mengenai tingkah laku cache anda berlaku. Anda boleh menambah jumlah data yang dilog masuk ke fail ini dengan pilihan Debug .

Fail log simpanan : Lalai bergantung pada sistem. Mencatat aktiviti pengurus storan. Menunjukkan objek mana yang dikeluarkan dari cache, dan objek mana yang disimpan dan berapa lama. Untuk melumpuhkan, masukkan "tidak ada". Tidak ada utiliti untuk menganalisis data ini, jadi anda boleh melumpuhkannya dengan selamat.

Fail metadata cache : Lalai adalah swap.log di setiap direktori cache. Fail log ini menyimpan metadata objek yang disimpan pada cakera. Ia digunakan untuk membina semula cache semasa permulaan. Biasanya fail ini berada di setiap direktori cache, tetapi anda boleh menentukan nama jalan alternatif di sini. Perhatikan bahawa anda mesti memberikan nama fail penuh, bukan hanya direktori. Oleh kerana ini adalah indeks untuk keseluruhan senarai objek, anda TIDAK BOLEH memutarnya secara berkala!

Sekiranya% s dapat digunakan dalam nama file maka itu akan diganti dengan representasi dari nama direktori cache di mana setiap / diganti dengan '.'. Ini diperlukan untuk memungkinkan menambah / menghapus baris direktori cache ketika log pertukaran cache alternatif sedang digunakan.

Sekiranya mempunyai lebih daripada satu direktori cache, dan% s tidak digunakan dalam namanya, log pertukaran ini akan mempunyai nama seperti:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

Sambungan bernombor (yang ditambahkan secara automatik) sesuai dengan susunan baris direktori cache dalam konfigurasi. Sekiranya anda mengubah susunan baris direktori cache dalam fail ini, maka fail log ini TIDAK akan sesuai dengan entri direktori cache yang betul (kecuali jika anda menamakannya secara manual). Kami mengesyorkan agar anda TIDAK menggunakan pilihan ini. Lebih baik menyimpan fail log ini di setiap direktori cache.

Gunakan format log HTTPD : Lalai ke tidak. Cache dapat meniru format fail log yang banyak digunakan oleh program 'httpd'. Untuk melumpuhkan / mengaktifkan emulasi ini, tetapkan atau matikan nilai ini. Lalai adalah menggunakan format log asli kerana ia merangkumi maklumat berguna yang digunakan penganalisis log khusus Squid.

Log header MIME : Lalai ke no. Cache dapat merekam permintaan dan header MIME respons untuk setiap transaksi HTTP. Header dikodkan dengan selamat dan akan muncul sebagai dua medan kurungan pada akhir log akses (untuk format log asli atau httpd-emulated). Untuk mendayakan pembalakan ini, hidupkan ini.

Fail log ejen pengguna : Lalai tidak ada. Sekiranya dikonfigurasi dengan pilihan konfigurasi "--enable-useragent-log", Squid akan menuliskan bidang User-Agent dari permintaan HTTP ke nama fail yang dinyatakan di sini.

Fail PID : Lalai bergantung pada sistem. Nama fail untuk menulis proses-id. Untuk melumpuhkan, masukkan "tidak ada".

Lakukan carian identifikasi RFC931 untuk ACL : Lalai tidak ada. Lakukan carian ident untuk mencari nama pengguna pelanggan yang mengakses apa sahaja di ACL ini.

Waktu tamat ident RFC931 : Lalai adalah 30 saat. Masa maksimum untuk menunggu permintaan IDENT. Sekiranya ini terlalu tinggi, dan anda mengaktifkan carian ident, anda mungkin mudah ditolak dengan banyak permintaan ident sekaligus.

Log nama hos penuh : Lalai adalah no. Mencatat nama host pelanggan sepenuhnya.

Netmask logging : Lalai adalah 255.255.255.255. Netmask untuk alamat pelanggan dalam fail log dan output cachemgr. Ubah ini untuk melindungi privasi klien cache anda. Netmask 255.255.255.0 akan mencatat semua IP dalam julat tersebut dengan digit terakhir ditetapkan ke '0'.

Pilihan debug : Lalai adalah "SEMUA, 1". Pilihan log ditetapkan sebagai bahagian, tahap di mana setiap fail sumber diberi bahagian unik. Tahap yang lebih rendah mengakibatkan debugging penuh (tahap 9) boleh menghasilkan fail log yang sangat besar, jadi berhati-hatilah. Kata ajaib "SEMUA" menetapkan tahap penyahpepijatan untuk semua bahagian. Kami mengesyorkan agar berjalan dengan "ALL, 1".

Jadual tajuk MIME : Pathname ke jadual MIME Squid. Anda tidak perlu mengubahnya, tetapi fail lalai mengandungi contoh dan maklumat pemformatan jika anda melakukannya.
