Verschiedene Optionen
Start-DNS-Testadressen : DNS-Server, die Squid abfragt, um zu testen, ob DNS-Lookups ordnungsgemäß funktionieren.

Rotationen der SIGUSR1-Protokolldatei : Standardmäßig 10 oder 0, abhängig vom System. Gibt die Anzahl der Protokolldateirotationen an, die ausgeführt werden sollen, wenn Sie "squid -k rotate" eingeben. Der Standardwert ist 10, der mit den Erweiterungen 0 bis 9 gedreht wird. Wenn Sie logfile_rotate auf 0 setzen, wird die Drehung deaktiviert, aber die Protokolldateien werden weiterhin geschlossen und erneut geöffnet. Auf diese Weise können Sie die Protokolldateien unmittelbar vor dem Senden des Rotationssignals selbst umbenennen.

Beachten Sie, dass der Befehl 'squid -k rotate' normalerweise ein USR1-Signal an den laufenden Squid-Prozess sendet. In bestimmten Situationen (z. B. unter Linux mit Async-E / A) wird USR1 für andere Zwecke verwendet, sodass -k rotate ein anderes Signal verwendet. Es ist am besten, sich daran zu gewöhnen, 'squid -k rotate' anstelle von 'kill -USR1' zu verwenden '.

Fehlermeldungen Text: HTML - Text in Fehlermeldungen enthält. Machen Sie dies zu einer "Mailto" -URL zu Ihrer Administratoradresse oder zu einem Link zu der Webseite Ihres Unternehmens.

Um dies in Ihre Fehlermeldungen aufzunehmen, müssen Sie die Fehlervorlagendateien (im Verzeichnis "Fehler") neu schreiben. Fügen Sie überall dort, wo die Zeile 'err_html_text' angezeigt werden soll, ein% L-Tag in die Fehlervorlagendatei ein.

Statistiken pro Client : Wenn Sie das Sammeln von Statistiken pro Client deaktivieren möchten, deaktivieren Sie hier client_db.

X-Forwarded-For-Header : Die Standardeinstellung ist "Ein". Wenn festgelegt, nimmt Squid die IP-Adresse oder den Namen Ihres Systems in die HTTP-Anforderungen auf, die es weiterleitet. Standardmäßig sieht es so aus, als ob die HTTP-Anforderungen es weiterleiten. Standardmäßig sieht es so aus:

X-Forwarded-For: 192.1.2.3

Wenn Sie dies deaktivieren, wird Folgendes angezeigt:

X-Forwarded-For: unbekannt

ICP-Abfragen protokollieren: Die Standardeinstellung ist "Ein". Wenn festgelegt, werden ICP-Abfragen in access.log protokolliert. Möglicherweise möchten Sie dies deaktivieren, wenn Ihre ICP-Last SEHR hoch ist, um die Arbeit zu beschleunigen oder die Protokollanalyse zu vereinfachen.

Minimale direkte Hops : Die Standardeinstellung ist 4. Wenn Sie das ICMP-Ping-Material verwenden, führen Sie direkte Abrufe für Sites durch, die nicht mehr als so viele Hops entfernt sind.

Bewahren Sie den Speicher für die zukünftige Verwendung auf : Die Standardeinstellung ist "Ja". Wenn festgelegt, hält Squid Pools des zugewiesenen (aber nicht verwendeten) Speichers für die zukünftige Verwendung verfügbar. Deaktivieren Sie dies, wenn Speicher auf Ihrem System eine Prämie darstellt und Sie der Meinung sind, dass Ihre Malloc-Bibliothek die Squid-Routinen übertrifft.

Speicherplatz : Die Standardeinstellung ist "Unbegrenzt". Bei einem Wert ungleich Null behält Squid höchstens die angegebene Grenze des zugewiesenen (aber nicht verwendeten) Speichers in Speicherpools bei. Alle free () - Anfragen, die dieses Limit überschreiten, werden von Ihrer malloc-Bibliothek bearbeitet. Squid weist keinen Speicher vorab zu, sondern verwahrt nur Objekte, die sonst frei wären () d. Daher ist es sicher, memory_pools_limit auf einen angemessen hohen Wert zu setzen, selbst wenn Ihre Konfiguration weniger Speicher benötigt.

Wenn nicht gesetzt (Standard) oder auf Null gesetzt, behält Squid den gesamten Speicher, den es kann. Das heißt, es gibt keine Begrenzung für die Gesamtmenge an Speicher, die zur sicheren Aufbewahrung verwendet wird.

Durchzuleitende Header : Die Standardeinstellung ist "Alle Header". Diese Option ersetzt die alte Option 'http_anonymizer' durch etwas, das viel konfigurierbarer ist. Sie können jetzt genau angeben, welche Header zugelassen oder welche aus ausgehenden Anforderungen entfernt werden sollen.

Es gibt zwei Methoden, um diese Option zu verwenden. Sie können entweder bestimmte Header zulassen (wodurch alle anderen verweigert werden), oder Sie können bestimmte Header ablehnen (wodurch alle anderen zugelassen werden).

Fake User-Agent : Standard ist 'none'. Wenn Sie den User-Agent-Header mit der Option " Header zum Durchlaufen" filtern, kann dies dazu führen, dass einige Webserver Ihre Anforderung ablehnen. Verwenden Sie dies, um eine zu fälschen.

WCCP-Router-Adresse : Verwenden Sie diese Option, um Ihren WCCP-Heimrouter für Squid zu definieren.

Eingehende WCCP-Adresse : Verwenden Sie diese Option, wenn WCCP-Nachrichten nur auf einer Schnittstelle empfangen werden sollen. Verwenden Sie diese Option NICHT, wenn Sie sich nicht sicher sind, wie viele Schnittstellen Sie haben oder wenn Sie wissen, dass Sie nur eine Schnittstelle haben.

Ausgehende WCCP-Adresse : Verwenden Sie diese Option, wenn WCCP-Nachrichten nur auf einer Schnittstelle gesendet werden sollen. Verwenden Sie diese Option NICHT, wenn Sie sich nicht sicher sind, wie viele Schnittstellen Sie haben oder wenn Sie wissen, dass Sie nur eine Schnittstelle haben.
