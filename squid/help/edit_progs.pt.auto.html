Programas auxiliares
Largura da coluna do FTP : o padrão é 32. Define a largura das listagens de ftp. Isso deve ser definido para caber na largura de um navegador padrão. Definir esse tamanho muito pequeno pode cortar nomes de arquivos longos ao navegar em sites FTP.

Logon no FTP da Anon : o padrão é 'Squid @'. Se você deseja que a senha de login anônima seja mais informativa (e habilite o uso de servidores FTP exigentes), defina-a como algo razoável para o seu domínio, como wwwuser@somewhere.net.

A razão pela qual isso é sem domínio por padrão é que a solicitação pode ser feita em nome de um usuário em qualquer domínio, dependendo de como o cache é usado. Alguns servidores ftp também validam que o endereço de email é válido (por exemplo, perl.com).

Programa DNS do Squid : O padrão é dnsserver. Especifique o local do executável para o processo dnslookup. O padrão é um servidor de nomes de armazenamento em cache sem bloqueio instalado por padrão com o Squid.

Número de programas DNS : o padrão é 5. O número de processos gerados para atender às pesquisas de nome DNS. Para caches muito carregados em servidores grandes, você provavelmente deve aumentar esse valor para pelo menos 10. O máximo é 32.

Anexar domínio a solicitações : o padrão está desativado. Normalmente, o 'dnsserver' desativa a opção do resolvedor RES_DEFNAMES (consulte res_init (3)). Isso evita que caches em uma hierarquia interpretem nomes de host de componente único localmente. Para permitir que o dnsserver manipule nomes de componente único, ative esta opção.

Endereços do servidor DNS : o padrão é usar o arquivo resolv.conf. Use isso se desejar especificar uma lista de servidores de nomes DNS (endereços IP) a serem usados em vez daqueles fornecidos no seu arquivo /etc/resolv.conf.

Programa de limpeza de cache : o padrão é 'unlinkd'. Especifique o local do executável para o processo de exclusão de arquivo. Isso pode não ser necessário se você estiver usando o async-io, pois ele é tratado por um encadeamento em algumas versões.

Squid ping program : O padrão é 'pinger'. Especifique o local do executável para o processo de transferência. Isso só é útil se você configurou o Squid (durante a compliação) com a opção '--enable-icmp'.

Programa de redirecionamento personalizado : o padrão é nenhum. Especifique o local do executável para o redirecionador de URL. Como eles podem executar praticamente qualquer função, não há uma incluída. Consulte as Notas da versão para obter informações sobre como escrever uma.

Número de programas de redirecionamento : o padrão é 5 se um redirecionador estiver configurado. O número de processos de redirecionador a serem gerados. Se você começar muito pouco, o Squid terá que esperar que eles processem um backlog de URLs, diminuindo a velocidade. Se você iniciar muitos, eles usarão RAM e outros recursos do sistema.
