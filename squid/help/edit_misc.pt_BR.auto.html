Opções diversas
Endereços de teste de DNS de inicialização : servidores DNS que o Squid consultará para testar se as pesquisas de DNS estão funcionando corretamente.

Rotações do arquivo de log SIGUSR1 : O padrão é 10 ou 0, dependendo do sistema. Especifica o número de rotações do arquivo de log a serem feitas quando você digita 'squid -k rotate'. O padrão é 10, que girará com as extensões de 0 a 9. A configuração de logfile_rotate como 0 desativará a rotação, mas os arquivos de log ainda serão fechados e reabertos. Isso permitirá que você renomeie os arquivos de log antes de enviar o sinal de rotação.

Note, o comando 'squid -k rotate' normalmente envia um sinal USR1 para o processo do squid em execução. Em certas situações (por exemplo, no Linux com E / S assíncrona), o USR1 é usado para outros fins, portanto -k rotate usa outro sinal. É melhor adquirir o hábito de usar 'squid -k rotate' em vez de 'kill -USR1 '

Texto das mensagens de erro : texto HTML a ser incluído nas mensagens de erro. Torne este um URL "mailto" para o seu endereço de administrador ou talvez apenas um link para a página da Web da sua organização.

Para incluir isso em suas mensagens de erro, você deve reescrever os arquivos de modelo de erro (encontrados no diretório "erros"). Onde quer que a linha 'err_html_text' apareça, insira uma marca% L no arquivo de modelo de erro.

Estatísticas por cliente : se você deseja desativar a coleta de estatísticas por cliente, desative aqui client_db.

Cabeçalho X-Forwarded-For : o padrão é 'on'. Se configurado, o Squid incluirá o endereço IP ou o nome do seu sistema nas solicitações HTTP que ele encaminha. Por padrão, parece que nos pedidos HTTP ele encaminha. Por padrão, fica assim:

X-Forwarded-For: 192.1.2.3

Se você desabilitar isso, ele aparecerá como:

X-Forwarded-For: desconhecido

Registrar consultas ICP : o padrão é 'on'. Se definido, as consultas ICP são registradas em access.log. Você pode desabilitar isso se sua carga de ICP for MUITO alta para acelerar as coisas ou simplificar a análise de logs.

Lúpulos diretos mínimos : o padrão é 4. Se você estiver usando o material de ping do ICMP, faça buscas diretas em sites que não tenham mais do que muitos saltos de distância.

Mantenha a memória para uso futuro : o padrão é 'yes'. Se configurado, o Squid manterá conjuntos de memória alocada (mas não utilizada) disponíveis para uso futuro. Se a memória é premium em seu sistema e você acredita que sua biblioteca malloc supera as rotinas do Squid, desative isso.

Quantidade de memória a ser mantida : o padrão é 'Ilimitado'. Se definido como um valor diferente de zero, o Squid manterá no máximo o limite especificado de memória alocada (mas não utilizada) em conjuntos de memórias. Todas as solicitações gratuitas () que excederem esse limite serão tratadas pela sua biblioteca malloc. O Squid não pré-aloca nenhuma memória, apenas guarda objetos que de outra forma estariam livres () d. Portanto, é seguro definir memory_pools_limit para um valor razoavelmente alto, mesmo que sua configuração use menos memória.

Se não estiver definido (padrão) ou definido como zero, o Squid manterá toda a memória possível. Ou seja, não haverá limite para a quantidade total de memória usada para proteção.

Cabeçalhos a serem passados : o padrão é 'Todos os cabeçalhos'. Esta opção substitui a antiga opção 'http_anonymizer' por algo que é muito mais configurável. Agora você pode especificar exatamente quais cabeçalhos devem ser permitidos ou quais devem ser removidos das solicitações de saída.

Existem dois métodos para usar esta opção. Você pode permitir cabeçalhos específicos (negando assim todos os outros) ou pode negar cabeçalhos específicos (permitindo assim todos os outros).

Agente do usuário falso : o padrão é 'none'. Se você filtrar o cabeçalho User-Agent com os cabeçalhos para passar a opção, isso poderá fazer com que alguns servidores Web recusem sua solicitação. Use isso para falsificar um.

Endereço do roteador WCCP : Use esta opção para definir seu roteador `` doméstico '' WCCP para o Squid.

Endereço de entrada WCCP : Use esta opção se você precisar que as mensagens WCCP sejam recebidas em apenas uma interface. NÃO use essa opção se não tiver certeza de quantas interfaces você possui ou se sabe que possui apenas uma interface.

Endereço de saída WCCP : Use esta opção se você precisar que as mensagens WCCP sejam enviadas em apenas uma interface. NÃO use essa opção se não tiver certeza de quantas interfaces você possui ou se sabe que possui apenas uma interface.
