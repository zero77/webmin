Contrôle d'accès
Listes de contrôle d'accès : les listes de contrôle d' accès sont des listes de termes à mettre en correspondance à l'aide d'expressions standard d'expressions littérales. Une ACL peut également être un fichier qui contient un élément ou une expression régulière par ligne. La zone ACL de cette page contient plusieurs champs qui peuvent être modifiés. De nouvelles ACL peuvent également être créées.

Les champs présents sont:

Nom est le nom par lequel l'ACL peut être identifiée. Lors de la création d'une restriction proxy, c'est le nom utilisé pour définir ce qui est restreint.

Type est le type d'information auquel l'ACL doit être comparée.

Correspondance .. est l'adresse, le port, l'URL, l'utilisateur, etc. auxquels la liste de contrôle d'accès correspondra.

Restrictions de proxy : les restrictions de proxy sont des règles qui autorisent ou refusent une demande donnée selon que son ACL correspond aux spécificités de la requête. Il y a trois champs dans cette case.

L'action est soit autoriser soit refuser .

Les ACL sont les ACL qui seront comparées pour décider si la demande est autorisée ou refusée. S'il est ajouté par un point d'exclamation, l'ACL sera annulée, en d'autres termes, tout sauf les membres de cette ACL sera autorisé ou refusé.

Déplacer permet l'ordre des restrictions dans la liste. L'ordre dans lequel ils apparaissent est important, car Squid ne lit la liste que jusqu'à ce qu'il ait trouvé la première correspondance.

Restrictions ICP : cette case sert à restreindre les demandes ICP. Les requêtes ICP sont des requêtes provenant d'autres caches voisins. Cette section fonctionne de la même manière que les restrictions de proxy .
