Opções administrativas
Executar como usuário Unix : O padrão é 'ninguém' ou 'Lula', dependendo do sistema. Se o cache for executado como raiz, ele alterará seu ID efetivo / real para esse valor.

Executar como grupo Unix : O padrão é 'nogroup' ou 'squid', dependendo do sistema. Se o cache for executado como raiz, ele alterará seu ID de grupo efetivo / real para esse valor.

Região de autenticação de proxy : especifica o nome da região que deve ser relatada ao cliente para autenticação de proxy (parte do texto que o usuário verá quando for solicitado seu nome de usuário e senha).

Endereço de email do gerenciador de cache : o padrão é 'webmaster' ou 'root' dependente do sistema. Endereço de email do gerenciador de cache local que receberá emails se o cache morrer.

Nome do host visível : se você deseja apresentar um nome de host especial nas mensagens de erro, etc, defina isso. Caso contrário, o valor de retorno de gethostname () será usado. Se você possui vários caches em um cluster e obtém erros sobre o encaminhamento de IP, configure-os para terem nomes individuais com essa configuração.

Nome de host exclusivo : se você deseja ter várias máquinas com o mesmo nome de host visível , deve fornecer a cada máquina um nome de host exclusivo diferente para que os loops de encaminhamento possam ser detectados.

Host de anúncio em cache : Nome do host para o qual as mensagens de anúncio serão enviadas. Saiba mais sobre hierarquias de cache nas Perguntas frequentes do Squid . O padrão é 'tracker.ircache.net' quando o anúncio é ativado, definindo um período de anúncio.

Porta de anúncio de cache : porta do host para a qual as mensagens de anúncio serão enviadas. O padrão é 3131 quando o anúncio está ativado.

Período do anúncio : Frequência dos anúncios. Ative os anúncios para uma hierarquia de cache, definindo-o como '1 dia'. O padrão é '0', que desativa o anúncio.
