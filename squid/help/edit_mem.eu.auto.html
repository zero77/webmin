Memoriaren erabilera
Memoriaren erabileraren muga : lehenetsi 8 MB.

OHARRA: parametro honek ez du prozesuaren gehieneko tamaina zehazten. Squid memoriaren erabileraren alderdi batean muga jartzen du. Txipiroiak beste gauza batzuetarako memoria ere erabiltzen du. Zure prozesua hemen jarri zenuen balioa bitan edo hiru aldiz handiagoa izango da.

Erabiliko den memoria kopuru egokia zehazten du:

  • In-Transit objektuak
  • Objektu beroak
  • Negatiboetan gordetako objektuak

Objektu horien datuak 4 kB bloketan gordetzen dira. Parametro honek esleitutako 4 kB blokeen guztizko neurriaren goiko muga aproposa zehazten du. In-Transit objektuek lehentasun handiena dute.

Trantsizioko objektuek lehentasuna dute besteekiko. Sarrerako datuak jartzeko espazio osagarria behar denean, negatiboak diren cacheak eta objektu beroak askatuko dira. Bestela esanda, cache negatiboko eta beroko objektuek trantsizioko objektuetarako beharrezko ez den espazioa beteko dute.

Zirkunstantziak behar izanez gero, muga hori gaindituko da. Zehazki, zure sarrerako eskaera tasak memoria kopuru hori baino gehiago eskatzen badu garraioko objektuak edukitzeko, Squid-ek muga hori gaindituko du eskaera berriak asetzeko. Karga gutxitzen denean, blokeak askatuko dira ur handiko marka lortu arte. Hortik aurrera, blokeak objektu beroak gordetzeko erabiliko dira.

FQDN cache tamaina : 1024 lehenetsi dira. Guztiz kualifikatutako domeinu-izenaren cache-sarreren gehienezko kopurua.

Memoria ur handiko marka : lehenetsitakoak% 95era. Puntu altu hori lortzen denean, objektu beroak eta iragazkiak diren objektuak memoria librera aterako dira.

Memoria ur gutxiko marka : lehenetsitakoak% 90era. Ur-marka handia lortu ondoren, Squid-ek memoria libratuko du erabilera marka honen azpitik jaitsi arte.

Disko ur handiko marka : lehenetsitakoak% 95era. Goi-marka hau lortzen denean, Squid-ek diskoa erasotzen hasiko da.

Disko ur gutxiko marka : lehenetsitakoak% 90era. Ur-marka handia lortu ondoren, Squid-ek diskoa iraintzen jarraituko du, erabilera marka honen azpitik jaitsi arte.

Gehienez gordetako objektuen tamaina : 4096 kB lehenetsi. Ezarpen honek cacheak gordeko dituen objektuen tamaina mugatzen du. Tamaina honetako edozein gauza ez da inoiz gordeko eta jatorrizko zerbitzaritik kargatu behar da behin.

IP helbideen cachearen tamaina : lehenetsitako 1024. IP cachean gordeko diren sarrera kopurua.

Ur handiko katxearen IP marka :% 95era lehenetsitakoak. Squid-ek IP cache sarrerak ixten hasten duen puntua.

IP cache ur baxuko marka : lehenetsitakoak% 90era. IP cacheen hustuketa hasi ondoren, IP cacheko sarrera kopurua ehuneko horren azpitik jaitsi arte jarraituko da.
