Opcije administracije
Pokreni kao Unix korisnik : Zadano je 'nitko' ili 'lignja' ovisi o sustavu. Ako se predmemorija pokrene kao root, promijenit će svoj efektivni / stvarni ID u ovu vrijednost.

Pokreni kao Unix grupa : Zadano je 'nogroup' ili 'lignja' ovisno o sustavu. Ako se predmemorija pokrene kao root, promijenit će svoj efektivni / stvarni ID grupe u ovu vrijednost.

Područje provjere autentičnosti proxy : Određuje ime kraljevstva koje se prijavljuje klijentu za proxy provjeru (dio teksta koji će korisnik vidjeti kad zatraži svoje korisničko ime i lozinku).

E-adresa upravitelja predmemorije : zadana postavka je "webmaster" ili "root" ovisno o sustavu. Adresa e-pošte lokalnog upravitelja predmemorije koji će primati poštu ako predmemorija umre.

Vidljivo ime računala : Ako želite predstaviti posebno ime hosta u porukama pogreške, itd., Definirajte to. U suprotnom, koristit će se povratna vrijednost gethostname (). Ako imate nekoliko predmemorija u klasteru i imate pogreške u vezi s prosljeđivanjem IP adrese, morate ih postaviti tako da imaju pojedinačna imena s ovom postavkom.

Jedinstveno ime računala : Ako želite imati više strojeva s istim Visible nazivom hosta, morate svakom stroju dati različito Jedinstveno ime računala kako bi se mogle detektirati petlje prosljeđivanja.

Keš najava hosta : Ime hosta gdje će se slati poruke najave. Saznajte više o hijerarhiji predmemorije putem FAQ-a za Squid . Zadano je "tracker.ircache.net" kada je najava omogućena postavljanjem razdoblja najave.

Port za predmemoriju najava : Luka hosta na koju će se slati poruke najave. Zadane vrijednosti 3131 kad je omogućena najava.

Razdoblje najava : Učestalost najava. Omogućite najave u predmemorijsku heirarhiju tako da postavite ovo na '1 dan'. Zadani je "0", što isključuje najavu.
