Ведення журналів
Файл журналу доступу : За замовчуванням залежить система, зазвичай десь у / var . Журнали активності запиту клієнта. Містить запис для кожного отриманого запиту HTTP та ICP.

Файл журналу налагодження : за замовчуванням залежить система. Файл журналу кешу. Тут іде загальна інформація про поведінку кешу. Ви можете збільшити кількість даних, зафіксованих до цього файлу, за допомогою параметрів налагодження .

Файл журналу зберігання : за замовчуванням залежить система. Реєструє діяльність менеджера зберігання. Показує, які об’єкти викидаються з кешу, а які об’єкти зберігаються та на скільки часу. Щоб відключити, введіть "немає". Не є насправді утиліти для аналізу цих даних, тому ви можете сміливо їх відключити.

Файл метаданих кешу : за замовчуванням є swap.log у кожному каталозі кеша. Цей файл журналу містить метадані об’єктів, збережених на диску. Він використовується для відновлення кешу під час запуску. Зазвичай цей файл знаходиться у кожному каталозі кешу, але ви можете вказати альтернативне ім'я шляху тут. Зверніть увагу, ви повинні вказати повне ім'я файлу, а не лише каталог. Оскільки це індекс для всього списку об’єктів, ви НЕ МОЖЕТЕ періодично обертати його!

Якщо у імені файлу може бути використаний% s, він буде замінений представленням імені каталогу кеша, де кожне / замінено на '.' Це потрібно, щоб дозволити додавання / видалення рядків каталогу кеша, коли використовується альтернативний журнал заміни кеша.

Якщо у вас є більше одного каталогу кешу, а% s не використовується в імені, то ці журнали своп будуть мати такі імена, як:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

Нумероване розширення (яке додається автоматично) відповідає порядку рядків кеш-каталогу в конфігурації. Якщо ви зміните порядок рядків каталогів кеша в цьому файлі, ці файли журналів НЕ будуть відповідати правильному запису каталогу кеша (якщо ви не перейменовуєте їх вручну). Ми рекомендуємо НЕ використовувати цю опцію. Краще зберігати ці файли журналів у кожному каталозі кешу.

Використовуйте формат журналу HTTPD : за замовчуванням немає. Кеш може імітувати формат файлу журналу, який використовують багато програм «httpd». Щоб відключити / увімкнути цю емуляцію, встановіть це значення на вимкнено або увімкнено. За замовчуванням застосовується власний формат журналу, оскільки він включає корисну інформацію, яку використовують аналізатори журналу для конкретних кальмарів.

Журнал MIME заголовків : за замовчуванням немає. Кеш може записувати як запити, так і MIME заголовки відповідей для кожної транзакції HTTP. Заголовки кодуються безпечно і відображатимуться як два скоплені у вікні поля в кінці журналу доступу (як для власного, так і для httpd-емульованого формату журналу). Щоб увімкнути цей журнал, увімкніть це.

Файл журналу агента користувача : за замовчуванням немає. Якщо налаштовано параметр "--enable-useragent-log", Squid запише поле User-Agent із запитів HTTP на вказане тут ім'я файлу.

PID-файл : за замовчуванням залежить система. Ім'я файлу, в яке слід записати ідентифікатор процесу. Щоб відключити, введіть "немає".

Виконайте пошук RFC931 ідентифікаторів для ACL : за замовчуванням немає. Виконайте ідентифікаційні пошуки, щоб знайти ім’я користувача клієнтів, що отримують доступ до будь-якого з цих ACL.

Час очікування ідентифікатора RFC931 : за замовчуванням 30 секунд. Максимальний час для очікування IDENT-запитів. Якщо це занадто високо, і ви ввімкнули будь-які пошукові ідентифікатори, то, можливо, ви будете сприйнятливі до відмови в обслуговуванні, зробивши відразу багато запитів ідентифікаторів.

Журнал повних імен хостів : за замовчуванням немає. Журнали повного імені хоста клієнтів.

Веб-маска журналу : за замовчуванням 255.255.255.255. Маска мережі для адрес клієнтів у логістичних файлах та кешмгр-виведенні. Змініть це, щоб захистити конфіденційність своїх клієнтів кешу. Маска мережі 255.255.255.0 запише всі IP-адреси в цьому діапазоні з останньою цифрою, встановленою на "0".

Параметри налагодження : за замовчуванням "ВСЕ, 1". Параметри журналу встановлюються як розділ, рівень, де кожному вихідному файлу призначається унікальний розділ. Нижчі рівні призводять до меншої Повна налагодження (рівень 9) може спричинити дуже великий файл журналу, тому будьте уважні. Чарівне слово "ВСІ" встановлює рівні налагодження для всіх розділів. Ми рекомендуємо нормально працювати з "ALL, 1".

Таблиця заголовків MIME : назва імені до таблиці MIME Squid. Вам не потрібно змінювати, але файл за замовчуванням містить приклади та інформацію про форматування, якщо це потрібно.
