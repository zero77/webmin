Opțiuni diverse
Adrese de testare DNS de pornire : serverele DNS pe care Squid le va interoga pentru a testa dacă căutările DNS funcționează corect.

Rotații ale fișierului de înregistrare SIGUSR1 : implicite la 10 sau 0, depind de sistem. Specifică numărul de rotații ale fișierului de înregistrare pe care îl introduceți atunci când introduceți „squid -k rotate”. Setarea implicită este 10, care se va roti cu extensiile de la 0 la 9. Setarea logfile_rotate pe 0 va dezactiva rotația, dar fișierele de înregistrare sunt încă închise și redeschise. Acest lucru vă va permite să redenumiți fișierele de logare chiar înainte de a trimite semnalul de rotire.

Notă, comanda „squid -k rotate” trimite în mod normal un semnal USR1 procesului de calamar în curs. În anumite situații (de exemplu, pe Linux cu Async I / O), USR1 este utilizat în alte scopuri, astfel încât -k rotate folosește un alt semnal. Cel mai bine este să obișnuiești să folosești „squid -k rotate” în loc de „ucid -USR1” “.

Textul mesajelor de eroare : text HTML care trebuie inclus în mesajele de eroare. Faceți din aceasta o adresă URL „mailto” la adresa dvs. de administrator sau poate doar un link către pagina web a organizațiilor dvs.

Pentru a include acest lucru în mesajele de eroare, trebuie să rescrieți fișierele de șabloane de eroare (găsite în directorul „erori”). Oriunde doriți să apară linia „err_html_text”, introduceți o etichetă% L în fișierul șablon de eroare.

Statistici per client : Dacă doriți să dezactivați colectarea statisticilor pentru fiecare client, atunci dezactivați client_db aici.

Antetul X-Forwarded-For : Default este „pornit”. Dacă este setat, Squid va include adresa IP sau numele sistemului dvs. în solicitările HTTP pe care le transmite mai departe. În mod implicit, se pare că în HTTP o solicită înainte. În mod implicit, arată așa:

X-Forwarded-For: 192.1.2.3

Dacă dezactivați acest lucru, acesta va apărea ca:

X-Forwarded-For: necunoscut

Jurnal de interogări ICP : implicit este „activat”. Dacă este setat, interogările ICP sunt înregistrate la access.log. Poate doriți să dezactivați acest lucru dacă încărcarea dvs. ICP este foarte mare pentru a accelera lucrurile sau pentru a simplifica analiza jurnalului.

Salt minim direct : implicit este 4. Dacă utilizați chestii pinging ICMP, efectuați preluări directe pentru site-uri care nu sunt mai mult decât atât de multe hamei.

Păstrați memoria pentru utilizarea viitoare : implicit este „da”. Dacă este setat, Squid va păstra grupurile de memorie alocate (dar nefolosite) disponibile pentru utilizare viitoare. Dacă memoria este premium pe sistemul dvs. și credeți că biblioteca dvs. malloc depășește rutinele Squid, dezactivați-o.

Cantitate de memorie de păstrat : implicit este „Nelimitat”. Dacă este setată la o valoare diferită de zero, Squid va păstra cel mult limita specificată de memorie alocată (dar nefolosită) în grupurile de memorie. Toate cererile gratuite () care depășesc această limită vor fi gestionate de biblioteca dvs. malloc. Squid nu alocă nicio memorie, ci păstrează în siguranță obiecte care altfel ar fi gratuite () d. Astfel, este sigur să setați memory_pools_limit la o valoare rezonabil de mare, chiar dacă configurația dvs. va utiliza mai puțină memorie.

Dacă nu este setat (implicit) sau setat la zero, Squid va păstra toată memoria pe care o poate. Adică, nu va exista nicio limită la cantitatea totală de memorie utilizată pentru păstrarea în siguranță.

Anteturi de trecut : Default este „Toate anteturile”. Această opțiune înlocuiește vechea opțiune „http_anonymizer” cu ceva mult mai configurabil. Acum puteți specifica cu exactitate care anteturi trebuie să fie permise sau care trebuie să fie eliminate din cererile de ieșire.

Există două metode de utilizare a acestei opțiuni. Puteți permite fie anteturi specifice (negând astfel toate celelalte), fie puteți refuza anumite anteturi (permițându-le astfel pe toate celelalte).

User-Agent fals : implicit este „none”. Dacă filtrați antetul User-Agent cu anteturile pentru a trece prin opțiunea aceasta poate determina unele servere Web să refuze solicitarea dvs. Folosiți acest lucru pentru a falsifica unul.

Adresa routerului WCCP : utilizați această opțiune pentru a defini routerul WCCP `` acasă '' pentru Squid.

Adresa de intrare WCCP : utilizați această opțiune dacă doriți să primiți mesaje WCCP pe o singură interfață. NU folosiți această opțiune dacă nu sunteți sigur câte interfețe aveți sau dacă știți că aveți o singură interfață.

Adresa de ieșire WCCP : Utilizați această opțiune dacă doriți ca mesajele WCCP să fie trimise pe o singură interfață. NU folosiți această opțiune dacă nu sunteți sigur câte interfețe aveți sau dacă știți că aveți o singură interfață.
