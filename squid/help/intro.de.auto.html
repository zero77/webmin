Einführung in Squid

Squid ist ein voll funktionsfähiger Web-Caching-Proxy-Server-Daemon. Mit Squid kann Ihr Server lokale Kopien entfernter Webobjekte speichern. Wenn jemand im lokalen Netzwerk erneut auf diese Objekte zugreift, werden die Objekte schneller geladen und verbrauchen nicht mehr Bandbreite, um das Objekt erneut vom Ursprungsserver abzurufen. Dies kann überfüllte Netzwerke entlasten und insgesamt eine reaktionsschnellere Netzwerkumgebung bieten.

Ein weiterer Vorteil der Verwendung von Squid ist die Zugriffskontrolle. Squid bietet umfangreiche Funktionen zum Blockieren des Zugriffs auf bestimmte Websites, Benutzer, Netzwerke, Arten von Dateninhalten usw. Squid kann im transparenten Modus, im herkömmlichen Proxy-Modus und im Reverse-Proxy-Modus (httpd Accelerator) betrieben werden.

Das Tintenfischmodul

Auf der Hauptseite dieses Moduls wird eine Tabelle mit Symbolen angezeigt, mit denen jeweils bestimmte Aspekte von Squid konfiguriert werden können. Bei der ersten Konfiguration von Squid werden alle Werte auf ihre Standardeinstellungen gesetzt. Informationen zu diesen Standardeinstellungen finden Sie auf der Hilfeseite für jede Konfigurationsseite.

Die meisten Hilfeseiten für dieses Modul wurden aus der Standarddatei squid.conf extrahiert, die in jeder Installation von Squid enthalten ist. Weitere Informationen finden Sie auf der Squid Homepage .
