Программы аутентификации

Программа пользовательской аутентификации : по умолчанию нет. Укажите команду для внешнего аутентификатора. Такая программа читает строку, содержащую «пароль пользователя» и отвечает «OK» или «ERR» в бесконечном цикле. Если вы используете аутентификатор, убедитесь, что у вас есть 1 acl типа proxy_auth.

Если вы хотите использовать традиционную аутентификацию прокси-сервера, установите для этой строки что-то вроде /usr/bin/ncsa_auth /usr/etc/passwd. Предполагая, что ncsa_auth был скомпилирован и установлен в вашей системе.

Количество программ аутентификации : по умолчанию 5, если аутентификатор включен.

Время для кеширования паролей : как долго Squid будет кешировать успешный вход в систему, прежде чем снова запрашивать программу аутентификации.

Время связать пользователя с IP-адресом для : Если один и тот же пользователь пытается войти в систему дважды с двух разных IP-адресов в течение этого периода, ему будет отказано. Это может быть использовано для предотвращения совместного использования паролей прокси между несколькими пользователями.

Если вы работаете с версией Squid 2.5 и выше, дополнительные параметры выше и другие могут быть установлены отдельно для разных типов аутентификации, таких как Digest и NTLM. Однако, если вы хотите использовать Webmin для редактирования списка прокси-пользователей, вы должны настроить обычную аутентификацию.
