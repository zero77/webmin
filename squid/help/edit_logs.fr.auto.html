Enregistrement
Fichier journal d'accès : la valeur par défaut dépend du système, généralement quelque part dans / var . Enregistre l'activité de demande du client. Contient une entrée pour chaque requête HTTP et ICP reçue.

Fichier journal de débogage : la valeur par défaut dépend du système. Fichier de journalisation du cache. C'est là que vont les informations générales sur le comportement de votre cache. Vous pouvez augmenter la quantité de données enregistrées dans ce fichier avec les options de débogage .

Fichier journal de stockage : la valeur par défaut dépend du système. Enregistre les activités du gestionnaire de stockage. Indique quels objets sont éjectés du cache, quels objets sont enregistrés et pendant combien de temps. Pour désactiver, entrez "aucun". Il n'y a pas vraiment d'utilitaires pour analyser ces données, vous pouvez donc les désactiver en toute sécurité.

Fichier de métadonnées du cache : la valeur par défaut est swap.log dans chaque répertoire de cache. Ce fichier journal contient les métadonnées des objets enregistrés sur le disque. Il est utilisé pour reconstruire le cache lors du démarrage. Normalement, ce fichier réside dans chaque répertoire de cache, mais vous pouvez spécifier un autre chemin ici. Notez que vous devez donner un nom de fichier complet, pas seulement un répertoire. Comme il s'agit de l'index de la liste complète des objets, vous NE POUVEZ PAS la faire tourner périodiquement!

Si% s peut être utilisé dans le nom du fichier, il sera remplacé par une représentation du nom du répertoire de cache où chaque / est remplacé par '.'. Cela est nécessaire pour permettre l'ajout / la suppression de lignes de répertoire de cache lorsqu'un autre journal d'échange de cache est utilisé.

Si vous disposez de plusieurs répertoires de cache et que% s n'est pas utilisé dans le nom, ces journaux de swap porteront des noms tels que:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

L'extension numérotée (qui est ajoutée automatiquement) correspond à l'ordre des lignes du répertoire de cache dans la configuration. Si vous modifiez l'ordre des lignes du répertoire de cache dans ce fichier, ces fichiers journaux NE correspondront PAS à l'entrée de répertoire de cache correcte (sauf si vous les renommez manuellement). Nous vous recommandons de NE PAS utiliser cette option. Il est préférable de conserver ces fichiers journaux dans chaque répertoire de cache.

Utiliser le format de journal HTTPD : par défaut à non. Le cache peut émuler le format de fichier journal utilisé par de nombreux programmes «httpd». Pour désactiver / activer cette émulation, définissez cette valeur sur off ou on. La valeur par défaut est d'utiliser le format de journal natif car il contient des informations utiles que les analyseurs de journaux spécifiques à Squid utilisent.

Enregistrer les en-têtes MIME : par défaut, non. Le cache peut enregistrer à la fois les en-têtes MIME de la demande et de la réponse pour chaque transaction HTTP. Les en-têtes sont encodés en toute sécurité et apparaîtront sous la forme de deux champs entre crochets à la fin du journal d'accès (pour les formats de journal natif ou émulé httpd). Pour activer cette journalisation, activez-la.

Fichier journal de l'agent utilisateur : par défaut, aucun. S'il est configuré avec l'option de configuration "--enable-useragent-log", Squid écrira le champ User-Agent des requêtes HTTP au nom de fichier spécifié ici.

Fichier PID : la valeur par défaut dépend du système. Un nom de fichier pour écrire l'ID de processus. Pour désactiver, entrez "aucun".

Effectuer des recherches d'identification RFC931 pour les ACL : la valeur par défaut est aucune. Effectuez des recherches d'identification pour trouver le nom d'utilisateur des clients accédant à quoi que ce soit dans ces ACL.

RFC931 ident timeout : la valeur par défaut est 30 secondes. Temps maximum d'attente pour les requêtes IDENT. Si cette valeur est trop élevée et que vous avez activé toutes les recherches d'identifiant, vous risquez d'être sensible au déni de service en effectuant de nombreuses demandes d'identification simultanément.

Enregistrer les noms d'hôtes complets : la valeur par défaut est non. Enregistre le nom d'hôte complet des clients.

Masque de journalisation : la valeur par défaut est 255.255.255.255. Un masque de réseau pour les adresses des clients dans les fichiers journaux et la sortie cachemgr. Modifiez cela pour protéger la confidentialité de vos clients de cache. Un masque de réseau de 255.255.255.0 enregistrera toutes les adresses IP de cette plage avec le dernier chiffre défini sur «0».

Options de débogage : la valeur par défaut est "ALL, 1". Les options de journalisation sont définies comme section, niveau auquel chaque fichier source se voit attribuer une section unique. Des niveaux inférieurs entraînent moins de débogage complet (niveau 9) peut entraîner un fichier journal très volumineux, alors soyez prudent. Le mot magique "ALL" définit les niveaux de débogage pour toutes les sections. Nous recommandons de fonctionner normalement avec "ALL, 1".

Table des en-têtes MIME : chemin d'accès à la table MIME de Squid. Vous ne devriez pas avoir besoin de changer, mais le fichier par défaut contient des exemples et des informations de mise en forme si vous le faites.
