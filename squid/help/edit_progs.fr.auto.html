Programmes d'assistance
Largeur de colonne FTP : la valeur par défaut est 32. Définit la largeur des listes ftp. Cela devrait être réglé pour s'adapter à la largeur d'un navigateur standard. La définition de cette valeur trop petite peut couper les noms de fichiers longs lors de la navigation sur des sites ftp.

Connexion FTP anon : la valeur par défaut est «Squid @». Si vous souhaitez que le mot de passe de connexion anonyme soit plus informatif (et activez l'utilisation de serveurs ftp pointilleux), définissez-le sur quelque chose de résonnable pour votre domaine, comme wwwuser@somewhere.net.

La raison pour laquelle cela est indémodable par défaut est que la demande peut être effectuée au nom d'un utilisateur dans n'importe quel domaine, selon la façon dont le cache est utilisé. Certains serveurs ftp vérifient également que l'adresse e-mail est valide (par exemple perl.com).

Programme DNS Squid : La valeur par défaut est dnsserver. Spécifiez l'emplacement de l'exécutable pour le processus dnslookup. La valeur par défaut est un serveur de noms de mise en cache non bloquant qui est installé avec Squid par défaut.

Nombre de programmes DNS : la valeur par défaut est 5. Nombre de processus générés pour gérer les recherches de nom DNS. Pour les caches lourdement chargés sur les grands serveurs, vous devez probablement augmenter cette valeur à au moins 10. Le maximum est 32.

Ajouter un domaine aux demandes : la valeur par défaut est désactivée. Normalement, le 'dnsserver' désactive l'option de résolveur RES_DEFNAMES (voir res_init (3)). Cela empêche les caches d'une hiérarchie d'interpréter localement les noms d'hôte à un seul composant. Pour permettre à dnsserver de gérer les noms à composant unique, activez cette option.

Adresses de serveur DNS : la valeur par défaut consiste à utiliser le fichier resolv.conf. Utilisez-le si vous souhaitez spécifier une liste de serveurs de noms DNS (adresses IP) à utiliser à la place de ceux indiqués dans votre fichier /etc/resolv.conf.

Programme d'effacement du cache : la valeur par défaut est «unlinkd». Spécifiez l'emplacement de l'exécutable pour le processus de suppression de fichiers. Cela peut ne pas être nécessaire si vous utilisez async-io car il est géré par un thread dans certaines versions.

Programme de squid ping : par défaut «pinger». Spécifiez l'emplacement de l'exécutable pour le processus Pinger. Ceci n'est utile que si vous avez configuré Squid (pendant la compliation) avec l'option '--enable-icmp'.

Programme de redirection personnalisé : par défaut, aucun. Spécifiez l'emplacement de l'exécutable pour le redirecteur d'URL. Puisqu'ils peuvent exécuter presque toutes les fonctions, aucune n'est incluse. Voir les Release-Notes pour savoir comment en écrire un.

Nombre de programmes de redirection : la valeur par défaut est 5 si un redirecteur est configuré. Nombre de processus de redirection à générer. Si vous démarrez trop peu de Squid, vous devrez attendre qu'ils traitent un backlog d'URL, le ralentissant. Si vous en commencez trop, ils utiliseront la RAM et d'autres ressources système.
