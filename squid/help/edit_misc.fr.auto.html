Options diverses
Adresses de test DNS de démarrage : serveurs DNS que Squid interrogera pour tester si les recherches DNS fonctionnent correctement.

Rotations du fichier journal SIGUSR1 : par défaut 10 ou 0, selon le système. Spécifie le nombre de rotations du fichier journal à effectuer lorsque vous tapez «Squid -k rotation». La valeur par défaut est 10, qui tournera avec les extensions 0 à 9. La définition de logfile_rotate sur 0 désactivera la rotation, mais les fichiers journaux sont toujours fermés et rouverts. Cela vous permettra de renommer les fichiers journaux vous-même juste avant d'envoyer le signal de rotation.

Notez que la commande 'squid -k rotation' envoie normalement un signal USR1 au processus de squid en cours d'exécution. Dans certaines situations (par exemple sous Linux avec Async I / O), USR1 est utilisé à d'autres fins, donc -k rotation utilise un autre signal. Il est préférable de prendre l'habitude d'utiliser 'squid -k rotation' au lieu de 'kill -USR1 ".

Texte des messages d'erreur : texte HTML à inclure dans les messages d'erreur. Faites-en une URL "mailto" vers votre adresse administrateur, ou peut-être simplement un lien vers la page Web de votre organisation.

Pour l'inclure dans vos messages d'erreur, vous devez réécrire les fichiers de modèle d'erreur (qui se trouvent dans le répertoire "erreurs"). Partout où vous souhaitez que la ligne «err_html_text» apparaisse, insérez une balise% L dans le fichier de modèle d'erreur.

Statistiques par client : si vous souhaitez désactiver la collecte de statistiques par client, désactivez client_db ici.

En-tête X-Forwarded-For : La valeur par défaut est «on». S'il est défini, Squid inclura l'adresse IP ou le nom de votre système dans les requêtes HTTP qu'il transmet. Par défaut, il ressemble aux requêtes HTTP qu'il transfère. Par défaut, cela ressemble à ceci:

X-Forwarded-For: 192.1.2.3

Si vous désactivez cela, il apparaîtra comme:

X-Forwarded-For: inconnu

Consigner les requêtes ICP : la valeur par défaut est «on». S'il est défini, les requêtes ICP sont enregistrées dans access.log. Vous pouvez désactiver cette option si votre charge ICP est TRÈS élevée pour accélérer les choses ou pour simplifier l'analyse des journaux.

Sauts directs minimum : la valeur par défaut est 4. Si vous utilisez le ping ICMP, effectuez des récupérations directes pour les sites qui ne sont pas plus éloignés que de nombreux sauts.

Conservez la mémoire pour une utilisation future : la valeur par défaut est «oui». S'il est défini, Squid gardera des pools de mémoire allouée (mais inutilisée) disponibles pour une utilisation future. Si la mémoire est une prime sur votre système et que vous pensez que votre bibliothèque malloc surpasse les routines Squid, désactivez-la.

Quantité de mémoire à conserver : la valeur par défaut est «Illimitée». S'il est défini sur une valeur non nulle, Squid conservera au maximum la limite spécifiée de mémoire allouée (mais inutilisée) dans les pools de mémoire. Toutes les demandes free () qui dépassent cette limite seront traitées par votre bibliothèque malloc. Squid ne pré-alloue aucune mémoire, juste des objets sûrs qui autrement seraient libres () d. Ainsi, il est sûr de définir memory_pools_limit à une valeur raisonnablement élevée même si votre configuration utilise moins de mémoire.

S'il n'est pas défini (par défaut) ou réglé sur zéro, Squid gardera toute la mémoire qu'il peut. Autrement dit, il n'y aura pas de limite sur la quantité totale de mémoire utilisée pour la sauvegarde.

En-têtes à passer : la valeur par défaut est «Tous les en-têtes». Cette option remplace l'ancienne option 'http_anonymizer' par quelque chose de beaucoup plus configurable. Vous pouvez maintenant spécifier exactement quels en-têtes doivent être autorisés ou qui doivent être supprimés des demandes sortantes.

Il existe deux méthodes d'utilisation de cette option. Vous pouvez soit autoriser des en-têtes spécifiques (refusant ainsi tous les autres), soit refuser des en-têtes spécifiques (autorisant ainsi tous les autres).

Faux utilisateur-agent : la valeur par défaut est «aucune». Si vous filtrez l'en-tête User-Agent avec l'option Headers to pass through, certains serveurs Web peuvent refuser votre demande. Utilisez-le pour en simuler un.

Adresse du routeur WCCP : utilisez cette option pour définir votre routeur `` domestique '' WCCP pour Squid.

Adresse entrante WCCP : utilisez cette option si vous souhaitez que les messages WCCP soient reçus sur une seule interface. N'utilisez PAS cette option si vous n'êtes pas sûr du nombre d'interfaces dont vous disposez ou si vous savez que vous n'avez qu'une seule interface.

Adresse sortante WCCP : utilisez cette option si vous souhaitez que les messages WCCP soient envoyés sur une seule interface. N'utilisez PAS cette option si vous n'êtes pas sûr du nombre d'interfaces dont vous disposez ou si vous savez que vous n'avez qu'une seule interface.
