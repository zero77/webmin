Verzögerungspool bearbeiten

Poolnummer
Dies ist eine Zahl ab 1, die von Webmin für Sie ausgewählt wird. Der Pool wird einfach so gekennzeichnet, dass mehr als ein Pool von Squid unterschiedlich konfiguriert und behandelt werden kann.

Poolklasse
Dies definiert den Klassentyp des Pools. Für jeden Verzögerungspool kann eine Klasse aus den Klassen 1, 2 oder 3 ausgewählt werden. Die Klassen haben die folgenden Bedeutungen: "1-Aggregat", "2-Aggregat und Individuum", "3-Aggregat, Individuum und Netzwerk". Die einfachste Klasse 1 gibt eine Bandbreite an, die alle Clients, zu denen dieser Pool gehört, nicht überschreiten können. Klasse 2 gibt sowohl eine Gesamtbandbreite als auch eine Bandbreite an, die jede Person verwenden kann, wodurch eine Hierarchie von Grenzwerten erstellt wird, bei der alle Personen zusammen in das Aggregat passen müssen, während jede Person durch die Einzelgrenze begrenzt ist. Schließlich gibt Klasse 3 zusätzlich zu einer Netzwerkbandbreite, die für ein gesamtes Subnetz gilt, eine aggregierte und eine einzelne Bandbreite an. Klasse 3 ist normalerweise nur in sehr großen Netzwerken mit mehr als einem Netzwerk-Subnetz nützlich.

Grenzwerte können in jeder bevorzugten Einheit angegeben werden, sie werden jedoch in Byte in der Squid-Konfigurationsdatei gespeichert. Das erste Feld ist der durchschnittliche Durchsatz, der dem Pool zugewiesen wird, während das zweite die Größe des verwendeten Buckets ist, was ungefähr einem Burstable-Bandbreitenmaximum entspricht.

Nach dem erstmaligen Erstellen eines Pools wird unten auf der Seite ein neuer Abschnitt angezeigt, in dem Sie auswählen können, auf welche ACLs der Pool angewendet werden soll. ACLs werden auf der Seite Zugriffssteuerung erstellt , und mit jeder ACL kann ausgewählt werden, auf welche Verbindungen ein Verzögerungspool angewendet werden soll. Durch das Zulassen einer ACL wird der Verzögerungspool auf die übereinstimmenden Verbindungen angewendet , während durch das Verweigern einer ACL übereinstimmende Verbindungen den Verzögerungspool umgehen. Sofern nicht auf alle Verbindungen derselbe Verzögerungspool angewendet wird, möchten Sie zwei ACLs hinzufügen. Die erste ist die ACL, auf die Sie den Verzögerungspool anwenden möchten, und die zweite ist die gesamte ACL dass nur Ihre ausgewählten Kunden eingeschränkt werden.

Vergessen Sie nicht, Änderungen anzuwenden, nachdem Sie einen neuen Pool erstellt und entsprechende ACLs hinzugefügt haben.
