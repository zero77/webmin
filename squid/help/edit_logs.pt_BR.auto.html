Exploração madeireira
Arquivo de log de acesso : o padrão depende do sistema, geralmente em algum lugar em / var . Registra a atividade de solicitação do cliente. Contém uma entrada para cada solicitação de HTTP e ICP recebida.

Arquivo de log de depuração : o padrão depende do sistema. Arquivo de log de cache. É aqui que as informações gerais sobre o comportamento do seu cache vão. Você pode aumentar a quantidade de dados registrados neste arquivo com as opções de Depuração .

Arquivo de log de armazenamento : O padrão depende do sistema. Registra as atividades do gerenciador de armazenamento. Mostra quais objetos são ejetados do cache e quais objetos são salvos e por quanto tempo. Para desativar, digite "nenhum". Na verdade, não existem utilitários para analisar esses dados, portanto você pode desativá-los com segurança.

Arquivo de metadados do cache : o padrão é swap.log em cada diretório de cache. Este arquivo de log contém os metadados dos objetos salvos no disco. É usado para reconstruir o cache durante a inicialização. Normalmente, esse arquivo reside em cada diretório de cache, mas você pode especificar um nome de caminho alternativo aqui. Observe que você deve fornecer um nome de arquivo completo, não apenas um diretório. Como esse é o índice de toda a lista de objetos, você NÃO pode girá-lo periodicamente!

Se% s puder ser usado no nome do arquivo, ele será substituído por uma representação do nome do diretório de cache em que cada / é substituído por '.'. Isso é necessário para permitir adicionar / remover linhas de diretório de cache quando um log alternativo de troca de cache estiver sendo usado.

Se tiver mais de um diretório de cache e% s não for usado no nome, esses logs de troca terão nomes como:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

A extensão numerada (que é adicionada automaticamente) corresponde à ordem das linhas do diretório de cache na configuração. Se você alterar a ordem das linhas do diretório de cache nesse arquivo, esses arquivos de log NÃO corresponderão à entrada correta do diretório de cache (a menos que você os renomeie manualmente). Recomendamos que você NÃO use esta opção. É melhor manter esses arquivos de log em cada diretório de cache.

Use o formato de log HTTPD : O padrão é não. O cache pode emular o formato do arquivo de log usado por muitos programas 'httpd'. Para desativar / ativar esta emulação, defina esse valor como desativado ou ativado. O padrão é usar o formato de log nativo, pois inclui informações úteis que os analisadores de log específicos do Squid usam.

Registrar cabeçalhos MIME : O padrão é não. O cache pode registrar os cabeçalhos MIME da solicitação e da resposta para cada transação HTTP. Os cabeçalhos são codificados com segurança e aparecerão como dois campos entre colchetes no final do log de acesso (para os formatos de log nativo ou emulado por httpd). Para habilitar esse log, configure-o como ativado.

Arquivo de log do agente do usuário : O padrão é nenhum. Se configurado com a opção de configuração "--enable-useragent-log", o Squid gravará o campo User-Agent de pedidos HTTP no nome do arquivo especificado aqui.

Arquivo PID : o padrão depende do sistema. Um nome de arquivo no qual escrever a identificação do processo. Para desativar, digite "nenhum".

Execute pesquisas de identificação do RFC931 para ACLs : o padrão é nenhum. Execute pesquisas de identificação para encontrar o nome de usuário dos clientes que acessam qualquer coisa nessas ACLs.

RFC931 ident timeout : O padrão é 30 segundos. Tempo máximo para aguardar solicitações de IDENT. Se isso for muito alto e você ativou qualquer pesquisa de identificação, pode ser suscetível à negação de serviço tendo muitas solicitações de identificação acontecendo ao mesmo tempo.

Registrar nomes de host completos : o padrão é não. Registra o nome completo do host dos clientes.

Máscara de rede de log : o padrão é 255.255.255.255. Uma máscara de rede para endereços de clientes em arquivos de log e saída cachemgr. Altere isso para proteger a privacidade dos seus clientes de cache. Uma máscara de rede 255.255.255.0 registrará todos os IPs nesse intervalo com o último dígito definido como '0'.

Opções de depuração : o padrão é "ALL, 1". As opções de log são definidas como seção, nível em que cada arquivo de origem recebe uma seção exclusiva. Níveis mais baixos resultam em menos Depuração completa (nível 9) pode resultar em um arquivo de log muito grande, portanto, tenha cuidado. A palavra mágica "ALL" define os níveis de depuração para todas as seções. Recomendamos que você execute normalmente com "ALL, 1".

Tabela de cabeçalhos MIME : Nome do caminho da tabela MIME do Squid. Você não precisa alterar, mas o arquivo padrão contém exemplos e informações de formatação, se o fizer.
