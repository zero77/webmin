메모리 사용량
메모리 사용 제한 : 기본값은 8MB입니다.

참고 :이 매개 변수는 최대 프로세스 크기를 지정하지 않습니다. Squid의 메모리 사용량의 한 측면에 제한이 있습니다. 오징어는 다른 것들에도 메모리를 사용합니다. 당신의 프로세스는 아마도 당신이 여기에 넣은 값의 두 배나 세 배가 될 것입니다.

다음에 사용할 이상적인 메모리 양을 지정합니다.

  • 운송 중 개체
  • 뜨거운 물체
  • 부정적인 캐치 객체

이러한 객체의 데이터는 4 kB 블록에 저장됩니다. 이 매개 변수는 할당 된 총 4 kB 블록의 이상적인 상한을 지정합니다. 전송 중 개체가 가장 우선합니다.

전송 중 개체가 다른 개체보다 우선합니다. 들어오는 데이터에 추가 공간이 필요한 경우 네거티브 캐시 및 핫 개체가 해제됩니다. 다시 말해서, 네거티브 캐싱 및 핫 오브젝트는 전송중인 오브젝트에 필요하지 않은 사용되지 않은 공간을 채 웁니다.

상황에 따라이 제한이 초과됩니다. 특히 들어오는 요청 속도가 전송중인 개체를 보유하기 위해이 메모리 양보다 많은 메모리를 필요로하는 경우 Squid는 새 요청을 충족시키기 위해이 제한을 초과합니다. 부하가 감소하면 최고 사용 표시 점에 도달 할 때까지 블록이 해제됩니다. 그 후, 블록은 뜨거운 물체를 저장하는 데 사용됩니다.

FQDN 캐시 크기 : 기본값은 1024입니다. 최대 정규화 된 도메인 이름 캐시 항목 수입니다.

메모리 최고 사용 표시 점 : 기본값은 95 %입니다. 이 높은 점수에 도달하면 뜨거운 물체와 운송중인 물체가 해제되어 메모리를 비 웁니다.

메모리 부족 워터 마크 : 기본값은 90 %입니다. 최고 사용 표시 점에 도달하면 Squid는 사용량이이 표시 아래로 떨어질 때까지 메모리를 비 웁니다.

디스크 최고 사용 표시 점 : 기본값은 95 %입니다. 이 높은 표시에 도달하면 Squid는 적극적으로 디스크를 플러시하기 시작합니다.

디스크 부족 워터 마크 : 기본값은 90 %입니다. 최고 사용 표시 점에 도달하면 Squid는 사용량이이 표시 아래로 떨어질 때까지 디스크를 계속 플러시합니다.

캐시 된 최대 개체 크기 : 기본값은 4096 kB입니다. 이 설정은 캐시에 의해 저장 될 객체의 크기를 제한합니다. 이 크기를 초과하는 것은 캐시되지 않으며 매번 오리진 서버에서로드되어야합니다.

IP 주소 캐시 크기 : 기본값은 1024입니다. IP 캐시에 저장 될 항목 수입니다.

IP 캐시 최고 수위 표시 : 기본값은 95 %입니다. Squid가 IP 캐시 항목을 플러시하기 시작하는 지점.

IP 캐시 최저 워터 마크 : 기본값은 90 %입니다. IP 캐시 플러시가 시작된 후에는 IP 캐시의 항목 수가이 백분율 아래로 떨어질 때까지 계속됩니다.
