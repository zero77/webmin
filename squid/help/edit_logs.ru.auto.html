логирование
Доступ к файлу журнала : по умолчанию зависит от системы, обычно где-то в / var . Регистрирует активность клиентских запросов. Содержит запись для каждого полученного запроса HTTP и ICP.

Файл журнала отладки : по умолчанию зависит от системы. Файл журнала кэша. Вот куда идет общая информация о поведении вашего кэша. Вы можете увеличить объем данных, записываемых в этот файл, с помощью параметров отладки .

Файл журнала хранилища : по умолчанию зависит от системы. Регистрирует действия менеджера хранилища. Показывает, какие объекты извлекаются из кэша, какие объекты сохраняются и как долго. Чтобы отключить, введите «нет». На самом деле нет никаких утилит для анализа этих данных, поэтому вы можете безопасно отключить их.

Файл метаданных кэша : по умолчанию используется swap.log в каждом каталоге кэша. Этот файл журнала содержит метаданные объектов, сохраненных на диске. Он используется для восстановления кеша при запуске. Обычно этот файл находится в каждом каталоге кеша, но вы можете указать здесь альтернативный путь. Обратите внимание, что вы должны указать полное имя файла, а не просто каталог. Поскольку это индекс для всего списка объектов, вы НЕ МОЖЕТЕ периодически поворачивать его!

Если в имени файла можно использовать% s, то оно будет заменено на представление имени каталога кеша, где каждый / заменяется на «.». Это необходимо для того, чтобы разрешить добавление / удаление строк каталога кэша, когда используется альтернативный журнал обмена кэшем.

Если у вас есть более одного каталога кэша, и% s не используется в имени, тогда эти журналы подкачки будут иметь такие имена, как:

  • cache_swap_log.00
  • cache_swap_log.01
  • cache_swap_log.02

Пронумерованное расширение (которое добавляется автоматически) соответствует порядку строк каталога кэша в конфигурации. Если вы измените порядок строк каталога кэша в этом файле, то эти файлы журнала НЕ будут соответствовать правильной записи каталога кэша (если вы не переименуете их вручную). Мы рекомендуем вам НЕ использовать эту опцию. Эти файлы журналов лучше хранить в каждом каталоге кеша.

Использовать формат журнала HTTPD : по умолчанию нет. Кэш может эмулировать формат файла журнала, который используют многие программы httpd. Чтобы отключить / включить эту эмуляцию, установите для этого значения значение off или on. По умолчанию используется собственный формат журнала, поскольку он содержит полезную информацию, которую используют специфичные для Squid анализаторы журнала.

Журналы MIME заголовков : по умолчанию нет. Кэш может записывать заголовки MIME запроса и ответа для каждой транзакции HTTP. Заголовки надежно кодируются и отображаются в виде двух заключенных в квадратные скобки полей в конце журнала доступа (как для собственного, так и для http-эмулируемого формата журнала). Чтобы включить это ведение журнала, установите его на.

Файл журнала агента пользователя : по умолчанию нет. При настройке с параметром конфигурации --enable-useragent-log Squid записывает поле User-Agent из HTTP-запросов в указанное здесь имя файла.

Файл PID : по умолчанию зависит от системы. Имя файла для записи идентификатора процесса. Чтобы отключить, введите «нет».

Выполните RFC931 для поиска ACL : по умолчанию нет. Выполните поиск идентификаторов, чтобы найти имя пользователя клиентов, обращающихся к чему-либо в этих списках ACL.

Тайм-аут RFC931 : по умолчанию 30 секунд. Максимальное время ожидания запросов IDENT. Если это слишком много и вы включили какие-либо поиски идентификаторов, то вы можете быть подвержены отказу в обслуживании, если одновременно выполняется много запросов на идентификацию.

Записать полные имена хостов : по умолчанию нет. Регистрирует полное имя хоста клиентов.

Маска ведения журнала : по умолчанию 255.255.255.255. Сетевая маска для клиентских адресов в лог-файлах и выходных данных cachemgr. Измените это, чтобы защитить конфиденциальность ваших клиентов кеша. Маска сети 255.255.255.0 будет регистрировать все IP-адреса в этом диапазоне с последней цифрой, установленной на «0».

Параметры отладки : по умолчанию «ВСЕ, 1». Параметры ведения журнала задаются как раздел, уровень, где каждому исходному файлу назначается уникальный раздел. Более низкие уровни приводят к меньшему. Полная отладка (уровень 9) может привести к очень большому файлу журнала, поэтому будьте осторожны. Волшебное слово «ВСЕ» устанавливает уровни отладки для всех разделов. Мы рекомендуем нормально работать с "ALL, 1".

Таблица заголовков MIME : Путь к таблице MIME Squid. Вам не нужно вносить изменения, но файл по умолчанию содержит примеры и информацию о форматировании, если вы это делаете.
