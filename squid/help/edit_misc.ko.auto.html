기타 옵션
시작 DNS 테스트 주소 : Squid가 DNS 조회가 올바르게 작동하는지 테스트하기 위해 쿼리 할 DNS 서버.

SIGUSR1 로그 파일 회전 : 시스템에 따라 기본값은 10 또는 0입니다. 'squid -k rotate'를 입력 할 때 수행 할 로그 파일 회전 수를 지정합니다. 기본값은 10이며 확장자는 0-9로 회전합니다. logfile_rotate를 0으로 설정하면 회전이 비활성화되지만 로그 파일은 여전히 닫히고 다시 열립니다. 회전 신호를 보내기 직전에 로그 파일 이름을 직접 바꿀 수 있습니다.

'squid -k rotate'명령은 일반적으로 USR1 신호를 실행중인 오징어 프로세스로 보냅니다. 특정 상황 (예 : 비동기 I / O가있는 Linux)에서는 USR1이 다른 용도로 사용되므로 -k rotate는 다른 신호를 사용합니다. 'kill -USR1 대신'squid -k rotate '를 사용하는 습관을들이는 것이 가장 좋습니다. '.

오류 메시지 텍스트 : 오류 메시지 에 포함 할 HTML 텍스트입니다. 이 주소를 관리자 주소의 "mailto"URL 또는 조직 웹 페이지의 링크로 만드십시오.

이것을 오류 메시지에 포함 시키려면 오류 템플릿 파일 ( "오류"디렉토리에 있음)을 다시 작성해야합니다. 'err_html_text'줄을 표시하려는 경우 오류 템플리트 파일에 % L 태그를 삽입하십시오.

클라이언트 별 통계 : 클라이언트 별 통계 수집을 사용하지 않으려면 여기서 client_db를 끄십시오.

X-Forwarded-For 헤더 : 기본값은 'on'입니다. 설정된 경우 Squid는 시스템의 IP 주소 또는 이름을 전달하는 HTTP 요청에 포함시킵니다. 기본적으로 전달되는 HTTP 요청과 같습니다. 기본적으로 다음과 같습니다.

X- 전달 : 192.1.2.3

이 기능을 비활성화하면 다음과 같이 나타납니다.

X-Forwarded-For : 알 수 없음

ICP 쿼리 기록 : 기본값은 '설정'입니다. 설정하면 ICP 쿼리가 access.log에 기록됩니다. 작업 속도를 높이거나 로그 분석을 단순화하기 위해 ICP로드가 매우 높은 경우이 기능을 비활성화 할 수 있습니다.

최소 직접 홉 : 기본값은 4입니다. ICMP 핑 항목을 사용하는 경우이 홉 수를 넘지 않는 사이트에 대해 직접 가져 오기를 수행하십시오.

나중에 사용할 수 있도록 메모리를 유지하십시오 . 기본값은 '예'입니다. 설정된 경우 Squid는 나중에 사용할 수 있도록 할당 된 (그러나 사용되지 않은) 메모리 풀을 계속 사용할 수있게합니다. 메모리가 시스템에서 프리미엄이며 malloc 라이브러리가 Squid 루틴보다 성능이 우수하다고 생각되면이를 비활성화하십시오.

보관할 메모리 용량 : 기본값은 '무제한'입니다. 0이 아닌 값으로 설정하면 Squid는 메모리 풀에서 할당 된 (그러나 사용되지 않은) 메모리의 지정된 제한을 최대한 유지합니다. 이 제한을 초과하는 모든 free () 요청은 malloc 라이브러리에서 처리합니다. Squid는 메모리를 미리 할당하지 않고 free () 된 객체를 안전하게 유지합니다. 따라서 구성에서 적은 메모리를 사용하더라도 memory_pools_limit를 상당히 높은 값으로 설정하는 것이 안전합니다.

설정하지 않거나 (기본값) 0으로 설정하면 Squid는 가능한 모든 메모리를 유지합니다. 즉, 안전 유지에 사용되는 총 메모리 양에는 제한이 없습니다.

통과 할 헤더 : 기본값은 '모든 헤더'입니다. 이 옵션은 이전 'http_anonymizer'옵션을 훨씬 더 구성 가능한 것으로 대체합니다. 이제 어떤 헤더를 허용하거나 발신 요청에서 제거할지 정확하게 지정할 수 있습니다.

이 옵션을 사용하는 방법에는 두 가지가 있습니다. 특정 헤더를 허용하거나 (따라서 모든 헤더를 거부 함) 특정 헤더를 거부 할 수 있습니다 (따라서 다른 헤더를 허용 함).

가짜 사용자 에이전트 : 기본값은 '없음'입니다. 통과 할 헤더 옵션으로 사용자 에이전트 헤더를 필터링하면 일부 웹 서버가 요청을 거부 할 수 있습니다. 이것을 사용하여 하나를 위조하십시오.

WCCP 라우터 주소 :이 옵션을 사용하여 오징어에 대한 WCCP``홈 ''라우터를 정의하십시오.

WCCP 수신 주소 : WCCP 메시지를 하나의 인터페이스에서만 수신해야하는 경우이 옵션을 사용하십시오. 사용중인 인터페이스 수를 잘 모르거나 인터페이스가 하나만있는 경우이 옵션을 사용하지 마십시오.

WCCP 발신 주소 : WCCP 메시지를 하나의 인터페이스에서만 발송해야하는 경우이 옵션을 사용하십시오. 사용중인 인터페이스 수를 잘 모르거나 인터페이스가 하나만있는 경우이 옵션을 사용하지 마십시오.
