Параметры кэша
Каталоги кеша : Вы можете указать несколько строк каталогов кеша для распределения кеша между различными разделами диска.

Каталог - это каталог верхнего уровня, в котором будут храниться файлы подкачки кэша. Если вы хотите использовать весь диск для кэширования, это может быть каталог точки монтирования. Каталог должен существовать и быть доступным для записи процессом Squid. Squid НЕ создаст этот каталог для вас.

Размер - это объем дискового пространства (МБ), используемого в этом каталоге. По умолчанию установлено 100 МБ. Измените это в соответствии с вашей конфигурацией.

DIR 1-го уровня - это количество подкаталогов первого уровня, которые будут созданы в каталоге . По умолчанию это 16.

Директории 2-го уровня - это количество подкаталогов второго уровня, которые будут создаваться в каждом каталоге первого уровня. По умолчанию 256.

Средний размер объекта : по умолчанию 13 кБ. Средний размер объекта, используемый для оценки количества объектов, которые может хранить ваш кеш.

Objects per bucket : по умолчанию 50. Целевое количество объектов на корзину в хэш-таблице магазина. Понижение этого значения увеличивает общее количество сегментов, а также скорость обслуживания хранилища.

Не кэшировать URL для списков ACL : по умолчанию нет. Выберите любые ACL, для которых вы не хотите, чтобы Squid кешировал данные. Полезно для определения внутреннего веб-сервера, который не должен кэшироваться.

Максимальное время кеширования : по умолчанию 1 год. Этот параметр определяет максимальный возраст LRU. Например, установка «1 неделя» приведет к удалению объектов, если к ним не обращались в течение недели или дольше.

Время кэширования невыполненных запросов По умолчанию 5 минут. Время жизни (TTL) для неудачных запросов. Некоторые типы сбоев (например, «отказано в соединении» и «404 не найден») кешируются отрицательно в течение настраиваемого промежутка времени. Обратите внимание, что это отличается от негативного кэширования DNS-запросов.

Время поиска в DNS : по умолчанию 6 часов (360 минут). Время жизни (TTL) для положительного кэширования успешных поисков DNS. Если вы хотите свести к минимуму использование ipcache в Squid, установите значение 1, а не 0.

Ошибка кэширования DNS : по умолчанию 5 минут. Время жизни (TTL) для отрицательного кэширования неудачных поисков DNS.

Время ожидания подключения : по умолчанию 120 секунд. На некоторые системы (особенно Linux) нельзя полагаться на правильное время ожидания запросов connect (2). Поэтому процесс Squid устанавливает свое собственное время ожидания для соединений с сервером. Этот параметр указывает, как долго ждать завершения соединения.

Тайм-аут чтения : по умолчанию 15 минут. Read_timeout применяется к соединениям на стороне сервера. После каждого успешного чтения () время ожидания увеличивается на эту величину. Если по истечении этого промежутка времени данные не считываются снова, запрос прерывается и регистрируется с помощью ERR_READ_TIMEOUT.

Тайм-аут выбора сайта : по умолчанию 4 секунды. Для URN для выбора нескольких URL-адресов.

Время ожидания запроса клиента : по умолчанию 15 минут. Read_timeout применяется к соединениям на стороне сервера. После каждого успешного чтения () время ожидания увеличивается на эту величину. Если по истечении этого промежутка времени данные не считываются снова, запрос прерывается и регистрируется с помощью ERR_READ_TIMEOUT.

Максимальное время подключения клиента : по умолчанию 1 день. Максимальное время, в течение которого клиент (браузер) может оставаться подключенным к процессу кэширования. Это защищает кэш от множества сокетов (и, следовательно, файловых дескрипторов), связанных в состоянии CLOSE_WAIT от удаленных клиентов, которые уходят без надлежащего завершения работы (либо из-за сбоя в сети, либо из-за плохой реализации клиента).

Максимальное время выключения : по умолчанию 30 секунд. При получении SIGTERM или SIGHUP кэш переводится в режим ожидания завершения работы до тех пор, пока все активные сокеты не будут закрыты. Это значение - время жизни, которое нужно установить для всех открытых дескрипторов в режиме выключения. Любые активные клиенты через столько секунд получат сообщение «Тайм-аут».

Полузакрытые клиенты : по умолчанию включено. Некоторые клиенты могут отключить отправляющую сторону своих TCP-соединений, оставив принимающие стороны открытыми. Иногда Squid не может определить разницу между полузакрытым и полностью закрытым TCP-соединением. По умолчанию полузакрытые клиентские подключения остаются открытыми до тех пор, пока read (2) или write (2) в сокете не вернут ошибку. Измените эту опцию на «off», и Squid немедленно закроет клиентские соединения, когда read (2) вернет «нет данных для чтения».

Постоянное время ожидания : по умолчанию 120 секунд. Тайм-аут для бездействующих постоянных соединений с клиентами, серверами и другими прокси. Постоянные соединения будут отключены, если это менее 10 секунд.

Узел ретрансляции WAIS : по умолчанию нет. Имя хоста или адрес для передачи запросов WAIS.

Порт реле WAIS : по умолчанию нет. Порт для прямой ретрансляции запросов WAIS на.
