Detectorul de intruziune Fail2Ban
Fail2Ban este un server care scanează fișierele de jurnal pentru înregistrările care indică autentificări eșuate sau alte atacuri, apoi efectuează acțiuni cum ar fi firewalling sau blochează altfel sursele acelor atacuri. Aceasta poate fi folosită pentru a preveni tentativele de ghicire a parolei cu forță brută, blocând atacatorul înainte de a putea încerca o gamă largă de parole.

Cele trei tipuri majore de obiecte de configurare din Fail2Ban sunt:

Filtre jurnal
Un filtru este practic o expresie obișnuită care poate fi aplicată unui fișier jurnal pentru a se potrivi cu autentificările eșuate sau cu alte atacuri.

Acțiuni de potrivire
O acțiune este un set de comenzi care sunt rulate pentru a bloca un atac. O acțiune poate defini, de asemenea, comenzile de deblocare a unui IP și comenzile care sunt executate atunci când Fail2Ban pornește sau oprește.

Filtrele de acțiune ale filtrului
Închisoarea este o combinație între un filtru, una sau mai multe acțiuni și unul sau mai multe fișiere de jurnal. Jurnalul este scanat continuu pentru liniile care se potrivesc filtrului și atunci când se găsește, se efectuează acțiunile selectate.
