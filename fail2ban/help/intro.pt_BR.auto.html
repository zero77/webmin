Detector de intrusão Fail2Ban
Fail2Ban é um servidor que verifica os arquivos de log em busca de entradas indicando logins com falha ou outros ataques e, em seguida, executa ações como firewall ou bloqueio das fontes desses ataques. Isso pode ser usado para impedir tentativas de adivinhação de senha de força bruta, bloqueando o invasor antes que ele possa tentar uma ampla variedade de senhas.

Os três principais tipos de objetos de configuração no Fail2Ban são:

Filtros de registro
Um filtro é basicamente uma expressão regular que pode ser aplicada a um arquivo de log para corresponder a logins com falha ou outros ataques.

Ações de correspondência
Uma ação é um conjunto de comandos que são executados para bloquear um ataque. Uma ação também pode definir comandos para desbloquear um IP e comandos que são executados quando o Fail2Ban é iniciado ou encerrado.

Cadeias de ação do filtro
Uma prisão é uma combinação de um filtro, uma ou mais ações e um ou mais arquivos de log. O log é varrido continuamente em busca de linhas que correspondam ao filtro e, quando um é encontrado, as ações selecionadas são executadas.
