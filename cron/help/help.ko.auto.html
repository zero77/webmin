예약 된 명령 및 작업

이 도움말 페이지가 불완전합니다

소개

이 모듈은 예약 된 시간에 자동으로 실행되도록 명령을 설정하는 데 사용할 수 있습니다. 표준 Unix cron 데몬을 사용합니다.이 데몬은 선택한 분, 시간, 요일, 요일 및 월에 다양한 사용자로 명령을 실행할 수 있습니다. 모든 작업은 사용자가 소유하며, 실행될 때 해당 사용자의 권한으로 실행됩니다.

모듈의 기본 페이지에는 알려진 모든 예약 된 작업이 사용자별로 그룹화되어 실행됩니다. 일부 시스템에는 root 또는 sys 와 같은 사용자가 소유 한 운영 체제의 일부인 작업이 있습니다. crontab 명령을 사용하여 시스템의 사용자가 작성한 작업이있을 수도 있습니다.

새 예약 된 작업 생성

새 작업을 만들려면 기존 작업 목록 아래에서 새 예약 작업 만들기를 클릭하십시오. 다음 매개 변수를 입력하기위한 양식이 표시됩니다.
  • 실행할 사용자
    작업이 실행될 Unix 사용자의 로그인 이름.

  • 유효한?
    작업이 활성 상태가 아닌 경우 알려진 작업 목록에는 여전히 표시되지만 일정대로 실행되지는 않습니다.

  • 명령
    스케줄 시간에 실행할 실제 Unix 명령. 이것은 명령 행에 입력하는 모든 명령이 될 수 있습니다.

  • 명령에 입력
    이 필드가 비어 있지 않으면 입력 한 모든 항목이 실행될 때 명령에 대한 입력으로 제공됩니다. 예를 들어, 명령이 mail foo@bar.com 이고 입력 hello world 인 경우 입력 텍스트는 foo@bar.com 으로 이메일로 전송됩니다.

  • 실행 시간
    명령이 실행되는 시간은 분, 시간, 일, 월, 일 요일을 지정하여 선택됩니다. 이들 각각에 대해 당신도 모두 선택하거나 명령을 실행해야 할 때 시간을 선택할 수 있습니다.

    예를 들어, 시간으로 5를 선택하고 요일로 화요일을 선택하면 명령은 매주 오전 5시에서 오전 5시 59 분까지 실행됩니다.
    또는 분으로 30을, 시간으로 14를 선택하면 매일 오후 2시 30 분에 명령이 실행됩니다.

양식을 작성한 후에는 페이지 하단의 작성 버튼을 클릭하십시오. 새 작업이 즉시 적용됩니다.

기존 예약 된 작업 편집

기존 작업을 편집하려면 기본 페이지의 작업 목록에서 명령을 클릭하십시오. 기존 작업 편집 양식은 위에서 설명한 새 작업 작성 양식과 동일합니다. 필요한 사항을 변경했으면 페이지 하단의 저장 버튼을 클릭하십시오.

작업을 삭제하려면 작업 편집 양식 하단의 삭제 버튼을 클릭하십시오. 확인을 요청하지 않고 작업을 즉시 삭제합니다.
